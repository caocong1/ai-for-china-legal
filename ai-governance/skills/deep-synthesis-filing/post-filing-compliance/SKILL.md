# 备案后合规管理（post-filing-compliance）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`，加载服务立场·合规严格度·监管关系。
> 2. 已完成备案提交（或用户已有有效备案）。
> 3. 读取 `ai-governance/skills/_shared/ai-law-citations.md`（§D.1 深度合成管理规定）。
> 4. 读取 `ai-governance/skills/_shared/legal-basis-conventions.md`。

---

## 目的

完成深度合成备案后的持续合规管理，包括备案信息变更时的更新义务、定期合规自查、监管配合、备案续期、AI治理法规跟踪等。

---

## 详细步骤

### 步骤 1：备案信息变更管理

#### 变更触发情形与分类

| 变更类型 | 触发情形 | 变更级别 | 变更要求 |
|---------|---------|---------|---------|
| **主体变更** | 企业名称变更 | 一般变更 | 须在变更后规定期限内更新备案信息 `[待验证]` |
| **主体变更** | 统一社会信用代码变更（如合并·分立） | 重大变更 | 须评估是否需要重新备案 `[待验证]` |
| **主体变更** | 服务运营主体变更（如业务转让） | 重大变更 | 原主体注销备案，新主体重新备案 `[待验证]` |
| **服务变更** | 深度合成服务类型新增（如从图像生成扩展到视频生成） | 重大变更 | 须评估是否需要补充备案或重新备案 `[待验证]` |
| **服务变更** | 服务应用场景发生重大变化（如从ToB扩展到ToC） | 重大变更 | 须评估影响并更新备案信息 `[待验证]` |
| **服务变更** | 服务对象发生变化（如新增面向未成年人） | 重大变更 | 须评估未成年人保护特别义务并更新备案 `[待验证]` |
| **算法变更** | 深度合成算法发生重大升级（模型架构变更） | 一般变更 | 须评估是否需要更新算法备案信息 `[待验证]` |
| **算法变更** | 深度合成算法完全替换 | 重大变更 | 须评估是否需要重新备案 `[待验证]` |
| **安全事项变更** | 安全管理制度重大修订 | 一般变更 | 须评估影响并更新备案信息 `[待验证]` |
| **安全事项变更** | 内容审核机制发生重大变化（如更换审核系统） | 一般变更 | 须评估影响并更新备案信息 `[待验证]` |
| **安全事项变更** | 发生数据安全事件或内容安全事件 | 紧急变更 | 须立即评估影响，按监管要求报告并更新备案 `[待验证]` |

#### 变更管理流程

```
变更识别 → 变更级别判定 → 影响评估 → 变更材料准备 → 备案系统更新提交 → 变更记录归档
```

**变更管理要点**：
- 建立变更监测机制：定期（每月）检查是否存在须变更备案信息的情形
- 变更级别判定：区分一般变更（更新备案信息即可）与重大变更（可能须补充或重新备案）
- 变更时限：一般变更须在变更后规定期限内完成更新 `[待验证具体时限]`；重大变更须立即启动评估
- 变更记录：所有变更均须记录在案，包括变更原因·变更内容·变更日期·审批人

### 步骤 2：定期合规自查

#### 自查体系

| 自查项目 | 自查内容 | 建议频次 | 责任人 | 输出 |
|---------|---------|---------|--------|------|
| **备案信息准确性** | 备案信息是否与实际情况一致 | 每季度 | [合规负责人] | 自查报告 |
| **内容安全** | 深度合成内容标识是否规范·内容审核机制是否有效运行·违规内容处置是否及时 | 每月 | [内容安全负责人] | 月度报告 |
| **数据安全** | 训练数据·用户数据保护是否合规·数据安全措施是否有效·是否发生数据安全事件 | 每季度 | [数据安全负责人] | 季度报告 |
| **用户保护** | 用户投诉处理是否及时·用户权利保障是否到位·用户协议与隐私政策是否需更新 | 每月 | [用户保护负责人] | 月度报告 |
| **管理制度执行** | 安全管理制度是否有效执行·制度是否需要修订·人员培训是否到位 | 每季度 | [合规负责人] | 季度报告 |
| **技术措施有效性** | 内容审核系统·风险监测系统·应急处置系统是否正常运行·是否需要升级 | 每季度 | [技术负责人] | 季度报告 |
| **深度合成标识合规** | 深度合成内容标识是否符合要求·标识是否被用户移除或规避 | 每月 | [内容安全负责人] | 月度报告 |
| **法规变化跟踪** | AI治理领域是否有新规发布·是否影响本企业备案义务 | 每月 | [合规负责人] | 法规跟踪简报 |

#### 自查结果处理

- 自查合格：记录自查结果，归档备查
- 自查发现一般问题：制定整改计划，在下次自查前完成整改
- 自查发现重大问题：立即启动整改，评估是否需要更新备案信息，必要时升级给执业律师
- 自查发现安全事件：按应急处置机制处理，评估是否需要向监管机关报告

### 步骤 3：年度合规审查

> **年度合规审查是备案后合规管理的核心环节，须全面回顾全年合规状况。**

#### 年度审查内容

| 审查项目 | 审查内容 | 审查方式 |
|---------|---------|---------|
| 备案信息全面核验 | 全部备案信息是否与当前实际情况一致 | 逐项对照 |
| 安全评估更新 | 是否需要更新安全评估报告（服务变化·法规变化·安全事件） | 综合评估 |
| 合规制度回顾 | 安全管理制度是否完善·是否需要修订 | 制度审查 |
| 安全事件回顾 | 全年安全事件汇总·处置情况·改进措施 | 事件台账审查 |
| 投诉举报回顾 | 全年投诉举报统计·处理情况·趋势分析 | 投诉台账审查 |
| 法规变化影响评估 | 全年AI治理法规变化对本企业的影响 | 法规跟踪汇总 |
| 技术措施升级评估 | 内容审核·风险监测·应急处置系统是否需要升级 | 技术评估 |

#### 年度审查输出

- 年度合规自查报告
- 安全评估更新报告（如需要）
- 制度修订建议（如需要）
- 下一年度合规工作计划

### 步骤 4：监管配合

#### 监管配合预案

| 监管场景 | 配合要点 | 响应时限 | 注意事项 |
|---------|---------|---------|---------|
| 监管机关问询 | 及时响应·如实回答·保留问询记录 | 收到问询后24小时内响应 `[待验证]` | 回复内容须经律师审核 |
| 监管机关检查 | 配合检查·提供所需材料·记录检查过程 | 按监管机关要求 | 保留检查记录与材料副本 |
| 监管机关责令改正 | 立即启动整改·制定整改方案·按期报告 | 按整改通知要求 | 标 `[被动应对 — 强标整改时限]` |
| 监管机关行政处罚 | 立即升级给执业律师·评估是否提起行政复议/诉讼 | 立即 | 标 `[强制升级 — 重大违规]` |
| 监管机关现场核查 | 配合现场核查·准备所需材料·安排相关人员 | 按监管机关要求 | 提前准备核查材料包 |

#### 监管配合注意事项

- 所有向监管机关提交的材料须经执业律师审核
- 保留与监管机关沟通的全部记录（函件·邮件·会议纪要）
- 整改完成后及时报告整改情况，保留整改证据
- 如被处罚，评估是否触发其他合规义务（如上市公司信息披露义务）

### 步骤 5：备案续期

关注备案有效期，在到期前及时办理续期：

- 确认备案有效期 `[待验证]`
- 在到期前规定时间内提交续期申请 `[待验证]`（建议到期前2-3个月启动）
- 更新安全评估报告（如要求）
- 更新备案信息（如有变更须一并处理）
- 续期申请提交后跟踪审核进度

**续期时间线建议**：

| 时间节点 | 操作 |
|---------|------|
| 到期前3个月 | 启动续期准备工作，评估是否需要更新材料 |
| 到期前2个月 | 完成安全评估更新（如需要），准备续期材料 |
| 到期前1个月 | 提交续期申请 |
| 到期前 | 确认续期审核结果 |

### 步骤 6：AI 治理法规跟踪

建立 AI 治理法规跟踪机制，及时响应法规变化：

**跟踪渠道**：
- 国家互联网信息办公室官网（cac.gov.cn）
- 工业和信息化部官网（miit.gov.cn）
- 国家法律法规数据库（flk.npc.gov.cn）
- 中国信息通信研究院（caict.ac.cn）
- 行业协会通知

**跟踪内容**：
- 深度合成管理规定配套细则更新
- 备案系统操作流程变化
- AI 安全评估标准更新
- 执法案例与行政处罚动态
- 地方性 AI 治理法规动态

**法规变化响应流程**：
```
法规监测 → 变化识别 → 影响评估 → 合规措施调整 → 内部通知 → 备案信息更新（如需）
```

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 服务发生重大变化（新增服务类型·服务对象重大变化） | 重新评估备案义务，可能需要补充或重新备案；标 `[重大变更 — 须评估备案影响]` |
| 监管机关发出整改通知 | 升级为被动应对模式，强标整改时限；标 `[被动应对 — 强标整改时限]` |
| 法规变化影响备案义务 | 评估影响范围，调整合规措施；如新规增加备案义务须及时响应 |
| 备案到期未续期 | 标 `[紧急 — 备案即将/已经到期]`，立即启动续期程序 |
| 发生安全事件（数据泄露·内容安全事故） | 按应急处置机制处理；评估是否需要向监管机关报告；评估是否需要更新备案信息 |
| 企业并购/重组导致主体变更 | 评估备案是否需要转移或重新办理；标 `[主体变更 — 须评估备案影响]` |
| 合规团队人员变动 | 确保合规职责交接完整，备案管理责任不中断 |
| 深度合成服务停止运营 | 评估是否需要注销备案；处理用户数据删除事宜 |

## 错误处理

| 错误/敏感情形 | 处理方式 |
|------------|---------|
| 被监管机关处罚 | 标 `[强制升级 — 重大违规]`，联系执业律师；评估是否提起行政复议/诉讼 |
| 行刑衔接 | 标 `[强制升级 — 行刑衔接]`；如安全事件可能构成刑事犯罪 |
| 备案信息未及时更新 | 评估风险等级，建议立即补充更新；如已超期标 `[超期未更新 — 合规风险]` |
| 自查发现系统性合规缺陷 | 标 `[系统性缺陷 — 须全面整改]`，制定全面整改计划，评估是否需要主动报告监管机关 |
| 续期申请被拒绝 | 分析拒绝原因，制定应对方案；如影响服务运营须立即升级 |
