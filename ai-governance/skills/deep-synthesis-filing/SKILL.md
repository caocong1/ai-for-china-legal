# deep-synthesis-filing — 深度合成备案编排入口

## 目的与整体流程概览

深度合成备案帮助企业（及其律师/合规官/法务）完成深度合成服务备案义务，包括识别深度合成服务类型、准备备案材料、完成安全评估与备案提交、备案后持续合规管理。

**整体备案流程（顺序调用，也可单独触发任一子技能）**：

```
深度合成服务识别与分类 → 备案材料准备 → 安全评估与备案提交 → 备案后合规管理
```

## 入口级护栏

以下护栏在编排入口层执行，不依赖子技能：

### 1. 行刑衔接升级

识别到 AI 服务违规可能升格刑事（侵犯公民个人信息·帮助信息网络犯罪活动·拒不履行信息网络安全管理义务等）时：
- **立即强制升级执业律师**，标 `[强制升级 — 行刑衔接]`
- 提示与 criminal-compliance 插件衔接
- 本护栏凌驾于服务立场与合规严格度配置之上，不可被任何立场配置覆盖

### 2. AI 治理合规立场护栏

- **绝不提供规避备案·虚假安全评估·绕过内容安全审查的方法**
- 识别此类需求即拒绝，标 `[合规边界 — 拒绝]`
- 具体拒绝场景包括：
  - 要求帮助设计规避深度合成备案义务的方案
  - 要求伪造安全评估报告或备案材料
  - 要求设计绕过内容安全标识义务的技术方案
  - 要求隐匿深度合成服务中的个人信息违规处理行为
  - 以「合规咨询」为名实质寻求规避监管路径

### 3. 深度合成特殊场景护栏

以下场景须特别关注并触发额外审查：

- **换脸/人脸替换服务**：识别到服务涉及人脸替换（如 AI 换脸应用）时，须额外评估是否涉及欺诈风险（电信诈骗·身份冒用），如涉及则标 `[高风险 — 行刑衔接审查]` 并评估帮助信息网络犯罪活动罪风险
- **深度伪造检测义务**：深度合成服务提供者应当具备深度合成内容检测能力，识别到企业未建立检测机制时标 `[合规缺口 — 检测义务]`
- **虚拟数字人/虚拟主播**：涉及虚拟数字人服务时，须评估是否同时涉及算法推荐服务（如个性化推送）从而触发算法备案义务
- **未成年人保护**：深度合成服务面向未成年人或可能被未成年人使用时，须评估未成年人保护特别义务
- **跨境深度合成服务**：服务涉及跨境提供时，须评估是否触发数据出境安全评估义务

### 4. 备案时限护栏

涉及备案义务或整改期限时：
- 强标备案时限提示：深度合成服务上线前须完成备案
- 不得静默放任时限流逝
- 被动应对模式下，强标监管整改期限并倒排工作计划

## 输出：完整备案流程产物

完整备案流程完成后，产出以下工作产物：

1. **深度合成服务识别与分类报告**（子技能 1 产出）：服务是否属于深度合成、分类结论、备案义务判断、交叉义务识别
2. **备案材料准备清单与草稿**（子技能 2 产出）：完整材料清单、各材料准备状态、待完善事项
3. **安全评估报告**（子技能 3 产出）：五维安全评估结果、风险控制措施、备案提交状态
4. **备案后合规管理方案**（子技能 4 产出）：变更管理流程、定期自查计划、监管配合预案、法规跟踪机制

输出继承实践配置文件 `## 输出` 中的工作产物标题和审查者注释块格式。非律师角色在备案材料定稿提交·安全评估报告定稿·向监管机关报送材料等步骤前须暂停并升级给执业律师确认。
