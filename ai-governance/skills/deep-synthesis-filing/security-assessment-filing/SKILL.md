# 安全评估与备案提交（security-assessment-filing）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`，加载服务立场与合规严格度。
> 2. 已完成备案材料准备（本子技能的前序子技能或单独触发时由用户提供已准备材料）。
> 3. 读取 `ai-governance/skills/_shared/ai-law-citations.md`（§D.1 深度合成管理规定）。
> 4. 读取 `ai-governance/skills/_shared/legal-basis-conventions.md`。

## 目的

完成深度合成服务安全评估，通过互联网信息服务算法备案系统完成备案提交，确保备案流程合规。

---

## 详细步骤

### 步骤 1：安全评估范围确定

确定安全评估的覆盖范围与评估深度：

| 评估维度 | 评估内容 | 评估方法 | 重点事项 |
|---------|---------|---------|---------|
| **内容安全** | 深度合成内容的合规性 | 滥用场景分析·审核机制测试·标识方案验证 | 是否可能生成违法违规内容·有害信息·虚假内容 |
| **数据安全** | 训练数据与用户数据保护 | 数据来源审查·合规性核验·安全技术评估 | 数据来源合法性·个人信息保护·生物识别信息保护·数据安全 |
| **算法安全** | 算法模型安全性 | 偏见测试·滥用场景分析·可解释性评估 | 算法偏见·算法滥用·深度伪造检测能力 |
| **用户保护** | 用户权益保障措施 | 用户协议审查·功能测试·投诉机制验证 | 用户知情权·选择权·投诉机制·未成年人保护 |
| **应急处置** | 应急响应能力 | 应急预案审查·演练记录检查·响应时间测试 | 违规内容处置·安全事件应急·监管配合能力 |

**评估范围确定原则**：
- 评估范围须覆盖深度合成服务的全部功能与使用场景
- 如涉及多种深度合成技术类型（如同时提供换脸和语音克隆），须对每种类型分别评估
- 如涉及敏感个人信息（人脸·声纹等生物识别信息），须加大数据安全评估深度
- 评估范围须在评估报告中明确记载

### 步骤 2：安全风险评估执行

**内容安全评估**：
- 滥用场景识别：
  - 换脸服务：是否可能被用于制作虚假视频·身份冒用·色情内容·欺诈内容
  - 语音克隆服务：是否可能被用于电信诈骗·冒充他人声音
  - 图像/视频生成：是否可能被用于生成虚假新闻·暴力恐怖内容·色情内容
  - 虚拟数字人：是否可能被用于虚假代言·误导性信息传播
- 内容审核机制有效性评估：
  - 审核覆盖率：自动审核能否覆盖全部生成内容
  - 审核准确率：误判率与漏判率指标
  - 审核时效：能否在内容发布前完成审核
  - 人工审核：审核团队规模·专业能力·培训情况
- 内容标识方案充分性评估：
  - 标识是否在生成内容的显著位置
  - 标识是否清晰可辨·不易移除
  - 不同场景的标识策略是否适当

**数据安全评估**：
- 训练数据来源合法性审查：
  - 训练数据的获取方式（公开数据集·授权数据·自行采集）
  - 训练数据中是否包含个人信息·敏感个人信息
  - 训练数据使用是否在授权范围内使用
  - 训练数据是否包含违法不良信息
- 个人信息处理合规性审查：
  - 用户输入数据（如上传的照片·录音）的收集·使用·存储·删除规则
  - 人脸信息·声纹信息等生物识别信息的特殊保护措施
  - 是否完成个人信息保护影响评估（PIA）
- 数据存储与传输安全评估：
  - 数据加密存储与传输措施
  - 访问控制与权限管理
  - 数据备份与灾难恢复

**算法安全评估**：
- 算法偏见与歧视风险评估：
  - 训练数据是否存在代表性偏差（如种族·性别·年龄分布不均）
  - 模型输出是否存在系统性偏见
  - 偏见监测与纠正机制
- 算法可解释性评估：
  - 算法决策过程的可解释程度
  - 向用户解释算法运行机制的方式
- 算法滥用防护评估：
  - 防止算法被用于生成违法内容的技术措施
  - 深度合成内容检测能力（是否具备检测深度合成内容真伪的技术）
  - API调用限制与异常行为监测

**用户保护评估**：
- 用户知情权保障：
  - 是否充分告知用户使用深度合成技术的情况
  - 用户协议·隐私政策中的告知是否完整
  - 深度合成内容标识是否清晰
- 用户选择权保障：
  - 用户是否可以选择关闭某些深度合成功能
  - 用户是否可以撤回同意并删除数据
- 投诉举报机制：
  - 投诉渠道是否畅通（在线客服·电话·邮件·应用内举报）
  - 投诉响应时限是否合理
  - 投诉处理记录是否完整
- 未成年人保护：
  - 是否有未成年人使用限制措施
  - 是否防止未成年人生成不适当的深度合成内容

### 步骤 3：安全评估报告撰写

撰写安全评估报告，包括以下章节：

1. **评估概述**：
   - 评估目的：为深度合成备案提供安全评估依据
   - 评估范围：覆盖的深度合成服务·功能·技术类型
   - 评估方法：文档审查·技术测试·人员访谈·功能验证
   - 评估时间：评估起止日期
   - 评估人员：评估团队成员及资质

2. **评估发现**：
   - 各评估维度的详细评估结果
   - 各项评估指标的具体数据
   - 评估中发现的问题与不足

3. **风险识别与评级**：
   - 已识别的风险点清单
   - 风险等级评定（🔴关键/🟠高/🟡中等/🟢低）
   - 风险影响范围与可能性分析

4. **控制措施**：
   - 已采取的风险控制措施及其有效性
   - 计划采取的风险控制措施及完成时间
   - 残余风险的可接受性评估

5. **评估结论**：
   - 总体安全评估结论
   - 是否具备安全条件提交备案
   - 遗留问题与整改建议

### 步骤 4：备案系统提交

通过互联网信息服务算法备案系统完成备案提交：

| 步骤 | 操作 | 注意事项 | 预计耗时 |
|------|------|---------|---------|
| 1 | 登录备案系统 | 确认使用正确的企业账号；首次登录须完成实名认证 `[待验证]` | 1个工作日 |
| 2 | 填写备案信息 | 按系统要求逐项填写：主体信息·服务信息·算法信息·安全信息；信息须与材料一致 | 1-2个工作日 |
| 3 | 上传备案材料 | 按清单上传所有材料（PDF/Word格式）；注意文件大小限制 `[待验证]` | 1个工作日 |
| 4 | 自查确认 | 提交前逐项核对：信息完整性·材料齐全性·信息一致性·真实性 | 0.5个工作日 |
| 5 | 提交审核 | 确认信息无误后提交；提交后部分信息可能不可修改 `[待验证]` | — |
| 6 | 跟踪审核进度 | 关注审核反馈；审核周期 `[待验证]`；及时补充材料 | 审核周期内持续 |

### 步骤 5：审核反馈处理

- **审核通过**：记录备案编号，归档备案材料，衔接备案后合规管理
- **审核退回**：
  - 分析退回原因（材料不完整·信息不一致·安全评估不充分 等）
  - 制定修改方案
  - 修改材料后重新提交
  - 记录退回原因与修改内容，作为后续备案经验
- **补充材料**：
  - 根据审核意见准备补充材料
  - 补充材料须与原备案信息保持一致
  - 在规定时限内提交补充材料 `[待验证具体时限]`

### 步骤 6：时限管理与进度跟踪

| 时间节点 | 时限要求 | 管理措施 |
|---------|---------|---------|
| 安全评估启动 | 备案材料准备完成后立即启动 | 制定评估计划，分配评估任务 |
| 安全评估完成 | 提交备案前须完成 | 设定评估完成截止日，倒排工作计划 |
| 备案材料提交 | 深度合成服务上线前须完成备案 `[待验证具体时限]` | 以服务上线日期为锚点倒推 |
| 补充材料提交 | 审核退回后须在规定时限内补充 `[待验证]` | 收到退回通知后立即启动修改 |
| 备案结果跟踪 | 审核周期内持续关注 `[待验证]` | 指定专人跟踪审核进度 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 安全评估发现高风险（🔴关键） | 标注 `[高风险 — 须在风险控制措施完善后方可提交备案]`；如涉及行刑衔接则强制升级 |
| 安全评估发现中高风险（🟠高） | 标注 `[中高风险 — 建议完善控制措施后提交]`；如合规严格度为保守型则须整改后提交 |
| 备案系统操作异常 | 记录异常情况（截图·错误信息），建议联系备案系统技术支持；记录延迟原因 |
| 审核被退回 | 分析退回原因，分类处理（材料问题·信息问题·安全问题）；修改后重新提交 |
| 服务已上线但未完成备案 | 标 `[紧急合规 — 服务已上线未备案]`，建议尽快完成备案；评估被监管机关发现的风险 |
| 安全评估发现训练数据合规问题 | 标注 `[数据合规缺口 — 须先整改]`，训练数据合法性问题须在提交备案前解决 |
| 企业无法完成安全评估（缺乏技术能力） | 建议委托第三方安全评估机构协助；标注 `[第三方评估 — 建议委托]` |
| 评估期间服务发生重大变更 | 重新评估变更部分的安全性，更新安全评估报告 |

## 错误处理

| 错误/敏感情形 | 处理方式 |
|------------|---------|
| 虚假评估需求 | 拒绝，标 `[合规边界 — 拒绝]`；如要求出具虚假安全评估结论·隐瞒风险发现·美化评估结果即拒绝 |
| 行刑衔接 | 标 `[强制升级 — 行刑衔接]`；如安全评估发现服务涉及换脸用于欺诈·语音克隆用于诈骗等场景 |
| 信息不足 | 列出需补充的信息清单，按评估维度分组；标注缺失信息对评估结论的影响 |
| 企业施压要求出具有利评估结论 | 保持评估独立性；标注 `[评估独立性 — 不受干预]`；如持续施压则升级给执业律师 |
| 备案系统无法访问或功能异常 | 记录异常情况，建议联系技术支持；标注 `[系统异常 — 备案时限可能受影响]` |

## 时限管理要点

> **备案时限是深度合成备案的关键合规节点，逾期可能面临行政处罚或服务下架风险。**

| 时限节点 | 要求 | 管理建议 |
|---------|------|---------|
| 服务上线前 | 须完成深度合成备案 `[待验证具体时限]` | 以计划上线日期为锚点，倒推至少4-6周启动备案准备 |
| 安全评估 | 须在提交备案前完成 | 安全评估通常需要1-2周，须预留充足时间 |
| 审核周期 | 提交后等待审核 `[待验证具体周期]` | 审核期间不得上线服务（如尚未上线） |
| 补充材料 | 审核退回后须在规定时限内补充 `[待验证]` | 收到退回通知后24小时内启动修改 |
| 被动应对模式 | 监管整改期限以整改通知为准 | 收到整改通知后立即标记期限，倒排工作计划 |
