# algorithm-filing — 算法备案编排入口

## 目的与整体流程概览

算法备案帮助企业（及其律师/合规官/法务）完成算法推荐服务备案义务，包括识别算法类型、分析备案要求、准备备案材料并完成提交、备案后变更管理。

**整体备案流程（顺序调用，也可单独触发任一子技能）**：

```
算法类型识别与分类 → 备案要求分析 → 备案材料准备与提交 → 备案后变更管理
```

---

## 事项上下文

算法备案是《互联网信息服务算法推荐管理规定》（2022，网信办等四部门）确立的核心合规义务。算法推荐服务提供者须通过互联网信息服务算法备案系统完成备案，否则面临行政处罚风险。

**适用主体**：在中国境内利用生成合成类、个性化推送类、排序精选类、检索过滤类、调度决策类等算法技术向用户提供互联网信息服务的企业。

**核心合规风险**：

| 风险类型 | 风险描述 | 法律后果 |
|---------|---------|---------|
| 未履行备案义务 | 算法推荐服务提供者未通过算法备案系统完成备案 | 行政处罚·责令改正·罚款 `[待验证]` |
| 算法歧视 | 利用算法实施不合理差别待遇（如大数据杀熟） | 行政处罚·民事赔偿·可能升格刑事 `[待验证]` |
| 算法影响舆论 | 算法推荐服务具有舆论属性或社会动员能力但未完成安全评估 | 行政处罚·责令暂停服务 `[待验证]` |
| 侵害用户权益 | 未保障用户知情权·选择权（未提供关闭算法推荐选项） | 行政处罚·责令改正 `[待验证]` |

## 入口级护栏

### 1. 行刑衔接升级

识别到算法服务违规可能升格刑事（侵犯公民个人信息·帮助信息网络犯罪活动·拒不履行信息网络安全管理义务等）时：
- **立即强制升级执业律师**，标 `[强制升级 — 行刑衔接]`
- 提示与 criminal-compliance 插件衔接

### 2. AI 治理合规立场护栏

- **绝不提供规避算法备案义务的方法**
- **绝不提供虚假安全评估材料或伪造算法透明度方案**
- **绝不提供绕过内容安全审查的方案**
- 识别此类需求即拒绝，标 `[合规边界 — 拒绝]`

### 3. 备案时限护栏

涉及备案义务或整改期限时：
- 强标备案时限提示
- 不得静默放任时限流逝

### 4. 算法歧视护栏

识别到以下算法歧视场景时，须明确提示合规风险：

| 场景 | 风险描述 | 处理要求 |
|------|---------|---------|
| **算法价格歧视（大数据杀熟）** | 利用算法对交易条件相同的消费者实施不合理差别定价 | 明确提示违反算法推荐管理规定及消费者权益保护法 `[待验证]`；绝不提供规避方法 |
| **算法推荐歧视** | 基于用户特征（地域·性别·消费习惯等）实施不公平的个性化推送 | 提示违反个人信息保护法自动化决策条款 `[待验证]` |
| **就业·信贷等领域算法歧视** | 利用算法在就业·信贷等领域实施歧视性决策 | 提示可能涉及多重法律责任 `[待验证]` |

### 5. 算法推荐影响舆论护栏

识别到算法推荐服务具有舆论属性或社会动员能力时：
- 提示须在备案前完成安全评估 `[待验证]`
- 提示算法推荐可能影响公共舆论的特殊合规义务
- 提示内容安全管理制度建设要求

## 输出

编排入口本身不直接输出法律分析，而是：

1. **加载配置**：读取实践配置与共享资源
2. **路由调度**：根据用户需求路由到对应子技能
3. **护栏检查**：在调度前完成入口级护栏检查
4. **汇总输出**：完整流程运行时，汇总各子技能输出为「算法备案全流程报告」

**路由规则**：

| 用户需求 | 路由目标 |
|---------|---------|
| 完整算法备案 | 按顺序执行全部四个子技能 |
| 识别算法类型 | 仅执行子技能 1 |
| 分析备案要求 | 仅执行子技能 2（须已有算法类型识别结果） |
| 准备备案材料 | 仅执行子技能 3（须已有备案要求分析结果） |
| 备案后变更管理 | 仅执行子技能 4（须已有有效备案） |
