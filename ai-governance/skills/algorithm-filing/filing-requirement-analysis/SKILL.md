# 备案要求分析（filing-requirement-analysis）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`。
> 2. 已完成算法类型识别与分类。

---

## 目的

根据已识别的算法类型，分析具体的备案要求，包括备案条件、备案流程、材料要求、安全评估要求，输出「算法类型—备案要求」对照表。

---

## 详细步骤

### 步骤 1：备案条件分析

| 备案条件 | 要求说明 | 判断标准 |
|---------|---------|---------|
| 服务主体 | 须为在中国境内合法设立的企业法人 | 核查营业执照 `[待验证]` |
| 服务类型 | 须属于算法推荐服务 | 步骤 1 已确认 |
| 服务状态 | 已上线或计划上线 | 确认服务状态 |
| 用户规模 | 是否达到备案门槛 | 须核实 `[待验证]` |

### 步骤 2：备案流程分析

**算法备案流程概览**：

```
准备材料 → 登录备案系统 → 填写信息 → 上传材料 → 提交审核 → 审核反馈 → 完成备案
```

| 流程节点 | 说明 | 注意事项 |
|---------|------|---------|
| 材料准备 | 按备案要求准备全部材料 | 确保材料完整准确 |
| 系统登录 | 通过互联网信息服务算法备案系统 | 确认企业账号 |
| 信息填写 | 按系统要求逐项填写 | 信息须与实际一致 |
| 审核 | 监管机关审核 | 关注审核反馈 |

### 步骤 3：安全评估要求分析

根据算法类型确定安全评估要求：

| 评估维度 | 评估内容 |
|---------|---------|
| **算法安全** | 算法偏见·歧视·滥用风险评估 |
| **内容安全** | 算法推荐内容合规性评估 |
| **数据安全** | 算法训练数据·用户数据保护评估 |
| **用户保护** | 用户知情权·选择权保障评估 |
| **可解释性** | 算法可解释性与透明度评估 |

### 步骤 4：特殊要求识别

| 特殊情形 | 额外要求 |
|---------|---------|
| 具有舆论属性或社会动员能力 | 须在备案前完成安全评估 `[待验证]` |
| 涉及个人信息处理 | 须满足个人信息保护法要求 `[待验证]` |
| 涉及未成年人 | 须满足未成年人保护特殊要求 `[待验证]` |
