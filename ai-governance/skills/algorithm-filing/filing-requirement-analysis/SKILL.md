# 备案要求分析（filing-requirement-analysis）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`，加载服务立场与合规严格度。
> 2. 读取 `ai-governance/skills/_shared/legal-basis-conventions.md`。
> 3. 读取 `ai-governance/skills/_shared/ai-law-citations.md`（§D.2 算法推荐管理规定）。
> 4. 已完成算法类型识别与分类（本子技能的前序子技能或单独触发时由用户提供算法类型信息）。

---

## 目的

根据已识别的算法类型，分析具体的备案要求，包括备案条件、备案流程、材料要求、安全评估要求、备案机关与时限要求，输出「算法类型—备案要求」对照表。

具体目标：

1. **备案条件逐项分析**：确认企业是否满足各项备案条件
2. **备案门槛判断**：分析是否达到备案触发门槛
3. **备案机关确定**：确定向国家网信部门还是地方网信部门备案
4. **备案流程分析**：明确备案系统操作流程
5. **安全评估要求分析**：确定安全评估的维度与深度
6. **时限要求分析**：明确备案时限与时间节点
7. **特殊要求识别**：识别舆论属性·未成年人保护等特殊要求

---

## 详细步骤

### 步骤 1：备案条件逐项分析

| 备案条件 | 要求说明 | 判断标准 | 常见不满足情形 |
|---------|---------|---------|-------------|
| 服务主体 | 须为在中国境内合法设立的企业法人 | 核查营业执照·统一社会信用代码 | 境外企业未在中国境内设立法人实体 `[待验证]` |
| 服务类型 | 须属于算法推荐服务 | 前序子技能已确认算法类型 | 算法不向用户提供互联网信息服务 |
| 服务状态 | 已上线或计划上线 | 确认服务状态（已上线/计划上线/开发中） | 服务尚在开发中但已接近上线 |
| 用户规模 | 是否达到备案门槛 | 须核实具体门槛 `[待验证]` | 用户规模接近门槛但未确认 |
| 技术能力 | 具备算法安全管理能力 | 是否有算法安全管理制度·技术措施 | 企业尚未建立算法安全管理制度 |

### 步骤 2：备案门槛详细分析

**备案触发条件分析**：

| 条件 | 触发备案 | 不触发备案 | 待确认 |
|------|---------|----------|-------|
| 提供算法推荐服务（生成合成类） | 利用算法生成文本·图像·音频·视频向用户提供 | 仅内部使用不向用户提供 | 通过 API 间接提供 |
| 提供算法推荐服务（个性化推送类） | 利用算法根据用户特征推送个性化信息 | 非算法驱动的随机推送 | 算法推送与人工推送混合 |
| 提供算法推荐服务（排序精选类） | 利用算法对信息进行排序·精选 | 纯人工编辑的内容排序 | 算法辅助人工编辑 |
| 提供算法推荐服务（检索过滤类） | 利用算法进行信息检索·内容过滤 | 纯关键词匹配（非算法驱动） | 简单规则与算法混合 |
| 提供算法推荐服务（调度决策类） | 利用算法进行派单·定价·资源调度 | 人工调度·固定定价 | 算法辅助人工决策 |

**备案门槛判断流程**：

```
步骤 A：确认企业是否提供算法推荐服务
├── 否 → 无备案义务，记录分析结论
└── 是 → 继续
    │
    ├─ 步骤 B：确认服务是否面向中国境内用户
    │   ├── 否 → 可能不适用 `[待验证]`
    │   └── 是 → 继续
    │       │
    │       ├─ 步骤 C：确认用户规模是否达到备案门槛
    │       │   ├── 是 → 触发备案义务
    │       │   ├── 否 → 可能暂不触发 `[待验证 — 门槛须核实]`
    │       │   └── 不确定 → 建议按保守策略完成备案
    │       │
    │       └─ 步骤 D：确认是否具有舆论属性或社会动员能力
    │           ├── 是 → 触发备案义务 + 安全评估义务
    │           └── 否 → 仅触发备案义务
```

### 步骤 3：备案机关确定

| 备案机关 | 适用情形 | 判断标准 |
|---------|---------|---------|
| **国家互联网信息办公室（国家网信部门）** | 具有舆论属性或社会动员能力的算法推荐服务；全国性服务的算法推荐服务；中央企业/央企的算法推荐服务 | 服务覆盖全国·用户规模大·涉及新闻舆论 `[待验证]` |
| **省/自治区/直辖市网信办（地方网信部门）** | 仅在特定行政区域内提供算法推荐服务；地方企业的算法推荐服务 | 服务覆盖特定区域·用户规模较小 `[待验证]` |
| **待确认** | 无法确定备案机关 | 建议咨询监管机关或律师 `[待验证]` |

**备案机关判断要点**：

1. 企业注册地 → 初步判断地方网信部门管辖范围
2. 服务覆盖范围 → 全国性服务可能由国家网信部门管辖
3. 服务性质 → 涉及新闻·舆论·社会动员的服务可能由国家网信部门管辖
4. 用户规模 → 大规模用户服务可能由国家网信部门管辖 `[待验证]`

### 步骤 4：备案流程分析

**算法备案流程概览**：

```
准备材料 → 注册备案系统账号 → 登录备案系统 → 填写算法信息 → 上传备案材料 → 提交审核 → 审核反馈 → 补充材料（如需）→ 完成备案 → 获取备案编号
```

| 流程节点 | 说明 | 注意事项 |
|---------|------|---------|
| 材料准备 | 按备案要求准备全部材料（详见备案材料准备子技能） | 确保材料完整准确，避免反复补充 |
| 系统注册 | 在互联网信息服务算法备案系统注册企业账号 | 确认企业账号信息与实际一致 |
| 信息填写 | 按系统要求逐项填写算法信息·服务信息 | 信息须与实际一致，不得虚假填报 |
| 材料上传 | 上传全部备案材料 | 确保文件格式·大小符合系统要求 |
| 提交审核 | 提交备案申请等待监管机关审核 | 关注审核进度，及时响应反馈 |
| 审核反馈 | 监管机关审核并反馈 | 可能需要补充材料或修改信息 |
| 完成备案 | 审核通过后完成备案，获取备案编号 | 备案编号须妥善保存，用于后续变更管理 |

### 步骤 5：安全评估要求分析

根据算法类型确定安全评估要求：

| 评估维度 | 评估内容 | 评估方法 | 适用算法类型 |
|---------|---------|---------|------------|
| **算法安全** | 算法偏见·歧视·滥用风险评估 | 算法公平性测试·偏见检测·滥用场景分析 | 全部类型 |
| **内容安全** | 算法推荐内容合规性评估 | 内容审核机制评估·违规内容识别能力测试 | 生成合成类·个性化推送类·排序精选类 |
| **数据安全** | 算法训练数据·用户数据保护评估 | 数据来源合规性审查·数据保护措施评估 | 全部类型 |
| **用户保护** | 用户知情权·选择权保障评估 | 用户告知机制检查·关闭推荐功能验证 | 个性化推送类·排序精选类 |
| **可解释性** | 算法可解释性与透明度评估 | 算法说明文档审查·公示内容检查 | 全部类型 |
| **舆论安全** | 算法推荐对公共舆论的影响评估 | 舆论属性识别·内容传播影响分析 | 具有舆论属性的算法推荐服务 |

**安全评估深度要求**：

| 算法风险等级 | 评估深度 | 适用情形 |
|------------|---------|---------|
| **高风险** | 全面安全评估·第三方评估 | 具有舆论属性或社会动员能力 `[待验证]` |
| **中风险** | 标准安全评估·企业自评估 | 用户规模较大·涉及个人信息处理 `[待验证]` |
| **低风险** | 基础安全评估·企业自评估 | 用户规模较小·不涉及敏感领域 `[待验证]` |

### 步骤 6：时限要求分析

| 时限事项 | 要求 | 法律依据 |
|---------|------|---------|
| 首次备案时限 | 算法推荐服务提供者须在规定期限内完成备案 `[待验证]` | 算法推荐管理规定 `[待验证]` |
| 变更备案时限 | 备案信息发生变更时须在规定期限内更新 `[待验证]` | 算法推荐管理规定 `[待验证]` |
| 安全评估时限 | 具有舆论属性的算法推荐服务须在上线前完成安全评估 `[待验证]` | 算法推荐管理规定 `[待验证]` |
| 定期自查 | 须定期开展算法安全评估 `[待验证]` | 算法推荐管理规定 `[待验证]` |

**时限合规提示**：
- 备案时限属于强制性要求，逾期未备案可能面临行政处罚 `[待验证]`
- 建议在服务上线前预留充足时间完成备案（建议提前 [X] 个月）`[待验证]`
- 变更备案须在变更后规定期限内完成，不得拖延 `[待验证]`

### 步骤 7：特殊要求识别

| 特殊情形 | 额外要求 | 法律依据 |
|---------|---------|---------|
| 具有舆论属性或社会动员能力 | 须在备案前完成安全评估 `[待验证]` | 算法推荐管理规定 `[待验证]` |
| 涉及个人信息处理 | 须满足个人信息保护法要求·可能需要个人信息保护影响评估 `[待验证]` | 个人信息保护法 `[待验证]` |
| 涉及未成年人 | 须满足未成年人保护特殊要求·不得利用算法向未成年人推送不适宜内容 `[待验证]` | 算法推荐管理规定·未成年人保护法 `[待验证]` |
| 涉及劳动者权益 | 调度决策类算法须保障劳动者知情权·休息权 `[待验证]` | 算法推荐管理规定·劳动法相关规定 `[待验证]` |
| 涉及新闻信息服务 | 须满足互联网新闻信息服务管理规定 `[待验证]` | 互联网新闻信息服务管理规定 `[待验证]` |
| 涉及金融信息服务 | 须满足金融信息服务相关监管要求 `[待验证]` | 金融信息服务相关规定 `[待验证]` |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 备案条件不明确 | 标注 `[待确认]`，建议咨询监管机关或执业律师 |
| 新型算法无明确备案指引 | 标注 `[新型算法 — 监管指引尚不明确]`，建议关注监管动态 |
| 备案机关不明确 | 标注 `[待确认]`，建议同时咨询国家和地方网信部门 |
| 用户规模接近门槛但未确认 | 建议按保守策略完成备案，避免逾期风险 |
| 服务尚未上线但计划上线 | 建议在上线前完成备案，预留充足时间 |
