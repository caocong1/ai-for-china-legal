# 备案后变更管理（post-filing-change-management）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`，加载服务立场与合规严格度。
> 2. 读取 `ai-governance/skills/_shared/legal-basis-conventions.md`。
> 3. 读取 `ai-governance/skills/_shared/ai-law-citations.md`（§D.2 算法推荐管理规定）。
> 4. 已完成算法备案（或已有有效备案）。

---

## 目的

完成算法备案后的变更管理，包括备案信息变更更新、算法重大变更的补充备案、算法注销/注销备案、定期合规自查、监管配合与法规跟踪、持续合规管理。

具体目标：

1. **变更识别与分类**：识别并分类备案后的各类变更（重大变更/一般变更/年度更新）
2. **变更报告触发与流程**：确定变更报告触发条件与操作流程
3. **算法注销/注销备案**：算法推荐服务终止时的注销备案流程
4. **定期合规自查**：建立定期合规自查机制
5. **监管配合与法规跟踪**：响应监管机关要求，跟踪法规变化
6. **持续合规管理**：确保算法推荐服务持续符合备案要求

---

## 详细步骤

### 步骤 1：变更识别与分类

**变更分类体系**：

| 变更类别 | 变更类型 | 触发情形 | 处理要求 | 时限要求 |
|---------|---------|---------|---------|---------|
| **重大变更** | 算法类型变更 | 算法推荐服务的算法类型发生变化（如从个性化推送类变更为生成合成类） | 须评估是否需要补充或重新备案 `[待验证]`；可能需要重新进行算法类型识别 | 变更后规定期限内 `[待验证]` |
| **重大变更** | 应用领域重大变更 | 算法应用领域发生重大变化（如从电商推荐扩展至新闻推荐） | 须评估是否影响备案义务与合规要求 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **重大变更** | 服务主体变更 | 算法推荐服务的运营主体发生变更（如企业合并·分立·转让） | 须评估是否需要重新备案 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **重大变更** | 舆论属性变化 | 算法推荐服务新增舆论属性或社会动员能力 | 须完成安全评估 `[待验证]` | 变更前完成 `[待验证]` |
| **一般变更** | 主体信息变更 | 企业名称·信用代码·联系方式·地址变更 | 须在备案系统中更新备案信息 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **一般变更** | 算法安全负责人变更 | 算法安全负责人发生变更 | 须更新备案信息 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **一般变更** | 算法参数调整 | 算法主要参数发生重大调整但不改变算法类型 | 须评估影响并更新备案信息 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **一般变更** | 安全管理制度变更 | 安全管理制度·技术措施发生重大变化 | 须更新备案信息 `[待验证]` | 变更后规定期限内 `[待验证]` |
| **年度更新** | 年度报告 | 定期更新备案信息·提交年度自查报告 | 按要求定期更新 `[待验证]` | 按规定时间 `[待验证]` |
| **年度更新** | 算法安全评估更新 | 定期开展算法安全评估并更新评估报告 | 须定期开展 `[待验证]` | 按规定频次 `[待验证]` |

**变更分类判断流程**：

```
变更发生
├── 是否涉及算法类型变化？
│   └── 是 → 【重大变更】→ 须评估是否需补充或重新备案
│
├── 是否涉及应用领域重大变化？
│   └── 是 → 【重大变更】→ 须评估是否影响备案义务
│
├── 是否涉及服务主体变更？
│   └── 是 → 【重大变更】→ 须评估是否需重新备案
│
├── 是否涉及舆论属性变化？
│   └── 是 → 【重大变更】→ 须完成安全评估
│
├── 是否为主体信息·负责人·参数·制度的一般变更？
│   └── 是 → 【一般变更】→ 在备案系统中更新信息
│
└── 是否为定期更新事项？
    └── 是 → 【年度更新】→ 按要求定期更新
```

### 步骤 2：变更报告触发与流程

**变更报告触发条件**：

| 触发条件 | 报告类型 | 报告内容 |
|---------|---------|---------|
| 算法类型发生变化 | 重大变更报告 | 变更原因·变更前后对比·影响评估·补救措施 |
| 应用领域发生重大变化 | 重大变更报告 | 变更原因·新增领域合规评估·安全评估更新 |
| 服务主体发生变更 | 重大变更报告 | 变更原因·新主体信息·备案转移方案 |
| 企业名称/信用代码变更 | 一般变更报告 | 变更前后信息对比·新营业执照 |
| 算法安全负责人变更 | 一般变更报告 | 新负责人信息·资质说明 |
| 安全管理制度重大修订 | 一般变更报告 | 修订内容·修订原因·新制度文件 |

**变更备案流程**：

```
变更识别 → 变更分类（重大/一般/年度） → 影响评估 → 准备变更材料 → 通过备案系统提交变更 → 跟踪审核进度 → 完成变更备案
```

| 流程节点 | 重大变更 | 一般变更 | 年度更新 |
|---------|---------|---------|---------|
| 变更识别 | 主动识别 + 监测触发 | 主动识别 | 定期触发 |
| 影响评估 | 全面评估（含重新备案必要性） | 简要评估 | 常规评估 |
| 材料准备 | 完整变更材料 + 更新的安全评估 | 变更材料 | 更新信息 |
| 提交方式 | 备案系统提交 + 可能需要补充说明 | 备案系统提交 | 备案系统提交 |
| 审核预期 | 较长（可能需补充材料） | 较短 | 常规 |

### 步骤 3：算法注销/注销备案

当算法推荐服务终止时，须完成注销备案：

| 注销情形 | 触发条件 | 处理要求 |
|---------|---------|---------|
| **服务终止** | 算法推荐服务停止运营 | 须在备案系统中注销备案 `[待验证]` |
| **企业注销** | 企业依法注销或解散 | 须在备案系统中注销备案 `[待验证]` |
| **算法下线** | 特定算法停止使用 | 须在备案系统中注销该算法备案 `[待验证]` |
| **主体转让** | 算法推荐服务转让至其他主体 | 原主体注销备案 + 新主体重新备案 `[待验证]` |

**注销备案流程**：

```
确认注销原因 → 准备注销材料 → 通过备案系统提交注销申请 → 跟踪注销进度 → 完成注销备案 → 保存注销备案记录
```

**注销备案注意事项**：

1. 注销备案前须确保用户权益已妥善处理（用户数据删除·服务迁移等）
2. 注销备案后须同步更新企业内部的算法备案管理记录
3. 注销备案不影响注销前已发生的违规行为的责任追究 `[待验证]`

### 步骤 4：定期合规自查

**合规自查体系**：

| 自查项目 | 自查内容 | 建议频次 | 自查方法 | 负责部门 |
|---------|---------|---------|---------|---------|
| 备案信息准确性 | 备案信息与实际是否一致 | 每季度 | 对照备案系统信息逐项核实 | 法务/合规部门 |
| 算法运行监测 | 算法推荐结果是否合规 | 每月 | 算法输出抽样检查·异常行为监测 | 技术部门 |
| 算法偏见检测 | 是否存在算法歧视·大数据杀熟 | 每季度 | 算法公平性测试·差别定价检测 | 技术部门 + 合规部门 |
| 用户权益保障 | 用户选择权·投诉机制是否有效 | 每月 | 功能验证·投诉数据分析 | 产品部门 + 客服部门 |
| 内容安全 | 推荐内容是否合规 | 每月 | 内容审核抽检·违规内容统计 | 内容安全部门 |
| 数据安全 | 用户数据保护是否到位 | 每季度 | 数据安全措施审计·访问日志检查 | 信息安全部门 |
| 算法透明度 | 公示内容是否准确·完整 | 每季度 | 公示内容与实际算法对比核实 | 法务/合规部门 |
| 管理制度执行 | 安全管理制度是否有效执行 | 每半年 | 制度执行情况审计 | 内审部门 |

**自查结果处理**：

| 自查结果 | 处理方式 |
|---------|---------|
| 全部合规 | 记录自查结果，归档备查 |
| 发现一般问题 | 制定整改计划，限期整改，记录整改结果 |
| 发现重大问题 | 立即启动应急响应，评估是否须主动报告监管机关，制定整改方案 |
| 发现可能违规 | 升级至法务/合规部门评估，必要时咨询执业律师 |

### 步骤 5：监管配合与法规跟踪

**监管配合**：

| 配合事项 | 处理方式 | 注意事项 |
|---------|---------|---------|
| 监管机关问询 | 及时响应，如实提供信息 | 响应须在时限内完成；涉及重大问题的须升级至律师 |
| 监管机关检查 | 配合检查，提供所需材料 | 检查前做好内部准备；检查中如实配合 |
| 监管整改通知 | 按整改要求制定整改方案并执行 | 整改须在期限内完成；整改完成后须报告 |
| 监管处罚 | 评估处罚影响，制定应对措施 | 标 `[强制升级 — 重大违规]`；须升级至律师 |

**法规跟踪**：

| 跟踪内容 | 跟踪方式 | 评估要求 |
|---------|---------|---------|
| AI 治理法规变化 | 关注网信办官网·行业协会通知 | 评估新规对备案义务的影响 |
| 备案系统要求变化 | 关注备案系统公告 | 评估是否需要更新备案信息 |
| 执法案例 | 关注监管机关发布的执法案例 | 评估企业是否存在类似风险 |
| 行业标准更新 | 关注 AI 安全评估相关标准更新 | 评估是否需要调整安全措施 |

### 步骤 6：持续合规管理

**持续合规管理框架**：

| 管理维度 | 管理内容 | 管理频率 |
|---------|---------|---------|
| 备案信息管理 | 确保备案信息始终与实际一致 | 持续 |
| 算法安全管理 | 确保算法安全措施持续有效 | 持续 |
| 用户权益保障 | 确保用户选择权·知情权持续保障 | 持续 |
| 内容安全管理 | 确保推荐内容持续合规 | 持续 |
| 数据安全保护 | 确保用户数据安全持续保障 | 持续 |
| 法规合规跟踪 | 确保始终符合最新法规要求 | 持续 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 算法重大变更 | 重新评估备案义务，可能需要重新进行算法类型识别并补充或重新备案 `[待验证]` |
| 监管整改通知 | 升级为被动应对模式，按整改要求制定整改方案并限期完成 |
| 企业合并/分立 | 评估备案义务是否转移，可能需要注销备案并重新备案 `[待验证]` |
| 算法推荐服务被责令暂停 | 立即执行暂停要求，评估整改方案，标 `[强制升级 — 重大违规]` |
| 算法推荐服务终止 | 启动注销备案流程，确保用户权益妥善处理 |
| 法规发生重大变化 | 评估新规对备案义务的影响，必要时调整备案信息与合规措施 |

## 错误处理

| 错误/敏感情形 | 处理方式 |
|------------|---------|
| 被监管处罚 | 标 `[强制升级 — 重大违规]`，联系执业律师 |
| 行刑衔接 | 标 `[强制升级 — 行刑衔接]`，联系执业律师 |
| 逾期未变更 | 评估逾期风险，尽快完成变更备案，标 `[合规风险 — 逾期]` |
| 信息不足 | 列出需补充的信息清单，标注 `[待确认]` |
