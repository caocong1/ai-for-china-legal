# 身份与企业画像访谈 — AI 治理

## 目的

通过结构化问卷采集用户身份信息与企业画像，写入 `ai-governance/CLAUDE.md` 的 `## 我们是谁` 和 `## 谁在使用` 两章。

**是否提供AI相关服务与AI服务类型描述是AI治理工作的核心变量**，须重点确认。

## 前置

- 确认 `ai-governance/CLAUDE.md` 存在；不存在则提示编排入口从模板创建。
- 如为增量更新，读取当前 `## 我们是谁` 与 `## 谁在使用` 章节。

## AI 企业画像扩展字段

### 模型与技术信息

以下字段在用户回答 Q4（是否提供AI相关服务）为「是」时进一步采集：

| 字段 | 说明 | 采集追问 | 对合规工作的影响 |
|------|------|---------|----------------|
| **模型类型** | 自研模型/开源模型微调/第三方API调用/混合 | 您使用的AI模型是自研还是基于第三方？ | 自研模型须关注训练数据合规；第三方API须关注API协议与责任划分 |
| **模型架构** | 大语言模型（LLM）/扩散模型/多模态/推荐模型/其他 | 模型架构类型是什么？ | 不同架构适用不同备案类型与安全评估标准 |
| **部署模式** | 公有云SaaS/私有化部署/混合部署/端侧部署 | AI服务如何部署？ | 部署模式影响数据安全监管要求·数据出境评估 |
| **用户规模** | 日活用户量级（<1万/1-10万/10-100万/100-1000万/>1000万） | 当前用户规模大约是多少？ | 用户规模影响备案义务·安全评估等级·社会动员能力判断 |
| **数据来源类型** | 公开数据集/自有采集数据/授权数据/用户生成内容/合成数据/混合 | 训练数据的主要来源是什么？ | 数据来源直接影响训练数据合规风险评估 |
| **第三方API使用** | 是否调用第三方AI API（如大模型API/图像识别API等） | 是否使用第三方AI服务的API？ | 使用第三方API须明确责任划分·数据处理协议 |
| **数据跨境** | 是否涉及数据跨境传输（训练数据出境/API调用出境等） | AI服务是否涉及数据跨境传输？ | 数据跨境须评估数据出境安全评估义务 `[待验证]` |
| **AI生成内容类型** | 文本/图像/音频/视频/代码/多模态/其他 | AI生成的内容类型有哪些？ | 内容类型影响内容安全评估重点·深度合成备案义务 |

## 企业规模与合规资源评估

| 企业规模 | 员工人数 | 合规团队特征 | 合规工作建议 |
|---------|---------|------------|------------|
| **初创公司** | <50人 | 通常无专职合规团队 | 优先完成必要备案·建立基本制度·借助外部律师 |
| **小型企业** | 50-200人 | 1-2名合规人员或法务兼管 | 建立合规制度框架·分步完成备案 |
| **中型企业** | 200-1000人 | 小型合规团队（2-5人） | 系统化合规建设·配备AI合规专人 |
| **大型企业** | 1000-5000人 | 专职合规部门 | 全面合规体系建设·定期自查·法规跟踪 |
| **大型互联网平台** | >5000人 | 大型合规·法务·安全团队 | 全面合规体系·算法审计·定期安全评估 |

## 访谈流程指引

### 开场话术

```
现在开始第一部分：身份与企业画像。
这部分大约需要10-15分钟，我会问您一些关于您的组织、角色和企业AI应用情况的问题。
您的回答将写入 ai-governance/CLAUDE.md 配置文件，所有 ai-governance 技能会读取这些配置来定制工作。
我们开始吧。
```

### 条件分支逻辑

| 触发条件 | 分支行为 |
|---------|---------|
| Q4 = 「是（生成式AI服务）」 | 进入 AI 技术画像全套问题（Q5a-Q5g） |
| Q4 = 「是（深度合成服务）」 | 进入 AI 技术画像，重点关注深度合成相关字段 |
| Q4 = 「是（算法推荐服务）」 | 进入 AI 技术画像，重点关注算法类型·用户规模 |
| Q4 = 「是（多类型AI服务）」 | 进入 AI 技术画像全套，追问各类型优先级 |
| Q4 = 「否（仅使用AI工具）」 | 跳过 AI 技术画像，仅采集基本信息与企业画像 |
| Q4 = 「待评估」 | 辅助判断：追问具体AI功能描述，帮助确定服务类型 |
| Q9 = 是（涉外） | 追问跨境数据流动·境外实体·境外用户情况 |
| Q10 = 「均未完成」且服务已上线 | 提示备案义务可能已触发，标 P0 关注 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 用户拒绝回答某字段 | 写入 `[PLACEHOLDER]`，不阻塞后续访谈 |
| 增量更新模式 | 仅替换用户确认修改的字段 |
| 用户无法确定AI服务类型 | 根据用户描述辅助判断，标注 `[待评估 — 须进一步确认]` |
| 用户同时是服务提供者与使用者 | 以服务提供者为主，同时记录使用者信息 |
| 企业为AI初创尚未上线 | 标注 `[研发阶段 — 关注上线前合规准备]` |
| 用户描述涉及多种AI服务 | 记录所有类型，追问核心业务与优先级 |
| 用户提供的信息前后矛盾 | 追问澄清，以用户最终确认为准 |
