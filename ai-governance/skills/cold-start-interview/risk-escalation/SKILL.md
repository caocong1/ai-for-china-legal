# 风险与升级访谈 — AI 治理

## 目的

采集AI合规风险等级定义、角色权限、强制升级触发条件，写入 `ai-governance/CLAUDE.md` 的 `## AI 合规风险校准` 和 `## 升级矩阵` 两章。

**重要提示**：行刑衔接强制升级为不可协商护栏——不可通过配置关闭。

## 前置

- 已完成身份与企业画像访谈、合规立场访谈。
- 读取 CLAUDE.md 当前 `## AI 合规风险校准` 与 `## 升级矩阵` 章节。

## 访谈问卷

### 第一部分：风险等级定义

**Q1. 对以下风险等级定义是否有调整？**

| 级别 | 含义 | 典型触发情形 |
|------|------|-------------|
| 🔴 关键 | 重大违规或行刑衔接风险 | 未备案被通知整改·AI服务涉刑事风险·被责令下架 |
| 🟠 高 | 重大合规风险 | 应备案未备案·训练数据来源存疑·内容安全缺失 |
| 🟡 中等 | 合规制度不完善 | 备案信息未更新·分类未完成 |
| 🟢 低 | 合规控制有效 | 已完成备案·训练数据合规·内容安全有效 |

- 追问：是否有行业特殊风险触发情形需纳入？
- 默认：采用现有定义

### 第二部分：强制升级条件

**Q2. 是否了解强制升级触发条件？** `[不可协商护栏]`

以下情形**必须强制升级给执业律师**：
- 被监管机关责令下架·关停服务
- 未备案且已被通知整改或处罚
- 行刑衔接风险（AI服务可能构成刑事犯罪）
- 用户重大权益损害
- 责任人人身自由风险

- 追问：是否有额外内部升级场景？
- 默认：采用法定条件

### 第三部分：备案时限护栏

**Q3. 是否了解备案时限护栏？** `[不可协商护栏]`

以下规则**强制执行**：
- 算法备案须在服务上线前完成
- 深度合成备案须在服务上线前完成
- 整改须在期限内完成
- 不得静默放任时限流逝

## 严重程度分类表

### 综合严重程度评估矩阵

| 维度 | 极低（1分） | 低（2分） | 中（3分） | 高（4分） | 极高（5分） |
|------|-----------|---------|---------|---------|-----------|
| **违规可能性** | 已有完善合规措施 | 合规措施基本到位 | 合规措施有缺陷 | 合规措施严重缺失 | 无任何合规措施 |
| **监管关注度** | 无监管关注 | 行业一般关注 | 行业重点关注 | 已被监管约谈 | 已被立案调查 |
| **影响范围** | 影响可忽略 | 影响少数用户 | 影响部分用户 | 影响大量用户 | 影响不特定公众 |
| **法律后果** | 无法律后果 | 可能被提醒 | 可能被责令改正 | 可能被行政处罚 | 可能被刑事追诉 |
| **声誉影响** | 无声誉影响 | 影响可控 | 可能引发关注 | 可能引发舆情 | 已引发重大舆情 |

### 综合评分与风险等级映射

| 综合评分 | 风险等级 | 处置要求 |
|---------|---------|---------|
| 4–8 分 | 🟢 低 | 记录并定期复查 |
| 9–14 分 | 🟡 中等 | 制定整改计划，限期完成 |
| 15–20 分 | 🟠 高 | 立即启动整改，建议升级执业律师 |
| 21–25 分 | 🔴 关键 | 立即处理，强制升级执业律师 |

> **评分示例**：某企业生成式AI服务已上线但未备案，训练数据来源不明，内容安全机制缺失。
> - 违规可能性：4分（合规措施严重缺失）
> - 监管关注度：3分（行业重点关注）
> - 影响范围：4分（影响大量用户）
> - 法律后果：4分（可能被行政处罚）
> - 声誉影响：3分（可能引发舆情）
> - 综合评分：18分 → 🟠 高风险

## 角色权限矩阵详细说明

### 各角色可处理事项与升级节点

| 角色 | 🟢 低风险 | 🟡 中等风险 | 🟠 高风险 | 🔴 关键风险 |
|------|---------|-----------|---------|-----------|
| 当事人本人（无律师） | 可独立了解流程 | 须升级找律师 | 须升级找律师 | 须升级找律师 |
| 当事人本人（有律师） | 可独立处理 | 可初步梳理，定稿前升级 | 须升级执业律师 | 须升级执业律师 |
| 企业合规官/法务 | 可独立处理 | 可独立处理，建议律师审核 | 可初步分析，定稿前升级 | 须升级执业律师 |
| 助理律师 | 可独立处理 | 可独立处理 | 可初步分析，主办律师定稿 | 须升级主办律师 |
| 主办AI合规律师 | 可独立处理 | 可独立处理 | 可独立处理 | 可独立处理，行刑衔接须合伙人确认 |
| 合伙人/高级律师 | 可独立处理 | 可独立处理 | 可独立处理 | 可独立处理 |

### 升级路由规则

```
识别风险等级
│
├─ 🟢 低风险
│   └─ 按角色权限处理，记录在案
│
├─ 🟡 中等风险
│   ├─ 当事人本人（无律师）→ 升级：建议寻找律师
│   ├─ 当事人本人（有律师）→ 可初步处理，定稿前升级
│   ├─ 企业合规官/法务 → 可独立处理，建议律师审核
│   └─ 律师角色 → 按权限处理
│
├─ 🟠 高风险
│   ├─ 非律师角色 → 强制升级执业律师
│   ├─ 助理律师 → 升级主办律师
│   └─ 主办律师/合伙人 → 可独立处理
│
└─ 🔴 关键风险
    ├─ 所有角色 → 强制升级执业律师
    ├─ 涉及行刑衔接 → 强制升级 + 衔接 criminal-compliance 插件
    └─ 涉及被责令下架 → 强制升级 + 合规时限护栏加强
```

```
现在开始第三部分：风险与升级。

请注意：其中部分内容属于法定护栏，不可通过配置关闭或削弱。这是为了保护您的合规安全。
```

### 不可协商护栏确认话术

```
以下内容为法定护栏，须确认您已了解：

1. 行刑衔接强制升级：当AI服务违规可能构成刑事犯罪（如侵犯公民个人信息罪·帮助信息网络犯罪活动罪等）时，必须强制升级给执业律师。此规则不可关闭。

2. 备案时限护栏：算法备案·深度合成备案须在服务上线前完成。整改须在监管要求的期限内完成。不得静默放任时限流逝。

您是否了解并确认上述规则？
```

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 用户要求关闭行刑衔接强制升级 | **拒绝**，说明法定护栏不可关闭，标 `[护栏保护 — 不可协商]` |
| 用户要求关闭备案时限护栏 | **拒绝**，说明法定护栏不可关闭 |
| 用户要求增加更多升级场景 | **允许**，按额外升级场景模板记录 |
| 用户要求调整风险等级定义 | **允许**，但法定护栏对应的触发情形不可删除 |
| 用户要求降低某场景的风险等级 | 允许调整自定义部分，法定触发情形（行刑衔接等）不可降级 |
| 增量更新 | 法定护栏仅允许补充不允许删除 |
| 用户对强制升级条件有异议 | 解释法律依据，记录异议但不修改护栏 |

## 错误处理

| 错误/敏感情形 | 处理方式 |
|------------|---------|
| 前置访谈未完成 | 提示先完成身份与企业画像·合规立场访谈 |
| 用户提供具体处罚条文 | 标 `[待验证]` |
| 用户要求删除所有升级条件 | 拒绝，法定护栏不可删除，标 `[合规边界 — 拒绝]` |
| 用户混淆风险等级与合规严格度 | 解释区别：风险等级是客观评估，合规严格度是主观偏好 |
