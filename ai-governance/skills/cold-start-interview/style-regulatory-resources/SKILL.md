## AI 治理文书模板清单

### 备案类文书

| 文书名称 | 适用场景 | 风格要求 | 说明 |
|---------|---------|---------|------|
| **深度合成备案申请表** | 深度合成服务备案 | 规范·完整·符合备案系统字段要求 | 按备案系统格式填写，不得遗漏必填项 |
| **算法备案申请表** | 算法推荐服务备案 | 规范·完整·符合备案系统字段要求 | 按备案系统格式填写，算法信息须准确 |
| **生成式AI备案申请表** | 生成式AI服务备案 | 规范·完整·符合备案系统字段要求 | 按备案系统格式填写，模型信息须详实 |
| **算法自评估报告** | 算法备案配套 | 客观·有据·结论明确 | 须包含评估方法·评估过程·评估结论 |
| **算法机制说明** | 算法备案配套 | 技术准确·表述清晰 | 须说明算法原理·运行机制·主要参数 |
| **安全评估报告** | 各类备案配套 | 客观·全面·结论有据 | 须覆盖内容安全·数据安全·用户保护 |
| **拟公示内容** | 算法备案配套 | 通俗易懂·面向用户 | 向用户公示的算法推荐服务基本信息 |
| **变更备案申请表** | 备案信息变更 | 规范·完整 | 说明变更事项·变更原因·变更内容 |

## 各类文书风格偏好详细说明

### 备案报告风格

**默认风格**：规范·完整·符合备案系统要求

| 风格维度 | 具体要求 |
|---------|---------|
| 结构 | 严格按备案系统要求的章节结构组织 |
| 用语 | 使用规范法律用语，避免口语化表达 |
| 技术描述 | 算法·模型描述须技术准确，同时兼顾审查人员理解 |
| 数据引用 | 引用数据须注明来源·统计口径·时间范围 |
| 结论表述 | 结论明确，避免模棱两可的表述 |
| 篇幅 | 完整但不冗余，重点突出 |

### 安全评估报告风格

**默认风格**：客观·有据·术语规范

| 风格维度 | 具体要求 |
|---------|---------|
| 评估方法 | 须明确说明评估方法（文档审查·技术测试·人员访谈等） |
| 评估过程 | 须完整记录评估过程，确保可追溯 |
| 证据引用 | 评估结论须有证据支撑，引用具体材料 |
| 风险表述 | 风险描述须客观，不夸大不缩小 |
| 结论表述 | 评估结论须明确（通过/有条件通过/不通过） |
| AI技术术语 | AI技术术语使用须准确，如模型架构·训练方法·评估指标 |

### 合规意见书风格

**默认风格**：客观·有据·术语规范

| 风格维度 | 具体要求 |
|---------|---------|
| 法律分析 | 须引用具体法律条文，分析法律适用 |
| 事实认定 | 须基于已确认的事实，区分已确认与待确认事实 |
| 风险提示 | 须区分确定性风险与可能性风险 |
| 建议表述 | 建议须具体可行，避免空泛建议 |
| 时效标注 | AI治理法规快速演进，须标注法规时效性 |
| 来源标签 | 须标注法律依据来源（`[法宝]`·`[法规网站]`·`[模型知识 — 需验证]`等） |

### 合规制度文本风格

**默认风格**：规范·可落地

| 风格维度 | 具体要求 |
|---------|---------|
| 条文结构 | 章—条—款—项结构清晰 |
| 用语 | 使用规范制度用语，「应当」「不得」「可以」区分明确 |
| 操作指引 | 关键条文须附操作指引或流程说明 |
| 责任分工 | 须明确各部门·各岗位的职责分工 |
| 时限要求 | 须明确各环节的时限要求 |
| 修订机制 | 须包含制度修订的触发条件与程序 |

|------|---------|
| 用户不确定监管机关 | 根据AI服务类型列出可能的机关，建议用户核实 |
