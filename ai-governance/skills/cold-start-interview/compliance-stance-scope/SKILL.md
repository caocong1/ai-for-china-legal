- 平衡型（重大合规风险标红，一般风险关注整改路径）
- 务实型（仅拦截明显高风险，一般风险关注可行控制措施）

- 追问：保守型重点是什么？如训练数据来源逐一核验/内容安全机制零容忍等。

### 第三部分：监管关系

**Q3. 当前监管关系？**

## 服务立场详细说明与 AI 场景示例

### 立场一：深度合成备案

适用情形：企业提供深度合成服务，包括人脸生成/替换/虚拟数字人/语音合成/图像生成等。

| AI 服务场景 | 是否触发深度合成备案 | 说明 |
|------------|-------------------|------|
| AI 换脸应用 | 是 | 人脸替换属于典型深度合成 `[待验证]` |
| 虚拟数字人直播 | 是 | 虚拟数字人属于深度合成服务范畴 `[待验证]` |
| AI 写真/图像生成 | 是 | 人脸生成/风格化属于深度合成 `[待验证]` |
| 语音克隆/TTS | 是 | 语音合成属于深度合成 `[待验证]` |
| 视频内容审核（不涉及生成） | 否 | 不涉及深度合成服务 |
| AI 文本对话（不涉及图像/音视频） | 否 | 纯文本生成不属于深度合成 |

### 立场二：算法备案

适用情形：企业提供算法推荐服务，包括个性化推送/排序精选/检索过滤/调度决策等。

| AI 服务场景 | 是否触发算法备案 | 说明 |
|------------|----------------|------|
| 信息流推荐算法 | 是 | 个性化推送属于算法推荐服务 `[待验证]` |
| 电商商品推荐 | 是 | 排序精选属于算法推荐服务 `[待验证]` |
| 搜索引擎排序 | 是 | 检索过滤属于算法推荐服务 `[待验证]` |
| 外卖/打车调度 | 是 | 调度决策属于算法推荐服务 `[待验证]` |
| 企业内部OA审批流 | 否 | 非面向公众的算法推荐服务 |
| 纯工具型AI（如翻译工具） | 视情况 | 如涉及个性化推荐则可能触发 |

### 立场三：生成式 AI 合规

适用情形：企业向公众提供生成式 AI 服务，包括大语言模型/图像生成/代码生成等。

| AI 服务场景 | 是否触发生成式AI合规 | 说明 |
|------------|-------------------|------|
| 对话式AI助手（面向公众） | 是 | 向公众提供生成式AI服务 `[待验证]` |
| AI 写作/文案生成工具 | 是 | 文本内容生成属于生成式AI `[待验证]` |
| AI 代码生成（面向开发者） | 是 | 代码生成亦属于生成式AI服务 `[待验证]` |
| AI 图像生成（面向公众） | 是 | 同时触发生成式AI + 深度合成备案 |
| 企业内部AI工具（不面向公众） | 可能不触发 | 须评估是否「向公众提供」`[待验证]` |
| API 提供给B端客户 | 视情况 | 如B端客户再向公众提供，上游可能触发 `[待验证]` |

### 立场四：AI 使用案例分类与合规评估

适用情形：企业已部署或计划部署多种 AI 应用，需全面盘点并分类评估合规风险。

| 评估维度 | 评估内容 |
|---------|---------|
| AI 应用场景盘点 | 列出所有AI应用场景，包括已上线与计划上线 |
| 服务类型识别 | 判断每个场景属于深度合成/算法推荐/生成式AI/其他 |
| 备案义务识别 | 判断每个场景是否触发备案义务 |
| 风险等级分类 | 按应用场景的风险等级进行分类 |
| 合规要求映射 | 将每个场景映射到适用的合规要求 |

### 立场五：混合多类型

适用情形：企业同时涉及上述多种立场。须确定优先级。

| 优先级设置 | 说明 |
|-----------|------|
| 优先级 1（主立场） | 当前最紧迫或最核心的合规工作方向 |
| 优先级 2 | 次重要的合规工作方向 |
| 优先级 3 | 其他合规工作方向 |

> **混合多类型提示**：当多个立场并存时，备案时限护栏按最紧迫的立场执行。例如：深度合成服务已上线但未备案，同时算法备案也在筹备中，则深度合成备案为优先级 1。

## 合规严格度详细说明

### 保守型

适用情形：对合规风险零容忍，一切疑点从严标红并升级。

| 维度 | 保守型处理方式 | 具体示例 |
|------|-------------|---------|
| 训练数据合规 | 每一数据来源须有明确授权或合法依据 | 即使是公开数据集，也须逐一核验授权范围 |
| 内容安全 | 内容安全机制零容忍 | AI生成内容须100%经过安全审核，不允许任何漏审 |
| 备案义务 | 有疑点即视为有义务 | 不确定是否触发备案时，按触发处理 |
| 用户保护 | 最严格的告知与同意 | 所有AI生成内容须显式标识，用户协议须逐项确认 |
| 风险标记 | 一切疑点标红 | 即使风险概率较低，也标为🟠或🔴 |

### 平衡型（默认）

适用情形：重大合规风险标红，一般风险关注整改路径。

| 维度 | 平衡型处理方式 | 具体示例 |
|------|-------------|---------|
| 训练数据合规 | 重点数据来源须核验，一般来源可分批核验 | 核心训练数据集须逐一核验，辅助数据集可抽样核验 |
| 内容安全 | 建立内容安全审核机制，关注重大违规 | 高风险内容类型须100%审核，低风险可抽样 |
| 备案义务 | 明确触发须备案，模糊地带建议备案 | 确定不触发的不标红，不确定的是否备案标🟡 |
| 用户保护 | 建立基本告知与投诉机制 | 用户协议·隐私政策·AI生成标识齐备 |
| 风险标记 | 重大风险标红，一般风险关注整改 | 🔴/🟠标红并建议整改路径，🟡/🟢记录并跟踪 |

### 务实型

适用情形：仅拦截明显高风险，一般风险关注可行控制措施。

| 维度 | 务实型处理方式 | 具体示例 |
|------|-------------|---------|
| 训练数据合规 | 明显侵权数据须排除，一般风险可逐步整改 | 已知未授权数据须排除，来源不明的可先使用后补授权 |
| 内容安全 | 建立基本内容安全机制 | 关键词过滤 + 人工审核兜底 |
| 备案义务 | 明确触发须备案，模糊地带评估后决定 | 不确定是否触发时，先评估再决定是否备案 |
| 用户保护 | 满足法定最低要求 | 基本告知 + 投诉渠道 |
| 风险标记 | 仅明显高风险标红 | 🔴标红，🟠视情况标红，🟡/🟢记录 |

## 监管关系详细说明

### 主动合规

特征：企业在未被监管机关采取任何措施的情况下，主动开展合规建设。

| 工作侧重 | 说明 |
|---------|------|
| 备案义务识别 | 主动盘点AI服务，识别备案义务 |
| 合规制度建设 | 主动建立安全管理制度·内容安全机制 |
| 安全评估 | 主动开展安全评估，确保服务上线前合规 |
| 法规跟踪 | 持续跟踪AI治理法规变化，及时调整合规措施 |

### 被动应对

特征：企业已收到监管机关的整改通知·行政处罚·约谈等。

| 工作侧重 | 说明 |
|---------|------|
| 整改时限识别 | **立即识别**监管通知送达日期，强标整改期限 |
| 整改方案制定 | 在整改期限内完成合规整改 |
| 监管配合 | 配合监管机关的检查·问询·现场核查 |
| 合规时限护栏加强 | 所有合规时限护栏自动加强，不得静默放任 |

> **被动应对自动触发**：识别监管关系为「被动应对」时，以下规则自动生效：
> 1. 合规时限护栏加强——所有备案·整改时限标红
> 2. 升级阈值降低——一般风险即可触发升级
> 3. 输出标注加强——所有输出增加时效性与紧迫性提示
