# 应用场景盘点（use-case-inventory）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`，加载企业画像与AI服务类型。

---

## 目的

全面盘点企业所有 AI 应用场景，建立 AI 应用场景台账，为后续风险分类与合规映射提供基础。

---

## 详细步骤

### 步骤 1：AI 应用场景分类框架

| 场景大类 | 场景子类 | 典型应用 |
|---------|---------|---------|
| **内容生成** | 文本生成 | AI写作·AI对话·翻译·摘要 |
| **内容生成** | 图像生成 | AI绘图·图像编辑·设计辅助 |
| **内容生成** | 音频生成 | 语音合成·音乐生成·配音 |
| **内容生成** | 视频生成 | AI视频·虚拟数字人·直播 |
| **推荐决策** | 个性化推荐 | 信息流推荐·商品推荐·广告推送 |
| **推荐决策** | 排序搜索 | 搜索排序·内容排序·热搜 |
| **推荐决策** | 调度决策 | 派单·定价·资源调度 |
| **自动化处理** | 文档处理 | OCR·合同审查·文档分类 |
| **自动化处理** | 客服交互 | 智能客服·AI助手 |
| **自动化处理** | 代码辅助 | AI编程·代码审查 |
| **数据分析** | 预测分析 | 风控预测·销售预测·趋势分析 |
| **数据分析** | 画像分析 | 用户画像·行为分析 |
| **生物识别** | 人脸识别 | 身份验证·安防·考勤 |
| **生物识别** | 语音识别 | 语音输入·声纹验证 |

### 步骤 2：应用场景逐一盘点

对每个 AI 应用场景收集以下信息：

| 信息项 | 内容 |
|--------|------|
| 场景名称 | 该 AI 应用场景的名称 |
| 场景描述 | 功能描述·应用方式 |
| 场景大类 | 按上述分类框架归类 |
| 服务对象 | ToC/ToB/内部 |
| 用户规模 | 大致用户数量 |
| 数据类型 | 处理的个人信息·敏感信息·重要数据 |
| 上线时间 | 已上线/计划上线/研发中 |
| 负责部门 | 负责该场景的业务部门 |
| 现有合规状态 | 已备案/备案中/未评估 |

### 步骤 3：AI 应用场景台账建立

将所有场景汇总建立台账：

```markdown
## AI 应用场景台账

| 序号 | 场景名称 | 场景大类 | 服务对象 | 数据类型 | 上线状态 | 合规状态 |
|------|---------|---------|---------|---------|---------|---------|
| 1 | [名称] | [大类] | [对象] | [数据] | [状态] | [合规] |
```

### 步骤 4：遗漏风险识别

检查是否有遗漏的 AI 应用场景：
- 各部门是否有独立引入的 AI 工具
- 是否有通过第三方服务间接使用的 AI
- 是否有研发中的 AI 项目尚未上线

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 无法获取完整AI应用信息 | 标注已知场景，标注信息缺失部分 |
| 第三方AI服务使用 | 标注第三方服务场景，评估间接合规义务 |
