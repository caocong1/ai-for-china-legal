# 合规要求映射（compliance-requirement-mapping）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`。
> 2. 已完成风险等级分类。
> 3. 加载共享资源：`_shared/legal-basis-conventions.md` + `_shared/ai-law-citations.md`。

---

## 目的

将每个 AI 应用场景映射到适用的合规要求，包括备案义务·安全评估·数据保护·内容安全·用户保护等，输出「场景—合规要求」映射表与合规差距分析。

**映射原则**：
- **穷尽映射**：每个场景须遍历所有合规要求类别，不得遗漏
- **规则描述**：合规要求一律描述规则 + 验证标注，不臆造条号
- **多规章交叉**：一个场景可能同时触发多个规章的合规义务，须逐一列示
- **动态更新**：法规快速演进，映射结论须定期复核

### 步骤 3：合规优先级评分方法

按以下评分方法确定每个合规差距的优先级：

| 优先级 | 评分规则 | 处置时限 | 说明 |
|--------|---------|---------|------|
| **P0 — 立即处置** | 满足以下任一条件：① 应备案未备案且已上线 ② 被监管机关通知整改 ③ 涉及行刑衔接风险 ④ 内容安全机制完全缺失 | 立即（24小时内启动） | 存在行政处罚·服务下架·刑事追诉的现实风险 |
| **P1 — 限期处置** | 满足以下任一条件：① 高风险场景的合规差距 ② 备案信息严重不完整 ③ 个人信息保护存在重大缺陷 ④ 安全评估未开展 | 15个工作日内 | 合规义务明确且风险较高 |
| **P2 — 计划处置** | 满足以下任一条件：① 较高风险场景的合规差距 ② 合规制度不完善但无紧迫风险 ③ 透明度义务未履行 | 30个工作日内 | 合规义务明确但风险可控 |
| **P3 — 持续改进** | 满足以下任一条件：① 中等/低风险场景的合规优化 ② 合规制度完善性提升 ③ 最佳实践对标 | 纳入季度合规计划 | 合规控制基本有效，优化提升 |

## 映射操作流程

### 映射输入要求

| 输入项 | 来源 | 必须/可选 | 说明 |
|--------|------|---------|------|
| AI 应用场景台账 | use-case-inventory 子技能输出 | 必须 | 场景基础信息 |
| 风险等级分类结果 | risk-level-classification 子技能输出 | 必须 | 影响优先级评分 |
| 企业画像 | CLAUDE.md | 必须 | 行业·AI服务类型·备案状态 |
| 合规严格度 | CLAUDE.md | 必须 | 影响映射范围宽严 |
| 已完成备案情况 | CLAUDE.md | 可选 | 影响合规差距判断 |

### 映射工作流

```
1. 加载输入（台账 + 风险分类 + CLAUDE.md 配置）
   │
2. 建立合规要求分类目录（步骤1）
   │
   ├─ AI 治理专项备案义务
   ├─ 安全评估义务
   ├─ 数据保护义务
   ├─ 内容安全与标识义务
   └─ 用户权益与透明度义务
   │
3. 逐场景填写合规映射矩阵（步骤2）
   │
   ├─ 遍历16项合规要求
   ├─ 判断是否适用（是/否/待判断）
   ├─ 记录适用依据
   ├─ 评估当前状态
   └─ 识别合规差距
   │
4. 按优先级评分方法确定优先级（步骤3）
   │
   ├─ P0：立即处置
   ├─ P1：限期处置（15工作日）
   ├─ P2：计划处置（30工作日）
   └─ P3：持续改进（季度计划）
   │
5. 逐场景输出合规差距分析（步骤4）
   │
6. 识别跨场景共性差距（步骤5）
```

### 映射结果校验

| 校验项 | 校验规则 | 不通过处理 |
|--------|---------|----------|
| 所有场景均已映射 | 台账中每个场景都有映射矩阵 | 返回补充映射 |
| 每项合规要求均已遍历 | 16项合规要求逐一判断 | 补充遗漏项 |

| 备案义务判断准确 | 与风险分类中的备案义务评分一致 | 修正偏差 |
| 行刑衔接已识别 | 涉及刑事风险的已标 P0 | 立即升级 |
| 所有映射有验证标注 | 标 `[待验证]` | 补充标注 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 合规要求不明确 | 标注 `[监管规则尚不明确 — 须持续关注]`，列示可能的适用方向 |
| 多规章交叉适用 | 逐一列示适用要求，说明各规章的触发条件与义务差异 |
| 新型 AI 应用场景无明确规章对应 | 标注 `[新型场景 — 映射参考最接近类型]`，从严映射 |
| 地方性 AI 法规 | 标注 `[地方 flagged]`，与中央级规章分别列示 |
| 临时模式运行 | 标 `[临时模式]`，映射范围可能偏宽（列示所有可能适用的规章） |
| 法规更新导致映射变化 | 标注 `[时效性 flagged — AI 治理法规快速演进，须核实现行版本]` |
| 用户对映射结论有异议 | 解释映射依据，允许用户提供补充信息后重新评估 |
| 合规要求之间存在冲突 | 列示冲突要求，建议从严适用并标注 `[法规冲突 — 须专业判断]` |

## 错误处理

| 错误/敏感情形 | 处理方式 |
|------------|---------|
| 行刑衔接 | 标 `[强制升级 — 行刑衔接]`，暂停映射输出，优先处理升级 |
| 规避合规需求 | 拒绝，标 `[合规边界 — 拒绝]` |
| 要求缩小映射范围以规避义务 | 拒绝，标 `[合规边界 — 拒绝]`，映射须穷尽所有可能适用的要求 |
| 提供虚假合规状态 | 标注 `[用户提供信息 — 须核实]`，不替代独立核实 |
