# 用户保护与备案管理（user-protection-filing）

> **前置**：
> 1. 读取 `ai-governance/CLAUDE.md`。
> 2. 已完成服务类型识别·训练数据合规·内容安全机制建设。

---

## 目的

建设生成式AI服务的用户权益保护机制，完成生成式AI服务备案，包括用户知情权保障、投诉举报机制、备案材料准备与提交。

---

## 详细步骤

### 步骤 1：用户知情权保障

| 告知事项 | 告知方式 | 说明 |
|---------|---------|------|
| **AI服务说明** | 服务页面·用户协议 | 告知用户正在使用AI生成服务 |
| **算法基本原理** | 适当方式公示 | 公示生成式AI的基本原理与运行机制 |
| **生成内容说明** | 内容标识·免责声明 | 说明内容为AI生成，可能存在不准确 |
| **个人信息处理** | 隐私政策 | 说明个人信息收集使用规则 |

### 步骤 2：用户选择权保障

| 权利 | 保障措施 |
|------|---------|
| **关闭权** | 提供关闭AI生成服务的选项（如适用）`[待验证]` |
| **反馈权** | 提供对生成内容的反馈渠道 |
| **投诉权** | 建立投诉举报机制 |

### 步骤 3：投诉举报机制建设

| 机制要素 | 建设要求 |
|---------|---------|
| **投诉渠道** | 提供便捷的投诉入口（电话·邮箱·在线） |
| **处理时限** | 建立投诉处理时限承诺 |
| **处理流程** | 受理→分类→处理→反馈→归档 |
| **记录保存** | 保存投诉记录与处理结果 |

### 步骤 4：备案材料准备

| 材料类别 | 材料名称 |
|---------|---------|
| **主体信息** | 企业基本信息 |
| **服务信息** | 生成式AI服务说明 |
| **技术信息** | 模型信息·训练数据概况 |
| **合规材料** | 安全评估报告·内容安全方案·用户保护方案 |
| **管理制度** | 安全管理制度·应急管理制度 |

### 步骤 5：安全评估

生成式AI服务上线前须开展安全评估：

| 评估维度 | 评估内容 |
|---------|---------|
| 内容安全 | 生成内容合规性评估 |
| 数据安全 | 训练数据·用户数据保护评估 |
| 用户保护 | 用户权益保障评估 |
| 应急处置 | 应急响应能力评估 |

### 步骤 6：备案提交

通过备案系统完成提交，跟踪审核进度。
