# 标准合同备案

> **可单独触发**：本子技能可脱离 cross-border-assessment 编排入口独立运行。
> 斜杠命令：`/data-compliance:standard-contract`
>
> **前置依赖**：
> - 读取 `data-compliance/CLAUDE.md` → 获取：CIIO 身份、数据处理者类型、跨境政策、首选出境路径、文书风格

## 目的

依据《个人信息出境标准合同办法》（2023年，网信办）及《个人信息保护法》（2021年）第38条第三路径，指导数据处理者完成：(1) 适用条件确认，(2) 标准合同范本核心条款审查，(3) 跨境前个人信息保护影响评估（PIA），(4) 向省级网信部门备案。

---

## 详细步骤

### 步骤一：适用条件判断

通过标准合同路径出境个人信息，须同时满足以下全部适用条件（**以《个人信息出境标准合同办法》最新版为准 `[待验证 — 条件与门槛数值随办法修订变动，须核实最新版]`**）：

| 适用条件 | 判断要素 | 结论 |
|---------|---------|------|
| **非关键信息基础设施运营者** | 数据处理者未被认定为 CIIO（来源：CLAUDE.md 或行业主管部门认定） | 是/否 |
| **处理个人信息未达强制安全评估规模** | 处理的个人信息总量未超过安全评估强制申报门槛（**门槛数值 `[待验证]`**） | 是/否 |
| **出境个人信息数量未超门槛** | 本次及历史出境个人信息累计数量未超过标准合同适用门槛（**累计出境人数门槛 `[待验证 — 数值随办法修订变动，须核实最新版标准合同办法]`**） | 是/否 |
| **出境敏感个人信息数量未超门槛** | 出境敏感个人信息人数未超过规定门槛（**数值 `[待验证]`**） | 是/否 |

> **⚠️ 重要合规警告**（标 🔴）：不得通过拆分业务或分批出境的方式规避数量门槛。如经累计计算超过适用门槛，须升级至路径①安全评估申报。蓄意拆分规避属违规行为。

**适用判断结论**：
- 全部条件满足 → 可适用标准合同路径；继续步骤二
- 任一条件不满足（尤其是 CIIO 身份或规模超门槛）→ 本路径不适用；须转路径①安全评估 (`security-assessment-filing`)
- 规模门槛不确定 → 临时模式：保守处理，建议向监管机构咨询或直接申报路径①

### 步骤二：标准合同范本核心条款审查

须使用网信办发布的标准合同范本（**具体范本格式以最新版标准合同办法为准，不得随意修改必备条款 `[待验证 — 须核实最新版范本文本]`**）。

**标准合同必须包含的核心条款对照表**：

| 条款类型 | 主要内容 | 审查要点 | 合规状态 |
|---------|---------|---------|---------|
| **处理者义务** | 数据处理者（境内）须对向境外接收方提供个人信息承担责任；须监督接收方按约定处理数据 | 是否保留对接收方的监督权 | ☐ |
| **接收方义务** | 境外接收方须按合同约定处理个人信息；须采取与中国法律法规要求相当的安全措施 | 义务范围是否覆盖全部个人信息处理活动 | ☐ |
| **个人作为第三方受益人** | 个人有权作为合同第三方受益人向接收方主张权利（查阅、复制、更正、删除）；接收方须建立个人权利行使机制 | 个人权利行使渠道是否明确（联系方式、响应时限） | ☐ |
| **接收方所在地数据保护情况** | 须在合同中说明接收方所在国/地区的数据保护法律情况，以及对实现合同约定目的的影响 | 是否客观披露，不得虚假陈述法律环境 | ☐ |
| **违约责任** | 接收方违约的责任承担；损害赔偿机制 | 是否有可执行的违约责任条款 | ☐ |
| **争议解决** | 争议解决方式（仲裁/诉讼）及适用法律 | 建议约定适用中国法律、在中国解决争议（保障个人权利可执行性） | ☐ |
| **数据安全事件处理** | 接收方须在发生数据安全事件时及时通知数据处理者，并协助处置 | 通知时限是否合理；协助义务是否具体 | ☐ |
| **合同终止后数据处理** | 合同终止/期满后，接收方须删除或退回个人信息的义务 | 删除/退回时限是否明确 | ☐ |

**合同定制条款**（标准范本框架内可附加协商内容）：

| 可附加事项 | 建议要点 |
|---------|---------|
| 数据转移限制 | 接收方不得将数据转移给第三方（或须经数据处理者书面同意） |
| 再转移条件 | 若允许再转移，须约定再转移须满足与标准合同同等保护水平 |
| 安全审计权 | 数据处理者有权对接收方进行合规审查或要求第三方审计 |
| 境外数据存储时限 | 约定接收方只在必要期限内存储数据 |

### 步骤三：跨境前个人信息保护影响评估（PIA）

《个人信息出境标准合同办法》要求在签订标准合同**之前**完成 PIA，评估报告须作为备案材料之一提交（**具体要求以最新版办法为准 `[待验证]`**）。

| PIA 评估内容 | 评估结论 |
|------------|---------|
| 出境处理活动的目的/方式是否合法正当必要 | — |
| 对个人权益的影响及安全风险（含敏感个人信息影响） | — |
| 接收方所在地数据保护法律政策对实现合同义务的影响 | — |
| 接收方承担合同义务的能力（技术能力/法律能力） | — |
| 数据主体权利在境外的可执行性 | — |
| 保护措施是否与风险程度相适应 | — |

**PIA 报告结论**（选择一项）：
- 风险可控，保护措施充分，符合通过标准合同出境条件 → 继续步骤四
- 存在可整改风险，整改后可出境 → 列出整改项，完成整改后继续
- 风险较高，当前不建议出境 → 升级处理或暂停出境计划

**PIA 报告留存**：评估报告和处理情况记录须至少保存三年（PIPL 第56条，已核实）。

### 步骤四：标准合同签订与备案

**签订顺序（严格遵守）**：
1. 完成 PIA 并形成评估报告
2. 与境外接收方签订标准合同（含必备条款，可含附加协商条款）
3. 向省级网信部门提交备案（**备案时限 `[待验证 — 须核实最新版标准合同办法]`**）

**备案材料框架**（**具体清单以最新版办法为准 `[待验证]`**）：

| 备案材料 | 内容 | 准备状态 |
|---------|------|---------|
| 备案申请材料（表格/说明） | 数据处理者基本信息、出境概况 | ☐ |
| 签订的标准合同 | 含全部必备条款的合同正式版本 | ☐ |
| 个人信息保护影响评估报告 | PIA 报告全文 | ☐ |
| 其他须提交材料 | **具体要求 `[待验证 — 以最新版办法及省级网信部门要求为准]`** | ☐ |

**备案注意事项**：
- 备案须在合同签订后按规定时限完成（**时限 `[待验证]`**），不得无限期拖延
- 备案后如出境情况发生重大变化，须重新评估并按需更新备案
- 备案材料须真实完整，不得提供虚假材料

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 累计出境数量超过标准合同适用门槛 | 路径③不适用（🔴）；须升级至路径①安全评估申报（`security-assessment-filing`） |
| 通过分批出境拆分数量规避门槛 | 属违规行为（🔴）；累计数量须合并计算 |
| 接收方不接受标准合同必备条款 | 视情况：必备条款不可协商删除（🔴）；可通过附加条款协商其他事项；必备条款缺失则不得以标准合同路径出境 |
| 合同签订后出境情况发生重大变化 | 需重新评估是否仍满足适用条件；如超出原合同范围须更新合同并重新备案 |
| 备案遭拒或省级网信部门要求整改 | 暂停出境活动；按要求整改后重新提交备案 |
| 出境目的地国/地区数据保护法律发生重大变化 | 重新开展 PIA；评估原标准合同义务是否仍可执行 |
| 数据出境范围超出原备案范围 | 须重新评估并补充备案（不得默许超范围出境） |

## 错误处理

| 错误类型 | 触发情形 | 处理方式 |
|---------|---------|---------|
| 适用门槛不确定 | 数据量难以准确统计，无法确认是否超门槛 | 临时模式：按超门槛处理；建议专业机构协助数据盘点；同时保守准备路径①材料 |
| 标准合同版本过时 | 使用的范本非最新版 | 以网信办官网最新版为准；已签订旧版合同须评估是否须更新 |
| PIA 未在签订合同前完成 | 先签合同后做 PIA | 属程序违规；须补开 PIA，如 PIA 发现高风险问题则须重新评估合同或暂停出境 |
| 备案超时 | 未在规定时限内完成备案 | 属合规缺口（🟠/🔴）；立即提交并说明延迟原因；视程度决定是否暂停出境 |
| 接收方发生数据安全事件 | 接收方所在地发生影响个人信息的事件 | 立即启动合同约定的事件处置流程；评估是否影响出境合规性；必要时暂停出境 |
