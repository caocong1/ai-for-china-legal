# 出境路径判定

> **可单独触发**：本子技能可脱离 cross-border-assessment 编排入口独立运行。
> 斜杠命令：`/data-compliance:pathway-determination`
>
> **前置依赖**：
> - 读取 `data-compliance/CLAUDE.md` → 获取：是否为 CIIO、数据处理者类型、跨境政策、首选出境路径、重要数据出境立场
> - 引用 `data-compliance/skills/_shared/data-protection-citations.md` → 第38-40条三路径与跨境单独同意/影响评估
> - 引用 `data-compliance/skills/_shared/legal-basis-conventions.md` → 来源标签与待验证规范
>
> **配置缺失处理**：若 `data-compliance/CLAUDE.md` 中「是否为 CIIO」「跨境政策」为 `[PLACEHOLDER]`，启用临时模式：
> - 临时模式：安全评估门槛按最保守处理（凡不确定是否触发强制安全评估，建议申报）
> - 建议用户运行 `/data-compliance:cold-start-interview` 完善配置
>
> **分工说明**：本子技能负责路径判定与出境数据初步识别；安全评估申报流程见 `security-assessment-filing`；标准合同签订与备案见 `standard-contract`；保护认证及最终合规清单见 `cross-border-assembly`。

---

## 目的

根据《个人信息保护法》（2021年）第38条（三路径）、第39条（跨境单独同意）、第40条（CIIO/达量处理者数据本地化+强制安全评估），结合第55条第(四)项+第56条（跨境前 PIA 义务）及数据出境安全评估办法（2022年）等相关规定，对拟出境数据进行分类识别，判定适用的合规路径（安全评估申报 / 保护认证 / 标准合同），并确认跨境单独同意与跨境前个人信息保护影响评估的完成义务。

---

## 详细步骤

### 步骤一：出境数据识别与分类

识别本次出境场景中涉及的所有数据类型，填入下表：

| 数据类型 | 示例 | 是否个人信息 | 是否敏感个人信息 | 是否重要数据 | 数据量级（预估） |
|---------|------|------------|----------------|------------|----------------|
| 员工信息 | 姓名、工号、职位、薪酬 | 是 | 否/是（薪酬） | 待识别 | — |
| 用户行为数据 | 浏览记录、消费记录 | 是 | 否 | 待识别 | — |
| 生物识别数据 | 人脸数据、指纹 | 是 | **是**（PIPL第28条） | 待识别 | — |
| 行踪轨迹数据 | GPS 轨迹、出行记录 | 是 | **是**（PIPL第28条） | 待识别 | — |
| 运营数据/业务数据 | 交易量、订单数据 | 部分 | — | **重点排查** | — |
| 技术数据 | 代码、算法参数 | 否 | — | **重点排查** | — |

**重要数据排查提示**：重要数据目录由各行业、各地区主管部门制定，须核查适用目录。`[行业监管 flagged — 需核实主管部门规定]`。若行业目录尚未公布或不确定，按保守标准推定（即：可能对国家安全、公共利益有影响的数据，先纳入重要数据范围排查，再确认）。重要数据出境须转入 `data-security-compliance/important-data-handling` 子技能联合评估。

### 步骤二：数据处理者身份判断

| 判断项目 | 结果 | 依据/说明 |
|---------|------|---------|
| 是否为关键信息基础设施运营者（CIIO） | 是 / 否 / 不确定 | 读取 data-compliance/CLAUDE.md；公共通信/能源/交通/水利/金融/公共服务/电子政务等重要行业领域 |
| 是否为处理个人信息达到规定数量规模的处理者 | 是 / 否 / 不确定 | 处理个人信息数量是否达到强制申报安全评估的门槛（**具体数量门槛 `[待验证 — 数值随办法修订变动，须核实最新版]`**） |
| 是否处理重要数据 | 是 / 否 / 待确认 | 结合步骤一排查结果 |

**判断结论**（从以下三种情形之一选择）：

- **情形A：强制安全评估** — CIIO 出境个人信息或重要数据；或处理个人信息达到规定数量门槛 `[待验证]`；或出境重要数据
- **情形B：可选安全评估、保护认证或标准合同** — 非 CIIO 且规模未达强制门槛，三路径可选（须结合路径适用条件判断）
- **情形C：数据中存在重要数据需单独处理** — 重要数据部分强制安全评估，个人信息部分另行判断路径

### 步骤三：三路径适用条件对照

依据《个人信息保护法》（2021年）第38条（**已核实锚点，建议复核**）：

| 路径 | 适用条件 | 主要场景 | 核心要求 | 参考下游子技能 |
|------|---------|---------|---------|--------------|
| **① 安全评估（政府申报）** | 强制：CIIO 出境个人信息或重要数据；处理个人信息达到规定数量；出境重要数据。**具体数量门槛 `[待验证 — 须核实最新版评估办法]`** | 大型平台、CIIO、处理重要数据的企业 | 向网信办申报，须通过政府安全评估 | `security-assessment-filing` |
| **② 保护认证（专业机构认证）** | 非强制申报情形；主要适用于集团内部跨境数据流转等特定场景。**具体适用条件 `[待验证 — 须核实认证规定最新版]`** | 跨国集团内部数据流转 | 经国家认监委认可机构取得认证 | `cross-border-assembly`（认证路径汇总） |
| **③ 标准合同（与接收方订立）** | 非 CIIO；处理个人信息未达强制申报门槛；出境个人信息数量未超过一定门槛。**具体门槛数值 `[待验证 — 数值随办法修订变动，须核实最新版标准合同办法]`** | 中小规模对外数据共享、SaaS 等 | 签订网信办标准合同范本 + 备案 + PIA | `standard-contract` |
| **④ 其他法律行政法规或网信办规定的条件** | 视具体法律行政法规或网信部门另行规定 | 特殊行业/场景 | 须核实相关专项规定 | — |

**路径选择决策树**：

```
是否为 CIIO？
├─ 是 → 安全评估申报（路径①强制）→ 转 security-assessment-filing
└─ 否
    ├─ 本次出境是否包含重要数据？
    │   ├─ 是 → 安全评估申报（路径①强制）→ 先确认重要数据范围（data-security-compliance/important-data-handling），再转 security-assessment-filing
    │   └─ 否（纯个人信息出境）
    │       ├─ 处理个人信息是否达到规定数量门槛？[待验证]
    │       │   ├─ 是 → 安全评估申报（路径①强制）→ 转 security-assessment-filing
    │       │   └─ 否
    │       │       ├─ 出境个人信息数量是否超过标准合同适用门槛？[待验证]
    │       │       │   ├─ 是 → 路径①安全评估 或 路径②保护认证（不可标准合同）
    │       │       │   └─ 否 → 路径②保护认证 或 路径③标准合同（按场景选择）
    │       │       └─ 是否为集团内部数据流转？
    │       │           ├─ 是 → 优先考虑路径②保护认证 → 转 cross-border-assembly
    │       │           └─ 否 → 路径③标准合同 → 转 standard-contract
```

> **[临时模式]**：当 CLAUDE.md 配置为 PLACEHOLDER 时，决策树按「是否为 CIIO」和「是否包含重要数据」两项保守判断；规模门槛不确定时，建议安全评估申报（路径①），避免因门槛误判导致合规缺口。

### 步骤四：跨境单独同意确认（PIPL 第39条）

依据《个人信息保护法》（2021年）第39条（**已核实锚点，建议复核**）：

向境外提供个人信息，须取得个人的**单独同意**，并须告知以下事项：

| 须告知事项 | 完成情况 | 落实位置 |
|---------|---------|---------|
| 境外接收方名称/联系方式 | ☐ | 隐私政策/单独同意书 |
| 向境外提供的目的/方式 | ☐ | 隐私政策/单独同意书 |
| 个人信息种类 | ☐ | 隐私政策/单独同意书 |
| 境外接收方保存期限 | ☐ | 隐私政策/单独同意书 |
| 个人向境外接收方行使权利的方式和途径 | ☐ | 隐私政策/单独同意书 |

**单独同意设计要点**：
- 跨境单独同意不可与其他同意合并，须单独取得
- 告知内容须在取得同意时披露，不得事后补充
- 须提供便捷的撤回同意机制
- 若已处理个人信息但未取得有效跨境单独同意：须暂停出境直至合规完成

### 步骤五：跨境前个人信息保护影响评估（PIPL 第55条第(四)项+第56条）

依据《个人信息保护法》（2021年）第55条第(四)项（**已核实锚点，建议复核**）及第56条（已核实）：

跨境提供个人信息属于须进行 PIA（个人信息保护影响评估）的法定情形。

| PIA 核心评估维度 | 评估要点 |
|--------------|---------|
| 处理目的/方式是否合法正当必要 | 出境目的是否合理；接收方处理活动是否限于必要范围 |
| 境外接收方保护水平 | 接收方所在国/地区数据保护法律环境；接收方内部安全措施 |
| 再转移风险 | 接收方能否向第三方再次提供数据；再转移约束机制 |
| 对个人权益的影响 | 个人能否在境外有效行使知情/查阅/更正/删除权利 |
| 保护措施是否与风险相适应 | 合同条款、技术措施、组织措施 |

**PIA 报告留存**：评估报告和处理情况记录须**至少保存三年**（PIPL 第56条，已核实）。

**跨境 PIA 与三路径的关系**：
- 路径①安全评估申报：自评估报告包含 PIA 要素，申报材料之一；可与申报准备工作并轨（见 `security-assessment-filing`）
- 路径③标准合同：标准合同办法要求在签订前开展 PIA，评估报告作为备案材料之一（见 `standard-contract`）
- 路径②保护认证：认证评审本身含 PIA 性质评估，但建议同时完成独立 PIA 文档

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 同一出境场景同时含重要数据和个人信息 | 重要数据部分强制路径①安全评估，个人信息部分单独判断路径；需分开处理或合并申报 |
| 重要数据出境，但行业重要数据目录尚未公布 | 按保守标准推定该数据为重要数据，建议路径①安全评估；标注 `[行业监管 flagged — 目录待公布]` |
| CIIO 身份不确定 | 按 CIIO 处理（路径①安全评估）；同时建议向行业主管部门确认关基认定情况 |
| 多次分批出境总量可能超门槛 | 须评估累计出境数量；不得通过分批出境规避强制申报门槛（标 🔴 违规风险） |
| 出境目的地位于数据保护水平存疑的国家/地区 | 提高 PIA 评估要求；路径③标准合同须在合同中加强接收方约束 |
| 数据出境对象为受托处理者（数据处理者角色） | 须确认委托方的出境合规安排；受托处理者自身亦须满足合规要求 |
| 出境数据中包含法律规定须境内存储的数据（如金融数据、健康数据等） | 境内存储义务优先，须先确认存储义务范围，再评估出境可行性；标 🔴 `[行业监管 flagged]` |

## 错误处理

| 错误类型 | 触发情形 | 处理方式 |
|---------|---------|---------|
| 数据类型识别不完整 | 用户提供的信息不足以完成全部数据类型识别 | 暂停路径判定；列出待补充信息清单，要求用户补充数据地图或处理活动记录 |
| 重要数据范围无法确认 | 行业目录未公布或用户不确定 | 先按保守推定完成路径判定，标 `[行业监管 flagged]`；同步建议启动 `data-security-compliance/important-data-handling` |
| CIIO 身份与处理规模均不确定 | CLAUDE.md 均为 PLACEHOLDER | 启用临时模式，按最保守路径判定（路径①安全评估）；明确标 `[临时模式]`，建议完善配置 |
| 出境数据含不确定是否为个人信息的数据 | 如匿名化数据是否满足 PIPL 匿名化标准存疑 | 不确定数据按个人信息处理；如确需排除，须提供匿名化技术说明及标准对照 |
| 路径判定后用户无法完成后续子技能操作 | 如无资源完成安全评估申报 | 提示升级处理；安全评估申报须由有资质团队操作 |
