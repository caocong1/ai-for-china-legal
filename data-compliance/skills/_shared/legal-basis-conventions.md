# 法律依据引用规范（数据合规版）

> 本文件定义 data-compliance 插件所有技能统一使用的法律依据引用规范。
> 后续计划（05-02、05-03）直接引用本规范，避免各技能重复维护。
> 数据合规法规更新频繁，本规范特别强调时效性标注与保守原则。

---

## 一、来源分类

所有法律依据必须归入以下类别之一，并在引用时明确标注类别标签。

### 1. 法律（全国人大及其常委会制定）

指全国人民代表大会及其常务委员会制定的法律，如：
- 《中华人民共和国个人信息保护法》（PIPL，2021年11月1日施行）
- 《中华人民共和国数据安全法》（DSL，2021年9月1日施行）
- 《中华人民共和国网络安全法》（CSL，2017年6月1日施行）
- 《中华人民共和国民法典》（2021年1月1日施行，第六章个人信息保护）

**法律层级最高，是数据合规分析的核心基础。** PIPL/DSL/CSL 的已核实锚点条号相对稳定，可在标注「建议复核」前提下引用。

**格式要求（法律最小引用格式）：**

```
《[法律全名]》（[施行/修订年份]）第[X]条[第X款]  [来源标签]  [验证状态]
```

示例：
```
《中华人民共和国个人信息保护法》（2021年）第13条  `[模型知识 — 需验证]`  已核实锚点，建议复核
```

### 2. 行政法规（国务院制定）

指国务院制定的行政法规，如：
- 《关键信息基础设施安全保护条例》（2021年9月1日施行，国务院）
- 《网络数据安全管理条例》（**施行状态须核实 `[待验证 — 须核实现行施行状态与条号]`**）

**格式要求：**

```
《[行政法规全名]》（[施行年份]，国务院）第[X]条  `[来源标签]`  `[待验证]`
```

**注意**：行政法规条号相对稳定但仍应标 `[待验证]`，施行状态须确认。

### 3. 部门规章与规范性文件（网信办/工信部/公安部等制定）

指国务院各部门制定的规章或规范性文件，如：
- 《数据出境安全评估办法》（2022年，网信办）
- 《个人信息出境标准合同办法》（2023年，网信办）
- 《个人信息保护认证》相关规定（网信办/国家认监委）
- 《网络安全审查办法》（网信办等多部门）
- 工信部电信行业数据安全相关规定
- 公安部网络安全等级保护相关规定

**数据合规核心纪律（必须遵守）：**

> **网信办规章/办法的具体条号若不确定，一律描述规则并标 `[待验证]`，绝不硬写裸条号。**
> 凡涉及申报材料清单具体项、备案时限、申报期限、阈值数值（如标准合同适用门槛数量）等具体内容，标 `[待验证 — 数值随办法修订变动，须核实最新版]`。
> 这是快速演进的监管领域，每隔一两年可能修订办法。

**格式要求：**

```
《[规章全名]》（[施行年份]，[制定机关]）[描述规则，不写条号]  `[模型知识 — 需验证]`  `[待验证]`
```

示例：
```
《数据出境安全评估办法》（2022年，网信办）规定：向境外提供达到一定规模的个人信息须向网信办申报安全评估，具体申报情形与数量门槛 `[待验证 — 须核实最新版]`  `[模型知识 — 需验证]`
```

### 4. 国家标准（GB/T 系列）

指全国信息安全标准化技术委员会（TC260）等发布的国家标准：
- 个人信息安全规范（GB/T 系列，推荐性标准，`[待验证]`）
- 数据出境安全评估指南（GB/T 系列，`[待验证]`）
- 网络安全等级保护基本要求（等保 2.0，`[待验证]`）
- 其他信息安全领域国家标准

**国家标准核心纪律（必须遵守）：**

> **国家标准的编号（GB/T XXXXX-YYYY）、发布年份、推荐性/强制性区分，一律标 `[待验证]`。**
> 推荐性标准（GB/T）与强制性标准（GB）效力不同，须在引用时说明区别。
> 推荐性标准本身不具有强制约束力，但在监管实践中往往作为合规评估基准。

**格式要求：**

```
[标准名称] GB/T XXXXX-YYYY `[待验证]`（推荐性/强制性 `[待验证]`）  `[模型知识 — 需验证]`
```

### 5. 案例（指导性案例与行政处罚决定）

数据合规领域以法条/行政法规/部门规章为主要依据，案例较少。可参考的案例类型：

- **最高院指导性案例**：数量少，仅作参考，不具判例法约束力
- **监管行政处罚决定**：网信办/公安部等发布的行政处罚决定，可作实践参考，但非判例，且执法标准可能随监管政策变化

**格式要求：**

```
[处罚决定文号或案例编号]  [裁决/处罚要点摘要]  [处罚机关/法院层级]  `[待验证]`
```

**警示**：数据合规案例少，模型回忆的案号与处罚决定文号风险极高，**所有案例引用默认标 `[待验证]`**。

### 6. 学说（学术观点）

学说为辅助参考，存在争议性，数据合规领域理论尚在形成中：

```
[作者姓名]，《[著作/论文名]》，[出版/期刊信息]  "[观点摘要]"  [是否有争议：有/无]  `[待验证]`
```

## 二、来源标签表

引用时，在条文/案例/观点末尾附加来源标签。复用五个标准标签，不新造冲突标签。

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| `[法宝]` | 来自北大法宝等法规数据库 | 检索工具获取的全文 |
| `[法规网站]` | 来自政府官方网站（网信办、全国人大、国务院等） | 官网原始文件 |
| `[网络搜索 — 需验证]` | 来自网络搜索结果 | 需对照原始来源核查 |
| `[模型知识 — 需验证]` | 来自训练数据回忆 | 风险较高，须优先核查；数据合规条文演进快，风险更高 |
| `[用户提供]` | 来自用户提供的文件 | 合规文件、内部政策等 |

**标签使用规则：**
1. 每条引用行**必须且只能**附加一个来源标签
2. 禁止剥离或折叠标签
3. `[模型知识 — 需验证]` 来源在依赖前**必须**对照原始来源检查——数据合规领域尤为重要

## 三、待验证标记规则

以下情形**必须**在引用行末追加 `[待验证]` 并说明验证途径：

| 触发情形 | 说明 | 建议验证途径 |
|---------|------|-------------|
| 来源标签为 `[模型知识 — 需验证]` | 模型回忆的条文号风险高 | 国家法律法规数据库、网信办官网 |
| **网信办规章/办法的任何具体条号** | 条号不确定，规章修订频繁 | 网信办官网 www.cac.gov.cn |
| **国家标准编号与版本** | GB/T 编号、年份、推荐性/强制性须核实 | 全国标准信息公共服务平台 |
| **部门规章阈值数值**（如申报数量门槛、备案时限、评估有效期、日志留存期限） | 具体数值随办法修订变动 | 核实最新版办法全文 |
| 距最后更新超过 6 个月的法规 | 法规可能已修订或废止 | 查询官方数据库最新版本 |
| 用户口述的条款号或阈值 | 可能记忆偏差或引用旧版 | 核查官方法规全文 |
| 所有案例引用（默认） | 数据合规案例少，案号极易错误回忆 | 中国裁判文书网、网信办执法信息 |
| 施行状态不确定的法规 | 部分法规施行状态须核实 | 国家法律法规数据库 |

**待验证标记格式（示例）：**

```
《数据出境安全评估办法》（2022年，网信办）规定申报情形之一：向境外提供重要数据
`[模型知识 — 需验证]`  `[待验证 — 具体条号与申报情形须核实最新版]`
```

## 四、快速演进监管领域时效性强标注

> **数据合规法规是快速演进的监管领域。** 除 PIPL/DSL/CSL 法律层级条号相对稳定外，部门规章/办法/国家标准的具体规则、条号、数值均可能频繁修订。

### 时效性标注规则

| 法规层级 | 时效性要求 |
|---------|----------|
| 法律（PIPL/DSL/CSL） | 已核实锚点条号可用；须说明「建议复核」；超 6 个月须标 `[时效性 flagged]` |
| 行政法规（国务院） | 施行状态须核实；条号标 `[待验证]` |
| **部门规章/办法（网信办等）** | **具体条号一律标 `[待验证]`；建议每季度复核** |
| **国家标准（GB/T）** | **编号、版本、推荐性/强制性一律标 `[待验证]`** |

**时效性 flag 格式：**

```
[引用内容]  `[时效性 flagged — 需验证最新法规版本]`
```

**触发时效性 flag 的情形：**
- 距最后更新超过 6 个月的规章/标准引用
- 涉及网信办办法（更新频繁）
- 涉及处罚金额、申报门槛等具体数值
- 用户询问距上次数据合规审查超过 6 个月的问题

**保守原则（硬性要求）：**

> **不确定即标待验证，不臆造。** 宁可描述规则并标待验证，不可为显示专业性而硬写无把握的条号。数据合规领域错误引用会直接误导出境路径选择、PIA 结论、等保定级等关键合规决策。

## 五、成文法优先与案例定位

### 成文法优先原则

1. **分析基础**：数据合规的所有结论，必须首先援引现行成文法（PIPL/DSL/CSL 及相关法规）
2. **不得以案例替代法条**：即使存在行政处罚决定，也必须同时引用对应法条
3. **法条缺失时**：若无直接法条，须明确说明「无直接成文法规定，参照以下行政执法决定/学说」

### 案例的辅助定位

1. **最高院指导性案例**：数据合规案例极少，仅作参考，标注「最高院指导，参照意义强」
2. **行政处罚决定**：网信办/公安部发布的执法决定可作实践参考，但**须注意执法标准随监管政策变化**，标「行政执法参考，非判例」
3. **明确说明非判例法**：中国采成文法体系，案例不具判例法约束力

### 数据合规案例的特殊定位

- 数据合规案例远少于商事合同/劳动争议
- 行政处罚决定（网信办、公安部等）可作参考，但执法尺度可能随监管优先级变化
- 不确定的处罚决定文号一律标 `[待验证]`，不依赖模型回忆

## 六、行业监管与机构定位

### 多头监管识别

数据合规领域存在多个监管机构，技能须识别主管部门：

| 监管机构 | 主要职能 | 适用领域 |
|---------|---------|---------|
| 国家互联网信息办公室（网信办）| 个人信息保护/数据出境/网络安全统筹 | PIPL/CSL 执法主体 |
| 工业和信息化部（工信部）| 电信和互联网行业数据安全 | 电信/互联网企业 |
| 公安部 | 网络安全等级保护/关基（部分）| 等保测评、网络安全审查 |
| 市场监督管理总局 | 电商/平台经济数据合规 | 电商平台 |
| 中国人民银行/金融监管总局 | 金融数据安全 | 金融机构 |
| 国家卫生健康委员会（卫健委）| 医疗健康数据 | 医疗机构 |
| 地方网信办 | 地方个人信息保护执法 | 地方性事务 |

**行业监管识别标注：**

```
[涉及特定行业时]  `[行业监管 flagged — 需核实主管部门规定]`
```

### 行业特殊规定

涉及以下行业时，须额外核查行业主管部门的特殊数据合规要求：
- **互联网/平台**：工信部/网信办双重监管，用户个人信息保护、算法推荐
- **金融**：人民银行/金融监管总局数据安全规定，金融数据分类分级
- **医疗健康**：卫健委健康医疗数据、基因数据特别保护
- **汽车**：汽车数据安全管理规定（网信办，具体条号 `[待验证]`）
- **教育**：未成年人个人信息特别保护（PIPL 未成年人专章）
