# 文书风格与跨境政策/行业监管访谈（style-cross-border-policy）

> **所属编排**：`data-compliance/skills/cold-start-interview/SKILL.md`（冷启动访谈）
> **遵循契约**：`data-compliance/skills/_shared/practice-profile-schema.md` 字段映射（§2.8-2.9 文书风格/输出/行业监管字段组）
> **写入目标**：`data-compliance/CLAUDE.md` — `## 文书风格` / `## 输出` / `## 共享护栏`（含 `### 行业监管识别`）
> **分工说明**：本子技能收集输出风格配置与行业监管偏好。身份与团队由 `identity-team` 收集（含行业字段）；合规立场（含跨境政策详细配置）由 `compliance-stance` 收集；风险升级规则由 `risk-escalation` 收集。本子技能侧重「如何输出」和「行业主管部门特殊要求」。
> **可单独触发**：`/data-compliance:style-cross-border-policy`（只更新 `## 文书风格` / `## 输出` / `## 共享护栏` 章节）

---

## 目的

通过结构化访谈收集律师/DPO 的工作产物风格偏好和行业监管识别配置，写入 `data-compliance/CLAUDE.md` 对应章节，使所有数据合规技能能够：

1. 按偏好风格生成合规报告/PIA/出境评估报告（结构化清单 vs 叙述式意见书）；
2. 自动识别行业主管部门并触发行业特殊要求标注（`[行业监管 flagged]`）；
3. 按工作产物标题规则生成律师版/非律师版输出；
4. 按监管沟通风格偏好定制与监管机构沟通的语气和格式。

## 访谈问题清单

### 第一部分：合规报告风格（写入 `## 文书风格`）

**Q1. 贵方偏好的合规报告/审查意见格式是什么？**

> 写入：`## 文书风格` → 合规报告风格

可选（可组合）：
- **结构化风险清单 + 整改建议**（适合快速决策，重点突出）
- **律师审查意见书格式**（叙述式，适合正式法律结论场景）
- **表格 + 条目标注**（适合数据合规审查场景，逐项标注合规状态）
- **混合风格**（清单 + 重要项详述）

*追问*：是否有内部合规报告模板？（若有，请描述格式要求）

---

**Q2. 贵方偏好的 PIA/出境评估报告格式是什么？**

> 写入：`## 文书风格` → PIA/出境评估报告风格

说明：PIA 报告有时须提交给管理层或作为安全评估申报材料；出境评估报告可能须作为标准合同备案附件。

可选：
- **符合网信办预期的正式报告格式**（适合申报材料场景，`[待验证网信办对报告格式的具体要求]`）
- **供内部决策使用的简明版本**（管理层审阅）
- **双轨制**：正式版（对外）+ 简明版（内部）

---

**Q3. 贵方的监管沟通风格偏好是什么？**

> 写入：`## 文书风格` → 监管沟通风格

可选：
- **正式书面**：避免推测性表述，强调合规诚意，适合监管问询回复
- **沟通说明式**：条理清晰，主动说明合规措施，适合主动沟通
- **最小化原则**：只提供被问及的信息，不主动披露

---

**Q4. 贵方工作产物存放在哪里？**

> 写入：`## 文书风格` → 工作产物存放位置

可选：
- 企业合规文件管理系统（请说明系统名称）
- 律所 / DMS 文档管理系统
- 共享目录（请说明路径规则）
- 企业微信/钉钉文档

---

**Q5. 贵方的合规报告/工作产物是否包含以下内容？**

> 写入：`## 文书风格` → 辅助内容偏好（新增字段）

| 内容项 | 是否包含 |
|-------|---------|
| 法条引用（引用具体条文号和内容）| 是 / 否 / 是（已核实条文）/ 是（含待验证） |
| 案例/行政处罚决定引用 | 是（须注明待验证）/ 否 |
| 双语版本（中英）| 是 / 否 / 视需求 |
| 审查者注释块（日期/范围/标记项）| 是（默认）/ 简化版 |
| 下一步决策树 | 是（默认）/ 否 |

## 法律事实合理性检查

| 口述内容 | 合理性检查 |
|---------|----------|
| 声称「PIA 报告符合网信办格式要求」| 核实网信办是否发布统一报告格式要求 `[待验证 — 须核实最新规定]` `[前提 flagged — 需验证]` |
| 声称「安全评估申报材料已有标准模板」| 核实网信办申报材料清单（具体清单 `[待验证]`），写入时保留待验证标注 |
| 声称某行业有具体数据合规条款 | 询问来源，标 `[行业监管 flagged — 须核实主管部门规定]` `[前提 flagged — 需验证]` |
| 声称跨境决策无需内部审批 | 提示安全评估申报等须有决策留档，建议建立内部审批流程 |
