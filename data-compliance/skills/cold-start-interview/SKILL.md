# 冷启动访谈（编排入口）

> **深化说明**：本技能已从 v0.1.0 的三部分浅骨架（基本信息/审查立场/文书风格）深化为四个可独立触发的访谈子技能。原有三部分访谈内容已无丢失迁移并深化到对应子技能中。

---

## 目的

通过完整的冷启动访谈，收集数据处理者/律所/DPO 的执业画像、合规立场、风险管理规则和文书偏好，将结果**填充到 `data-compliance/CLAUDE.md`** 对应章节，完成数据合规实践档案初始化。

**访谈结果写入 `data-compliance/CLAUDE.md`**（散文形式，唯一真相来源），而非另存独立 YAML 文件或 JSON 配置。`data-compliance/CLAUDE.md` 是律师/DPO 可直接阅读和编辑的配置文件，为所有数据合规技能提供个性化设置。

---

## 访谈子技能编排顺序

| 步骤 | 子技能 | 收集什么 | 填充目标章节 | 子技能路径 | 单独触发命令 |
|------|-------|---------|------------|----------|------------|
| 1 | **身份与团队** | 组织名称/行业/数据处理者角色/是否 CIIO/个人信息处理规模/是否处理敏感信息/是否有出境场景/合规团队规模/DPO/最大痛点/执业角色/联系人/集成工具 | `## 我们是谁` / `## 谁在使用` / `## 可用集成` | `data-compliance/skills/cold-start-interview/identity-team/SKILL.md` | `/data-compliance:identity-team` |
| 2 | **合规立场** | 整体合规偏好（保守/平衡/进取）/敏感数据政策（处理门槛/单独同意方式/未成年人信息/生物识别立场）/跨境政策（是否允许/首选路径/本地化立场/接收方偏好/重要数据立场）/数据安全立场（分类分级策略/重要数据识别/等保定级倾向）| `## 合规立场`（含 `### 敏感数据政策` / `### 跨境政策` / `### 数据安全立场`）| `data-compliance/skills/cold-start-interview/compliance-stance/SKILL.md` | `/data-compliance:compliance-stance` |
| 3 | **风险与升级** | 风险等级定义（🔴🟠🟡🟢）/升级矩阵（角色权限/升级条件）/自动升级触发条件/监管报告触发规则 | `## 风险校准` / `## 升级矩阵`（含 `### 角色权限表` / `### 自动升级触发条件` / `### 监管报告触发`）| `data-compliance/skills/cold-start-interview/risk-escalation/SKILL.md` | `/data-compliance:risk-escalation` |
| 4 | **文书风格与跨境政策/行业监管** | 合规报告风格/PIA 出境评估报告风格/监管沟通风格/工作产物存放位置/法条引用偏好/行业主管部门特殊要求/跨境决策审批流程/时效性检查频率/工作产物标题 | `## 文书风格` / `## 输出` / `## 共享护栏`（含 `### 行业监管识别`）| `data-compliance/skills/cold-start-interview/style-cross-border-policy/SKILL.md` | `/data-compliance:style-cross-border-policy` |

> **流程说明**：步骤 1 → 步骤 2 → 步骤 3 → 步骤 4，顺序完成完整档案初始化。各子技能可**单独触发**，用于只更新配置的某个部分（如行业主管部门新出规定，只需重跑步骤 4）。每个步骤写入 `data-compliance/CLAUDE.md` 后即时生效，下次技能调用自动读取。

## 访谈目的与完成标准

**完成本次冷启动访谈后，`data-compliance/CLAUDE.md` 将填充以下章节（替换 `[PLACEHOLDER]`）：**

- `## 我们是谁`：组织名称/行业/数据处理者角色/CIIO 状态/个人信息规模/是否处理敏感信息/是否有出境/合规团队/DPO/痛点
- `## 谁在使用`：执业角色/律师联系人
- `## 可用集成`：各集成工具状态
- `## 合规立场`：整体合规偏好 + 敏感数据政策 + 跨境政策 + 数据安全立场
- `## 风险校准`：风险等级表（组织定义版）
- `## 升级矩阵`：角色权限表 + 自动升级触发条件 + 监管报告触发规则
- `## 文书风格`：合规报告风格/PIA 报告风格/监管沟通风格/工作产物存放位置
- `## 输出`：工作产物标题（律师版/非律师版）
- `## 共享护栏` → `### 行业监管识别`：行业主管部门特殊要求表
- `## 共享护栏` → `### 时效性触发`：时效性检查频率调整

## 重跑/部分更新/直接编辑说明

### 重跑完整访谈

```
/data-compliance:cold-start-interview
```

顺序运行全部四个访谈子技能，覆盖全部章节。适用于：首次初始化/组织/团队重大变化后全面更新配置。

### 部分更新（只更新某个部分）

| 更新需求 | 触发命令 |
|---------|---------|
| 只更新组织画像（新行业/新 CIIO 状态/团队变化）| `/data-compliance:identity-team` |
| 只更新合规立场（新跨境政策/新分类分级策略）| `/data-compliance:compliance-stance` |
| 只更新风险升级矩阵（角色变化/触发条件调整）| `/data-compliance:risk-escalation` |
| 只更新文书风格与行业监管（新行业规定/风格调整）| `/data-compliance:style-cross-border-policy` |

### 直接编辑

律师/DPO 可直接编辑 `data-compliance/CLAUDE.md`，无需重跑访谈。适用于：
- 更新 DPO/负责人联系人
- 调整升级矩阵（如新增角色、调整权限）
- 根据新版办法更新跨境政策中的路径偏好（须注意时效性）
- 修正数据处理规模估算

### git 版本控制

```bash
# 查看配置变更历史
git log --oneline data-compliance/CLAUDE.md

# 查看具体变更内容
git diff HEAD~1 data-compliance/CLAUDE.md

# 回滚到历史版本
git checkout <commit-hash> -- data-compliance/CLAUDE.md
```

## 临时模式提示

如在完整访谈前须立即使用数据合规技能，可对技能说「**临时模式**」：

> 技能将使用以下默认值运行，并在输出上标注 `[临时模式 — 配置你的数据合规档案以获得定制输出]`：
>
> - 整体合规偏好：平衡型
> - 执业角色：数据合规律师
> - 数据处理者角色：个人信息处理者（非 CIIO）
> - 处理规模：不明（逐案判断出境门槛）
> - 敏感数据政策：按严格门槛标记常见风险类别
> - 跨境政策：逐案判定，不预设首选路径
> - 数据安全立场：按保守标准识别（不确定即标重要数据风险）
>
> 建议完成本次冷启动访谈后，运行技能以获取基于实际画像的定制输出。
