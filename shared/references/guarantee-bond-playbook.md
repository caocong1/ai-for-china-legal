# 保函/担保案件专用 Playbook

本 playbook 由真实任务验证经验沉淀，用于指导所有涉及保函、担保公司、独立保函认定的案件。

## 一、触发信号

当案件材料中出现以下任一信号时，**自动展开本 playbook 的检索**：

- `保函`、`见索即付`、`无条件支付`、`投标担保`、`履约担保`
- 出函主体含 `融资担保公司`、`担保有限公司`、`地方金融组织`
- 用户要求 `诉讼方案`、`案由`、`管辖`、`索赔`、`起诉状`
- 本地材料中有 `独立保函`、`不可撤销`、`无条件支付` 等容易误导模型的表述

## 二、核心法律判断流程

```
文书中出现"见索即付"/"无条件支付"
         │
         ▼
  出函主体是谁？
         │
    ┌────┴────┐
    │         │
 银行/     融资担保公司/
 非银金融机构  担保公司/
              地方金融组织
    │         │
    ▼         ▼
 可能构成    通常不构成独立保函
 独立保函    → 按保证合同处理
              → 分析保证方式
              → 连带 or 一般？
```

## 三、融资担保公司主体资格 — 已验证裁判共识

**2024-2025 年各级法院已形成共识**：

1. **融资担保公司不属于"银行或非银行金融机构"**（法释〔2016〕24号第1条）
2. 其出具的载有"见索即付"、"无条件支付"的保函 **不构成独立保函**
3. 通常按 **保证合同** 处理
4. 保证方式依《担保制度解释》第25条按保函措辞判断

### 关键法规

| 法规 | 条文 | 结论 |
|------|------|------|
| 法释〔2016〕24号 | 第1条 | 开立人限 **银行或非银行金融机构** |
| 金融监管总局令〔2023〕第3号 | 全文 | 非银行金融机构范围 **不含** 融资担保公司 |
| 国务院令第683号 | 全文 | 融资担保公司业务范围 **不含** 开立独立保函 |
| 民法典 | 第686条 | 保证方式分类（一般 vs 连带） |
| 民法典 | 第688条第一款 | 连带责任保证的定义 |
| 民法典 | 第688条第二款 | 债权人可向债务人或保证人主张 — 单独起诉担保人的法律基础 |
| 担保制度解释 | 第25条第二款 | "无条件承担保证责任" → **连带责任保证** |
| 担保制度解释 | 第21条 | 合并起诉时按 **主合同** 确定管辖 |
| 民事诉讼法 | 第24条 | 被告住所地 / 合同履行地 |

### 已知入库案例

| 编号 | 来源 | 核心认定 |
|------|------|----------|
| `2024-10-2-358-001` | 人民法院案例库 | 融资担保公司不属于可开立独立保函的主体 |
| 成渝金融法院判例 | 公开判决 | 融资担保公司出具保函按保证合同处理 |

## 四、保证方式分析 — "见索即付"条款的正确处理

### 常见错误

❌ **错误做法**：看到"见索即付"就直接警告"可能被认定为一般保证，有先诉抗辩权风险"

✅ **正确做法**：

1. 首先检索是否支持 **连带责任保证**
2. 保函载有 `见索即付`、`无条件支付`、`固定期限付款` 等表述
3. 这些表述表明 **无先诉抗辩安排**，应倾向 **连带责任保证**
4. 仅在保函措辞确实不明时，才适用民法典第686条第二款的"约定不明→一般保证"

### 连带责任保证论证路径

保函措辞 → 是否含"无条件支付"等表述 → 是 → 检索支持连带责任的案例和依据 → 适用担保制度解释第25条

## 五、案由锁定 — 唯一首选

### 强制要求

- 最终诉讼方案必须写明一个 **唯一的 `首选案由`**
- **不得**以 `保证合同纠纷或担保合同纠纷` 这种未决方式收尾
- 替代案由只能作为备选，不能与首选并列

### 案由选择路径

```
出函主体是融资担保公司？
    │
    ├── 是 → 首选案由：保证合同纠纷
    │         └── 备选：担保合同纠纷（如法院认为保证合同不成立）
    │
    └── 否（银行/非银金融机构）→ 首选案由：独立保函纠纷
```

## 六、证据目录分组

**必须按以下 3 大类分组列示**（每类至少 1 项）：

### 第一类：基础交易关系类
- 招标公告/招标文件、投标文件、资格审查承诺
- 审计报告、监管机关认定函
- 取消中标资格/解除合同等官方文书及送达凭证

### 第二类：保函法律关系及索赔事实类
- 保函原件、索赔函原件
- 索赔函送达凭证（EMS/邮政/当面签收）

### 第三类：被告违约事实类
- 催告记录（邮件/微信/通话录音）
- 对方拒付或拖延的会议纪要/函件
- 律师催告函及回执

## 七、风险段覆盖

**必须覆盖以下 5 大类**（缺一即不通过）：

1. **损失认定风险**：成渝地区"提交保函即视为已交保证金"的隐藏路径
2. **主观故意认定风险**：被告用"非主动故意"抗辩
3. **保函有效期/索赔时效风险**：含 OCR 误读 sanity check
4. **保证方式被改判风险**：被改判为一般保证的后果
5. **执行风险**：被告是否已为被执行人/失信

## 八、OCR/扫描件 Sanity Check

凡保函系扫描件，下列关键字段必须 **交叉核验**（与请示文/备忘录/招标文件对照）：

| 字段 | 错读后果 | 核验方法 |
|------|----------|----------|
| 保函编号 | 索赔函附错保函、立案被驳 | 与索赔函、内部备忘录对照 |
| 担保金额 | 诉请金额错 | 与招标文件保证金金额对照 |
| **保函有效期截止日** | 索赔时效错估、风险段误判 | 与"投标有效期"+N日重新推算 |
| 索赔通知期 | 利息起算日错 | 与索赔函载明的付款期限对照 |
| 出函人/受益人/申请人名称 | 当事人错列 | 与工商登记、营业执照对照 |
| 管辖约定 | 错告法院 | 重新读保函原条款 |

发现两源不一致时：
1. 在研究底稿 `无法获取/存疑事实` 段记录冲突
2. 标 `OCR 待人工核验`
3. 在风险段写入"该字段经多模态识别得出，可能存在识别误差"

## 九、地方司法实务反例预警

成渝地区（重庆、四川）法院存在以下"反受益人"路径，必须主动检索：

| 反例路径 | 检索词 | 应对 |
|----------|--------|------|
| "提交保函即视为已交保证金，需自证损失" | `投标保函 视为 履行 交保证金 损失` | 准备重新招标成本证据 |
| "基础合同无效 → 保函一并无效" | `基础合同 无效 保函 无效 投标` | 论证基础关系是招投标关系 |
| "弄虚作假需主观故意" | `弄虚作假 主观 故意 投标 抗辩` | 准备权威机关认定函 |
| "管辖约定不明" | `管辖约定 不明 投标保函` | 论证按民诉法第24条确定 |

**未检索到反例 ≠ 不存在**，必须在研究底稿明示"已检索 X、Y 关键词未发现反例"。
