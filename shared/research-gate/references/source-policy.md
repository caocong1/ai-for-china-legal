# 引用策略与失败回退链

本文件规定中国法律研究的引用规则和来源不可达时的回退策略。

## 引用标记

所有法律研究输出中的引用必须带来源标记：

| 标记 | 含义 |
|------|------|
| `[L1-法院]` | 最高人民法院官网全文 |
| `[L1-法规]` | 国务院/部门规章全文 |
| `[L2-入库]` | 人民法院案例库入库案例 |
| `[L2-指导]` | 指导性案例/公报案例 |
| `[L3-判决]` | 地方法院公开判决 |
| `[L4-监管]` | 监管机构文件 |
| `[L5-律所]` | 律所文章（仅作线索） |
| `[L5-公众号]` | 法律公众号（仅作线索） |
| `[搜索摘要-未抓全文]` | 仅有搜索引擎摘要，未核验全文 |
| `[待律师复核]` | 无可靠权威来源，需人工补检索 |

## 禁止引用

- ❌ 百度百科、知乎、个人博客作为法律依据
- ❌ 未经过 L1/L2 核验的 L5 文章中的法律结论
- ❌ 模型训练记忆中的法条（除非用工具检索验证）

## 失败回退链

| 失败信号 | 第一回退 | 第二回退 | 最终处理 |
|----------|----------|----------|----------|
| `flk.npc.gov.cn` 拿不到正文 | court.gov.cn 同条文 | WebSearch `法条名 第N条 site:gov.cn` | 标 `[搜索摘要-未抓全文]` |
| `wenshu.court.gov.cn` 维护 | 各省高院/中院/金融法院 site:gov.cn | 律所文章引述（标 L5） | 标 `[待律师复核]` |
| 公众号链接打不开 | 搜狗微信搜标题 | WebSearch 标题精确匹配 | 标 `公众号原文不可达，仅记录标题` |
| 律所文章 timeout | 换同主题其他律所 | WebSearch 摘要（标 L5） | 标 `[待律师复核]` |
| 任何全文都拿不到 | — | — | 标 `本案该论点缺权威源支持，待律师人工补检索` |

## 两段式检索协议

由于 `flk.npc.gov.cn`（国家法律法规数据库）是动态 SPA，headless 环境无法直接抓取，**必须**使用两段式：

### 阶段 1：WebSearch 定位

- 法条/司法解释：`"完整法律名" 第N条 site:court.gov.cn OR site:gov.cn`
- 案例：`"案件关键词" "案由" site:court.gov.cn` 或 `"入库编号"`
- 实务理解：`"争议焦点" 实务`（不限 site，但产出当 L5 看待）

### 阶段 2：WebFetch 全文

- **首选**：`court.gov.cn/zixun/xiangqing/{id}.html`（已实测稳定）
- **备选**：`www.gov.cn/gongbao/...`（行政法规）
- **落败**：WebSearch 摘要（必须标注 `[搜索摘要-未抓全文]`）

### 阶段 3：引用格式

```
《XX法》第X条 全文链接：<court.gov.cn URL>
《最高人民法院关于XX的规定》（法释〔YYYY〕NN号）第X条 全文链接：<court.gov.cn URL>
```

## 环境限制

| 限制 | 影响 | 应对 |
|------|------|------|
| flk.npc.gov.cn 动态 SPA | WebFetch 返回空 | 转 court.gov.cn 或 WebSearch site: |
| 裁判文书网维护中 | 无法获取判决 | 转地方法院网站或案例库 |
| 微信公众号需登录态 | 直链不稳 | 搜狗微信 + WebSearch 限定 site: |
| headless 无 X server | chrome-devtools 不可用 | 仅依赖 WebFetch + WebSearch |
