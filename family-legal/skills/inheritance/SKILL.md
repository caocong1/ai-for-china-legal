# 继承权分析（编排入口）

> **迁移说明**：原继承权分析深度内容（法定继承顺序与份额、遗嘱继承效力、遗产范围与债务清偿、代位继承与转继承）已拆分迁移至四个子技能，每个子技能可独立触发。本文件为编排入口，负责前置护栏、代理立场加载与子技能调用顺序。子技能路径：`family-legal/skills/inheritance/<子技能名>/SKILL.md`。

---

## 目的

继承权分析覆盖被继承人死亡后遗产分配的全流程法律分析：法定继承的顺序与份额确定、遗嘱继承的效力认定与优先顺位、遗产范围的析产认定与债务限额清偿、代位继承与转继承的区分适用。本技能按代理立场（代理某继承人/中立）调整份额主张侧重，适用于婚姻家事律师、个人法务、继承人或立遗嘱人。

**主要法律框架**：民法典继承编（第1119-1163条）及最高法继承编司法解释(一)（具体条号 `[待验证]`）。

### 3. 代理立场加载（核心变量）

**代理立场决定份额主张侧重**——同一继承案件，代理不同继承人的侧重截然不同。

| 代理立场 | 份额主张侧重 | 重点关注事项 |
|---------|-----------|-----------|
| **代理某继承人（已说明）** | 积极主张该继承人的应继份额、可多分情形、资格确立 | 丧失继承权情形排除、代位/转继承资格确立、尽主要扶养义务举证 |
| **中立审查** | 双向列示各继承人权益，识别争议风险 | 潜在纠纷点识别、遗产范围争议、债务分担合理性 |

## 入口级护栏

### 当事人画像与亲属关系确认

在开始继承分析前，须确认以下信息（从 `## 我们是谁` 与用户输入读取）：

- **被继承人基本信息**：姓名、死亡时间（继承开始时间点）、婚姻状态
- **已知继承人名单**：亲属关系类型（配偶/子女/父母/兄弟姐妹等）及每人的生死状态
- **是否有遗嘱或遗赠扶养协议**
- **财产构成概况**（影响析产复杂度）

信息不完整时，提示用户补充上述关键事实，再启动子技能分析。

### 代理立场加载（重申）

再次确认：本次分析代理哪位继承人，还是中立审查？代理立场是继承权分析的核心变量，直接影响份额主张侧重。若仍未明确，提示并等待用户选择，然后标注当前立场后继续。

### 敏感情形识别与强制升级

以下情形须在进入子技能前识别并触发相应护栏：

| 敏感情形 | 处理策略 | 强制升级 |
|---------|---------|---------|
| **涉刑情形**（故意杀害被继承人/为争夺遗产杀害继承人） | 立即停止继承份额分析，提示刑事风险 | ✅ 是——须刑事律师主导 |
| **丧失继承权争议**（继承人之间相互主张丧失继承权） | 提示须通过法院认定程序，不得自行宣布 | ✅ 是——须律师提起诉讼 |
| **遗产隐匿转移迹象** | 提示财产保全路径 | ✅ 是——须律师申请保全 |
| **继承人纠纷激化（肢体冲突/人身安全风险）** | 暂停复杂争议分析，提示安全保障 | ✅ 视情况 |
| **涉及未成年继承人重大利益受损** | 以最有利于未成年子女为首要原则（第1084条，建议复核） | ✅ 是——须律师确认 |
| **当事人情绪危机（丧亲悲痛/危机信号）** | 中立克制措辞，提示寻求心理支持，暂停复杂争议分析 | 视情况 |

> **涉刑情形优先于一切护栏：** 识别到故意杀害等涉刑情形，须**立即强制升级刑事律师**，停止继承权分析。

### 当事人情绪护栏（丧亲）

婚姻家事继承案件中，当事人处于丧亲悲痛中是普遍情形。全程须：

- **中立、克制**：不在分析中激化继承人之间的对立情绪
- **非煽动**：不替当事人作情绪化判断，不使用加剧冲突的语言
- **识别危机信号**：若当事人表达自我伤害或极度绝望，暂停法律分析，优先提示心理支持资源
- **尊重丧失**：在涉及被继承人时，以尊重的措辞呈现

### 目的地与特权检查

分析结果的目的地：

- **内部使用/律师工作产物**：全量分析，标题为「律师工作产物 — 受律师-当事人特权保护 / 保密」
- **当事人本人（非律师角色）**：标题为「研究笔记 — 非法律建议，供律师审查」；在遗产分割定案、起诉、遗嘱定稿等重大法律行动前，须暂停并升级律师
- **其他继承人/对方**：不得直接发送未经脱敏的分析，须提供脱敏版本

### 公证/不动产登记路径提示

继承涉及不动产时，提示以下路径：

- **继承权公证**：向公证机构申请继承权公证书（证明继承人身份与继承资格）
- **不动产过户**：凭继承权公证书向不动产登记机构办理继承过户登记
- **公证机构建议**：复杂继承（多继承人/遗产范围争议/遗嘱真实性存疑）建议通过公证处理，减少后续争议

### 地方性规定提示

特别关注：房产继承过户的当地不动产登记要求（各地材料要求不同）、遗产税/契税（地区差异 `[待验证]`）、公证机构具体程序。

## 子技能编排顺序

律师/法务/继承人可单独触发任一子技能，也可通过本入口运行完整继承分析流程：

| 顺序 | 子技能 | 职责 | 子技能路径 | 单独触发命令 |
|------|-------|------|---------|-----------|
| 1 | **estate-scope-debt** | 遗产范围认定与债务清偿：先析产、遗产正向认定与排除、限额清偿原则 | `skills/inheritance/estate-scope-debt/SKILL.md` | `/family-legal:estate-scope-debt` |
| 2 | **testate-succession-validity** | 遗嘱继承效力：继承优先顺位确认、遗嘱有效要件审查、数份遗嘱最后为准（公证不再优先）、遗赠扶养协议优先 | `skills/inheritance/testate-succession-validity/SKILL.md` | `/family-legal:testate-succession-validity` |
| 3 | **legal-succession-order** | 法定继承顺序与份额：第一二顺序核查、份额均等与酌情多分少分、丧偶儿媳女婿地位判定 | `skills/inheritance/legal-succession-order/SKILL.md` | `/family-legal:legal-succession-order` |
| 4 | **subrogation-transmission** | 代位继承与转继承：触发条件与份额计算、代位vs转继承区分、丧失继承权审查 | `skills/inheritance/subrogation-transmission/SKILL.md` | `/family-legal:subrogation-transmission` |

**顺序说明**：遗产范围析产在前（明确可继承基础）→ 遗嘱效力第二（确定遗嘱继承还是法定继承/混合）→ 法定继承顺序第三（无有效遗嘱时确定继承人与份额）→ 代位与转继承最后（处理继承人先死等特殊情形）。

**单独触发场景**：律师可仅针对单一问题调用子技能。例如：

- 仅需判断某份遗嘱是否有效：`/family-legal:testate-succession-validity`
- 仅需计算法定继承份额（有遗嘱但部分法定）：`/family-legal:legal-succession-order`
- 仅需确认遗产范围与债务清偿：`/family-legal:estate-scope-debt`
- 仅需判断代位继承是否适用：`/family-legal:subrogation-transmission`

## 输出格式

输出须包含：

1. **当前分析依据**：代理立场、遗产类型（有遗嘱/无遗嘱/混合）
2. **继承分析结论**：各继承人资格与份额，标注可能的争议点
3. **敏感情形处理**：已触发的护栏与升级情况
4. **下一步建议**：继承权公证 / 遗产分割协议拟定 / 继承诉讼 / 遗嘱公证（如有遗嘱）

**工作产物标题**（律师角色）：

```
律师工作产物 — 受律师-当事人特权保护 / 保密
```

**工作产物标题**（非律师角色）：

```
研究笔记 — 非法律建议，供律师审查
```
