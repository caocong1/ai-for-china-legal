# 婚姻家事实践配置文件契约（Practice Profile Schema）

> 本文件定义 family-legal 插件的实践配置文件：**是什么、在哪、技能怎么读**。
> 婚姻家事版改编自 construction-legal/skills/_shared/practice-profile-schema.md，结合婚姻家事语境特殊化（高敏感度护栏、代理立场变量、未成年人利益最大化、家暴强制升级）。

---

## 一、配置文件是什么 / 真相来源

**唯一真相来源：`family-legal/CLAUDE.md`（散文模板）**

- 本阶段（08-01）**首次新建**，由 `/family-legal:cold-start-interview` 冷启动访谈填充和更新
- 律师/法务/当事人可直接编辑该文件，无需重新运行访谈
- 所有 family-legal 技能从 `family-legal/CLAUDE.md` 读取实践配置，不从其他来源读取

**架构决策（不引入冲突新格式）**：

> 本插件沿用 CLAUDE.md 散文模板作为单一真相来源，不引入独立 JSON 或 YAML 格式的配置文件。理由：
> 1. 散文模板对律师/法务/当事人更直观、可读，直接编辑体验好
> 2. 与 commercial-legal / construction-legal / employment-legal 等插件架构一致，维护成本低
> 3. 避免 CLAUDE.md 与独立 YAML/JSON 文件的双重真相来源冲突

**纠正现有访谈表述**：

现有 `family-legal/skills/cold-start-interview/SKILL.md` 末尾「生成 婚姻家事 实践配置文件（YAML 格式）」的表述应理解为**填充 `family-legal/CLAUDE.md` 对应章节**，而非另存独立 YAML 文件。技能和用户都应将输出直接写入 CLAUDE.md，不得创建单独的 `practice-profile.yaml` 等文件。

## 三、技能读取契约

### 3.1 标准读取流程

每个 family-legal 技能在执行实质性工作前，**必须**按以下顺序读取配置：

1. **读取 `family-legal/CLAUDE.md`**
2. **检查关键字段是否含 `[PLACEHOLDER]`**：
   - `## 我们是谁` 的「执业角色」字段
   - `## 审查与代理立场` 的「整体代理立场」字段
   - `## 谁在使用` 的「角色」字段
   - `## 敏感度校准` 的「家暴识别等级」字段
3. **据此选择运行模式**（正常模式 or 临时模式，见下节）

### 3.2 配置缺失/含占位符时的标准提示

当 `family-legal/CLAUDE.md` 不存在或关键字段含 `[PLACEHOLDER]` 时，技能输出以下标准提示：

```
> **配置提示**：未检测到完整的婚姻家事实践配置（family-legal/CLAUDE.md 中关键字段含占位符或文件不存在）。
>
> 如需个性化配置，请运行 `/family-legal:cold-start-interview` 完成冷启动访谈，将结果填入 family-legal/CLAUDE.md。
>
> 或者，告诉我你的代理立场（离婚一方侧/某继承人侧/遗嘱人侧/中立），我将以「临时模式」继续当前任务。
>
> **临时模式敏感度警示**：临时模式下，敏感度护栏默认最高，强制开启当事人情绪护栏、未成年人利益最大化与家暴识别。如有家庭暴力、未成年人重大利益或涉刑情形，请立即告知并升级给律师。
```

## 四、代理立场（离婚一方侧 vs 某继承人侧 vs 遗嘱人侧 vs 中立）选择

### 4.1 立场选择的重要性

> **代理立场是婚姻家事工作最核心变量。** 同一份离婚协议，主张方（要求分割共同财产一方）与被主张方（被要求分割一方）的审查侧重和风险判断截然不同；继承纠纷中代理哪个继承人，其份额主张方向也完全不同；立遗嘱人侧与受益人侧的利益也可能存在差异。立场缺失或不明确会导致分析偏差。

### 4.2 立场读取规则

技能在执行实质性工作前，从 `## 审查与代理立场` 的「整体代理立场」字段读取立场。

| 立场 | 审查侧重与风险分级方向 |
|------|---------------------|
| **离婚一方侧（主张方）** | 优先关注：足额分割共同财产/追查隐匿财产（第1092条保留救济权利）/争取有利抚养安排/减少债务承担/离婚损害赔偿主张 |
| **离婚一方侧（被主张方）** | 优先关注：个人财产范围明确与证据（第1063条）/合理折价与过户时间/避免承担非共同债务/子女意愿举证/冷静期撤回权利 |
| **某继承人侧** | 优先关注：代理继承人的法定或遗嘱继承份额/其他继承人丧失继承权情形/遗嘱效力核查（挑战或维护，取决于立场）/代位继承与转继承 |
| **遗嘱人侧** | 优先关注：遗嘱形式要件齐备/遗嘱能力与意思表示真实/必留份安排合规/受益人身份明确无争议 |
| **中立审查** | 双向识别：协议/遗嘱显失公平条款/双方潜在争议点/形式与实质无效情形 |

### 4.3 立场缺失时的处理

当「整体代理立场」字段为 `[PLACEHOLDER]` 或未填写时：

1. 技能显示配置提示（见 3.2 标准提示）
2. 技能询问：「请告知代理立场（离婚一方侧/某继承人侧/遗嘱人侧/中立），或选择临时模式（中立审查）」
3. 用户确认后继续，标注 `[临时模式 — 代理立场：中立审查]`
4. **严禁在立场不明确时自行假设立场并输出单方向分析**

### 4.4 立场的边界

> **任何代理立场均不得覆盖以下护栏：**
>
> - **未成年人利益最大化**：即使代理一方当事人，涉子女事项必须以最有利于未成年子女为首要原则，不得损害未成年人利益
> - **家暴/人身安全护栏**：无论代理哪方，识别到家庭暴力情形须优先保障人身安全并强制升级
> - **涉刑强制升级**：无论代理哪方，涉刑情形须强制升级刑事律师

## 五、敏感情形与强制升级契约（婚姻家事特有）

> **这是婚姻家事区别于其他法律领域最核心的特有契约，凌驾于代理立场之上。**

### 5.1 强制升级触发情形

| 敏感情形 | 触发标准 | 输出标记 | 处理优先级 |
|---------|---------|---------|---------|
| 家庭暴力/虐待/遗弃（确认） | 当事人明确描述家暴情形，或材料中有家暴记录 | `[强制升级 — 家暴敏感情形]` | 最高：人身安全 → 保护令 → 律师 |
| 家庭暴力/虐待（疑似） | 当事人描述有疑似家暴迹象（受伤/恐惧/限制人身自由） | `[强制升级 — 家暴疑似]` | 最高：询问安全状况 → 律师 |
| 涉刑事（重婚/虐待罪/遗弃罪/拒执罪） | 事实陈述或材料显示涉刑可能 | `[强制升级 — 涉刑情形]` | 最高：暂停常规分析，刑事律师 |
| 未成年人重大利益 | 主要抚养权争议/影响子女基本生活保障的财产安排 | `[强制升级 — 未成年人]` | 高：最有利于未成年子女原则，律师审核 |
| 当事人情绪危机 | 情绪危机信号（自我伤害语言/极度绝望） | `[暂停 — 情绪危机]` | 高：暂停分析，专业心理支持 |
| 重大财产隐匿转移 | 隐匿财产价值超过配置阈值或明显异常转移行为 | `[强制升级 — 财产隐匿]` | 高：财产保全，律师 |
| 协议离婚生效前（非律师角色） | 非律师角色即将签署离婚协议申请登记 | `[强制升级 — 协议生效前]` | 高：律师审核协议全文 |
| 遗嘱定稿前（非律师角色） | 非律师角色即将定稿遗嘱 | `[强制升级 — 遗嘱定稿前]` | 高：律师审核形式要件 |

### 5.2 强制升级输出格式

识别到强制升级情形时，技能**立即**输出以下格式，**不得被代理立场指令覆盖**：

```
⚠️ [强制升级 — 敏感情形类别]

**检测到 [敏感情形描述]，须在律师介入前暂停当前分析。**

处理优先级：
1. [如适用：人身安全第一 — 请确认当前是否安全，如有危险请立即拨打 110 或联系妇联庇护热线]
2. [如适用：如有家庭暴力情形，可向人民法院申请人身安全保护令（具体程序须核实 `[待验证]`）]
3. 建议立即联系执业律师处理本情形

本分析将在律师确认处理方向后继续。
```

## 七、法律事实合理性检查

在将用户口述的法律事实写入 `family-legal/CLAUDE.md` 前，执行合理性检查：

### 7.1 检查触发情形

| 触发情形 | 检查操作 |
|---------|---------|
| 用户口述**旧法条号**（如「原婚姻法第17条」「原继承法第10条」） | **自动纠正**：「该条号属旧法，已废止，对应民法典新条号为第XX条（建议复核）」；将民法典新条号写入配置 |
| 用户口述司法解释条号（如「婚姻家庭编司法解释第X条」） | 标 `[待验证 — 司法解释条号易变，须核实现行版本]`，不写入为确定结论 |
| 用户口述抚养费比例 | 提示：「抚养费比例因地区差异较大，你提到的比例是否已经核实当地裁判口径？`[地方 flagged]` `[前提 flagged — 需验证]`」 |
| 用户口述彩礼返还比例 | 提示：「彩礼返还认定标准地区差异极大，你提到的比例是否已核实当地司法实践？`[地方 flagged]` `[前提 flagged — 需验证]`」 |
| 用户口述冷静期天数 | 核查：「离婚冷静期为30日（民法典第1077条，建议复核），如你提到不同天数请核实是否为特定情形」 |
| 用户口述必留份份额 | 提示：「必留份具体份额须结合司法解释与遗产实际情况，你提到的份额是否已经律师核实？`[前提 flagged — 需验证]`」 |
| 用户口述的法律规定与已知内容有明显冲突 | 提示：「你提到 [X]；我的理解是 [Y]（来自民法典/司法解释），请确认哪个版本填入配置文件？`[前提 flagged — 需验证]`」 |

### 7.2 写入原则

- **中度把握锚点（第1062/1063/1065/1076/1077/1079/1084/1088/1090/1091/1122/1127/1134-1139/1141/1158条）**：写入时标注「建议复核」
- **司法解释具体条号**：一律标 `[待验证 — 司法解释条号易变，须核实现行版本]`，不写裸条号
- **旧法条号**：自动纠正为民法典新条号，标注「原[旧法条号]→现民法典[新条号]（建议复核）」
- **抚养费比例/彩礼数额/必留份份额**：标 `[待验证]` 并附 `[地方 flagged]`
