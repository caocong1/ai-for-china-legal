# 建设工程实践配置文件契约（Practice Profile Schema）

> 本文件定义 construction-legal 插件的实践配置文件：**是什么、在哪、技能怎么读**。
> 建设工程版改编自 ip-legal/skills/_shared/practice-profile-schema.md，结合建设工程语境特殊化。

---

## 一、配置文件是什么 / 真相来源

**唯一真相来源：`construction-legal/CLAUDE.md`（散文模板）**

- 本阶段（07-01）新建，由 `/construction-legal:cold-start-interview` 冷启动访谈填充和更新
- 律师/法务可直接编辑该文件，无需重新运行访谈
- 所有 construction-legal 技能从 `construction-legal/CLAUDE.md` 读取实践配置，不从其他来源读取

**架构决策（不引入冲突新格式）**：

> 本插件沿用 CLAUDE.md 散文模板作为单一真相来源，不引入独立 JSON 或 YAML 格式的配置文件。理由：
> 1. 散文模板对律师/法务更直观、可读，直接编辑体验好
> 2. 与 commercial-legal / ip-legal / employment-legal 等插件架构一致，维护成本低
> 3. 避免 CLAUDE.md 与独立 YAML/JSON 文件的双重真相来源冲突

**纠正现有访谈表述**：

现有 `construction-legal/skills/cold-start-interview/SKILL.md` 末尾「生成 建设工程 实践配置文件（YAML 格式）」的表述应理解为**填充 `construction-legal/CLAUDE.md` 对应章节**，而非另存独立 YAML 文件。技能和用户都应将输出直接写入 CLAUDE.md，不得创建单独的 `practice-profile.yaml` 等文件。

## 三、技能读取契约

### 3.1 标准读取流程

每个 construction-legal 技能在执行实质性工作前，**必须**按以下顺序读取配置：

1. **读取 `construction-legal/CLAUDE.md`**
2. **检查关键字段是否含 `[PLACEHOLDER]`**：
   - `## 我们是谁` 的「建设工程角色」字段
   - `## 审查与争议立场` 的「整体代理立场」字段
   - `## 谁在使用` 的「角色」字段
3. **据此选择运行模式**（正常模式 or 临时模式，见下节）

### 3.2 配置缺失/含占位符时的标准提示

当 `construction-legal/CLAUDE.md` 不存在或关键字段含 `[PLACEHOLDER]` 时，技能输出以下标准提示：

```
> **配置提示**：未检测到完整的建设工程实践配置（construction-legal/CLAUDE.md 中关键字段含占位符或文件不存在）。
>
> 如需个性化配置，请运行 `/construction-legal:cold-start-interview` 完成冷启动访谈，将结果填入 construction-legal/CLAUDE.md。
>
> 或者，告诉我你的代理立场（发包人侧/承包人侧/中立/购房人侧），我将以「临时模式」继续当前任务。
```

|------|-------------|------|

## 四、代理立场选择

### 4.1 立场选择的重要性

> **代理立场是建设工程审查的核心变量。** 同一份施工合同，发包人侧与承包人侧的审查侧重和风险判断截然不同；商品房买卖中购房人侧与开发商侧的关注点也完全相反。立场缺失或不明确会导致审查结论偏差。

### 4.2 立场读取规则

技能在执行实质性工作前，从 `## 审查与争议立场` 的「整体代理立场」字段读取立场。

| 立场 | 审查侧重与风险分级方向 |
|------|---------------------|
| **发包人侧** | 优先关注：承包人资质核查/合同无效风险（防范黑白合同被承包人利用）/工期违约追责/质量缺陷追责/结算高估冒算防控/签证审批权限/进度款拒付理由 |
| **承包人侧** | 优先关注：合同无效后折价补偿路径/工程款足额结算保障/工期顺延签证权利/变更签证举证/优先受偿权行使期限与条件/进度款迟付追偿/解除后已完工程价值保障 |
| **中立审查** | 双向识别：合同显失公平条款/双方潜在争议点/无效情形的双向后果/合同缺口 |
| **购房人侧** | 优先关注：开发商主体资质/预售许可五证核查/逾期交房违约金条款/面积误差条款/退房条件/贷款不成解除/网签备案保护 |

### 4.3 立场缺失时的处理

当「整体代理立场」字段为 `[PLACEHOLDER]` 或未填写时：

1. 技能显示配置提示（见 3.2 标准提示）
2. 技能询问：「请告知代理立场（发包人侧/承包人侧/中立/购房人侧），或选择临时模式（中立审查）」
3. 用户确认后继续，标注 `[临时模式 — 代理立场：中立审查]`
4. **严禁在立场不明确时自行假设立场并输出单方向分析**

## 五、重跑 / 编辑 / 版本控制

### 5.1 完整重跑

```bash
# 重新运行冷启动访谈，全量更新配置
/construction-legal:cold-start-interview
```

### 5.2 单独更新某章节

```bash
# 只更新审查与争议立场（如更换代理项目立场）
/construction-legal:review-stance
# 或直接编辑
vim construction-legal/CLAUDE.md
# 找到 ## 审查与争议立场 章节，修改「整体代理立场」字段
```

### 5.3 git 版本控制与回滚

```bash
# 查看历史版本
git log construction-legal/CLAUDE.md

git diff HEAD~1 construction-legal/CLAUDE.md

git checkout HEAD~1 construction-legal/CLAUDE.md

git show <commit-hash>:construction-legal/CLAUDE.md
```

## 六、法律事实合理性检查

在将用户口述的法律事实写入 `construction-legal/CLAUDE.md` 前，执行合理性检查：

### 6.1 检查触发情形

| 触发情形 | 检查操作 |
|---------|---------|
| 用户口述司法解释条号（如「施工合同司法解释第 X 条」） | 标 `[待验证 — 司法解释条号易变，须核实现行版本]`，不写入为确定结论 |
| 用户口述优先受偿权行使期限 | 提示：「优先受偿权行使期限须核实现行司法解释版本，你提到的期限是否已经过核实？`[前提 flagged — 需验证]`」 |
| 用户口述必须招标规模标准（金额） | 提示：「招标规模标准因项目类型与地方规定有差异，你提到的金额是否已核实？`[前提 flagged — 需验证]`」 |
| 用户口述面积误差处理比例 | 提示：「3% 误差处理规则的具体算法须核实现行商品房司法解释版本 `[前提 flagged — 需验证]`」 |
| 用户口述逾期违约金率或上限 | 提示：「违约金率须与合同约定及司法解释调整规则对照，你提到的比例是否需要核实？`[前提 flagged — 需验证]`」 |
| 用户口述某法律规定与已知内容有明显冲突 | 提示：「你提到 [X]；我的理解是 [Y]。建设工程司法解释修订频繁，请确认哪个填入配置文件？`[前提 flagged — 需验证]`」 |

### 6.2 写入原则

- **已核实中度把握锚点（第788/799/807条 + 通则 585条第2款/588/563/533等）**：写入时标注「建议复核」
- **司法解释具体条号**：一律标 `[待验证 — 司法解释条号易变，须核实现行版本]`，不写裸条号
- **规模标准金额/期限阈值**：标 `[待验证]`，写入描述规则而非具体数字
