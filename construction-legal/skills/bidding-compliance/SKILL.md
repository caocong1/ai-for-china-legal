# 建设工程招投标合规

> **迁移说明**：本技能已深化拆分为四个可独立触发的子技能，存放于 `skills/bidding-compliance/<子技能名>/SKILL.md`。原三步纲要（需求收集/法律分析/输出生成）已无丢失迁移并大幅深化到各子技能。本文件现为编排入口。

---

## 目的与整体流程

系统处理建设工程招投标合规问题，覆盖必须招标判定、程序合规审查、违规行为识别、以及中标无效与救济四个核心维度。

**整体招投标合规审查流程概览：**

1. **必须招标判定**（mandatory-bidding-scope）→ 判断是否须招标、适用何种招标方式
2. **程序合规审查**（bidding-procedure）→ 审查招标投标全流程程序合规性
3. **串通/虚假识别**（collusion-fraud）→ 识别串通投标与弄虚作假行为
4. **中标无效与异议投诉**（bid-invalidity-complaint）→ 处置中标无效情形与启动投诉机制

> **顺序说明**：四个子技能可整体顺序运行，也可**按合规审查目的单独触发**——建设工程实践中往往只需处理其中一个维度（如仅判断是否必须招标，或仅核查某项目的评标程序合规性）。

## 入口级护栏

### 项目类型与资金来源确认

读取 `construction-legal/CLAUDE.md` 中的 `## 我们是谁 → 主要项目类型` 与 `## 共享护栏 → 项目类型识别`：

| 项目类型 | 必须招标风险 | 行政监督主要部门 |
|---------|-----------|------------|
| 市政基础设施 | 通常属必须招标范围 | 住建主管部门 |
| 房屋建筑 | 国有资金/国家融资时通常须招标 | 住建主管部门 |
| 公路工程 | 通常属必须招标范围 | 交通运输主管部门 |
| 水利工程 | 通常属必须招标范围 | 水利主管部门 |
| 社会资本私人项目 | 低于规模标准时可不招标 `[待验证规模标准]` | 视情况 |

### 立场加载

- **招标人侧（合规）**：关注程序合规防控法律风险，防止招标文件歧视性条款，确保评标程序公正
- **投标人侧（维权）**：关注招标文件公平性，保留异议投诉权利，及时在期限内行权

### 与 contract-review 的分工提示

| 问题 | 处理技能 |
|------|---------|
| 是否因未招标导致施工合同无效 | contract-review → subject-qualification-validity |
| 黑白合同（中标后另行签订背离实质性内容的合同）| bidding-compliance → bidding-procedure（识别），payment-dispute → settlement-basis-dual-contract（处理结算）|
| 招投标全流程合规 | **本技能（bidding-compliance）各子技能** |
| 中标无效后的工程价款折价补偿 | payment-dispute → settlement-basis-dual-contract（无效合同路径）|

### 行政监督部门按行业识别

> 自动从 `construction-legal/CLAUDE.md` 的 `## 共享护栏 → 项目类型识别` 中读取项目类型，匹配对应行政监督部门（住建/水利/交通/发改委等）。
>
> 投诉招投标违规时须向对应行政监督部门提交，不同部门的受理要求可能有差异。

### 目的地/特权检查

- 内部分析材料 → 标「律师工作产物 — 受律师-当事人特权保护 / 保密」
- 面向行政监督部门（投诉书）→ 准备去除特权保护的正式版本

### 地方性规定提示

招投标受地方规定影响，涉及具体地区时标：`[地方 flagged — 需核实地方性规定与地方住建主管部门要求]`。
地方特别关注：地方必须招标规模标准（部分省市高于国家标准 `[待验证]`）、地方招标投标监督管理办法、地方工程建设项目招投标专项规定。

### 时效性提示

招标投标法实施条例可能修订，必须招标的工程建设项目规定可能调整规模标准。引用规模阈值与程序期限前须核实现行规定版本 `[待验证]`。
