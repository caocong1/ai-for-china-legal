# 风险等级与升级矩阵采集

## 目的

通过结构化访谈，采集用户的风险等级定义、角色权限划分、自动升级触发条件与诉讼/仲裁/优先受偿权行使触发规则，并将结果填充到 `construction-legal/CLAUDE.md` 的 `## 风险校准` 与 `## 升级矩阵` 章节。

> **升级矩阵是建设工程实践中的关键安全网。** 建设工程合同无效风险、工程款优先受偿权期限、群体性商品房退房等高度专业化场景，内部法务和助理律师很容易超出权限处理——升级矩阵的作用是在正确的时间点将事务升级给有资质的律师，防止因处理不当造成更大损失。

**可单独触发**：`/construction-legal:risk-escalation`（只更新风险校准与升级矩阵部分），或由 `/construction-legal:cold-start-interview` 编排入口作为第三步调用。

### 第一部分：风险等级定义

> **风险等级定义**说明：construction-legal 技能使用四级风险等级（🔴关键 / 🟠高 / 🟡中等 / 🟢低）。以下问题帮助你为这四个等级设定符合你实践场景的触发条件——默认触发条件已在 `construction-legal/CLAUDE.md` 中预设，可以保留或修改。

**Q1.1 — 🔴 关键风险（须立即处理）**

> 在建设工程业务中，哪些情形构成「必须立即处理」的关键风险？（可参考默认项勾选或自行补充）

```
默认触发情形（供参考，可修改）：
A. 合同因转包/违法分包/必须招标但未招标/黑白合同可能无效（须律师出具效力意见）
B. 工程款优先受偿权期限临近（须核实现行司法解释版本 [待验证]）
C. 收到合同解除通知或起诉/仲裁申请
D. 工程质量存在重大安全隐患（危及人员安全）
E. 商品房群体性退房风险（多户购房人同时主张）
F. 其他：[自行补充]
```

**Q1.2 — 🟠 高风险（须采取行动）**

> 哪些情形构成「须采取行动但不需立即升级」的高风险？

A. 承包人资质存在问题（证书不全/超范围施工）
B. 结算依据不足或双方对结算金额存在重大争议
C. 变更签证未按时确认（超出约定期限 [待验证]）

E. 逾期交房/办证违约金数额较大

G. 其他：[自行补充]
```

*字段映射*：→ `## 风险校准` > 🟠 高风险级 > 建设工程典型触发情形

**Q1.3 — 🟡 中等风险（建议进一步核实）**

> 哪些情形构成「值得关注但不紧急」的中等风险？

```
默认触发情形（供参考，可修改）：
A. 合同约定不明确，可能引发纠纷
B. 工期顺延条件限制较多，承包人主张顺延困难
C. 面积误差接近 3% 临界值（±2%~3% 范围）
D. 造价鉴定申请条件存在争议

F. 其他：[自行补充]
```

*字段映射*：→ `## 风险校准` > 🟡 中等风险级 > 建设工程典型触发情形

**Q1.4 — 🟢 低风险（正常推进）**

> 哪些情形表示风险可接受，可正常推进？

C. 商品房五证齐全且预售许可在效期内

```

### 第二部分：角色权限划分

> **角色权限划分**决定了哪些人可以独立处理哪些事务、何时须升级。以下问题帮助你设定建设工程团队的权限层级——如团队规模较小，可简化为两级。

**Q2.1 — 初级角色（内部法务/助理律师）**

> 你的团队中最初级的建设工程法律人员（如内部法务助理、初级律师）通常可以独立处理哪些事项？

```
常见可独立处理事项（供参考）：
A. 合同初步审查（资质查询、条款对比、标准条款分析）
B. 招标文件合规初步审核
C. 逾期通知/催款函草稿准备
D. 工程款主张材料整理
E. 商品房审查信息收集（五证核查、备案查询）
F. 其他：[自行补充]
```

**Q2.2 — 初级角色需升级的条件**

> 以下哪些情形，你认为初级法律人员应当升级给主办律师/合伙人？（可多选）

```
常见需升级条件（供参考）：
A. 合同可能无效（转包/违法分包/必须招标但未招标/黑白合同）
B. 工程款重大争议（超过 [PLACEHOLDER — 如：500 万]）
C. 作出合同解除判断或发送解除通知
D. 行使优先受偿权（需确认期限与程序合规）
E. 重大索赔诉讼/仲裁策略决策
F. 商品房群体性退房
G. 其他：[自行补充]
```

*字段映射（Q2.1+Q2.2）*：→ `## 升级矩阵 > ### 角色权限表`

### 第三部分：自动升级触发条件

> **自动升级**指无论风险等级、无论角色权限，一旦触发即必须升级的情形。这些触发条件优先于角色权限划分。

**Q3.1 — 必须自动升级的情形**

> 在你的实践中，以下哪些情形应当无论如何自动升级给主办律师/合伙人？（默认全选，可调整）

```
默认自动升级清单（供选择是否全部保留）：
A. 合同可能无效：发现转包/违法分包/必须招标但未招标/黑白合同情形
B. 工程款数额重大：超过设定金额阈值（见 Q2.3）
C. 拟行使优先受偿权：在启动前须律师确认期限与程序合规（期限须核实 [待验证]）
D. 收到解除通知或起诉/仲裁申请：任何正式法律程序须律师主导
E. 商品房群体性退房：多户购房人同时主张退房
F. 优先受偿权期限临近（离截止日 30 天内 [待验证]）：须主动提示并升级
```

> 如需新增触发条件，请说明：

*字段映射*：→ `## 升级矩阵 > ### 自动升级触发条件`

```
常见触发条件（供参考）：
A. 催款函发送后 [PLACEHOLDER — 如：30 天] 无回应
B. 明确拒绝支付
C. 优先受偿权期限临近，需结合行使评估
D. 金额超过阈值（参见 Q2.3）
E. 其他：[自行补充]
```

*字段映射*：→ `## 升级矩阵 > ### 诉讼/仲裁/优先受偿权行使触发` > 「工程款拖欠且拒绝协商」行

**Q4.2 — 优先受偿权行使决策**

> 在工程款拖欠后评估是否行使优先受偿权时，你的决策依据是什么？

```
常见决策因素（供参考）：
A. 优先受偿权期限（须核实现行司法解释 [待验证]）——首要考虑
B. 是否已进入诉讼/仲裁程序
C. 工程款金额大小
D. 发包人资产状况（是否有可供执行财产）
E. 项目是否已竣工/交付（影响行使条件 [待验证]）
```

*字段映射*：→ `## 升级矩阵 > ### 诉讼/仲裁/优先受偿权行使触发` > 「工程款拖欠且拒绝协商」行

**Q4.3 — 收到合同解除通知后的行动规则**

常见处理步骤（供参考）：
A. 立即升级主办律师（已在自动升级触发中）
B. 评估解除条件是否成就（有无法定/约定解除事由）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 用户团队规模很小（如独立执业律师） | 简化为两级权限（本人 vs 须转委托），调整角色权限表 |
| 用户无商品房业务 | Q4.4 跳过；升级矩阵中商品房相关条目保留但标「不适用」 |
| 用户口述的优先受偿权期限与常见理解不符 | 触发合理性检查，标 `[前提 flagged — 须核实]`，提示核实现行司法解释版本 |
| 用户不确定工程款升级阈值 | 保留 `[PLACEHOLDER]`，提示后续通过直接编辑 construction-legal/CLAUDE.md 设定 |
| 用户主要为开发商侧，无优先受偿权需求 | 优先受偿权相关问题标「不适用」；商品房升级触发条件调整为开发商侧视角 |
