# 文书风格与地方规定采集

## 目的

通过结构化访谈，采集用户的文书风格偏好、工作产物存放位置、造价鉴定资源与地方建设主管部门偏好，并将结果填充到 `construction-legal/CLAUDE.md` 的 `## 文书风格`、`## 输出` 与 `## 共享护栏` 章节。

> **地方性规定是建设工程审查的重要变量。** 建设工程受地方建设主管部门规章与地方性法规影响较大，各省市在必须招标规模标准、施工许可条件、工程造价计价定额、农民工工资保障等方面均可能有高于国家标准的地方性要求，配置地方性规定偏好有助于技能自动触发 `[地方 flagged]` 提示。

**可单独触发**：`/construction-legal:style-local-rules`（只更新文书风格与地方规定部分），或由 `/construction-legal:cold-start-interview` 编排入口作为第四步调用。

## 访谈问题清单

### 第一部分：合同审查意见书风格

> 建设工程合同审查意见书是主要工作产物之一（contract-review 技能输出）。以下问题帮助你设定默认输出风格。

**Q1.1 — 合同审查意见书结构偏好**

> 你偏好哪种合同审查意见书结构？

```
选项：
A. 结构化分析（按模块逐项：主体资质→工期质量→价款结算→违约解除）+ 风险等级清单 + 修改建议
   适合：对手方需要清晰可追踪的审查文件；需要说明修改理由
B. 摘要式 — 仅列重要风险项，不逐条分析
   适合：内部快速审查；合同条款数量多但只需高亮关键风险
C. 对照表式 — 合同原文 vs 建议修改对照，适合谈判阶段
   适合：谈判前审查，逐条列出原文与建议修改版本
D. 简报式 — 领导汇报用，3-5 个要点 + 风险结论
   适合：向管理层或客户高层简报
E. 其他（请说明）：
```

*字段映射*：→ `## 文书风格` > 「合同审查意见书风格」

**Q1.2 — 风险标注方式偏好**

> 在合同审查意见书中，你偏好如何标注风险？

```
选项：
A. 使用颜色/图标（🔴🟠🟡🟢 四级 + 文字说明）
B. 纯文字（关键风险/高风险/一般风险/低风险）
C. 仅标关键和高风险（中低风险不单独标记）
D. 视输出目的地（对外呈现用 A 或 B，内部用 C 或 D）
```

*字段映射*：→ `## 文书风格` > 合同审查意见书风格（补充说明）

### 第二部分：工程款主张/优先受偿权函件风格

**Q2.1 — 工程款主张函件语气**

> 在起草工程款主张函件（如催款函、逾期通知、优先受偿权行使通知）时，你倾向于哪种语气？

```
选项：
A. 正式强硬 — 措辞坚定，明确金额与期限，不保留协商空间
B. 正式但留余地 — 坚定立场的同时保留协商空间（适合长期合作方）
C. 阶梯式 — 第一封友好提醒，第二封正式催告，第三封法律声明
D. 视情况 — 按关系深浅与拖欠金额选择
```

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 用户无商品房业务 | Q4.1 跳过；商品房审查报告风格保留 `[PLACEHOLDER — 不适用]` |
| 用户业务跨多个省市 | 记录全部省市；提示「涉及具体地区时技能将自动触发地方性规定检查」 |
| 用户不熟悉造价定额 | 说明造价定额的作用（影响结算审核与鉴定），建议咨询配合的造价咨询机构确认 |
| 用户已在 identity-project 完成造价机构填写 | Q6.1 显示已采集值，询问是否需要更新；快速确认即可 |
| 用户选择「视项目所在地」的地方定额 | 写入「视项目所在地，每次确认」；同时提示知名省市定额差异较大 |
