# 冷启动访谈编排入口 — 建设工程

## 目的

初始化或更新建设工程实践配置档案（`construction-legal/CLAUDE.md`），为所有 construction-legal 技能（contract-review / payment-dispute / bidding-compliance / commercial-housing-review）提供个性化基础配置：

- **主体画像与项目组合**：建设工程角色（发包人/承包人/开发商/购房人代理）、项目类型、团队规模
- **审查与争议立场**：整体代理立场（发包人侧/承包人侧/中立/购房人侧）、合同审查严格度、商品房审查立场
- **风险校准与升级矩阵**：四级风险定义、角色权限、自动升级触发条件、诉讼/仲裁/优先受偿权触发
- **文书风格与地方规定**：合同审查意见书/工程款函件/争议分析/商品房审查报告风格、造价鉴定资源、地方性规定

## 访谈结果写入说明

访谈结果写入 `construction-legal/CLAUDE.md`（**散文模板，非独立 YAML 文件**）：

- 采集的答案**覆盖** `construction-legal/CLAUDE.md` 中对应章节的 `[PLACEHOLDER]` 占位符
- 用户跳过的问题保留 `[PLACEHOLDER]`
- 涉及司法解释具体条号或法律事实的字段保留 `[待验证]` 标注，不因用户口述而移除
- 访谈完成后，`construction-legal/CLAUDE.md` 即成为所有 construction-legal 技能的**唯一配置来源**

**支持的运行方式**：

| 方式 | 命令 | 说明 |
|------|------|------|
| 完整访谈（首次配置） | `/construction-legal:cold-start-interview` | 顺序运行全部四个子技能，完整初始化 |
| 单独更新某一部分 | `/construction-legal:identity-project` 等 | 直接触发单个子技能，只更新该章节 |
| 直接编辑 | 编辑 `construction-legal/CLAUDE.md` | 可直接修改文件中任何字段 |
| git 版本控制 | `git log construction-legal/CLAUDE.md` | 可追踪配置变更历史 |

## 访谈的重要性说明

> **为什么建设工程审查特别需要配置立场？**
>
> 建设工程审查是立场驱动性最强的法律专业领域之一：同一份施工合同，发包人侧审查侧重控价/质量/违约追责，承包人侧审查侧重足额结算/签证/优先受偿权——立场不同，结论截然相反。商品房审查同样如此：购房人侧关注五证/逾期/面积误差/退房权，开发商侧关注合规确认与违约限制。
>
> 此外，建设工程受地方建设主管部门规章影响极大（地方必须招标标准/造价定额/施工许可条件差异显著），正确配置地方性规定偏好有助于技能自动触发 `[地方 flagged]` 提示，避免遗漏关键合规风险。

## 跳过访谈（临时模式）

如不希望完成访谈，可告诉我代理立场（发包人侧/承包人侧/购房人侧/中立），我将以「临时模式」继续，使用以下默认值（均标注 `[临时模式]`）：

| 字段 | 临时模式默认值 |
|------|-------------|
| 审查立场 | 中立审查（双向呈现风险） |
| 合同审查严格度 | 保守型 |
| 商品房审查立场 | 双向提示（购房人与开发商风险均衡呈现） |
| 风险校准 | 标准四级（🔴🟠🟡🟢，建设工程默认触发情形） |
| 文书风格 | 结构化分析（含风险等级清单与修改建议） |
| 地方性规定 | 每次触发地方相关问题时主动提醒核实 |

临时模式可在任何时候通过运行 `/construction-legal:cold-start-interview` 替换为个性化配置。
