# 主体画像与项目组合采集

## 目的

通过结构化访谈，采集用户（律师/法务/开发商/购房人代理）的主体画像与项目组合信息，并将结果填充到 `construction-legal/CLAUDE.md` 的对应章节。本访谈结果是所有 construction-legal 技能个性化的基础——尤其是工作产物标题（律师特权 vs 法务研究笔记）、项目类型识别与资质要求判断。

**可单独触发**：`/construction-legal:identity-project`（只更新主体画像与项目组合部分），或由 `/construction-legal:cold-start-interview` 编排入口作为第一步调用。

---

### 引用脊柱

- 引用规范：`construction-legal/skills/_shared/legal-basis-conventions.md`（用于法律事实合理性检查）
- 配置契约：`construction-legal/skills/_shared/practice-profile-schema.md`（字段映射表）

### 写入目标

本访谈结果填充 `construction-legal/CLAUDE.md` 以下章节：

| 填充章节 | 对应字段 |
|---------|---------|
| `## 我们是谁` | 组织/律所/企业名称；实体类型；建设工程角色；典型合同对手方 |
| `## 我们是谁 > ### 建设工程角色` | 主要代理角色；典型合同对手方 |
| `## 我们是谁 > ### 项目类型组合` | 主要项目类型；典型项目规模与造价区间；主要痛点；团队规模；负责人/联系人 |
| `## 谁在使用` | 角色（律师/法务/监理/造价）；律师联系人 |
| `## 可用集成` | 工程造价咨询/鉴定机构；全国建筑市场监管平台；中国招标投标公共服务平台；企业信用信息系统；网签备案系统；中国裁判文书网；项目管理/OA系统；文档存储；企业微信/钉钉 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 用户跳过某个问题 | 该字段保留 `[PLACEHOLDER]`；访谈继续推进 |
| 用户选「多角色混合」 | 追问主要比例；提示后续在 `review-stance` 子技能中逐项设定立场 |
| 用户为非律师且无律师可咨询 | 明确提醒：所有输出为研究笔记，须律师审查方可作出法律结论；工作产物使用「研究笔记」标题 |
| 用户实体类型与角色不一致（如律所但选「购房人代理」）| 无冲突，保留双重描述 |
| 项目类型含商品房买卖 | 提示：还需通过 `review-stance` 访谈设定商品房审查立场（购房人侧/开发商侧） |
