# 违法解除救济与风险预防（unlawful-termination-remedy）

> **前置**：
> 1. 读取 `employment-legal/CLAUDE.md`（实践配置文件），加载 `## 审查立场 > 解雇审查`
>    和 `## 谁在使用`（服务对象：用人单位方/劳动者方；立场影响救济方向）。
>    若含占位符，进入**临时模式**（见下方）。
> 2. 读取 `employment-legal/skills/_shared/legal-basis-conventions.md`（引用规范）。
> 3. 读取 `employment-legal/skills/_shared/labor-law-citations.md`（引用库，
>    重点：§A.11 违法解除2N第87条、§A.10 经济补偿N第47条）。
> 4. 分工说明：本子技能专注「解错了如何救济和预防」（违法解除后的救济路径与风险预防）；
>    `grounds-legality` 负责「能不能解」（事由合法性，已评估违法风险）；
>    `severance-calculation` 负责「给多少」（合法解除时的补偿计算，2N计算也在此）；
>    `termination-procedure` 负责「怎么解才合规」（程序合规）。
>    本子技能聚焦违法解除认定后的救济路径与用人单位的事前预防。
> 5. 本子技能可被 `employment-legal/skills/termination-review/SKILL.md` 编排入口调用，
>    也可通过 `/employment-legal:unlawful-termination-remedy` 单独触发。

---

**临时模式**：若实践配置文件含占位符，使用以下默认值并标注 `[临时模式]`：

- 违法解除时劳动者可选择：(a)恢复劳动关系 或 (b)2N赔偿金，二者择一 `[临时模式]`
- 用人单位对解除合法性承担举证责任 `[临时模式]`
- 风险预防：事由证据固化 + 规章制度合规 + 通知送达留痕 + 工会程序 `[临时模式]`

## 目的

在解除被认定为违法的情形下，分析劳动者的救济路径（恢复劳动关系 vs. 2N赔偿金），
核查举证责任分配，并为用人单位提供事前风险预防清单，
实现「解错了怎么救济、怎么预防」的完整闭环。

---

## 详细步骤

### 步骤 1：立场确认与违法解除认定来源

首先确认使用立场，决定救济分析方向：

| 立场 | 关注重点 |
|------|---------|
| 用人单位方 | 能否重新合规：撤销解除、补程序？不能则如何将损失最小化（接受2N谈判和解） |
| 劳动者方 | 主张恢复劳动关系还是2N赔偿金，哪个对劳动者更有利？ |

**违法解除常见来源**（须确认触发原因）：

| 来源 | 说明 |
|------|------|
| 解除事由不符合法定情形 | 由 `grounds-legality` 评估得出 |
| 程序瑕疵（未通知工会/未提前通知） | 由 `termination-procedure` 评估得出 |
| 解除对象在禁止解除情形中 | 医疗期/孕产哺乳期/工伤等 |
| 解除事由与实际情形不符 | 虚构解除理由 |

### 步骤 2：劳动者救济路径分析

违法解除时，劳动者有两条法定救济路径（**第87条已核实锚点**），**二者择一，不可兼得**：

#### 路径 A：恢复劳动关系（继续履行劳动合同）

**适用条件**：

- [ ] 劳动者要求继续履行劳动合同（须明确表达该意愿）
- [ ] 劳动合同**能够**继续履行（满足以下全部条件）：
  - 用人单位仍存续（未注销/未破产）
  - 岗位未撤销或可恢复（若岗位已不存在，可能不支持恢复）
  - 劳动关系不存在其他不能继续的客观障碍（如双方矛盾不可调和 `[待验证]`）

**法律后果**：

| 项目 | 内容 |
|------|------|
| 继续履行劳动合同 | 按原合同条款继续（职位/工资/待遇） |
| 中间期工资 | 支付解除之日至恢复劳动关系期间的工资（不低于原工资标准） |
| 是否支付2N | **不支付2N**（恢复劳动关系和2N赔偿金二者择一） |
| 社保补缴 | 通常须补缴中间期社保 `[待验证]` |

**不支持恢复的情形**（须在报告中明确提示）：

| 情形 | 处理 |
|------|------|
| 劳动合同已到期（解除时合同期限已终止） | 一般不支持恢复，转向2N `[待验证]` |
| 用人单位已注销 | 无法恢复，转向2N（向清算组或股东主张） |
| 岗位已不存在（如整个业务线撤销） | 争议，法院/仲裁委裁量 `[待验证]` |
| 合同期已届满（在仲裁/诉讼期间期满） | 转为支付2N（合同期届满后不能恢复） `[待验证]` |

#### 路径 B：违法解除赔偿金 2N（第87条，已核实锚点）

**适用条件**：

- 劳动者不要求恢复劳动关系，或
- 客观上劳动关系已不能继续履行（见上方不支持恢复情形）

**2N 计算**：按 `severance-calculation` 子技能中的2N计算规则执行：

| 步骤 | 说明 |
|------|------|
| 先算N | 按第47条（**已核实锚点**）：月工资基数 × 工龄年限（含封顶规则） |
| 2N = N × 2 | 违法解除赔偿金 = 经济补偿N × 2 |
| 与N互斥 | 支付2N，不另支付经济补偿N |

**重要说明**：

- 2N是**最低法定赔偿**，用人单位不得低于此标准；双方可协商高于2N的赔偿（和解谈判）
- 2N是**赔偿金**性质，与经济补偿N性质不同（一合法一违法），不可叠加

#### 两条路径对比

| 维度 | 路径A：恢复劳动关系 | 路径B：2N赔偿金 |
|------|-------------------|--------------|
| 适用前提 | 劳动者要求 + 合同能继续 | 劳动者不要求恢复 或客观不能恢复 |
| 经济补偿 | 不支付2N，但须支付中间期工资 | 支付2N，与N互斥 |
| 社保 | 须补缴中间期 | 按协议/判决处理 |
| 劳动者利益分析 | 关系存续价值 + 中间期工资；适合长工龄、临近退休、关系重要的情形 | 一次性了结；适合双方关系已破裂、工作已找到的情形 |
| 执行难度 | 用人单位不配合时执行难度较高 | 执行相对简单（金钱给付） |

### 步骤 3：举证责任分析

**核心原则**：**用人单位对解除合法性承担举证责任**（`[待验证]`——此规则在劳动争议实践中高度确定，具体条号须核查）。

| 举证负担 | 由谁承担 | 内容 |
|---------|---------|------|
| 解除事由的合法性 | 用人单位 | 须举证劳动者存在约定情形（如严重违纪的具体事实和证据） |
| 程序合规性 | 用人单位 | 须举证已完成通知工会/提前通知/出具证明等程序 |
| 劳动者签收解除通知 | 用人单位 | 须保留送达记录（签收/快递回执/电子记录） |
| 规章制度公示 | 用人单位 | 须举证规章制度经民主程序制定并告知劳动者 |
| 劳动者过错事实 | 用人单位 | 须有具体事实证据（非仅结论性评价） |

**举证不能的后果**：

| 情形 | 可能结论 |
|------|---------|
| 无法证明解除事由存在 | 解除被认定违法，须支付2N或恢复劳动关系 |
| 无法证明规章制度公示 | 不能以违反规章制度为由解除 |
| 无法证明工会通知 | 程序违法，可能导致解除无效（各地不一 `[待验证]`） |
| 无法证明通知送达 | 解除效力存疑，可能被认为解除通知未生效 |

### 步骤 4：用人单位风险预防清单（事前预防）

> 此清单供用人单位方在解除**前**使用，防患于未然。

#### 4.1 事由证据固化

- [ ] 过失行为的书面记录（违纪记录、处理意见、当事人签字确认）
- [ ] 绩效考核结论的书面证明（含考核标准、结果和劳动者知晓）
- [ ] 培训/调岗的书面记录（「不能胜任工作」前置程序的完整证明链）
- [ ] 客观情况变化的外部证据（政府政策文件/市场公告等）
- [ ] 劳动者欺诈/违规的具体证据（时间/地点/方式/损害金额）

#### 4.2 规章制度合规

- [ ] 规章制度是否经民主程序制定（职工大会或职工代表大会讨论/工会协商）
- [ ] 规章制度是否经公示（公告栏/系统发布/书面告知/入职培训签到）
- [ ] 「严重」标准是否在制度中明确量化（金额/次数/情节描述）
- [ ] 末位淘汰制度是否与「不能胜任工作」认定程序脱钩
- [ ] 违纪行为定义是否清晰，避免歧义

#### 4.3 通知与送达留痕

- [ ] 书面解除通知送达证明（首选当面签收，其次EMS留回执）
- [ ] 解除通知载明解除依据（具体情形，非泛称「违纪」）
- [ ] 非过失性辞退：提前30天通知或代通知金支付记录
- [ ] 协商解除：协议书原件留存（含双方签字、日期）

#### 4.4 工会程序

- [ ] 解除**前**向工会书面通知（留存通知记录）
- [ ] 工会意见是否收到（有书面回复的，留存；无回复的，记录通知已发）
- [ ] 工会异议是否有书面处理说明（若提出异议）

#### 4.5 全流程时间线记录

- [ ] 从事由发现到解除决定的时间线清晰（防止「时过境迁」证据失效）
- [ ] 各程序节点均有书面时间戳（通知日期/工会告知日期/解除生效日期）
- [ ] 与劳动者沟通记录留存（包括协商、培训、调岗等关键节点）

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 恢复劳动关系诉求实际难以执行（用人单位拒不配合） | 告知劳动者：仲裁委/法院可裁决支持恢复，用人单位不执行则视为违法拒绝履行，可强制执行或转为2N赔偿（各地实践 `[待验证]`）；建议优先考虑2N路径以便执行 |
| 仲裁/诉讼期间劳动合同期限届满 | 合同已届满则恢复劳动关系不再可行，转为支付2N赔偿金 `[待验证]` |
| 劳动者主张恢复劳动关系+2N双重请求 | 明确两者不可兼得，须在程序中作出明确选择；通知劳动者须确定一种主张 |
| 用人单位愿意协商和解但劳动者要求高于2N | 2N为法定最低标准，高于2N的和解为合法约定；根据案件风险评估建议合理协商区间 |
| 程序瑕疵但事由合法（如未通知工会但过失事由充分） | 纯程序违法时各地口径不一——有认为程序瑕疵即违法须2N，有认为事由合法可酌情处理 `[待验证]`；建议标🟠并在当地确认口径 |

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 双方对解除合法性争议大，无法判断是否违法 | 出具双路径分析：「若合法解除：N = ¥[  ]；若违法解除：2N = ¥[  ]，或恢复劳动关系」，提示最终结论需仲裁委/法院认定 |
| 用人单位已实际支付N但被认定违法解除 | 须补差额（2N - 已付N）= 额外支付N；已付N一般可抵扣 `[待验证]` |
| 禁止解除情形下强行解除 | 标🔴：强行解除医疗期/孕期等情形的，违法性清晰，支持恢复劳动关系或2N的概率极高；建议立即暂停执行解除方案，重新评估 |
| 劳动者未及时申请仲裁（时效问题） | 提示：劳动争议仲裁时效1年（《劳动争议调解仲裁法》第27条，**已核实锚点**），从知道或应当知道权利受侵害之日起算，迟延申请可能丧失仲裁权利 |
