# 审查立场访谈（review-stance）

> **所属编排**：`employment-legal/skills/cold-start-interview/SKILL.md`（冷启动访谈）
> **遵循契约**：`employment-legal/skills/_shared/practice-profile-schema.md` 字段映射（§ 2.2–2.4）
> **写入目标**：`employment-legal/CLAUDE.md` — `## 审查立场`（`### 雇佣审查`、`### 解雇审查`）
> **分工说明**：身份与团队由 `identity-team` 收集；风险偏好/升级矩阵由 `risk-escalation` 收集；文书风格由 `style-local-rules` 收集。

## 目的

通过结构化访谈明确律师/HR 的审查立场（用人单位方或劳动者方）及各审查维度的标准立场，写入 `employment-legal/CLAUDE.md` 的 `## 审查立场` 章节，使所有劳动人事技能：

1. 在雇佣审查中按试用期/工时/社保/竞业的标准立场生成定制建议；
2. 在解雇审查中按经济补偿倾向（N/N+1/2N）和法定解除情形内部认定标准运行；
3. 正确加载用人单位方（合规风险预防）或劳动者方（权益主张）视角。

## 访谈问题清单

### 第一部分：服务对象（核心立场）

**Q1. 在劳动法审查中，你主要代表哪一方？**

> 写入：`## 谁在使用` → 服务对象（跨 identity-team 的字段，如未填充则在此收集）

| 选项 | 加载视角 |
|------|---------|
| **用人单位方（雇主）** | 合规与风险预防视角：条款是否暴露用人单位于劳动争议风险；如何在合法前提下保护用人单位利益 |
| **劳动者方（员工）** | 权益主张视角：条款是否侵犯劳动者法定权利；无效条款识别；可主张的权利与救济路径 |
| **双方（依事项）** | 按事项提示，运行时询问「本次审查代表哪一方？」 |

---

### 第二部分：雇佣审查立场（收集 → 写入 `## 审查立场 → ### 雇佣审查`）

**Q2. 关于试用期审查，你们的标准立场是什么？**

> 写入：`## 审查立场` → `### 雇佣审查` → `#### 试用期` → 标准立场 + 特殊情况

合理性检查基准（《劳动合同法》第19条，已核实）：
- 合同期限 3个月以上不满1年 → 试用期最长 1 个月
- 1年以上不满3年 → 最长 2 个月
- 3年以上及无固定期限 → 最长 6 个月

典型立场选项：
- 用人单位方：「按合同期限在法定上限内约定，倾向约定至上限」
- 劳动者方：「核实是否超过法定上限，超过部分无效，另支付工资差额」

---

**Q3. 关于工时制度，你们通常使用哪种工时制？审查立场是什么？**

> 写入：`## 审查立场` → `### 雇佣审查` → `#### 工时制度`

| 工时制类型 | 说明 |
|----------|------|
| 标准工时制 | 每日不超过8小时，每周不超过40小时（行政法规规定） |
| 综合计算工时制 | 须经劳动行政部门批准，适用特殊行业 `[待验证]` |
| 不定时工时制 | 须经劳动行政部门批准，适用特殊岗位 `[待验证]` |

合理性提示：「综合计算工时制/不定时工时制须经劳动行政部门批准；未经批准自行约定，加班费计算仍按标准工时制处理 `[前提 flagged — 需验证]`」

---

**Q4. 关于社会保险，你们的标准立场是什么？**

> 写入：`## 审查立场` → `### 雇佣审查` → `#### 社会保险`

合理性检查基准（已核实规则）：
- 「自愿放弃社保」约定无效，用人单位仍须依法缴纳五险
- 以「社保补贴」代替缴纳无效
- 缴纳基数须合规（不低于法定下限）

典型立场选项：
- 用人单位方：「严格按法定标准缴纳，拒绝任何合同中的免缴约定」
- 劳动者方：「核实是否足额缴纳，追缴欠缴社保及利息 `[待验证]`」

---

**Q5. 关于竞业限制，你们适用于哪些人员？经济补偿标准是多少？期限和范围如何定义？**

> 写入：`## 审查立场` → `### 雇佣审查` → `#### 竞业限制`

合理性检查基准（《劳动合同法》第23-24条，已核实）：
- 适用人员：高级管理人员、高级技术人员和其他负有保密义务人员（不得扩大到全体员工）
- 竞业期限：不得超过 **2 年**（用户若口述3年，提示：「第24条规定上限2年，3年约定中超出部分无效 `[前提 flagged — 需验证]`」）
- 须按月支付经济补偿（补偿标准参考地方指引，一般不低于离职前12个月平均工资的一定比例 `[管辖 flagged — 需验证地方性规定]`）
- 竞业范围：须限于与本单位有竞争关系的业务范围

若用户口述竞业补偿比例过低（如5%以下），提示：「补偿过低可能被认定为无效或劳动者有权单方解除竞业义务（具体标准参照地方指引 `[待验证]`）`[前提 flagged — 需验证]`」

### 第三部分：解雇审查立场（收集 → 写入 `## 审查立场 → ### 解雇审查`）

**Q6. 关于过失性辞退，你们内部的认定标准是什么？**

> 写入：`## 审查立场` → `### 解雇审查` → `#### 法定解除情形` → 过失性辞退

合理性提示：过失性辞退须依据已生效并公示的规章制度认定，不得事后补充 `[前提 flagged — 需验证]`

---

**Q7. 关于非过失性辞退（医疗期满/不胜任/客观情况变化），你们的内部处理流程是什么？**

> 写入：`## 审查立场` → `### 解雇审查` → `#### 法定解除情形` → 非过失性辞退

合理性提示：须提前30天书面通知（或额外支付一个月工资代替通知）`[前提 flagged — 需验证]`

---

**Q8. 关于经济性裁员，你们有什么审批流程要求？**

> 写入：`## 审查立场` → `### 解雇审查` → `#### 法定解除情形` → 经济性裁员

合理性提示：裁员20人以上或占10%以上，须提前向工会或全体职工说明并听取意见 `[第41条，待验证]`

---

**Q9. 关于经济补偿，你们的计算倾向是什么？**

> 写入：`## 审查立场` → `### 解雇审查` → `#### 经济补偿`

| 倾向 | 说明 |
|------|------|
| 法定 N | 每满一年支付一个月工资（《劳动合同法》第47条，已核实） |
| N+1 | 法定 N 基础上额外支付一个月工资，常见于非过失性辞退（提前通知代偿） |
| 2N | 违法解除赔偿金（《劳动合同法》第87条，已核实）；注意 2N 与 N 互斥 |
| 高于法定（如主动提升补偿以降低争议风险） | 填写具体标准 |

合理性检查：
- 「高于三倍平均工资部分，经济补偿年限封顶12年，月工资按三倍封顶」（《劳动合同法》第47条，已核实）
- 若用户口述 2N 用于合法解除，提示：「2N 是违法解除的赔偿金，不适用于合法解除；合法解除支付 N 或 N+1 `[前提 flagged — 需验证]`」

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 用人单位方与劳动者方同时服务 | 记录为「双方（依事项）」，运行时每次询问当次立场 |
| 竞业限制期限超过2年 | 标注 `[前提 flagged — 需验证]`，提示法定上限为2年（第24条） |
| 经济补偿倾向未明确（如「视情况而定」） | 记录为「依法定标准，具体情形具体判断」，后续技能按法定 N 计算并提示确认 |
| 用户不清楚自己代表哪方 | 询问：「在你通常处理的劳动事务中，你是为用人单位提供服务（帮助公司合法用工、降低劳动争议风险）还是为劳动者提供服务（帮助员工主张权利、索取赔偿）？」 |

## 错误处理

| 错误场景 | 处理方案 |
|---------|---------|
| 竞业限制期限口述为3年 | 提示第24条规定不超过2年；询问是按2年写入还是保留3年并标 `[前提 flagged]` |
| 经济补偿口述「违法解除也支付N就够了」 | 提示：违法解除须支付2N（第87条，已核实）；N 与 2N 互斥，不可选低标准 |
| 将「社保补贴」作为标准立场写入 | 警告：以社保补贴代替缴纳无效（已核实规则），建议修改立场为「按法定标准缴纳，不约定补贴代缴」 |
