# 风险偏好与升级矩阵访谈（risk-escalation）

> **所属编排**：`employment-legal/skills/cold-start-interview/SKILL.md`（冷启动访谈）
> **遵循契约**：`employment-legal/skills/_shared/practice-profile-schema.md` 字段映射（§ 2.5–2.6）
> **写入目标**：`employment-legal/CLAUDE.md` — `## 风险校准` / `## 升级矩阵`
> **分工说明**：身份信息由 `identity-team` 收集；审查立场由 `review-stance` 收集；文书风格由 `style-local-rules` 收集。
> **可单独触发**：`/employment-legal:risk-escalation`（只更新风险校准与升级矩阵，不影响其他章节）

---

## 目的

通过结构化访谈确认律师/HR 团队的风险偏好等级定义，以及升级矩阵（谁可以独立决策、什么情况下必须升级），写入 `employment-legal/CLAUDE.md`，使所有劳动人事技能：

1. 用统一口径标注风险等级（🔴/🟠/🟡/🟢 的含义基于你的团队定义）；
2. 在产生升级信号时，自动路由到正确的升级联系人；
3. 识别自动升级触发条件（竞业限制、解雇、群体性事件等高风险场景）。

---

1. **读取 `employment-legal/CLAUDE.md`** — 检查 `## 风险校准`、`## 升级矩阵` 是否已有非 `[PLACEHOLDER]` 内容（若已有，提示是否覆盖更新）

## 访谈问题清单

### 第一部分：风险等级确认（收集 → 写入 `## 风险校准`）

**Q1. 在劳动法审查中，以下风险等级对你们而言含义是否需要调整？**

> 写入：`## 风险校准` → 劳动争议风险等级表

系统默认风险定义（可确认或调整）：

| 等级 | 默认含义 | 你们的调整（如有） |
|-----|---------|----------------|
| 🔴 关键 | 明显违反强制性规定，几乎肯定败诉；例：医疗期内违法解除、三期解雇、未付经济补偿2N | |
| 🟠 高 | 有较大法律风险，可能败诉；例：竞业补偿明显过低、规章制度未经民主程序 | |
| 🟡 中等 | 存在争议空间，取决于证据和仲裁员/法官裁量；例：过失认定证据不充分、日工资基数计算方式有争议 | |
| 🟢 低 | 合规或有充分依据；法定标准完全符合 | |

> 大多数团队无需调整默认定义，确认后直接写入即可。

**Q1 跟进问题**：在你们的实务中，有没有某类劳动法问题，你们视之为「必须升级不可自行处理」而默认定义未包含的？

> 如有，记录为额外触发条件

### 第二部分：升级矩阵（收集 → 写入 `## 升级矩阵`）

**Q2. 你们团队的主要角色有哪些？各自处理劳动事务的权限是什么？**

> 写入：`## 升级矩阵` → 角色 / 可独立处理事项

常见角色示例（用户可修改）：

| 角色 | 参考可独立处理范围 |
|------|----------------|
| HR 专员/助理 | 日常入职文件签署、常规假期审批、社保登记 |
| HR 经理/法务助理 | 一般劳动合同审查、常规离职手续、病假医疗期管理 |
| 主办律师/法务经理 | 解雇方案审查、经济补偿计算、竞业限制执行评估 |
| 法务总监/合伙人 | 无上限，包括群体性争议、仲裁/诉讼策略、重大解雇决策 |

---

**Q3. 什么情况下必须升级给上一级？（需升级条件）**

> 写入：`## 升级矩阵` → 需升级条件

请填写每个角色的升级触发条件。建议至少包含：

- HR 专员/助理升级条件：__（如：涉及解雇、竞业限制、劳动争议申诉）
- 主办律师/法务经理升级条件：__（如：集体性争议、超出X万元的经济补偿决策、诉讼立案风险）
- 升级到法务总监/合伙人条件：__（如：群体性争议、媒体关注风险、监管调查）

---

**Q4. 以下场景是否自动触发升级给最高层（不论处理者级别）？**

> 写入：`## 升级矩阵` → 自动升级触发条件

系统默认自动升级触发（可确认或补充）：

| 触发场景 | 默认设置 | 你们的调整 |
|---------|---------|---------|
| 竞业限制纠纷（涉及违约金主张或竞业协议有效性争议） | 自动升级 | |
| 解雇或终止劳动合同（非常规离职） | 自动升级 | |
| 群体性劳动事件（多人同时主张权益、集体停工风险） | 自动升级 | |
| 仲裁/诉讼立案（已进入争议程序） | 自动升级 | |
| 监管检查/劳动局稽查 | 自动升级 | |
| 媒体/社交媒体曝光风险 | 确认后升级 | |

---

**Q5. 有没有需要「立即响应」的高优先场景（时效要求极高）？**

> 写入：`## 升级矩阵` 附加说明

劳动争议领域常见时效触发：
- 仲裁时效：1年（《劳动争议调解仲裁法》第27条，已核实），接近到期须立即响应
- 不服裁决起诉：收到裁决书后 15 日内 `[待验证]`
- 一裁终局用人单位申请撤销：裁决书送达后 30 日内 `[待验证]`

## 法律事实合理性检查

本访谈子技能涉及时效参照，如用户口述以下内容，须进行合理性检查：

| 用户口述场景 | 检查提示 |
|------------|---------|
| 「仲裁时效是3年」 | 劳动争议仲裁时效为1年（《劳动争议调解仲裁法》第27条，已核实），非民法典的3年时效；记录并标注 `[前提 flagged — 需验证]` |
| 「不服裁决有1个月时间起诉」 | 一般为15日，但一裁终局用人单位申请撤销为30日 `[待验证]`；提示确认具体情形 |
| 「群体性纠纷我们自己处理就好」 | 提示建议列为自动升级触发，风险较高 `[前提 flagged — 建议确认]` |

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 团队只有1-2人 | 升级矩阵简化为「独立处理」和「须外部法律建议」两层 |
| 无明确升级联系人 | 记录为「未配置，升级时询问」并在配置文件中标注 |
| 风险偏好「保守型」 | 🟡 中等风险标记提升为需关注项，建议修改而不仅是提示 |
| 风险偏好「进取型」 | 🟠 高风险项降低为提醒项，但🔴关键项始终保持升级触发 |

## 错误处理

| 错误场景 | 处理方案 |
|---------|---------|
| 升级矩阵中角色职位不清晰（如「某人」） | 询问职位/头衔，确保升级路由可操作 |
| 用户设置「任何事项都自己处理」 | 保留记录，但提示：「建议至少保留群体性事件/仲裁立案为自动升级触发，以控制风险」 |
| 风险等级定义与已核实规则冲突（如「违法解除只是🟡」） | 提示：「违法解除（2N赔偿金，第87条已核实）通常为🔴，设为🟡可能导致风险提示不足。是否确认调整？」 |
