# 法律依据引用规范（监管合规版）

> 本文件定义 regulatory-legal 插件所有技能统一使用的法律依据引用规范。
> 后续计划（10-02、10-03）直接引用本规范，避免各技能重复维护引用规则。
> **监管合规直接关涉企业与个人的行政责任、罚款与许可存续，以及潜在的行刑衔接风险。引用错误条号、文号或数额会直接误导处罚应对策略与合规判断。本规范特别强调行政处罚法 2021 条号重排保守标注纪律与行业监管规章快速演进标注纪律。**

---

## 一、来源分类

所有法律依据必须归入以下类别之一，并在引用时明确标注类别标签。

### 1. 法律（全国人大及其常委会制定）

指全国人民代表大会及其常务委员会制定的法律，监管合规语境中主要包括：

- 《中华人民共和国行政处罚法》（2021年修订，全国人大常委会；**条号已重排，须用 2021 修订后新条号，中度把握锚点见本规范第四章**）
- 《中华人民共和国行政许可法》（全国人大常委会制定；具体条号把握不足的**一律描述规则 + `[待验证]`**）
- 《中华人民共和国行政强制法》（全国人大常委会制定；具体条号**一律描述规则 + `[待验证]`**）
- 《中华人民共和国行政复议法》（含修订版本；复议申请期限与前置情形的具体条号**一律 `[待验证]`**）
- 《中华人民共和国行政诉讼法》（含修订版本；起诉期限、举证责任倒置等具体条号**一律 `[待验证]`**）
- 《中华人民共和国反不正当竞争法》《中华人民共和国产品质量法》《中华人民共和国广告法》《中华人民共和国安全生产法》《中华人民共和国环境保护法》《中华人民共和国个人信息保护法》《中华人民共和国数据安全法》等（作为行政违法的前置行为规范参照；具体条号**一律标 `[待验证]`**）

**法律层级最高，是监管合规分析的核心成文法基础。**

> **监管合规法律引用特别提示（必须遵守）**：行政处罚法 2021 修订对条文进行了较大重排，中度把握锚点（第4/9/28/29/32/33/36/44/45/58/63条）可在标「已核实锚点（中度把握），建议复核」前提下引用，**务必用 2021 修订后新条号，不得使用旧版条号**；行政许可法·行政强制法·行政复议法·行政诉讼法的具体条号（复议前置·起诉期限·举证责任倒置等）作者把握不足的，**一律描述规则 + `[待验证]`，绝不臆造条号**。

**格式要求（法律最小引用格式）：**

```
《[法律全名]》第[X]条  [来源标签]  [验证状态]
```

示例（行政处罚法 2021 中度把握锚点）：
```
《中华人民共和国行政处罚法》第9条  `[模型知识 — 需验证]`  已核实锚点（中度把握），建议复核
```

示例（其他行政法律 — 一律待验证）：
```
《中华人民共和国行政复议法》规定：当事人不服行政决定可申请行政复议  `[模型知识 — 需验证]`  `[待验证]`
```

### 2. 行政法规（国务院制定）

指国务院制定的条例、规定等，监管合规语境中主要包括：

- 各行业监管领域的国务院条例（如食品安全相关条例、互联网信息服务相关规定、药品管理相关法规等）

**核心纪律（必须遵守）**：

> **行政法规的具体条号把握不足时，一律描述规则并标 `[待验证]`，不臆造条号。** 行政法规层级仅次于法律，是行业监管的重要依据。

**格式要求：**

```
《[行政法规全名]》（国务院）规定：[描述规则]  [来源标签]  `[待验证]`
```

### 3. 部门规章（各监管机关制定）

指各行业监管机关制定的规章，监管合规语境中主要包括：

- 市场监管总局发布的各类监管规章（市场竞争·广告·产品质量·价格·反垄断·消费者权益等）
- 国家互联网信息办公室（网信办）发布的网络数据安全与算法规章
- 国家工信部发布的电信业务与工业产品规章
- 金融监管总局·中国人民银行·中国证监会发布的金融业务规章
- 国家药品监督管理局发布的药品医疗器械规章
- 生态环境部发布的环境保护规章
- 应急管理部发布的安全生产规章

**核心纪律（必须遵守）**：

> **各行业监管部门规章属快速演进文件，一律描述规则并标 `[待验证 — 行业监管规章快速演进，须核实现行版本]`，不硬写文号、发文日期、具体条款编号、罚款数额区间。** 行业监管规章数量庞大、修订频繁，错误的文号细节会直接误导合规判断。

**格式要求：**

```
[监管机关]发布的关于[监管事项]的规章规定：[描述规则]  [来源标签]  `[待验证 — 行业监管规章快速演进，须核实现行版本]`
```

### 4. 规范性文件（行政处罚裁量基准·行业合规指引等）

指各行政机关发布的规范性文件，监管合规语境中主要包括：

- 各地各部门制定的行政处罚裁量基准（将罚款数额与情节按阶次细化，地方差异大）
- 各监管机关发布的行业合规指引与自查指南
- 行政执法程序规范性文件

**核心纪律（必须遵守）**：

> **规范性文件与行政处罚裁量基准属快速演进且地方差异大的文件，一律描述规则并标 `[待验证]` 和 `[地方 flagged]`，不硬写文号·裁量数值·罚款区间。** 裁量基准的数值具体适用须以当地现行版本核实。

**格式要求：**

```
[规范性文件名称描述]规定：[描述规则]  [来源标签]  `[待验证 — 规范性文件快速演进，须核实现行版本]`  `[地方 flagged]`（如涉地方版本）
```

### 5. 国家标准（GB/T 系列及行业标准）

指国家市场监督管理总局·国家标准化管理委员会发布的推荐性国家标准，以及行业主管部门发布的行业标准，监管合规语境中主要包括：

- 企业合规管理体系国家标准（GB/T 系列，规定合规管理体系的要素）
- 相关行业国家标准（信息安全·产品质量·环境管理等）

**核心纪律（必须遵守）**：

> **国家标准编号与版本须核实，描述规则 + `[待验证]`，不臆造标准号。** GB/T 标准有版本演进，错误的标准号直接误导合规体系建设。

**格式要求：**

```
企业合规管理体系相关国家标准（GB/T [待验证]）规定：[描述规则]  [来源标签]  `[待验证 — 标准编号与版本须核实]`
```

### 6. 案例（最高院行政指导性案例·行政复议与诉讼典型案例）

监管合规领域可参考的案例类型：

- **最高院行政指导性案例**：最高人民法院发布，展示行政处罚合法性审查·陈述申辩·不予处罚等典型情形，**仅作参考，不具判例法约束力**
- **各监管机关典型案例**：各行业监管机关公开发布的执法典型案例，**仅作参考，不具判例法约束力**

**核心纪律（必须遵守）**：

> **所有案例案号一律标 `[待验证]`，未经检索核验不得移除该标注。** 中国采成文法体系，案例不具判例法约束力，须明确说明。

**格式要求：**

```
[案例名称或案号 `[待验证]`]  [要点摘要]  [发布机关]  参考，不具判例法约束力  `[待验证]`
```

### 7. 学说（学术观点）

学说为辅助参考，存在争议性，引用时须标注来源和争议性：

```
[作者姓名]，《[著作/论文名]》，[出版/期刊信息]  "[观点摘要]"  [是否有争议：有/无]  `[待验证]`
```

监管合规常见学说争议点：行政处罚比例原则的适用、一事不再罚的认定边界、不予处罚「无主观过错」的举证、合规管理体系有效性认定、行刑衔接的触发标准等。

## 二、来源标签表

引用时，在条文/案例/观点末尾附加来源标签。复用五个标准标签，不新造冲突标签。

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| `[法宝]` | 来自北大法宝等法规数据库 | 检索工具获取的全文 |
| `[法规网站]` | 来自政府官方网站（全国人大·各监管机关官网） | 官网原始文件 |
| `[网络搜索 — 需验证]` | 来自网络搜索结果 | 需对照原始来源核查 |
| `[模型知识 — 需验证]` | 来自训练数据回忆 | 风险较高，须优先核查；行业监管规章文号·罚款数额·国家标准号风险尤为突出 |
| `[用户提供]` | 来自用户提供的文件 | 处罚决定书·合规制度文件·监管通知等 |

**标签使用规则：**

1. 每条引用行**必须且只能**附加一个来源标签
2. 禁止剥离或折叠标签
3. `[模型知识 — 需验证]` 来源在依赖前**必须**对照原始来源检查——行业监管规章文号·罚款数额·国家标准号风险突出
4. 来源标签不替代 `[待验证]` 标注——两者可同时出现在一行

## 三、待验证标记规则

以下情形**必须**在引用行末追加 `[待验证]` 并说明验证途径：

| 触发情形 | 说明 | 建议验证途径 |
|---------|------|-------------|
| 来源标签为 `[模型知识 — 需验证]` 的具体条号 | 模型回忆的条文号风险高 | 国家法律法规数据库 flk.npc.gov.cn |
| **行政许可法·行政强制法·行政复议法·行政诉讼法的具体条号** | 作者把握不足，极易错误回忆 | 国家法律法规数据库核实现行版本 |
| **各行业监管部门规章/规范性文件的文号·发文日期·条款编号** | 快速演进，文号细节极易过时 | 各监管机关官网核实现行版本 |
| **行政处罚裁量基准的罚款数额区间·裁量阶次数值** | 地方差异大，快速演进 | 当地监管机关或政府官网核实 |
| **企业合规管理体系国家标准（GB/T）编号与版本** | 标准有版本演进 | 国家标准全文公开系统核实 |
| 司法解释具体条号 | 行政诉讼司法解释修订频繁 | 最高人民法院官网 court.gov.cn |
| 距最后更新超过 6 个月的法规 | 法规可能已修订或废止 | 查询官方数据库最新版本 |
| 所有案例引用（默认） | 案例案号极易错误回忆 | 中国裁判文书网、各机关官网 |

## 四、行政处罚法 2021 条号重排保守标注纪律

> **这是监管合规引用最重要的纪律之一，必须严格遵守。**

### 4.1 背景：行政处罚法 2021 修订条号重排

行政处罚法 2021 年修订对条文进行了较大重排，旧版条号与 2021 修订后新条号不同。模型回忆旧版条号的风险极高，**必须用 2021 修订后新条号并标建议复核，不确定就描述规则**。

### 4.2 行政处罚法 2021 中度把握锚点（可引用，须标「建议复核」）

以下条号为行政处罚法（2021 年修订）中度把握锚点，可在标「已核实锚点（中度把握），建议复核」前提下引用，**务必用以下 2021 修订后新条号**：

| 锚点 | 条号（2021 新） | 规则摘要 |
|------|---------------|---------|
| 处罚法定原则 | **第4条** | 没有法定依据或者不遵守法定程序的，行政处罚无效 |
| 行政处罚种类 | **第9条** | 警告·通报批评·罚款·没收违法所得没收非法财物·暂扣或者吊销许可证件·降低资质等级·限制开展生产经营活动·责令停产停业·责令关闭·限制从业·行政拘留 等 |
| 责令改正 | **第28条** | 实施行政处罚时应当责令当事人改正或者限期改正违法行为 |
| 一事不再罚 | **第29条** | 对当事人的同一个违法行为，不得给予两次以上罚款的行政处罚 |
| 从轻减轻处罚 | **第32条** | 主动消除或者减轻危害后果·受他人胁迫诱骗·主动供述配合查处有立功表现 等情形应当从轻或者减轻 |
| 不予处罚 | **第33条** | 违法行为轻微并及时改正没有造成危害后果不予处罚；初次违法且危害后果轻微并及时改正可以不予处罚；当事人有证据足以证明没有主观过错不予处罚 |
| 追责时效 | **第36条** | 违法行为在二年内未被发现的不再给予行政处罚；涉及公民生命健康安全·金融安全且有危害后果的延长至五年 |
| 陈述申辩权 | **第44条、第45条** | 当事人有权进行陈述和申辩；行政机关不得因当事人陈述申辩而给予更重的处罚 |
| 法制审核 | **第58条** | 在作出重大行政处罚决定之前应当进行法制审核 |
| 听证 | **第63条** | 责令停产停业·吊销许可证件·较大数额罚款 等较重处罚，当事人要求听证的应当组织听证 |

### 4.3 其他行政法律具体条号一律待验证

**关键纪律（硬性要求）**：

> **行政许可法·行政强制法·行政复议法·行政诉讼法的具体条号（复议前置·起诉期限·举证责任倒置等）作者把握不足的，一律以规则描述呈现并标 `[待验证]`，绝不臆造具体条号。**

须谨慎处理的高频事项（一律规则描述 + 条号待验证）：

| 事项类型 | 处理原则 |
|---------|---------|
| 行政复议申请期限 | 描述规则（自知道行政行为之日起有一定期限内申请复议），条号 `[待验证]` |
| 行政复议前置情形 | 描述规则（部分情形须先复议再诉讼），条号 `[待验证]` |
| 行政诉讼起诉期限 | 描述规则（自知道行政行为之日起有一定期限内起诉），条号 `[待验证]` |
| 行政诉讼举证责任倒置 | 描述规则（被告对作出的行政行为负举证责任），条号 `[待验证]` |
| 行政强制措施种类与程序 | 描述规则（查封·扣押等的法定程序），条号 `[待验证]` |
| 行政许可申请·延续·变更程序 | 描述规则，条号 `[待验证]` |

## 五、行业监管规章快速演进标注纪律

> **这是监管合规引用的另一关键纪律，必须严格遵守。**

### 5.1 快速演进背景

各行业监管部门规章·规范性文件·行政处罚裁量基准数量庞大、修订频繁、地方差异大。市场监管·网信·金融·药监·生态环境·应急管理等领域的规章快速迭代，具体文号、发文日期、条款编号、罚款数额区间、裁量阶次随时可能更新。

### 5.2 快速演进标注规则

**一律描述规则并标 `[待验证 — 行业监管规章快速演进，须核实现行版本]`，不硬写文号、发文日期、具体条款编号、罚款数额区间、裁量阶次具体数值。**

| 文件类型 | 快速演进程度 | 处理方式 |
|---------|------------|---------|
| 各监管机关部门规章 | 高（修订频繁） | 描述规则，`[待验证 — 行业监管规章快速演进]` |
| 行政处罚裁量基准 | 高（地方差异大，数值快速变化） | 描述规则，`[待验证]`，标 `[地方 flagged]` |
| 行业合规指引与自查指南 | 高（持续更新） | 描述规则，`[待验证 — 行业监管规章快速演进]` |
| 国家标准（GB/T） | 中（有版本演进） | 描述规则，`[待验证 — 标准编号与版本须核实]` |

## 六、成文法优先与案例定位

### 成文法优先原则

1. **分析基础**：监管合规法律分析的所有结论，必须首先援引现行成文法（行政处罚法·行政许可法·行政强制法·行政复议法·行政诉讼法及相关法律）作为规范依据
2. **不得以案例替代法条**：即使存在典型案例，也必须同时援引对应法条或规则描述
3. **法条缺失时**：若无直接成文法，须明确说明「无直接成文法规定，参照以下规范性文件/典型案例/学说」

### 案例的辅助定位

1. **最高院行政指导性案例**：标注「最高院指导，参照意义强，不具判例法约束力」
2. **各监管机关典型案例**：标注「[机关]典型案例，参照，不具判例法约束力」
3. **地方典型案例**：参考价值有限，标注「地方案例，参照，不具判例法约束力，地区差异大」
4. **明确说明非判例法**：中国采成文法体系，行政指导性案例/典型案例不具判例法约束力

## 七、法院/行政复议机关/监管机构层级与定位

### 法院层级（行政诉讼）

| 层级 | 说明 | 行政案件特点 |
|------|------|------------|
| 最高人民法院 | 全国最高审判机关 | 指导性案例与司法解释发布；行政再审监督 |
| 高级人民法院 | 省级，行政上诉二审 | 重大行政案件二审 |
| 中级人民法院 | 地市级，部分行政案件一审；行政上诉二审 | 中央机关被告的行政案件通常由中级法院管辖 |
| 基层人民法院 | 县区级，**一般行政案件通常由基层法院一审** | 大多数行政处罚、行政许可类案件的一审 |

### 行政复议机关（行政复议）

| 情形 | 复议机关 |
|------|---------|
| 对县级以上地方各级政府工作部门的决定不服 | 该部门的本级政府或者上一级主管部门 `[待验证]` |
| 对地方各级政府的决定不服 | 上一级政府 `[待验证]` |
| 对国务院各部委的决定不服 | 作出决定的部委或者国务院 `[待验证]` |

> **行政复议前置情形（复议前置须先复议再起诉）**：具体法定情形及对应条号 `[待验证]`，引用时须核实现行行政复议法与相关行业法律中的前置规定。

**重要说明**：行政复议是行政诉讼的重要前置程序（部分情形），也是获得权利救济的高效途径——费用低、周期相对短。复议决定后不服可再提起行政诉讼（复议后再诉，具体衔接方式 `[待验证]`）。

### 行业监管机构（中央与地方两级）

| 监管领域 | 中央机构 | 地方对应机构 | 主要监管权力 |

## 八、监管合规立场定位

> **这是监管合规插件的根本立场，任何立场不得覆盖本章节的护栏规则。**

### 8.1 本插件定位

本插件（regulatory-legal）**仅用于行政监管合规预防、行政处罚依法应对、合规体系建设与监管报告起草**。所有输出立足于帮助企业事前合规、依法应对处罚、如实报送监管报告。

### 8.2 绝对禁止提供的内容

**绝不提供以下任何内容：**

- 逃避监管·规避检查的方法
- 虚假报送·伪造合规证据的建议
- 阻碍行政执法·对抗调查的方案
- 隐匿违法事实·销毁证据的操作
- 任何以「合规」为名实质上规避监管义务的方法

**识别到上述需求时，一律拒绝，并提示**：监管合规的正道是依法合规·如实申报·依法行使陈述申辩与复议诉讼救济权利，输出标 `[合规边界 — 拒绝]`。

### 8.3 行刑衔接强制升级

识别到行政违法**可能升格刑事**（如违法所得数额·情节达到刑事追诉标准、监管机关移送公安）、重大处罚（吊销许可·责令停产停业·较大数额罚款·限制从业）或涉及**责任人人身自由风险**（如行政拘留）时，**强制升级给执业律师**，输出标 `[强制升级 — 行刑衔接]` 或 `[强制升级 — 重大处罚]`。本插件不替代刑事辩护工作，行刑衔接情形提示与 criminal-compliance 插件衔接。
