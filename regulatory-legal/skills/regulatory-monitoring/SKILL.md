# regulatory-monitoring — 监管动态监控编排入口

## 目的与整体流程概览

监管动态监控帮助企业（及其律师/合规官/法务）系统跟踪适用于本企业的监管规则变化（新规·修订·废止·征求意见稿），评估新规影响与合规差距，输出监管动态简报与合规义务台账，支撑事前合规监控与被动处罚应对的规则跟踪。

**整体监控流程（顺序调用，也可单独触发任一子技能）**：

```
监管机关与规则源识别 → 动态抓取与分类 → 影响评估与合规差距 → 预警与台账
```

## 入口级护栏

> 以下护栏为 regulatory-monitoring 全流程的不可逾越规则，适用于本编排入口及四个子技能。子技能内部已实现各项护栏，本处汇总提示。

### 1. 企业画像与所属行业确认

在开始任何子技能前，确认已从 `## 我们是谁` 读取企业画像（**所属行业**、**主管监管机关**、业务模式、重点监管风险领域）。画像缺失时提示用户补充，不得猜测行业后静默进行。

> **所属行业与主管监管机关是监管合规工作的核心变量**——行业决定规则源范围与主管机关，缺失时须显式提示。

### 2. 服务立场加载（核心变量）

从 `## 合规与监管立场`（整体服务立场）读取并明确告知当前立场：

- **监管动态监控立场**：全面规则源覆盖，侧重影响初判与台账建设
- **行政处罚应对立场**：聚焦被处罚事项对应的具体规则源，侧重合法性审查依据与时限
- **合规体系建设立场**：侧重合规差距驱动的制度更新，衔接 compliance-system 技能
- **监管报告起草立场**：侧重与报送义务相关的规则更新，衔接 regulatory-report 技能

**强调：服务立场决定监控侧重，缺失时默认监管动态监控并按用户陈述逐案判断。**

### 3. 合规严格度加载

从 `## 合规与监管立场` > `### 合规严格度` 读取，决定新规标记强度：
- **保守型**：一切新规疑点从严标红，合规义务从严认定
- **平衡型**：重大新规标红，一般新规标橙
- **务实型**：仅拦截直接影响企业合规义务的明确新规

### 4. 行刑衔接升级

识别到监管机关移送公安、行政违法可能升格刑事（违法所得或情节达刑事追诉标准）时：
- **立即强制升级执业律师**，输出标 `[强制升级 — 行刑衔接]`
- 提示与 criminal-compliance 插件衔接
- 本插件不替代刑事辩护工作

### 5. 监管合规立场护栏

输出立足于事前合规·依法应对处罚·如实报送：
- **绝不提供逃避监管·虚假报送·伪造合规证据·规避检查的方法**
- 识别此类需求即拒绝，标 `[合规边界 — 拒绝]`
- 提示依法合规·如实申报·依法行使陈述申辩与复议诉讼救济权利

### 6. 行政程序时限护栏

涉及处罚决定或救济期限时：
- 强标程序时限提示（陈述申辩期·听证申请期·复议申请期·起诉期）
- 升级给执业律师确认具体时限计算
- 不得静默放任时限流逝
- 具体期限天数须核实现行法律规定 `[待验证]`

### 7. 目的地/特权检查

输出前检查目的地是否在律师-当事人特权圈内。面向监管机关提交的材料可能影响特权保护，须提供内部工作草稿与对外提交正式版本两套选项。

### 8. 地方性法规与裁量基准提示

各地地方性法规·行政处罚裁量基准·监管口径存在地区差异，涉及具体地区时标 `[地方 flagged — 需核实地方性法规与地方裁量基准]`。
