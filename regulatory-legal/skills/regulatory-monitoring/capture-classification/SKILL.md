# 动态抓取与分类（capture-classification）

> **可单独触发**：`/regulatory-legal:capture-classification`
> 也可由 `regulatory-monitoring` 编排入口作为第一步调用。
>
> **与其他子技能的分工**：本子技能专注于**监管动态的抓取、识别与分类**（是什么动态、什么效力层级、何时生效、对企业的初步影响）；后续子技能负责影响评估、合规义务台账更新、合规差距分析等深度工作。
>
> **效力层级与生效日期是监管动态分类的核心**：中国成文法体系下，法律 > 行政法规 > 地方性法规 > 部门规章 > 规范性文件，不同效力层级的规则变化对企业合规义务的影响路径截然不同；公布日与施行日的区别、过渡期的有无直接决定企业合规调整的紧迫程度。

## 目的

对企业面临的监管规则变化进行**系统抓取与结构化分类**，覆盖四类动态：

1. **新规出台**：新发布的法律、行政法规、部门规章、规范性文件
2. **修订**：对现行规则的修改（含修正案、修订版）
3. **废止**：现行规则被明确废止或失效
4. **征求意见稿**：尚处于征求意见阶段的规则草案

分类维度包括**效力层级**（法律 > 行政法规 > 地方性法规 > 部门规章 > 规范性文件）、**行业归属**、**生效日期**与**过渡期**识别，为企业合规义务台账更新与合规差距分析提供基础数据。

## 详细步骤

### 一、监管动态类型分类

**四类动态的识别要点与初步意义**：

| 动态类型 | 识别要点 | 对企业的初步意义 |
|---------|---------|----------------|
| **新规出台** | 标题含「规定」「办法」「条例」「决定」等新发布字样；文号为新编年份序号；官方公告明确「现予发布」「自X年X月X日起施行」 | 可能新增合规义务（许可申请·报告报送·内控制度建设·数据分类分级等）；须评估是否适用于本企业 |
| **修订** | 标题含「修订」「修改」「修正案」；或同一文号不同版本；官方公告说明「对XX作出修改」 | 已适用的合规义务可能发生变化（义务范围扩大/缩减·罚款数额调整·程序要求变更）；须对比新旧条文确认具体变化 |
| **废止** | 标题含「废止」「失效」；或在新规定末尾列明「XX同时废止」；官方公告明确废止清单 | 原合规义务解除（许可取消·报告要求取消·禁止事项放开）；但须确认是否有替代规则衔接，不可简单认为「无需合规」 |
| **征求意见稿** | 标题含「征求意见稿」「草案」；官方公告说明「现向社会公开征求意见」；附有征求意见截止日期 | **仅作趋势预判，不产生合规义务**；标 `[征求意见稿 — 未生效，仅供预判]`；可用于提前规划合规调整方向 |

### 二、效力层级与生效日期提取

**每条动态须提取以下结构化信息**：

| 动态 | 制定机关 | 效力层级 | 公布日 | 施行日 | 过渡期 |
|------|---------|---------|-------|-------|-------|
| [规则名称] | [全国人大/国务院/XX部委/省级人大/地方政府] | [法律/行政法规/地方性法规/部门规章/规范性文件] | [YYYY-MM-DD 或 待确认] | [YYYY-MM-DD 或 待确认] | [有：X个月/无/待确认] |

**效力层级识别规则**：

| 制定机关 | 效力层级 | 典型文件名称 |
|---------|---------|------------|
| 全国人民代表大会及其常委会 | 法律 | 「XX法」 |
| 国务院 | 行政法规 | 「XX条例」「XX规定」「XX办法」 |
| 省级/设区的市级人大及其常委会 | 地方性法规 | 「XX省/市XX条例」「XX省/市XX规定」 |
| 国务院各部委·直属机构（市场监管总局·网信办·金融监管总局·药监局·生态环境部·应急管理部·工信部等） | 部门规章 | 「XX规定」「XX办法」 |
| 县级以上地方政府及其部门 | 地方规范性文件 | 「XX通知」「XX意见」「XX细则」 |
| 各监管机关内设司局·办公室 | 规范性文件（非规章） | 「XX指导意见」「XX合规指引」「XX自查指南」 |

**公布日与施行日提取要点**：

- **公布日**：以官方发布载体（政府公报·监管机关官网公告）标注的发布日期为准
- **施行日**：规则正文末条或公告中明确的「自X年X月X日起施行」
- **公布日 ≠ 施行日**：多数规则公布后有一定间隔期（常见30日至数月），此间隔即为**过渡期**
- **即时施行**：少数规则「自公布之日起施行」，无过渡期，合规调整时间紧迫

### 三、征求意见稿处理

**征求意见稿不构成现行合规义务**，但对趋势预判有重要参考价值。

| 处理要素 | 规则 |
|---------|------|
| **标识标记** | 一律标 `[征求意见稿 — 未生效，仅供预判]`，不得与现行生效规则混淆 |
| **效力定位** | 不产生合规义务；企业无遵守义务；不得作为合规依据引用 |
| **预判价值** | 反映监管趋势与立法方向；可据此提前规划合规调整（如预判数据分类分级要求、预判许可范围变化） |
| **征求意见** | 企业可通过法定渠道提交书面意见（征求意见公告中载明的提交方式与截止日期） |
| **转正跟踪** | 征求意见稿正式发布后，须更新为正式规则并重新分类（公布日·施行日·效力层级以正式版为准） |
| **差异对比** | 正式版与征求意见稿可能存在差异（条文增删·义务范围调整·施行日期变化），须对比标注 |

### 四、服务立场对分类侧重的影响

| 服务立场 | 分类侧重 | 监控范围 |
|---------|---------|---------|
| **监管动态监控** | 全量分类——新规出台·修订·废止·征求意见稿均须完整分类，按效力层级与行业归入动态简报 | 全面覆盖适用于本企业的所有监管规则变化 |
| **行政处罚应对** | 侧重处罚依据规章变化——关注处罚种类·罚款数额·裁量基准的修订与废止，直接影响正在应对的处罚案件 | 以处罚依据的规则源为核心 |
| **合规体系建设** | 侧重合规义务来源规则变化——关注新增合规义务（内控制度·报告报送·数据分类等）与合规标准更新 | 以合规管理体系对标规则为核心 |
| **监管报告起草** | 侧重报告相关规则更新——关注定期报告·专项报告·重大事项报告的报送要求变化（报送时限·内容格式·数据口径） | 以报告义务来源规则为核心 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| **征求意见稿与正式发布版差异** | 征求意见稿正式发布后，须对比正式版与征求意见稿的差异（条文增删·义务范围变化·施行日期调整），以正式版为准重新分类，标注「正式版与征求意见稿主要差异：...」 |
| **公布日与施行日的区分** | 不得将公布日等同于施行日。规则正文中「自X年X月X日起施行」的日期为施行日；未明确施行日的，须核实是否「自公布之日起施行」或另有规定，标 `[待验证]` |
| **中央版本与地方版本的差异** | 同一领域可能存在中央部门规章与地方性法规/地方规章并存的情形。地方版本可在上位法范围内作出更严格规定，须分别标注并标 `[地方 flagged — 需核实地方性法规与地方裁量基准]` |
| **规则修订但尚未施行** | 修订版已公布但尚未到施行日的，标注「已公布·未施行」，注明过渡期截止日；过渡期内新旧规则并行，企业须同时遵守旧规则并为新规则做准备 |
| **废止但设有过渡条款** | 部分规则废止时设有过渡条款（如「废止前已取得许可的，在有效期内继续有效」），须提取过渡条款内容并标注 |
| **联合发布的规则** | 多个监管机关联合发布的规则，效力层级以牵头机关级别为准，所有联合发布机关均须列明 |

## 错误处理

| 错误/敏感情形 | 识别信号 | 处理方式 |
|------------|---------|---------|
| **行刑衔接升级** | 新规中涉及刑事责任衔接条款（如「构成犯罪的，依法追究刑事责任」）；或修订提高行政违法追诉标准使更多行为可能升格刑事 | 🔴 标注 `[强制升级 — 行刑衔接]`，提示行刑衔接风险，建议联系执业律师；涉及刑事追诉标准变化时提示与 criminal-compliance 插件衔接 |
| **生效日期误判纠正** | 将公布日误认为施行日；或未识别到「自公布之日起施行」导致遗漏过渡期紧迫性 | 纠正分类，重新标注公布日与施行日；如属即时施行且对企业有重大影响，标 🔴 高亮提示「即时施行，合规调整窗口极短」 |
| **地方差异标记遗漏** | 中央规则已分类但遗漏了对应的地方版本（地方性法规·地方规章·地方裁量基准） | 补充地方版本分类，标 `[地方 flagged — 需核实地方性法规与地方裁量基准]`，提示各地可能有更严格规定 |
| **征求意见稿误标为现行规则** | 将尚在征求意见阶段的规则作为现行合规义务列入台账 | 立即纠正，移除合规义务引用，标 `[征求意见稿 — 未生效，仅供预判]`，提示合规义务仅以正式生效规则为依据 |
| **废止规则误认为仍有效** | 已废止的规则仍列在合规义务台账中 | 更新分类为「废止」，检查是否有替代规则衔接，提示清理合规义务台账中的过期条目 |
| **文号·罚款数额裸写** | 行业监管规章文号或罚款数额未标 `[待验证]` 直接写入 | 纠正为规则描述方式，标 `[待验证 — 行业监管规章快速演进，须核实现行版本]`，不硬写裸文号裸数额 |
