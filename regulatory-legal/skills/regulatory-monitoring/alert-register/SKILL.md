# 预警与台账（alert-register）

> **与 authority-rule-source-identification / capture-classification / impact-gap-assessment 的分工**：
> 前序三个子技能分别完成「主管监管机关与规则源识别」「监管动态采集与效力分类」「新规影响评估与合规差距分析」。
> 本子技能是 regulatory-monitoring 的收口子技能——**汇总整合**前序产出，输出两类核心产物：
> 1. **监管动态简报**：本期规则变化分类列示·重点提示·待办清单
> 2. **合规义务台账**：逐项义务·规则源·主管机关·生效或截止时限·责任部门·合规状态
>
> 同时基于台账中时限字段生成分级**预警**提醒（红·橙·黄），确保不遗漏临近失权或重大新规生效。
>
> **Phase 15 边界说明**：本子技能仅输出静态台账与提醒清单（markdown 表格）。动态预警代理（定时检查·自动推送·企微/钉钉通知）属于 Phase 15 的自动化代理范畴，本子技能不涉及。
>
> **单独触发说明**：本子技能可通过 `/regulatory-legal:alert-register` 单独触发，用于更新现有台账或生成新一期简报。

## 目的

本子技能整合 regulatory-monitoring 前序三个子技能的产出，完成以下工作：

1. **监管动态简报**：将本期监管动态按效力层级与行业分类列示，突出重大新规与生效日期，输出待办清单
2. **合规义务台账**：将影响评估中识别的合规义务逐项录入台账（多列结构），形成可持续更新的合规义务总表
3. **预警提醒**：基于台账中的生效日期·整改截止时限·程序时限，生成分级预警（红·橙·黄），确保临近失权或重大新规不遗漏
4. **服务立场适配**：根据整体服务立场调整简报与台账的输出范围（详见步骤四）

### 步骤二：建立合规义务台账

#### 2.1 台账表结构

合规义务台账采用以下多列结构，每行记录一项合规义务：

| 列名 | 说明 | 数据来源 |
|------|------|---------|
| **义务编号** | 台账内部编号（格式：OBL-[YYYY]-[序号]） | 自动生成 |
| **义务项** | 合规义务的简要描述（如「数据安全保护义务」「排污许可持证排污」） | impact-gap-assessment |
| **规则源描述** | 义务所依据的法律·法规·规章·规范性文件的描述（规则描述方式，不硬写文号） | authority-rule-source-identification + capture-classification |
| **来源标签** | 规则源的信息来源标签（`[法规网站]` / `[模型知识 — 需验证]` / `[用户提供]` 等） | legal-basis-conventions.md |
| **验证状态** | `[待验证]` / 已核实锚点（中度把握），建议复核 | legal-basis-conventions.md |
| **主管机关** | 负责该义务的监管机关（中央·地方） | authority-rule-source-identification |
| **生效或截止时限** | 规则生效日期 / 整改截止日期 / 许可有效期届满日 / 报送截止日 | capture-classification + impact-gap-assessment |
| **责任部门** | 企业内部负责该义务落实的部门（如：合规部·技术部·生产部·人事部） | 用户确认 |
| **合规状态** | 合规 / 存在差距 / 不合规 / 已整改 / 持续监控 | impact-gap-assessment |
| **预警等级** | 红 / 橙 / 黄 / 绿（由步骤三计算） | 本子技能计算 |
| **备注** | 过渡期·地方版本差异·行刑衔接提示·关联其他义务等 | 综合 |

#### 2.2 台账示例

```markdown
| 义务编号 | 义务项 | 规则源描述 | 来源标签 | 验证状态 | 主管机关 | 生效或截止时限 | 责任部门 | 合规状态 | 预警等级 | 备注 |
|---------|-------|----------|---------|---------|---------|-------------|---------|---------|---------|------|
| OBL-2026-001 | 数据安全保护义务 | 《数据安全法》规定：数据处理者须履行数据安全保护义务 | `[模型知识 — 需验证]` | `[待验证]` | 网信办 / XX省网信办 | YYYY-MM-DD 生效 | 技术部 | 存在差距 | 🟡 | 须完成数据分类分级 |
| OBL-2026-002 | 排污许可持证排污 | 生态环境部门规章规定：须依法取得排污许可证 | `[模型知识 — 需验证]` | `[待验证 — 行业监管规章快速演进]` | 生态环境部 / XX市生态环境局 | YYYY-MM-DD 许可到期 | 生产部 | 合规 | 绿 | 许可延续须在到期前申请 |
```

#### 2.3 台账录入规则

| 规则 | 说明 |
|------|------|
| 规则源描述方式 | 一律描述规则（如「XX法规定：XX活动须取得许可」），不硬写文号·条号·罚款数额 |
| 来源标签必填 | 每条台账必须附加来源标签，遵循 legal-basis-conventions.md 规范 |
| 验证状态必填 | 行政处罚法 2021 中度把握锚点标「已核实锚点（中度把握），建议复核」；其他一律 `[待验证]` |
| 行政处罚法追责时效参考 | 台账中涉及违法行为追责时效的，参考行政处罚法第36条（2021修订）：一般2年；涉及公民生命健康安全·金融安全且有危害后果的延长至5年（建议复核） |
| 不臆造文号数额 | 行业监管规章文号·罚款数额·裁量阶次数值·国家标准号一律不硬写，标 `[待验证]` |
| 地方版本差异 | 同一义务有中央与地方版本的，分别记录或备注 `[地方 flagged]` |

### 步骤三：生成预警提醒

#### 3.1 预警分类表

基于台账中「生效或截止时限」与今日日期计算剩余天数，触发分级预警：

| 预警等级 | 颜色 | 触发条件 | 典型情形 | 处置要求 |
|---------|------|---------|---------|---------|
| **红色预警** | 🔴 红 | 临近失权或重大新规即将生效 | 整改截止日期剩余≤15天；重大新规生效日期剩余≤30天；许可有效期届满剩余≤30天；行政程序时限（陈述申辩期·听证申请期·复议申请期·起诉期）临近 | **紧急 — 须立即行动或升级律师**；不得静默放任 |
| **橙色预警** | 🟠 橙 | 较紧迫的合规义务时限 | 整改截止日期剩余16–45天；新规生效日期剩余31–60天；许可有效期届满剩余31–60天 | **须在本月内启动整改或准备工作** |
| **黄色预警** | 🟡 黄 | 需关注但暂不紧迫 | 整改截止日期剩余46–90天；新规生效日期剩余61–120天；许可有效期届满剩余61–120天 | **纳入近期工作计划，开始准备** |
| **绿色正常** | 🟢 绿 | 时限充裕 | 剩余>120天 | 常规监控，按季度更新台账 |

#### 3.2 行政程序时限护栏联动

预警等级计算中，涉及行政程序时限（陈述申辩期·听证申请期·复议申请期·起诉期）的条目，适用更严格的预警阈值：

| 程序时限类型 | 红色预警阈值 | 说明 |
|------------|-----------|------|
| 陈述申辩期 | 剩余≤规定期限的一半（具体天数 `[待验证]`）| 逾期可能被视为放弃陈述申辩权 |
| 听证申请期 | 任何剩余天数（须在期限内申请） | 逾期丧失听证权 |
| 行政复议申请期 | 剩余≤规定期限的一半（具体天数 `[待验证]`）| 逾期丧失复议权 |
| 行政诉讼起诉期 | 剩余≤规定期限的一半（具体天数 `[待验证]`）| 逾期丧失诉权 |

> **行政程序时限护栏**：以上具体天数须核实现行法律规定 `[待验证]`，不得自行推算裸数字。以当事人实际收到文书日期为起算点。逾期将丧失救济权利，不得静默放任时限流逝。

### 步骤四：服务立场适配输出范围

整体服务立场影响简报与台账的输出范围：

| 服务立场 | 简报输出范围 | 台账输出范围 | 预警侧重 |
|---------|-----------|-----------|---------|
| **监管动态监控** | 完整简报：全量本期动态分类列示·重点提示·待办 | 全量合规义务台账：所有已识别的合规义务均录入 | 全量预警：所有义务的生效日期·整改时限·许可到期均设预警 |
| **行政处罚应对** | 定向简报：聚焦与当前处罚相关的规则源·同类执法动态·裁量基准变化 | 定向台账：仅录入与当前处罚事项直接相关的合规义务（含整改义务·程序时限义务） | 程序时限预警优先：陈述申辩期·听证申请期·复议申请期·起诉期设为红色预警；整改期限联动 |
| **合规体系建设** | 完整简报 + 合规体系建设专项提示：标注哪些新规需要新建或修订合规制度 | 全量台账 + 制度建设待办：台账中新增「制度建设状态」列（制度已建/在建/待建）| 制度缺口预警：识别有合规义务但无对应制度条目的，标橙色预警 |
| **监管报告起草** | 定向简报：聚焦与报告义务相关的规则变化（报送口径变化·新增报告义务·合规声明要求变化） | 定向台账：仅录入与报告义务相关的合规义务项（报告类型·报送机关·报送频次·截止时限） | 报送截止日预警：定期报告·专项报告·重大事项报告的报送截止日设为预警触发点 |
| **混合多类型** | 完整简报（各立场均覆盖） | 全量台账 | 全量预警 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 台账需动态更新（新规则发布·旧规则废止） | 本子技能输出当前快照；下次调用时基于新数据重新生成；动态推送属于 Phase 15 |
| 规则生效日期发生变更（延期·提前·废止） | 在台账中标注变更日期与新日期；原预警等级重新计算；废止的规则标记「已废止」并移出活跃预警 |
| 跨部门义务归属（同一义务涉及多个责任部门） | 台账中按部门拆分为多行（同一义务编号，不同责任部门），或在一行备注中标注「多部门协同：[部门A]·[部门B]·[部门C]」 |
| 中央规则与地方版本差异 | 分别录入中央版本与地方版本；地方版本标 `[地方 flagged — 需核实地方版本]`；如地方版本更严格，预警等级取高 |
| 前序子技能产出不完整 | 在简报与台账中标注缺失项（如「规则源识别未完成，以下条目主管机关待确认」），不静默填补 |
| 配置文件缺失或含占位符 | 走临时模式（默认监管动态监控立场·保守型严格度）；输出标 `[临时模式 — 请运行冷启动访谈完成配置]` |
| 台账条目过多（如>30项） | 优先显示红色/橙色预警条目；绿色条目折叠至附录 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 简报遗漏临近程序时限的条目 | **行政程序时限护栏触发**：如识别到存在行政程序时限（陈述申辩期·听证申请期·复议申请期·起诉期）临近的条目但未出现在预警清单中，立即补充并标红色预警；不得遗漏任何程序时限临近项——此为行政程序时限护栏的强制要求 |
| 行刑衔接风险识别 | 如台账中某义务的不合规状态可能导致行政违法升格刑事（违法所得数额·情节达刑事追诉标准·监管机关可能移送公安），立即标 `[强制升级 — 行刑衔接]`，提示联系执业律师，并提示与 criminal-compliance 插件衔接 |
| 地方版本差异导致义务冲突 | 当中央规则与地方版本对同一义务有不同要求时，分别列出并标 `[地方 flagged]`；建议从严合规（取更严格版本），并注明「以[地名]现行版本为准，建议核实现行地方性法规与裁量基准」 |
| 前序子技能未执行 | 提示须先执行前序三个子技能（authority-rule-source-identification → capture-classification → impact-gap-assessment），或单独触发本子技能时由用户直接提供规则源清单与影响评估结果 |
| 规则源信息不足以建立台账条目 | 遵循无静默补充原则：报告缺失信息，列出已知部分，标 `[待补充 — 需执行前序子技能或手动提供]`，不自行编造规则源 |
| 台账中义务项的追责时效临近（行政处罚法第36条） | 参考行政处罚法第36条（2021修订，建议复核）：一般追责时效2年；涉及公民生命健康安全·金融安全且有危害后果的延长至5年。如台账中记录的违法行为首发日期距今接近时效，在备注中标注「追责时效参考：行政处罚法第36条，[日期] 届满 `[建议复核]`」 |
