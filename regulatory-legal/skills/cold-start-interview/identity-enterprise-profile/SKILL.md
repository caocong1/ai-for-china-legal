# 身份与企业画像访谈 — 监管合规

## 目的

通过结构化问卷采集用户身份信息与企业画像，写入 `regulatory-legal/CLAUDE.md` 的 `## 我们是谁` 和 `## 谁在使用` 两章。

**所属行业与主管监管机关是监管合规工作的核心变量**，须在访谈中重点确认。

## 前置

- 确认 `regulatory-legal/CLAUDE.md` 存在；不存在则提示编排入口从模板创建。
- 如为增量更新，读取 CLAUDE.md 当前 `## 我们是谁` 与 `## 谁在使用` 章节内容，作为默认值展示给用户确认或修改。

## 访谈问卷

### 第一部分：基本信息

**Q1. 组织/律所/企业名称？**

- 追问：该组织是律所合规/争议业务部、企业合规部、企业法务部还是个人当事人？
- 默认：`[PLACEHOLDER]`

**Q2. 您的主要执业角色是什么？**

选项：
- 监管合规律师
- 企业合规官
- 企业法务
- 当事人本人（有律师可咨询）
- 当事人本人（无律师可咨询）
- 非律师但有律师联系人
- 非律师且无律师可咨询

- 追问：如为非律师角色，是否已有律师联系人？请提供姓名与联系方式。
- 默认：根据用户回答确定

> **角色影响**：执业角色决定输出标题格式（律师工作产物 vs 研究笔记）、升级路由与风险提示强度。

### 第二部分：企业画像（核心变量）

**Q3. 企业所属行业是什么？** `[核心变量]`

- 追问：是否有跨行业经营？如有，请列出所有相关行业。
- 追问：行业是否属于强监管行业（金融·医药·食品·危化品·能源等）？
- 默认：`[PLACEHOLDER]`

> **核心变量说明**：所属行业决定适用的行业监管规章体系、规则源识别方向、行政处罚裁量基准适用范围。须准确填写。

**Q4. 主管监管机关（中央级）是哪个？** `[核心变量]`

- 追问：是否有多个中央级主管监管机关？（如互联网平台可能同时受市场监管总局与网信办监管）
- 追问：主管监管机关（地方级）是哪个？如 XX 省市场监管局、XX 市网信办等。
- 默认：`[PLACEHOLDER]`

> **核心变量说明**：主管监管机关决定处罚决定来源、行政复议机关、规则源官网、裁量基准适用版本。须准确填写。

**Q5. 企业规模如何？**

- 追问：员工人数、年营业额（大致范围）？是民营企业、国有企业还是上市公司？
- 默认：`[PLACEHOLDER]`

**Q6. 主要业务模式是什么？**

- 追问：是否有跨境业务？是否部分涉外？
- 默认：`[PLACEHOLDER]`

**Q7. 是否有涉外业务？**

选项：是（跨境业务）/ 否 / 部分涉外
- 追问：如涉及跨境业务，涉及哪些国家或地区？是否涉及出口管制或制裁合规？
- 默认：否

**Q8. 是否有历史受罚情况？**

选项：无历史处罚 / 曾受警告或小额罚款 / 曾受重大处罚 / 待评估
- 追问：如曾受处罚，请简要描述处罚种类、处罚机关、大致时间。
- 追问（法律事实检查）：如用户提供具体处罚条号或罚款数额，标记 `[待验证]` 并提示核实。
- 默认：待评估

**Q9. 重点监管风险领域有哪些？**

- 追问：可从以下领域选择或补充——反垄断合规 / 数据安全与个人信息保护 / 广告合规 / 产品质量 / 安全生产 / 环保排污 / 食品药品安全 / 金融业务许可。
- 默认：`[PLACEHOLDER]`

**Q10. 当前主要痛点是什么？**

- 追问：可从以下选项选择或补充——监管规则变化快跟不上 / 收到行政处罚决定不知如何应对 / 合规体系缺失 / 监管报告规范不清楚。
- 默认：`[PLACEHOLDER]`

**Q11. 团队规模如何？**

- 追问：负责监管合规的律师/合规官/法务各几人？
- 默认：`[PLACEHOLDER]`

**Q12. 负责人/联系人是谁？**

- 默认：`[PLACEHOLDER]`

### 第三部分：使用者信息

**Q13. 谁在使用本插件？**

- 追问：实际使用者的角色与 Q2 是否一致？如不一致，请分别说明。
- 追问：如使用者为非律师角色，是否有律师联系人供审查？
- 默认：与 Q2 一致

## 检查清单

- [ ] 所有 PLACEHOLDER 字段已填写或用户明确跳过

## 边界条件

- 用户拒绝回答某字段：写入 `[PLACEHOLDER]`，不阻塞后续访谈。
- 用户无法确定主管监管机关：根据所属行业列出可能的监管机关供选择，仍未确定则写入 `[待确认]`。
- 用户角色为「非律师且无律师可咨询」：写入配置并在 `## 谁在使用` 附加特别提示——所有输出为合规研究，不替代律师意见。
- 增量更新模式：仅替换用户确认修改的字段，其余保留原值。

## 错误处理

- CLAUDE.md 不存在：提示编排入口先创建文件，本技能暂停。
- 用户提供具体处罚条号但为旧版（2021 修订前）：提醒更新为新条号，标 `[待验证]`，以规则描述方式记录。
- 用户提供罚款数额但无法确认地方裁量基准：标 `[待验证 — 地方 flagged]`，不写入裸数字。
- 用户口述国家标准编号：标 `[待验证 — 标准号须核实]`。
