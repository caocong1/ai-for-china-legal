# 行政复议与诉讼衔接（review-litigation-bridge）

> **可单独触发**：`/regulatory-legal:review-litigation-bridge`
> 也可由 `penalty-response` 编排入口作为子步骤调用。
>
> **前置**：
> 1. 读取 `regulatory-legal/CLAUDE.md`（实践配置文件），加载以下关键字段：
>    - `## 我们是谁` → 所属行业、主管监管机关（中央·地方）、规模、业务模式
>    - `## 合规与监管立场` → 整体服务立场、合规严格度、监管关系
>    - `## 谁在使用` → 角色（决定输出标题与升级路由）
>    - `## 升级矩阵` → 强制升级触发条件
>    - `## 可用集成` → 行政复议机关、行政审判法院配置
>    - 若配置文件含 `[PLACEHOLDER]`，进入**临时模式**
> 2. 读取 `regulatory-legal/skills/_shared/legal-basis-conventions.md`（监管合规引用规范）
> 3. 读取 `regulatory-legal/skills/_shared/administrative-law-citations.md`（监管合规法律引用库）
> 4. 读取 `regulatory-legal/skills/_shared/practice-profile-schema.md`（实践配置文件契约）
> 5. 分工说明：本子技能负责行政复议与诉讼路径选择与衔接；
>    合法性审查见 `legality-review`；
>    陈述申辩与听证见 `statement-defense-hearing`；
>    从轻减轻不予处罚见 `mitigation-no-penalty`。
>
> **配置缺失处理**：若关键字段含 `[PLACEHOLDER]`，启用临时模式：
> - 合规严格度默认保守型
> - 行政复议机关与管辖法院按用户陈述逐案判断
> - 建议用户运行 `/regulatory-legal:cold-start-interview` 完善配置
>
> **特别提示**：行政复议法·行政诉讼法的具体条号作者把握不足，
> 本技能一律以规则描述方式呈现并标 `[待验证]`，绝不臆造具体条号。
> 所有时限天数须核实现行法律规定，不写裸数字。

---

## 目的

协助当事人在行政处罚决定作出后选择合适的救济路径：

1. **路径选择**：根据案件情况，评估行政复议与行政诉讼各自的优劣，建议最优路径
2. **复议前置识别**：识别是否属于法律规定的复议前置情形
3. **期限守护**：严格守护复议申请期限与起诉期限，逾期将丧失救济权利
4. **材料准备**：准备行政复议申请书或行政诉讼起诉状
5. **程序衔接**：处理复议与诉讼之间的程序衔接（复议后起诉·共同被告等）

> **行政程序时限警示**：行政复议申请期限与行政诉讼起诉期限是法定时限，
> 逾期将丧失救济权利。必须以当事人实际收到处罚决定书（或复议决定书）的日期为起算点，
> 核实具体期限天数 `[待验证]`。

### 步骤四：复议 vs 诉讼路径选择

**复议 vs 诉讼路径选择对照表**：

| 比较维度 | 行政复议 | 行政诉讼 |
|---------|---------|---------|
| **受理机关** | 行政复议机关（上一级主管部门或本级政府）`[待验证]` | 有管辖权的人民法院 `[待验证]` |
| **审理方式** | 书面审理为主，可听证 `[待验证]` | 开庭审理 `[待验证]` |
| **费用** | 不收费 `[待验证]` | 诉讼费（行政案件收费标准 `[待验证]`） |
| **审理期限** | 受理后须在法定期限内作出复议决定 `[待验证]` | 一审须在法定期限内审结 `[待验证]` |
| **审查范围** | 合法性审查 + 合理性审查 `[待验证]` | 以合法性审查为原则 `[待验证]` |
| **举证责任** | 被申请人（行政机关）负举证责任 `[待验证]` | 被告（行政机关）负举证责任 `[待验证]` |
| **变更权** | 复议机关可以变更原行政行为 `[待验证]` | 法院可以判决变更（限于行政处罚明显不当等情形）`[待验证]` |
| **终局性** | 复议决定一般非终局，可再起诉 `[待验证]` | 一审判决可上诉；二审判决终审 `[待验证]` |
| **优势** | 费用低·周期相对短·可审查合理性 | 司法独立·程序保障更强·终局裁判 |
| **劣势** | 复议机关与原机关的隶属关系可能影响公正 | 周期长·费用相对高·仅审查合法性 |

**路径选择建议逻辑**：

| 案件情况 | 建议路径 | 理由 |
|---------|---------|------|
| 事实争议较大，需全面审查 | 先复议后诉讼（两步走） | 复议可审查合理性，为后续诉讼积累有利认定 |
| 程序违法明显 | 可直接诉讼 | 程序违法是明确的法律问题，诉讼即可解决 |
| 时间紧迫，先争取快速救济 | 先复议 | 复议周期相对短，可快速获得救济 |
| 裁量合理性争议 | 先复议 | 复议机关可审查合理性并变更 |
| 需要司法终局裁判 | 直接诉讼或复议后诉讼 | 法院判决具有终局效力 |
| 复议前置情形 | 必须先复议 | 法定前置条件 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| **复议申请期限/起诉期限已过** | 标 🔴 **紧急 — 时限已逾期**；提示丧失救济权利的风险；建议立即联系执业律师评估是否有补救途径（如不可抗力·其他正当理由） |
| **复议前置但当事人直接起诉** | 标 `[复议前置 — 须先申请复议，直接起诉可能被驳回]`；建议先申请行政复议 |
| **同时申请复议和起诉** | 标 `[前提 flagged — 同时启动复议和诉讼的处理规则须核实现行法律]` |
| **复议机关逾期不作决定** | 标 `[复议不作为 — 可对原行政行为起诉或对复议不作为起诉]`；提示起诉期限的计算方式 `[待验证]` |
| **管辖权争议** | 标 `[管辖存疑 — 须核实具体管辖法院]`；建议向律师或有管辖权的法院咨询确认 |
| **重大处罚（吊销许可·责令停产停业等）** | 触发 `[强制升级 — 重大处罚]`；建议由执业律师主导复议/诉讼策略 |
| **行刑衔接情形** | 触发 `[强制升级 — 行刑衔接]`；复议/诉讼策略须与刑事辩护策略协调 |
| **涉外因素** | 涉外行政案件的管辖与程序可能有特殊规定 `[待验证]`；建议咨询有涉外经验的律师 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| **无法读取配置文件** | 标 `[临时模式]`；按保守型执行；复议机关与管辖法院按用户陈述逐案判断 |
| **处罚决定书关键信息缺失** | 标 `[前提 flagged — 信息不完整]`；列出需补充的信息清单 |
| **行刑衔接升级** | 立即标 `[强制升级 — 行刑衔接]`；暂停常规分析；提示联系执业律师 |
| **监管合规边界触发**：用户要求伪造复议/诉讼材料 | 标 `[合规边界 — 拒绝]`；拒绝提供相关建议 |
| **行政程序时限临近（不足5天）** | 强标 🔴 紧急；立即建议联系执业律师；优先提交申请/起诉材料 |
| **条号臆造风险** | 行政复议法·行政诉讼法的条号一律规则描述 + `[待验证]`，不得自行回忆裸条号 |
| **复议/诉讼策略超出本技能范围** | 详细诉讼策略须由执业律师制定；本技能提供路径分析与材料准备辅助 |
| **复议决定书与处罚决定不一致** | 仔细对比两份文书，确定起诉对象与被告 |
