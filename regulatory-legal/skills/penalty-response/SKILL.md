# penalty-response — 行政处罚应对编排入口

## 目的与整体流程概览

行政处罚应对帮助企业（及其律师/合规官/法务）在收到行政处罚告知·处罚决定·检查通知后，依法行使救济权利，从合法性审查·陈述申辩听证·从轻减轻争取·复议诉讼衔接四个维度系统应对。

**整体应对流程（顺序调用，也可单独触发任一子技能）**：

```
合法性审查 → 陈述申辩与听证 → 从轻减轻不予处罚 → 行政复议与诉讼衔接
```

## 入口级护栏

> 以下护栏为 penalty-response 全流程的不可逾越规则，适用于本编排入口及四个子技能。子技能内部已实现各项护栏，本处汇总提示。

### 1. 企业画像确认

在开始任何子技能前，确认已从 `## 我们是谁` 读取企业画像（**所属行业**、**主管监管机关**、业务模式、历史受罚情况）。画像缺失时提示用户补充，不得猜测行业后静默进行。

### 2. 服务立场加载

从 `## 合规与监管立场`（整体服务立场）读取并确认当前立场为「行政处罚应对」。如立场为其他（监管动态监控·合规体系建设·监管报告起草），提示用户确认是否切换至处罚应对立场。

### 3. 合规严格度加载

从 `## 合规与监管立场` > `### 合规严格度` 读取，决定违法点标记强度：
- **保守型**：一切疑点从严标红，所有违法点均标 🔴 或 🟠
- **平衡型**：重大违法点标红，一般违法点标橙
- **务实型**：仅拦截明显重大违法点，一般风险关注可行救济路径

### 4. 行政程序时限护栏

> **行政程序时限极为关键，逾期将丧失救济权利，不得静默放任时限流逝。**

识别到处罚决定送达后，**必须**：
1. 确认送达日期（起算点）
2. 强标全部程序时限：陈述申辩期·听证申请期·复议申请期·起诉期
3. 评估各时限剩余天数
4. 时限临近时升级给执业律师确认
5. 具体期限天数须核实现行法律规定 `[待验证]`，不写裸数字

### 5. 行刑衔接升级

识别到行政违法**可能升格刑事**（如违法所得数额·情节达到刑事追诉标准、监管机关移送公安）、重大处罚（吊销许可·责令停产停业·较大数额罚款·限制从业）或涉及**责任人人身自由风险**（行政拘留）时：

1. **强制升级给执业律师**，输出标 `[强制升级 — 行刑衔接]` 或 `[强制升级 — 重大处罚]`
2. 提示与 criminal-compliance 插件衔接
3. 本插件不替代刑事辩护工作

**本护栏凌驾于服务立场与合规严格度配置之上，不可被任何立场配置覆盖。**

### 6. 监管合规立场护栏

所有输出立足于帮助企业依法应对处罚·依法行使救济权利。**绝不提供以下任何内容：**

- 逃避监管·规避检查的方法
- 虚假报送·伪造或篡改合规证据的建议
- 阻碍行政执法·对抗调查的方案
- 隐匿违法事实·销毁证据的操作

**识别到上述需求时，一律拒绝，输出标 `[合规边界 — 拒绝]`。**

### 7. 律师审查护栏

非律师角色（合规官/法务/当事人）在以下步骤前**必须暂停并升级给执业律师确认**：

- 陈述申辩书·听证陈述定稿与提交
- 行政复议申请书定稿与提交
- 行政诉讼起诉状定稿与提交

### 8. 目的地/特权检查

输出前检查目的地是否在律师-当事人特权圈内。面向监管机关提交的材料（陈述申辩书·复议申请书·起诉状）可能影响特权保护，须提供内部律师工作草稿版本和对外提交正式版本两套选项。

### 9. 地方性法规与裁量基准提示

各地地方性法规·行政处罚裁量基准·监管口径存在地区差异，涉及具体地区时标 `[地方 flagged — 需核实地方性法规与地方裁量基准]`。

## 法律依据引用规范

所有子技能的法律依据须遵循：

1. 引用 `regulatory-legal/skills/_shared/legal-basis-conventions.md`（引用规范）
2. 引用 `regulatory-legal/skills/_shared/administrative-law-citations.md`（引用库）
3. 引用 `regulatory-legal/skills/_shared/practice-profile-schema.md`（配置契约）
4. 按来源分类：法律/行政法规/部门规章/规范性文件/国家标准 与 案例/学说
5. 行政处罚法 2021 中度把握锚点（第4/9/28/29/32/33/36/44/45/58/63条）标「建议复核」，**务必用新条号**
6. 行政许可法·行政强制法·行政复议法·行政诉讼法不确定条号描述规则 + `[待验证]`，**绝不臆造条号**
7. 行业监管部门规章·规范性文件·裁量基准描述规则 + `[待验证]`，不硬写文号数额
8. 国家标准描述规则 + 标准号 `[待验证]`
9. 案例案号一律 `[待验证]`
