# 合规声明与报送（compliance-statement-submission）

> **可单独触发**：`/regulatory-legal:compliance-statement-submission`
> 也可由 `regulatory-report` 编排入口作为第四步调用。
>
> **前置**：
> 1. 读取 `regulatory-legal/CLAUDE.md`（实践配置文件），加载以下关键字段：
>    - `## 我们是谁` → **所属行业**（核心变量）、主管监管机关（中央·地方）、业务模式、规模
>    - `## 合规与监管立场` → 整体服务立场、合规严格度、监管关系
>    - `## 谁在使用` → 角色（决定输出标题与升级路由·签章权限）
>    - `## 升级矩阵` → 强制升级触发条件·角色权限表
>    - 若配置文件含 `[PLACEHOLDER]`，进入**临时模式**
> 2. 读取 `regulatory-legal/skills/_shared/legal-basis-conventions.md`（监管合规引用规范）
> 3. 读取 `regulatory-legal/skills/_shared/administrative-law-citations.md`（监管合规法律引用库）
> 4. 分工说明：本子技能负责合规声明·签章·报送·归档；
>    报告类型识别见 `report-type-requirement-identification`；
>    数据收集与事实梳理见 `data-fact-gathering`；
>    报告结构与起草见 `structure-drafting`。
>
> **配置缺失处理**：若关键字段含 `[PLACEHOLDER]`，启用临时模式：
> - 合规严格度默认保守型
> - 所属行业与主管监管机关按用户陈述逐案判断
> - 建议用户运行 `/regulatory-legal:cold-start-interview` 完善配置

---

## 目的

本子技能完成监管报告定稿前的合规声明·签章·报送·归档全流程：

1. **合规声明**：由报送主体声明报告内容真实·准确·完整
2. **签章程序**：完成法定代表人/授权人签字与单位盖章
3. **报送渠道**：确认正确的报送渠道与方式
4. **回执与归档**：获取报送回执并完整归档
5. **虚假报告法律风险提示**：明确告知虚假报送·遗漏·伪造的法律后果

> **核心护栏 — 虚假报告法律风险提示**：本子技能的核心护栏是确保报送主体充分知悉虚假报告的法律后果。
> 如识别到用户寻求虚假报送·瞒报·漏报·伪造数据，**一律拒绝**，标 `[合规边界 — 拒绝]`。

## 详细步骤

### 步骤一：虚假报告法律风险提示（强制前置）

> **⚠️ 本步骤为强制前置步骤，不得跳过。**

在起草合规声明与报送前，**必须**向报送主体明确告知虚假报告的法律后果：

#### 虚假报告法律风险提示表

| 风险类型 | 法律后果（规则描述） | 法律依据 | 严重程度 |
|---------|-----------------|---------|---------|
| **虚假报送** | 报告内容与事实不符，构成提供虚假材料/虚假报告 | 行政处罚法相关规定（建议复核）·行业监管规章 `[待验证]` | 🔴 严重 |
| **重大遗漏** | 报告故意遗漏应当披露的重要事项 | 行政处罚法相关规定（建议复核）·行业监管规章 `[待验证]` | 🔴 严重 |
| **伪造数据** | 报告中使用伪造·篡改的数据 | 行政处罚法相关规定（建议复核）·行业监管规章 `[待验证]` | 🔴 严重 |
| **行政处罚后果** | 警告·罚款·没收违法所得·责令停产停业·吊销许可证件（行政处罚法第9条，2021修订，建议复核） | 行政处罚法第9条（建议复核） | 🔴 严重 |
| **信用惩戒** | 被列入严重违法失信名单·公示行政处罚信息 | 行政处罚法第48条（建议复核）·相关规章 `[待验证]` | 🟠 高 |
| **加重处罚** | 虚假报送本身可作为从重处罚情节 | 行政处罚裁量基准 `[待验证]` `[地方 flagged]` | 🟠 高 |
| **刑事责任** | 虚假报告情节严重可能构成犯罪（如提供虚假证明文件罪·妨害公务罪等） | 刑法相关规定 `[待验证]` | 🔴 极严重 |
| **个人责任** | 直接负责的主管人员和其他直接责任人员可能承担个人责任 | 行政处罚法相关规定（建议复核）·行业规章 `[待验证]` | 🔴 严重 |

> **虚假报告后果告知确认**：在继续下一步骤前，须确认报送主体已知悉上述法律后果。
>
> **行刑衔接升级**：如识别到报告内容可能涉及虚假且情节严重（可能构成犯罪），
> **立即强制升级给执业律师**，标 `[强制升级 — 行刑衔接]`，暂停报送流程。

#### 行政处罚法处罚种类锚点（2021 修订，建议复核）

行政处罚法第9条（2021修订，建议复核）规定的行政处罚种类包括：
1. 警告·通报批评
2. 罚款·没收违法所得·没收非法财物
3. 暂扣许可证件·降低资质等级·吊销许可证件
4. 限制开展生产经营活动·责令停产停业·责令关闭·限制从业
5. 行政拘留
6. 法律·行政法规规定的其他行政处罚

> **虚假报告可能触发的处罚种类**：上述第1-6项均可能因虚假报告而适用，具体以行业监管规章为准 `[待验证]`。

### 步骤二：起草合规声明

确认报送主体已知悉虚假报告法律后果后，起草合规声明：

#### 合规声明模板

```markdown
## 合规声明

**报送单位**：[企业全称]（统一社会信用代码：[代码]）
**报告名称**：[报告全称]
**报告期间**：[起止日期]
**报送机关**：[监管机关全称]

[报送单位]郑重声明：

一、本报告所载内容真实·准确·完整，不存在虚假记载·误导性陈述或重大遗漏。

二、本报告所依据的数据均来源于可追溯的原始记录，数据口径一致，计算方法符合
[行业监管规章名称/标准名称] `[待验证]` 的规定。

三、本报告已经本单位 [内部审核程序描述] 审核。

四、本单位知悉：提供虚假报告·重大遗漏·伪造数据将承担相应法律责任，
包括但不限于行政处罚·信用惩戒·刑事责任。

**法定代表人/授权代表人签字：** ________________
**单位盖章：**
**日期：** [YYYY-MM-DD]
```

> **非律师角色提示**：非律师角色（合规官/法务/当事人）在合规声明定稿前**必须暂停并升级给执业律师确认**。
> 律师审查护栏不可跳过。

### 步骤三：签章程序

确认签章要求：

**签章要素**：

| 签章要素 | 要求 | 备注 |
|---------|------|------|
| **法定代表人/授权代表人签字** | 须由法定代表人或其书面授权代表人签字 | 授权书须留存备查 |
| **单位公章** | 加盖单位公章（不得使用部门章替代） | 部分监管机关要求加盖多枚印章 `[待验证]` |
| **骑缝章** | 多页报告须加盖骑缝章 | 部分监管机关有此要求 `[待验证]` |
| **日期** | 签署日期 | 须在报送期限前 |

> **报送时限护栏**：如报送期限临近（不足 3 个工作日），标 🔴 **紧急 — 报送期限临近**，
> 建议立即完成报送。如报送期限已过，标 🔴 **紧急 — 报送期限已逾期**，
> 提示逾期报送的法律后果并建议立即联系执业律师。

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 报送主体拒绝签署合规声明 | 提示不签署的后果（无法完成报送义务）；如原因是知悉虚假报告法律后果后仍坚持虚假内容，标 `[合规边界 — 拒绝]` |
| 报送期限已逾期 | 标 🔴 **紧急 — 报送期限已逾期**；提示逾期报送的法律后果；建议立即联系执业律师评估补救途径 |
| 报送期限临近（不足 3 个工作日） | 标 🔴 **紧急 — 报送期限临近**；建议立即完成报送 |
| 发现报告内容可能存在虚假 | **立即暂停报送流程**；标 `[合规边界 — 报告内容存疑]`；建议查明后再决定是否报送 |
| 用户要求跳过虚假报告法律风险提示 | 拒绝跳过；本步骤为强制前置步骤，不可省略 |
| 用户要求隐瞒已知问题后报送 | 拒绝，标 `[合规边界 — 拒绝]`；提示隐瞒的法律后果 |
| 用户要求伪造签章 | 拒绝，标 `[合规边界 — 拒绝]`；伪造签章本身构成违法行为 |
| 报送渠道不确定 | 标 `[待验证 — 报送渠道须核实]`；建议通过监管机关官网确认 |
| 非律师角色未在定稿前升级律师 | 触发律师审查护栏；暂停报送；提示升级 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 无法读取配置文件 | 标 `[临时模式]`；按保守型执行；建议运行 `/regulatory-legal:cold-start-interview` |
| 报告内容在审核中发现错误 | 暂停报送流程；返回 `structure-drafting` 修正后重新进入本子技能 |
| 用户要求在合规声明中使用模糊表述 | 标 `[合规边界 — 拒绝]`；合规声明须明确·具体·无歧义 |
| 行政处罚法条号使用旧版 | 提醒 2021 修订条号已重排；须使用新条号并标「建议复核」 |
| 用户口述具体罚款数额或文号 | 标 `[待验证]`；不直接写入裸数值 |
| 行刑衔接风险被识别但未触发升级 | **强制纠正**：立即标 `[强制升级 — 行刑衔接]`；暂停报送；提示联系执业律师 |
| 报送回执未获取 | 标 `[待获取 — 报送回执]`；跟进获取回执作为报送凭证 |
| 用户要求协助瞒报或漏报 | 标 `[合规边界 — 拒绝]`；拒绝协助；提示如实报送义务与虚假报送的法律后果 |
