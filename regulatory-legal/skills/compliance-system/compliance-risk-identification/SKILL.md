# 合规风险识别（compliance-risk-identification）

> **与 compliance-system 其他子技能的分工**：
> 本子技能是合规体系建设的第一步（PDCA 的 P 阶段——风险识别与评估），
> 识别企业在各维度的行政合规风险，输出合规风险地图与重点领域速查表，
> 为后续的合规制度与岗责设计（policy-duty-design）提供风险基础。
>
> **与 regulatory-monitoring 的关系**：
> 监管动态监控（regulatory-monitoring）输出的新规变化与合规义务台账驱动风险识别更新——
> 新监管规则发布时，须重新评估受影响维度的风险等级。
>
> **单独触发说明**：本子技能可通过 `/regulatory-legal:compliance-risk-identification` 单独触发，
> 用于定期风险复查或特定行业合规风险排查。

## 目的

本子技能按多维度系统性识别企业面临的行政合规风险，完成以下工作：

1. **多维度风险扫描**：按业务流程·岗位·区域·行业监管要求·关联方五个维度逐一扫描合规风险
2. **合规风险地图**：将识别的风险点按领域归类，评估风险等级，输出合规风险地图
3. **重点合规领域速查**：根据所属行业从速查表中提取重点合规领域及其可能的违规后果
4. **风险与处罚种类关联**：将风险点对应到行政处罚法第9条（2021修订，建议复核）的处罚种类，使风险后果具体化

## 详细步骤

### 步骤一：读取所属行业，确定重点合规领域

| 所属行业 | 典型重点合规领域 | 典型主管监管机关 |
|---------|--------------|--------------|
| 制造业 | 产品质量·安全生产·环保排污 | 市场监管总局·应急管理部·生态环境部 |
| 互联网平台 | 数据安全·个人信息保护·广告合规·反垄断 | 网信办·市场监管总局 |
| 金融机构 | 金融业务许可·反洗钱·消费者权益保护 | 金融监管总局·人民银行·证监会 |
| 药品医疗器械 | 生产经营许可·药品质量·广告合规 | 药监局·市场监管总局 |
| 食品饮料 | 食品安全·产品质量·广告合规 | 市场监管总局 |
| 房地产建设 | 建设工程·环保排污·安全生产 | 住建部·生态环境部·应急管理部 |
| 能源化工 | 安全生产·环保排污·排污许可 | 应急管理部·生态环境部 |
| 广告传媒 | 广告合规·网络信息内容·消费者权益 | 市场监管总局·网信办 |

### 步骤二：多维度风险扫描

按以下五个维度逐一扫描合规风险：

#### 合规风险识别维度表

| 维度 | 扫描内容 | 典型风险来源 | 输出 |
|------|---------|-----------|------|
| **业务流程维度** | 逐一审视企业核心业务流程（采购·生产·销售·售后·财务·人事等）中的合规风险 | 许可缺失·超范围经营·合同违规·产品质量缺陷·虚假宣传·财务违规 | 流程风险清单 |
| **岗位维度** | 识别高风险岗位（法定代表人·高管·采购·销售·财务·安全·环保等）的合规风险 | 高管个人责任·岗位授权不当·利益冲突·商业秘密泄露·岗位违规操作 | 岗位风险清单 |
| **区域维度** | 识别企业注册地·经营地·项目所在地的地方性法规与裁量基准差异 | 地方性法规更严格·地方裁量基准差异·跨区域经营许可·地方监管口径差异 | 区域风险清单 `[地方 flagged]` |
| **行业监管要求维度** | 对照主管监管机关的行业规章·规范性文件·合规指引，识别制度性合规缺失 | 许可资质缺失·报告义务未履行·合规制度缺失·标准不符合·行业禁入 | 行业监管风险清单 |
| **关联方维度** | 识别关联企业·供应商·经销商·外包方·合作方带来的连带合规风险 | 供应商违规连带·经销商违规宣传·关联方交易·外包方环保/安全违规 | 关联方风险清单 |

每个维度扫描完成后，记录风险点并初步评估风险等级。

**风险等级评定规则**：

| 等级 | 评定标准 | 对应 CLAUDE.md 风险校准 |
|------|---------|---------------------|
| 🔴 关键 | 涉及吊销许可·责令停产停业·较大数额罚款·行刑衔接风险 | 与行政违法风险校准 🔴关键 一致 |
| 🟠 高 | 存在重大合规缺失，处罚风险大但尚未现实化 | 与行政违法风险校准 🟠高 一致 |
| 🟡 中等 | 合规制度缺失或不完善，一般违法风险 | 与行政违法风险校准 🟡中等 一致 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 所属行业字段含 `[PLACEHOLDER]` | 提示用户确认所属行业，不得猜测行业后静默进行；以临时模式按用户陈述逐案判断 |
| 企业业务流程信息不完整 | 基于已知信息识别风险，标注缺失维度（如「采购流程信息未提供，该维度风险待补充」），不静默填补 |
| 涉及多个行业的混合型企业 | 按主要行业与次要行业分别识别，取较高风险等级 |
| 历史受罚情况为「曾受重大处罚」 | 在风险地图中将同类风险点自动升至 🟠高 或以上，并提示行刑衔接评估 |
| 监管关系为「被动应对」 | 风险识别侧重已受处罚事项相关的领域，同时扫描其他维度防止遗漏 |
| 风险点涉及行刑衔接 | 立即标 `[强制升级 — 行刑衔接]`，提示联系执业律师，与 criminal-compliance 插件衔接 |
| 用户需求涉及逃避监管或伪造合规证据 | 拒绝，标 `[合规边界 — 拒绝]`，提示监管合规的正道是依法合规 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 行业监管规章信息不足以评估某领域风险 | 遵循无静默补充原则：报告该领域信息不足，标 `[待补充 — 须核实监管规章]`，建议调用 regulatory-monitoring 补充 |
| 用户口述具体罚款数额或文号 | 标 `[待验证]`，提示须核实现行版本；不直接写入裸数值 |
| 行政处罚法条号使用旧版条号 | 提醒用户 2021 修订条号已重排，须使用新条号并标「建议复核」 |
| 风险等级评估存在争议 | 按合规严格度处理：保守型取高，务实型取低，平衡型取中并注明争议 |
| 合规义务台账已有记录（来自 alert-register） | 直接引用台账中的风险等级与合规状态，避免重复评估；如有新发现则更新台账 |
| 行刑衔接风险被识别但未触发升级 | **强制纠正**：立即标 `[强制升级 — 行刑衔接]`，提示联系执业律师；本护栏凌驾于服务立场与合规严格度之上 |
