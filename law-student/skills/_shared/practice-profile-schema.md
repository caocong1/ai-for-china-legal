# 法考培训学习者配置文件契约（Practice Profile Schema）

> 本文件定义 law-student 插件的学习者配置文件契约：字段定义、存储位置、技能读取规则，
> 以及与 `law-student/CLAUDE.md` 学习配置模板的映射关系。
>
> **所有法考培训技能通过本契约读取个性化配置。** 冷启动访谈子技能按本契约填充配置文件。

---

## 一、配置文件是什么

### 真相的来源

学习者配置文件的真相来源是 **`law-student/CLAUDE.md`**（散文形式）。该文件：

- 由 `/law-student:cold-start-interview` 冷启动访谈生成和更新
- 可由学习者直接编辑（无需重新运行访谈）
- 为所有法考培训技能提供个性化设置的读取来源

### 架构决策

**决策：不引入与现有体系冲突的新配置文件格式。**

**选择理由：**

1. `law-student/CLAUDE.md` 已是学习者可读可编辑的人类友好格式
2. 配置文件读取本质是文本理解，不需要机器解析的结构化格式
3. 引入新的 JSON/YAML 文件会产生双重真相来源
4. 现有模式已验证可行

**结论：** `law-student/CLAUDE.md` 是唯一真相来源。

## 三、技能读取契约

### 3.1 技能读取时机

**在执行任何实质性工作之前**，每个技能（或其编排入口）必须：

1. 读取 `law-student/CLAUDE.md`
2. 检查关键字段是否已填充（不含 `[PLACEHOLDER]`）
3. 根据检查结果选择运行模式

### 3.2 配置缺失或含占位符时的处理

当技能检测到 `law-student/CLAUDE.md` 缺失或含 `[PLACEHOLDER]` 时，**不得静默使用猜测值**，须按以下逻辑处理：

**提示词（标准格式）：**

> 我注意到你还没有配置法考学习档案——这是让我根据你的身份（在校生/在职考生）、
> 薄弱学科、学习风格和目标考试时间来定制训练内容的关键。
>
> **两个选择：**
> - 运行 `/law-student:cold-start-interview`（约5分钟）来配置你的学习档案。
> - 说 **「临时模式」**，我会使用通用默认值——并在输出上标记
>   `[临时模式 — 配置你的档案以获得定制训练]`。

### 3.3 临时模式默认值

| 字段 | 临时模式默认值 |
|-----|-------------|
| 身份 | 法学本科在校生（首次备考） |
| 目标分数 | 通过线（客观题180分/主观题108分，具体以当年大纲为准 `[待验证]`） |
| 薄弱学科 | 无预设（逐案询问） |
| 学习风格 | 读写型（偏文字输出） |
| 考试侧重 | 客观题与主观题均衡 |
| 每日学习时间 | 4小时 |
| 教材 | 无预设（逐案确认） |
| 备考阶段 | 基础阶段 |

## 四、教育护栏

### 4.1 学习者画像写入前检查

在将用户口述的信息写入配置文件之前，须注意：

| 口述场景 | 检查提示 |
|---------|---------|
| 用户口述的具体分数线 | 提示：「分数线以当年司法部公告为准，建议确认 `[前提 flagged — 需验证]`」 |
| 用户口述的考试日期 | 提示：「考试日期以当年司法部公告为准 `[前提 flagged — 需验证]`」 |
| 用户口述的大纲变化 | 提示：「大纲变化请以司法部官方发布为准 `[前提 flagged — 需验证]`」 |

### 4.2 矛盾回答澄清

若用户在同一访谈会话中提供矛盾信息（如先说「非法本零基础」后说「已通过客观题」），在写入前澄清。

## 五、重跑 / 编辑 / 版本控制

### 5.1 重跑访谈

学习者可随时重新运行完整访谈或任一子技能：

- **完整重跑**：`/law-student:cold-start-interview`
- **部分更新**：单独触发某一子技能，只更新对应章节

### 5.2 直接编辑

学习者可编辑 `law-student/CLAUDE.md`，无需重跑访谈。编辑后即时生效。

### 5.3 版本控制

- `law-student/CLAUDE.md` 纳入 git 版本控制
- 支持通过 git 查看历史和回滚
