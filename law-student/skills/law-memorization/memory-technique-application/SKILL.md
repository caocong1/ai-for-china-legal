# 记忆技巧应用（memory-technique-application）

> **所属编排**：`law-student/skills/law-memorization/SKILL.md`（法条记忆）
> **遵循规范**：`law-student/skills/_shared/legal-basis-conventions.md`（法考培训引用规范）
> **分工说明**：本子技能负责**记忆技巧的选取与应用**。体系框架由 `legal-system-mapping` 处理；
> 重点识别由 `key-provision-identification` 处理；间隔复习由 `spaced-repetition-review` 处理。
> **可单独触发**：`/law-student:memory-technique-application`

## 目的

为学习者提供多种记忆技巧，帮助高效编码和记忆法考核心法条。
不同记忆技巧适用于不同类型的知识点，须根据知识点特征和学习者偏好选择合适的方法。

## 详细步骤

### 步骤 1：记忆技巧类型总览

#### 1.1 技巧类型与适用场景

| 记忆技巧 | 适用场景 | 优势 | 局限 |
|---------|---------|------|------|
| **对比记忆** | 易混淆的法条/概念 | 区分度高 | 须先理解两者 |
| **联想记忆** | 抽象概念、构成要件 | 生动形象 | 联想可能偏离原意 |
| **口诀记忆** | 多项列举（如权利类型、期限） | 朗朗上口 | 过度简化风险 |
| **案例串联** | 关联性强的多个法条 | 情境化记忆 | 案例本身须准确 |
| **图表记忆** | 体系结构、程序流程 | 视觉化 | 制作成本较高 |
| **数字编码** | 含数字的法条（期限/比例/数额） | 数字敏感 | 适用范围有限 |

### 步骤 2：对比记忆法

#### 2.1 对比记忆表模板

| 对比项 | 概念A | 概念B | 关键区别 |
|-------|-------|-------|---------|
| 定义 | [A的定义] | [B的定义] | [核心区别] |
| 适用条件 | [A的条件] | [B的条件] | [条件差异] |
| 法律效果 | [A的效果] | [B的效果] | [效果差异] |
| 记忆关键词 | [A的关键词] | [B的关键词] | [区分关键词] |

#### 2.2 常见对比记忆组合

| 对比组 | 学科 | 关键区别 |
|-------|------|---------|
| 无效 vs 可撤销 vs 效力待定 | 民法 | 自始无效/撤销后无效/追认后有效 |
| 一般保证 vs 连带保证 | 民法 | 先诉抗辩权有无 |
| 盗窃 vs 侵占 vs 抢劫 | 刑法 | 占有转移方式不同 |
| 犯罪中止 vs 犯罪未遂 | 刑法 | 是否出于意志以外原因 |
| 行政拘留 vs 刑事拘留 vs 司法拘留 | 多科 | 性质和决定机关不同 |
| 诉讼时效 vs 除斥期间 | 民法 | 可变期间vs不变期间 |
| 上诉 vs 申诉 vs 申请再审 | 诉讼法 | 程序和效果不同 |

### 步骤 3：联想记忆法

#### 3.1 联想记忆方法

将抽象的法条内容通过生动的联想进行编码：

| 法条内容 | 联想方式 | 联想说明 |
|---------|---------|---------|
| [法条A] | 谐音联想 | [谐音编码] |
| [法条B] | 场景联想 | [场景化描述] |
| [法条C] | 故事串联 | [编成小故事] |

#### 3.2 联想记忆注意事项

- 联想只是辅助记忆工具，不能替代对法条原文的理解
- 联想编码须个人化，适合自己的记忆方式才有效
- 避免联想产生对法条的错误理解

### 步骤 4：口诀记忆法

#### 4.1 口诀编写原则

1. **准确性**：口诀须准确反映法条核心内容
2. **简洁性**：便于记忆，不超过10个字为佳
3. **完整性**：不遗漏关键要素

#### 4.2 常见口诀示例

| 知识点 | 口诀 | 口诀含义 | 注意事项 |
|-------|------|---------|---------|
| 民事法律行为有效要件 | 「能力真实不违反」 | 行为能力+意思表示真实+不违反强制性规定 | 须回归第143条原文 `[待验证]` |
| 合同法定解除情形 | 「不可抗力预期违，催告迟延目的灭」 | 不可抗力+预期违约+迟延经催告+目的不能实现 | 须回归第563条原文 `[待验证]` |
| 正当防卫要件 | 「正在不法为防卫」 | 正在进行+不法侵害+防卫行为 | 须回归第20条原文 `[待验证]` |

> **注意**：口诀仅为记忆辅助，法考答题时须引用完整法条内容，不能仅写口诀。

### 步骤 5：案例串联法

#### 5.1 案例串联方法

用一个综合案例将多个关联法条串联起来记忆：

```
案例：甲与乙签订买卖合同 → [合同成立/效力法条]
     → 甲交付货物但质量不合格 → [违约/质量瑕疵法条]
     → 乙发现货物为丙所有 → [善意取得/物权法条]
     → 丙要求返还 → [物权请求权/侵权法条]
```

#### 5.2 案例串联的注意事项

- 案例须符合法律逻辑，不能为串联而扭曲法律关系
- 案例中的事实须合理（不出现不合常理的情节）
- 每个串联的法条须在案例中有明确的对应关系

### 步骤 6：图表记忆法

#### 6.1 流程图（适用于程序性知识）

```
起诉 → 受理 → 举证 → 开庭 → 判决 → 上诉（15日内）→ 二审 → 执行
```

#### 6.2 思维导图（适用于体系性知识）

用树状图展开学科知识结构（参见 `legal-system-mapping` 子技能的图谱格式）。

#### 6.3 表格对比（适用于并列知识点）

将多个并列知识点整理成对比表格（参见对比记忆法的表格格式）。

## 记忆卡片格式

为每个需要记忆的知识点生成记忆卡片：

```markdown
### 记忆卡片：[知识点名称]

**法条依据：** 《XX法》第X条 `[待验证]`
**核心内容：** [简要规则描述]
**记忆技巧：** [选用的记忆方法及具体编码]
**易混淆点：** [与哪些知识点容易混淆]
**自测问题：** [用于自测的问题]
```

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 学习者偏好某种记忆方法 | 优先使用其偏好的方法 |
| 知识点不适合口诀化 | 改用对比记忆或案例串联 |
| 口诀可能导致误解 | 明确标注「口诀简化，须回归原文」 |
