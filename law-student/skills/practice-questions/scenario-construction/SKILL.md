# 案例情景构建（scenario-construction）

> **所属编排**：`law-student/skills/practice-questions/SKILL.md`（模拟题生成）
> **遵循规范**：`law-student/skills/_shared/legal-basis-conventions.md`（法考培训引用规范）
> **分工说明**：本子技能负责**案例情景的构建**。题型设计由 `question-type-design` 处理；
> 答案解析由 `answer-key-explanation` 处理；难度校准由 `difficulty-calibration` 处理。
> **可单独触发**：`/law-student:scenario-construction`

---

## 目的

为法考模拟题中的案例型题目构建合理、准确、有考查价值的法律情景，
包括人物设定、事实背景、法律关系和时间线设计。

---

## 前置

1. **读取 `law-student/CLAUDE.md`** — 检查学习者薄弱学科
2. **读取 `law-student/skills/_shared/legal-basis-conventions.md`** — 法考培训引用规范
3. 本子技能产出是案例情景材料，**不构成法律建议**
4. **理想输入**：来自 `question-type-design` 的知识点覆盖计划

## 详细步骤

### 步骤 1：案例情景要素设计

#### 1.1 案例要素清单

| 要素 | 要求 | 检查项 |
|------|------|-------|
| **人物设定** | 人物身份须与法律关系匹配 | 主体资格是否合规（如公司须有法定代表人） |
| **事实背景** | 须包含所有必要的法律事实 | 是否有遗漏的关键事实 |
| **时间线** | 时间节点须影响法律后果 | 是否涉及时效/期间问题 |
| **法律关系** | 须明确体现考查的法律关系 | 法律关系是否清晰可辨 |
| **干扰信息** | 适量干扰信息考查信息筛选 | 干扰信息不过多导致偏离考查目标 |
| **争议点** | 须有明确的法律争议 | 争议点是否有法律依据支撑 |

#### 1.2 人物命名规范

- 使用常见的中文姓名（如甲、乙、丙或张三、李四、王五）
- 法人使用通用公司名称（如A公司、B公司）
- 人物数量与法律关系复杂度匹配

### 步骤 2：常见案例情景模板

#### 2.1 合同纠纷案例模板

```markdown
甲公司与乙公司于[日期]签订[合同类型]合同，约定[主要条款]。
合同约定[价款/工期/其他关键条款]。
[履行情况：甲已履行/乙未履行/双方争议]。
[争议焦点事件发生]。
现甲/乙拟主张[诉讼请求]。
```

#### 2.2 侵权纠纷案例模板

```markdown
[时间]，[侵权人]在[地点]实施了[侵权行为]。
该行为导致[被侵权人]遭受[损害结果]。
[因果关系描述]。
[过错分析相关事实]。
[可能的抗辩事由事实]。
```

#### 2.3 刑事案件案例模板

```markdown
[时间]，[行为人]在[地点]实施了[行为描述]。
[行为手段和方式]。
[结果描述：人身损害/财产损失等]。
[主观方面相关事实：故意/过失/目的]。
[可能的法定情节：自首/立功/未遂等]。
```

### 步骤 3：案例质量控制

#### 3.1 法律逻辑检查

| 检查项 | 标准 | 不合规处理 |
|-------|------|-----------|
| 主体资格 | 人物须有相应民事行为能力 | 修正主体设定 |
| 事实完整性 | 法律构成要件的每个事实须有对应 | 补充缺失事实 |
| 时间线合理 | 事件先后顺序须符合逻辑 | 调整时间线 |
| 金额合理 | 涉及金额须在合理范围内 | 调整金额 |
| 因果关系 | 行为与结果之间须有因果关系 | 补充因果链条 |

#### 3.2 考查价值评估

| 评估维度 | 标准 | 评分 |
|---------|------|------|
| 知识点覆盖 | 是否有效考查目标知识点 | 1-5 |
| 干扰度 | 是否有合理的干扰信息 | 1-5 |
| 区分度 | 能否区分不同水平的考生 | 1-5 |
| 真实性 | 案例是否符合实际法律场景 | 1-5 |

### 步骤 4：案例难度层次

#### 4.1 难度层次设计

| 难度 | 案例特征 | 适用学习者 |
|------|---------|-----------|
| **基础** | 单一法律关系、事实清晰、无干扰 | 初学者、基础阶段 |
| **中等** | 2-3个法律关系、适度干扰 | 有一定基础的学习者 |
| **进阶** | 多法律关系交叉、多干扰信息 | 冲刺阶段、高水平学习者 |
| **挑战** | 涉及学说争议、法律空白 | 追求高分的学习者 |

---

## 检查清单

- [ ] 人物设定与法律关系匹配
- [ ] 事实包含所有必要的法律事实
- [ ] 时间线合理（涉及时效的已标注）
- [ ] 法律关系清晰可辨
- [ ] 干扰信息适量（不过多不过少）
- [ ] 案例符合法律逻辑
- [ ] 案例难度已标注

## 输出模板

```markdown
[教育声明：本案例仅供学习训练，不构成法律意见。]

# 案例情景 — [考查知识点]

## 案例材料

[案例正文]

## 案例设计说明（仅供出题参考，不展示给学习者）

- **考查知识点：** [知识点]
- **涉及法律关系：** [关系列表]
- **关键事实：** [事实列表]
- **干扰信息：** [干扰项列表]
- **难度：** [基础/中等/进阶/挑战]
```

---

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 案例涉及多个学科 | 标注各学科考查点，建议分步解答 |
| 案例涉及最新法规 | 标注法规版本，提醒核实现行有效性 |
| 案例情节过于巧合 | 调整使情节更贴近实际 |
