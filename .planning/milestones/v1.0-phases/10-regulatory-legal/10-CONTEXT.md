## Domain

本阶段将监管合规插件（regulatory-legal）从 v0.1.0 的 3 个扁平浅骨架深化为完整深层子技能集，沿用 Phase 2 商事合同 / Phase 3 诉讼仲裁 / Phase 4 劳动人事 / Phase 5 数据合规 / Phase 6 知识产权 / Phase 7 建设工程 / Phase 8 婚姻家事 / Phase 9 刑事合规 已验证的深化模式：将每个主技能拆分为多个可独立触发的子技能，新增 2 个全新主技能（合规体系建设 compliance-system、监管报告起草 regulatory-report），建立全插件复用的监管合规法律引用脊柱（行政处罚法 2021 修订 / 行政许可法 / 行政强制法 / 行政复议法 / 行政诉讼法 + 各行业监管规章规范性文件 + 企业合规管理体系国家标准 的 法律/行政法规/部门规章/规范性文件/国家标准 分类 + 待验证标记 + 行政处罚法 2021 条号重排保守标注 + 行业监管规章快速演进标注），定义实践配置文件契约使所有技能从同一处个性化读取，并**新建** regulatory-legal/CLAUDE.md 实践配置模板。

**本插件定位（关键）**：监管合规是**预防性 / 应对性**执业领域——服务于为企业提供行政监管动态跟踪、行政处罚应对、合规体系建设、监管报告起草的律师与企业合规官/法务。本插件聚焦**行政法层面的监管合规**（行政处罚、行政许可、行政强制、行政复议、行政诉讼），与刑事责任（criminal-compliance）相区分；当行政违法可能升格为刑事案件（行刑衔接）、或涉及人身自由风险时**强制升级**给执业律师并提示行刑衔接风险。

当前 regulatory-legal 是 v0.1.0，仅有 3 个**扁平浅骨架**技能（每个 ~10-17 行），且**尚无 regulatory-legal/CLAUDE.md 实践配置模板**：

- **regulatory-monitoring（监管动态监控）** — 现 ~17 行浅骨架，仅列「需求收集 / 法律分析 / 输出生成」三步纲要，须深化拆分为 监管机关与规则源识别 / 动态抓取与分类 / 影响评估与合规差距 / 预警与台账 四个子技能。
- **penalty-response（行政处罚应对）** — 现 ~17 行浅骨架，仅列三步纲要，须深化拆分为 处罚决定合法性审查 / 陈述申辩与听证 / 从轻减轻不予处罚争取 / 行政复议与诉讼衔接 四个子技能。
- **cold-start-interview（冷启动访谈）** — 现 ~15 行三部分访谈（且末尾「生成 YAML 格式」表述须纠正），须拆为可独立触发的访谈子技能并填充 regulatory-legal/CLAUDE.md。

并按 ROADMAP 覆盖的 5 个能力面，**新增 2 个全新主技能**：

- **compliance-system（合规体系建设，新增）** — 合规风险识别 / 合规制度与岗责 / 合规运行与培训 / 合规评估与改进，四个子技能 + 编排入口。
- **regulatory-report（监管报告起草，新增）** — 报告类型与监管要求识别 / 数据收集与事实梳理 / 报告结构与起草 / 合规声明与报送，四个子技能 + 编排入口。

覆盖能力（对齐 ROADMAP Phase 10 五个能力面）：监管动态监控（regulatory-monitoring）、行政处罚应对（penalty-response）、合规体系建设（compliance-system，新增）、监管报告起草（regulatory-report，新增）、冷启动访谈（cold-start-interview）。

## Decisions

### 技能深度和范围

**决策**：深层子技能拆分（沿用 Phase 2 / 3 / 4 / 5 / 6 / 7 / 8 / 9 模式）

- 每个主技能拆分为 4 个可独立触发的子技能
- 每个子技能包含：目的、详细步骤、检查清单、输出模板、边界条件、错误处理、按来源分类的法律依据（每个子技能 150+ 行的实质内容，非占位骨架）
- 拆分与新增示例：
  - **regulatory-monitoring（监管动态监控）** 拆分为：监管机关与规则源识别（按企业所属行业识别主管监管机关 市场监管总局·网信办·金融监管·药监·生态环境·应急管理 等 + 规则源 法律·行政法规·部门规章·地方性法规·规范性文件 + 官方发布渠道）/ 动态抓取与分类（新规·修订·废止·征求意见稿的抓取要点、按效力层级与行业分类、生效日期与过渡期识别）/ 影响评估与合规差距（新规对企业的适用性判断、合规差距分析、整改优先级）/ 预警与台账（监管动态预警机制、合规义务台账、到期与生效提醒）（4 子技能 + 编排入口）
  - **penalty-response（行政处罚应对）** 拆分为：处罚决定合法性审查（处罚法定原则=行政处罚法第4条 建议复核、处罚种类=第9条 建议复核、主体职权·事实证据·法律适用·程序合法性审查、一事不再罚=第29条 建议复核）/ 陈述申辩与听证（陈述申辩权=第44/45条 建议复核、听证=第63条 建议复核、听证申请条件与程序、不得因申辩加重处罚）/ 从轻减轻不予处罚争取（从轻减轻=第32条 建议复核、不予处罚 无主观过错·初次违法危害轻微及时改正=第33条 建议复核、责令改正=第28条 建议复核、追责时效=第36条 建议复核）/ 行政复议与诉讼衔接（行政复议法·行政诉讼法路径选择、复议前置情形、起诉期限、举证责任倒置）（4 子技能 + 编排入口）
  - **compliance-system（合规体系建设 / 新增）** 拆分为：合规风险识别（业务·岗位·区域·行业监管维度全面行政合规风险梳理 + 风险地图 + 与 regulatory-monitoring 衔接）/ 合规制度与岗责（合规政策与制度体系、重点领域专项制度 反垄断·数据·广告·产品质量·安全生产·环保·财税、合规岗责与审批·禁止性清单）/ 合规运行与培训（合规组织架构 首席合规官·合规部门·业务合规员、三道防线、合规流程嵌入业务、培训考核与合规文化）/ 合规评估与改进（合规有效性评估指标 参照企业合规管理体系国家标准 描述规则·待验证、定期审计与自查、问题整改闭环、持续改进）（4 子技能 + 编排入口）
  - **regulatory-report（监管报告起草 / 新增）** 拆分为：报告类型与监管要求识别（识别报告类型 定期报告·专项报告·重大事项报告·整改报告·自查报告、对应监管机关与法定报送要求·频次·时限）/ 数据收集与事实梳理（数据口径与来源核验、事实时间线梳理、数据一致性校验、留痕与可追溯）/ 报告结构与起草（报告结构设计、事实陈述·合规说明·问题与整改、客观准确不夸大、术语规范）/ 合规声明与报送（合规声明与签署、报送渠道与方式、报送回执与归档、虚假报告法律风险提示）（4 子技能 + 编排入口）
  - **cold-start-interview（冷启动访谈）** 拆分为：身份与企业画像（执业角色 监管合规律师/企业合规官/法务/当事人 + 服务对象 + 企业画像 所属行业·主管监管机关·规模·业务模式·是否涉外·历史受罚情况·重点监管风险领域）/ 合规立场与服务范围（整体服务立场 监管动态监控/行政处罚应对/合规体系建设/监管报告起草 + 合规严格度 保守/平衡/务实 + 监管关系 主动合规·被动应对）/ 风险与升级（行政违法风险等级 + 角色权限 + 升级触发 重大处罚·吊销许可·行刑衔接·责任人人身自由风险强制升级律师 + 行政程序时限护栏）/ 文书风格与监管资源（文书风格 + 主管监管机关·行业协会·法规数据库资源 + 地方性法规与地方裁量基准差异）（4 子技能 + 编排入口）

**为什么**：监管合规律师、企业合规官/法务、被处罚企业需要详细、可操作、可单独触发的技能。监管合规场景中往往只需单点能力（如「只跟踪这家药企本月的药监新规」「只审查这份市场监管局的行政处罚决定书合法性」「只设计这家平台企业的反垄断合规制度」「只起草这份网信办要求的数据安全自查报告」），不必每次跑完整流程。现有 3 个扁平技能均为浅骨架（~10-17 行），深度远不足以支撑实质执业。

**如何应用**：规划时为每个主技能创建子技能目录 `skills/<主技能>/<子技能>/SKILL.md`，每个子技能有独立 SKILL.md 与 frontmatter；原 SKILL.md 改为编排入口。现有浅骨架内容无丢失迁移到子技能并深化。

### 冷启动访谈策略

**决策**：完整访谈流程 → 写入实践配置文件（regulatory-legal/CLAUDE.md）

- 冷启动访谈拆分为可独立触发的子技能，按部分收集身份与企业画像 / 合规立场与服务范围 / 风险与升级 / 文书风格与监管资源
- 访谈结果写入实践配置文件 `regulatory-legal/CLAUDE.md`（散文模板，**本阶段新建**，律师/法务/合规官可读可编辑），而非引入新文件格式
- 每个技能从该配置文件读取个性化设置（执业角色与企业画像、所属行业与主管监管机关、整体服务立场 监管监控/处罚应对/合规建设/报告起草、合规严格度、监管关系、行政违法风险等级、升级矩阵 含重大处罚·吊销许可·行刑衔接强制升级、可用监管机关/法规数据库资源、地方性法规与裁量基准差异）
- 支持重新运行访谈、单独更新某一部分、直接编辑、git 版本控制

**为什么**：与 Phase 2-9 一致——监管合规律师/合规官/法务希望技能按其服务立场（主动合规监控 vs 被动处罚应对、企业侧合规 vs 应对代理）与所属行业（不同行业主管监管机关与规则源差异巨大）个性化，而非固定模板。所属行业与主管监管机关是监管合规工作的核心变量。沿用 CLAUDE.md 散文真相来源避免双重真相来源。注意现有 cold-start-interview 末尾「生成 YAML 格式」表述须在 schema 中纠正为写入 regulatory-legal/CLAUDE.md。

**如何应用**：本阶段新建 regulatory-legal/CLAUDE.md（监管合规语境，章节：我们是谁=执业角色与企业画像 含所属行业与主管监管机关 / 谁在使用 / 可用集成 监管机关·行业协会·法规数据库资源 / 合规与监管立场（监管动态监控/行政处罚应对/合规体系建设/监管报告起草 + 合规严格度 + 监管关系）/ 行政违法风险校准 / 升级矩阵（重大处罚·吊销许可·行刑衔接·人身自由风险强制升级律师）/ 文书风格 / 输出 / 共享护栏（监管合规立场护栏 / 行政程序时限护栏 / 行刑衔接升级护栏 / 律师审查护栏 / 地方性法规与裁量基准提示））；定义 practice-profile-schema（监管合规版）记录「配置文件是什么、在哪、技能怎么读」与字段映射；访谈子技能填充 regulatory-legal/CLAUDE.md 对应章节。

### 技能结构和组织

**决策**：独立技能包（可独立触发）

- 每个子技能是完整的 SKILL.md，含 frontmatter（name/description/argument-hint）
- 子技能可单独触发（`/regulatory-legal:<子技能名>`），也可由编排入口顺序调用
- 技能之间通过共享的 `regulatory-legal/CLAUDE.md` 实践配置文件协调
- 共享引用脊柱放在 `regulatory-legal/skills/_shared/`

**为什么**：律师/合规官/法务希望技能模块化、按需调用，单点能力不被整流程绑架。

**如何应用**：编排入口 SKILL.md 保留跨子技能护栏（服务立场加载、临时模式、企业画像与所属行业确认、行政程序时限识别、监管合规立场护栏、行刑衔接升级护栏），正文改为子技能编排顺序表，逐一指向子技能路径并说明可单独触发。

### 中国法律适配深度

**决策**：全面法律研究（监管合规语境，成文法为主）

- 引用法律：行政处罚法（2021 修订，**条号已重排，须用新条号并标建议复核**：处罚法定原则 第4条、处罚种类 第9条、责令改正 第28条、一事不再罚 同一违法行为不得两次罚款 第29条、从轻减轻 第32条、不予处罚 无主观过错·初次违法危害轻微及时改正 第33条、追责时效 一般2年·涉及公民生命健康安全金融安全且有危害后果的5年 第36条、陈述申辩权 第44/45条、重大执法决定法制审核 第58条、听证 第63条）、行政许可法、行政强制法、行政复议法、行政诉讼法（路径与期限锚点）
- 引用行政法规与部门规章：各行业监管法规规章（市场监管·网信办·金融监管·药监·生态环境·应急管理 等，**部门规章/规范性文件快速演进，一律描述规则 + `[待验证]`，不硬写文号/发文日期/条款编号**）
- 引用规范性文件：行政处罚裁量基准、行业监管规范性文件、合规指引（**属规范性文件且快速演进、地方差异大，一律描述规则 + `[待验证]` + 地方标注**）
- 引用国家标准：企业合规管理体系国家标准（GB/T 系列，**标准编号与版本须核实，描述规则 + `[待验证]`**）
- 引用案例：最高院行政指导性案例、行政复议·行政诉讼典型案例作参考（**案号一律标 `[待验证]`**），不具判例法约束力
- 对不确定内容标 `[待验证]`，所有引用按来源标记（法律/行政法规/部门规章/规范性文件/国家标准 + 案例/学说）+ 来源标签（[法宝]/[法规网站]/[网络搜索 — 需验证]/[模型知识 — 需验证]/[用户提供]）

**已核实/中度把握锚点（须统一标「已核实锚点（中度把握），建议复核」）**：

行政处罚法（2021 修订）：
- 处罚法定原则（没有法定依据或者不遵守法定程序的，行政处罚无效）= 第4条 — 中度把握，建议复核
- 行政处罚种类（警告·通报批评·罚款·没收违法所得没收非法财物·暂扣或者吊销许可证件·降低资质等级·限制开展生产经营活动·责令停产停业·责令关闭·限制从业·行政拘留 等）= 第9条 — 中度把握，建议复核
- 责令改正（实施行政处罚时应当责令当事人改正或者限期改正违法行为）= 第28条 — 中度把握，建议复核
- 一事不再罚（同一个违法行为不得给予两次以上罚款的行政处罚）= 第29条 — 中度把握，建议复核
- 从轻减轻处罚（主动消除或者减轻危害后果·受他人胁迫诱骗·主动供述·配合查处有立功表现 等情形应当从轻或者减轻）= 第32条 — 中度把握，建议复核
- 不予处罚（违法行为轻微并及时改正没有造成危害后果不予处罚；初次违法且危害后果轻微并及时改正可以不予处罚；当事人有证据足以证明没有主观过错不予处罚）= 第33条 — 中度把握，建议复核
- 追责时效（违法行为在二年内未被发现的不再给予行政处罚；涉及公民生命健康安全·金融安全且有危害后果的延长至五年）= 第36条 — 中度把握，建议复核
- 当事人陈述申辩权（当事人有权陈述和申辩；行政机关不得因当事人申辩而加重处罚）= 第44条、第45条 — 中度把握，建议复核
- 重大执法决定法制审核（在作出重大行政处罚决定之前应当进行法制审核）= 第58条 — 中度把握，建议复核
- 听证（责令停产停业·吊销许可证件·较大数额罚款 等较重处罚，当事人要求听证的应当组织听证）= 第63条 — 中度把握，建议复核

**关键纪律**：**不臆造条号。** 行政处罚法 2021 修订上述总则锚点可在标「建议复核」前提下引用（**务必用 2021 修订后的新条号，不得使用旧版条号**）；但**行政许可法·行政强制法·行政复议法·行政诉讼法的具体条号（复议前置·起诉期限·举证责任等）作者把握不足的，一律描述规则 + `[待验证]`，不臆造条号**。**各行业监管部门规章/规范性文件、行政处罚裁量基准属规范性文件且快速演进、地方差异大，一律描述规则并标 `[待验证]`，不硬写文号、发文日期、具体条款编号、罚款数额区间。** 企业合规管理体系国家标准（GB/T）的标准编号与版本须核实，描述规则 + `[待验证]`。司法解释具体条号、案例案号一律标 `[待验证]`，描述规则呈现，绝不臆造。

**为什么**：行政处罚法 2021 修订对条文进行了较大重排（旧版与新版条号不同），模型回忆旧版条号风险极高，**必须用 2021 修订后新条号并标建议复核，不确定就描述规则**。各行业监管规章、规范性文件、裁量基准数量庞大、修订频繁、地方差异大，硬写文号、罚款数额区间风险极高。错误条号或数额会直接误导处罚应对策略与合规判断。律师/合规官/法务需要全面、可核验、明确标注待验证的法律研究支持。

**如何应用**：建立监管合规版引用脊柱（legal-basis-conventions 改编，含「法律/行政法规/部门规章/规范性文件/国家标准」的来源分类与保守标注规则 + 「行政处罚法 2021 条号重排保守标注」纪律 + 「行业监管规章快速演进标注」纪律 + administrative-law-citations 引用库覆盖 行政处罚法 2021 锚点 + 行政许可法/行政强制法/行政复议法/行政诉讼法 规则描述 + 各行业监管规章占位 + 企业合规管理体系国家标准描述），每个子技能输出含 `## 法律依据` 章节，按来源分类引用脊柱并标待验证。

### 监管合规立场与护栏

**决策**：监管合规立场护栏 + 行政程序时限护栏 + 行刑衔接升级护栏 + 律师审查护栏（监管合规特有，须贯穿全部技能）

- **监管合规立场护栏**：本插件**用于行政监管合规预防、行政处罚依法应对、合规体系建设与监管报告起草**。所有输出立足于「帮助企业事前合规、依法应对处罚、如实报送监管报告」，**绝不提供逃避监管、虚假报送、伪造合规证据、规避检查的方法**；识别此类需求时拒绝并提示依法合规、如实申报、依法行使陈述申辩与复议诉讼救济权利。
- **行政程序时限护栏**：行政处罚陈述申辩期、听证申请期、行政复议申请期、行政诉讼起诉期等**程序时限极为关键，逾期将丧失救济权利**。技能识别到处罚决定送达、复议诉讼期限临近时，强标程序时限提示并升级律师，不得静默放任时限流逝。
- **行刑衔接升级护栏**：识别到行政违法可能升格刑事（如违法所得数额·情节达到刑事追诉标准、监管机关移送公安）、或涉及责任人人身自由风险时，**强制升级给执业律师**并提示行刑衔接风险与 criminal-compliance 插件衔接，本插件不替代刑事辩护工作。
- **律师审查护栏**：陈述申辩书·听证陈述·行政复议申请书·行政诉讼起诉状定稿与提交、监管报告定稿报送、合规制度定稿等重大法律后果步骤前，非律师角色须暂停并升级给执业律师确认。

**为什么**：监管合规兼具高程序性（行政救济程序时限严格、逾期失权）、高时效性（行业监管规则快速演进）、潜在行刑衔接风险（行政违法可能升格刑事）三重特殊性。模板化输出若被误用于逃避监管或虚假报送，将构成严重的执业伦理与法律风险；处罚应对若错过程序时限或未识别行刑衔接，可能损害当事人重大权益。必须将「监管合规立场」「程序时限」「行刑衔接升级」固化为不可逾越的护栏。

**如何应用**：在 regulatory-legal/CLAUDE.md 设 `## 合规与监管立场` 章节与 `## 共享护栏` 下的 监管合规立场护栏 / 行政程序时限护栏 / 行刑衔接升级护栏 / 律师审查护栏 / 地方性法规与裁量基准提示；practice-profile-schema 字段映射含监管关系与升级触发；每个子技能「前置」与「错误处理」均体现行政程序时限识别与行刑衔接升级，并在涉及报送、检查应对、合规声明等处显式声明监管合规边界；编排入口入口级护栏含行政程序时限识别、监管合规立场护栏、行刑衔接升级护栏。
