<objective>
延续监管合规插件深化：拆分行政处罚应对（penalty-response）为四个可独立触发的子技能，并**新增** compliance-system（合规体系建设）全新主技能。

本计划交付两块内容：
1. **行政处罚应对拆分** — 按 CONTEXT「深层子技能拆分」决策，将 penalty-response（现 ~17 行浅骨架）拆为四个可独立触发的子技能：处罚决定合法性审查（主体职权·事实证据·法律适用·程序合法性，处罚法定第4条·处罚种类第9条·一事不再罚第29条 建议复核）/ 陈述申辩与听证（陈述申辩权第44/45条·听证第63条 建议复核·不得因申辩加重处罚）/ 从轻减轻不予处罚争取（从轻减轻第32条·不予处罚第33条·责令改正第28条·追责时效第36条 建议复核）/ 行政复议与诉讼衔接（复议前置·起诉期限·举证责任倒置 描述规则待验证）。现有三步纲要（需求收集 / 法律分析 / 输出生成）拆分迁移并大幅深化，原 SKILL.md 改为编排入口。
2. **新增 compliance-system（合规体系建设）主技能** — 按 CONTEXT「新增 1 个全新主技能」决策，创建四个可独立触发的子技能 + 编排入口：合规风险识别（业务·岗位·区域·行业监管维度全面风险梳理 + 风险地图 + 与 regulatory-monitoring 衔接）/ 合规制度与岗责（合规政策与制度体系 + 重点领域专项制度 + 合规岗责与审批与禁止性清单）/ 合规运行与培训（合规组织架构 + 三道防线 + 流程嵌入 + 培训考核与合规文化）/ 合规评估与改进（有效性评估指标 参照企业合规管理体系国家标准 + 定期审计与自查 + 整改闭环 + 持续改进）。

Purpose: 关闭「ROADMAP 第二能力面 penalty-response 未拆分 + 第三能力面 compliance-system 全新主技能缺失」的 gap。本计划与 10-01 文件无重叠（10-01 改 regulatory-monitoring 与 _shared 与 CLAUDE.md，本计划改 penalty-response 与新增 compliance-system），可与 10-01 并行（wave 1）；引用脊柱与 CLAUDE.md 由 10-01 创建，本计划子技能引用它们——执行顺序上须在 10-01 完成后（或 10-01 的 Task 1/2 产出可用后）引用，但因 10-01/10-02 同 wave 由执行编排保证 10-01 先到 _shared 与 CLAUDE.md，本计划在子技能「前置」中引用 _shared 路径与 CLAUDE.md。
Output: 4 个 penalty-response 子技能 SKILL.md + 1 个改造后的 penalty-response 编排入口 + 4 个 compliance-system 子技能 SKILL.md + 1 个新增 compliance-system 编排入口。

中国化原则约束（来自 CLAUDE.md，必须遵守）：成文法为主（行政处罚法 2021·行政许可法·行政强制法·行政复议法·行政诉讼法 为分析基础，各行业监管规章·企业合规管理体系国家标准 为补充）；指导性案例作参考非判例法（行政处罚·复议·诉讼典型案例可参考，案号标待验证）；法院层级识别（基层/中院/高院/最高院四级 + 复议机关）；行业监管特色（市场监管·网信办·金融监管·药监·生态环境·应急管理 等）；执业环境适配（监管合规律师 / 企业合规官 / 企业法务 / 当事人本人 / 非律师但有律师可咨询）。命名约定：技能名 kebab-case；斜杠命令 /regulatory-legal:<skill>。文件格式：JSON 2 空格缩进、文件以换行符结尾、无尾部空格、markdown 表格列数一致。

法律内容硬性要求：引用 _shared/administrative-law-citations.md，按 法律/行政法规/部门规章/规范性文件/国家标准 与 案例/学说 分类，带来源标签。行政处罚法 2021 中度把握锚点（标「已核实锚点（中度把握），建议复核」，**用 2021 修订后新条号**）：处罚法定第4条·处罚种类第9条·责令改正第28条·一事不再罚第29条·从轻减轻第32条·不予处罚第33条·追责时效第36条·陈述申辩第44/45条·法制审核第58条·听证第63条。**行政许可法·行政强制法·行政复议法·行政诉讼法具体条号（复议前置·起诉期限·举证责任倒置等）一律描述规则 + `[待验证]`，绝不臆造条号。** **各行业监管部门规章/规范性文件/裁量基准一律描述规则 + `[待验证]`，不硬写文号·发文日期·条款编号·罚款数额区间·裁量阶次数值。** 企业合规管理体系国家标准（GB/T）标准编号与版本须核实，描述规则 + `[待验证]`，不臆造标准号。司法解释条号、案例案号一律标 `[待验证]`。识别行刑衔接（违法升格刑事·移送公安·人身自由风险）即强制升级律师；识别逃避监管·虚假报送·伪造合规证据·规避检查需求即拒绝（监管合规边界）。
