# 09-CONTEXT.md — 新增插件深化：刑事合规

## Domain

本阶段将刑事合规插件（criminal-compliance）从 v0.1.0 的 4 个扁平浅骨架深化为完整深层子技能集，沿用 Phase 2 商事合同 / Phase 3 诉讼仲裁 / Phase 4 劳动人事 / Phase 5 数据合规 / Phase 6 知识产权 / Phase 7 建设工程 / Phase 8 婚姻家事 已验证的深化模式：将每个主技能拆分为多个可独立触发的子技能，新增 1 个全新主技能（刑事合规体系建设 compliance-system-building），建立全插件复用的刑事合规法律引用脊柱（刑法 / 刑事诉讼法 / 涉案企业合规相关规范性文件 / 认罪认罚从宽制度 的 法条/司法解释/规范性文件 分类 + 待验证标记），定义实践配置文件契约使所有技能从同一处个性化读取，并**新建** criminal-compliance/CLAUDE.md 实践配置模板。

**本插件定位（关键）**：刑事合规是**防御性 / 预防性**执业领域——服务于为企业提供合规风险预防、合规不起诉（合规整改换取从宽处理）、合规体系建设的律师与企业法务。**本插件用于合规预防与刑事辩护，绝不提供任何规避侦查、伪造证据、逃避法律责任、对抗调查的方法**。涉及重大刑事风险（已立案、被采取强制措施、面临侦查、可能构成具体罪名）时**强制升级给执业刑事辩护律师**。

当前 criminal-compliance 是 v0.1.0，仅有 4 个**扁平浅骨架**技能（每个 ~10-16 行），且**尚无 criminal-compliance/CLAUDE.md 实践配置模板**：

- **risk-assessment（刑事风险评估）** — 现 ~16 行浅骨架，仅列「需求收集 / 法律分析 / 输出生成」三步纲要，须深化拆分为 行业高发罪名识别 / 岗位与行为风险点 / 风险等级与可能性评估 / 风险清单与控制建议输出 四个子技能。
- **non-prosecution（合规不起诉方案）** — 现 ~16 行浅骨架，仅列三步纲要，须深化拆分为 适用条件评估 / 合规计划制定 / 第三方监督评估配合 / 不起诉申请与听证 四个子技能。
- **corporate-crime（单位犯罪预防）** — 现 ~16 行浅骨架，仅列三步纲要，须深化拆分为 单位犯罪构成识别 / 常见单位犯罪类型 / 内控与防范机制 / 涉案应对 四个子技能。
- **cold-start-interview（冷启动访谈）** — 现 ~10 行三部分访谈（且末尾「生成 YAML 格式」表述须纠正），须拆为可独立触发的访谈子技能并填充 criminal-compliance/CLAUDE.md。

并按 ROADMAP 覆盖的 5 个能力面，**新增 1 个全新主技能**：

- **compliance-system-building（刑事合规体系建设，新增）** — 合规风险识别 / 合规制度与流程设计 / 合规组织与培训 / 合规有效性评估与整改，四个子技能 + 编排入口。

覆盖能力（对齐 ROADMAP Phase 9 五个能力面）：刑事风险评估（risk-assessment）、合规不起诉方案（non-prosecution）、单位犯罪预防（corporate-crime）、刑事合规体系建设（compliance-system-building，新增）、冷启动访谈（cold-start-interview）。

## Decisions

### 技能深度和范围

**决策**：深层子技能拆分（沿用 Phase 2 / 3 / 4 / 5 / 6 / 7 / 8 模式）

- 每个主技能拆分为 4 个可独立触发的子技能
- 每个子技能包含：目的、详细步骤、检查清单、输出模板、边界条件、错误处理、按来源分类的法律依据（每个子技能 150+ 行的实质内容，非占位骨架）
- 拆分与新增示例：
  - **risk-assessment（刑事风险评估）** 拆分为：行业高发罪名识别（按行业识别高发刑事风险点——如制造/贸易企业的虚开增值税专用发票、走私、假冒注册商标；金融/投资的非法集资 非法吸收公众存款·集资诈骗、操纵市场；医药的商业贿赂；环保的污染环境；安全生产的重大责任事故等，**具体罪名条号一律 `[待验证]`，描述构成要件与行为特征**）/ 岗位与行为风险点（按岗位 采购·销售·财务·高管·法务 识别职务侵占、挪用资金、单位行贿、对非国家工作人员行贿、内幕交易等行为风险点）/ 风险等级与可能性评估（风险发生可能性×影响程度矩阵、刑事风险等级 关键/高/中/低、是否已现实化 苗头·线索·立案）/ 风险清单与控制建议输出（风险清单 风险点·罪名·岗位·可能性·影响·控制建议·责任部门，输出刑事风险评估报告）（4 子技能 + 编排入口）
  - **non-prosecution（合规不起诉方案）** 拆分为：适用条件评估（涉案企业合规适用范围与条件——企业涉嫌经济·职务等犯罪、认罪认罚、愿意合规整改、可能判处的刑罚区间；不起诉类型 法定不起诉·酌定相对不起诉·存疑不起诉 刑事诉讼法第177条 建议复核；认罪认罚从宽 刑事诉讼法第15条 建议复核；**涉案企业合规试点与第三方监督评估机制属规范性文件且快速演进，描述规则并标 `[待验证]`，不硬写文号细节**）/ 合规计划制定（针对涉案问题制定专项合规整改计划 整改措施·时间表·组织保障·可验证成果）/ 第三方监督评估配合（第三方监督评估机制的启动·组成·评估流程·配合要点，**规则描述 + `[待验证]`**）/ 不起诉申请与听证（向检察机关提交合规整改报告·不起诉申请、拟不起诉公开听证配合）（4 子技能 + 编排入口）
  - **corporate-crime（单位犯罪预防）** 拆分为：单位犯罪构成识别（单位犯罪定义=刑法第30条 建议复核、双罚制 对单位判处罚金并对直接负责的主管人员和其他直接责任人员判刑=第31条 建议复核、单位犯罪与自然人犯罪区分 以单位名义·为单位利益·违法所得归单位、单位犯罪与个人犯罪借单位之名的区分）/ 常见单位犯罪类型（单位可构成的常见罪名类别——走私类、金融类 非法吸收公众存款等、税收类 虚开增值税专用发票等、商业贿赂类 单位行贿·对非国家工作人员行贿、扰乱市场秩序类、环境类 污染环境、安全生产类，**具体罪名条号一律 `[待验证]`，描述构成要件**）/ 内控与防范机制（事前防范 制度·审批·留痕·禁止性清单、重点环节风险控制 采购销售财务·关联交易·对外捐赠赞助、岗位制衡与举报机制）/ 涉案应对（单位涉嫌犯罪后的应对——配合调查的合规边界、自首与立功 第67/68条 建议复核、单位与责任人员的责任切割、**强制升级刑事辩护律师**）（4 子技能 + 编排入口）
  - **compliance-system-building（刑事合规体系建设 / 新增）** 拆分为：合规风险识别（全面刑事合规风险梳理——业务·岗位·区域·关联方维度、风险地图与重点领域、与 risk-assessment 衔接）/ 合规制度与流程设计（合规政策与行为准则、重点领域专项合规制度 反商业贿赂·反垄断·数据·安全生产·财税、合规审批与禁止性清单、合规流程嵌入业务）/ 合规组织与培训（合规组织架构 首席合规官·合规部门·业务合规员、三道防线、合规培训与考核、合规文化）/ 合规有效性评估与整改（合规体系有效性评估指标、定期审计与自查、问题整改闭环、合规体系与合规不起诉「有效合规」的衔接，**有效性是涉案企业合规从宽的关键，但具体认定标准属规范性文件 `[待验证]`**）（4 子技能 + 编排入口）
  - **cold-start-interview（冷启动访谈）** 拆分为：身份与企业画像（执业角色 刑事合规律师/企业法务/合规官/当事人 + 服务对象 企业侧合规预防/涉案企业辩护 + 企业画像 行业·规模·业务模式·是否涉外·历史涉案情况·重点刑事风险领域）/ 合规立场与服务范围（整体服务立场 事前合规体系建设/风险评估/涉案合规不起诉/单位犯罪辩护 + 合规严格度 保守 一切疑点从严标红/平衡/务实 + 是否已涉案 预防 vs 应对）/ 风险与升级（刑事风险等级定义 + 角色权限 + **强制升级触发 已立案·强制措施·具体罪名风险·重大涉案一律强制升级刑事辩护律师** + 涉案应对的合规边界护栏）/ 文书风格与监管资源（文书风格 + 当地检察机关·公安·行业监管·第三方监督评估机制资源 + 地方涉案企业合规试点口径地区差异）（4 子技能 + 编排入口）

**为什么**：刑事合规律师、企业合规官/法务、涉案企业需要详细、可操作、可单独触发的技能。刑事合规场景中往往只需单点能力（如「只评估这家制造企业采购岗位的刑事风险点」「只判这起涉案企业能否适用合规不起诉、需要怎么整改」「只识别这个业务模式可能构成的单位犯罪类型」「只设计这家企业的反商业贿赂合规制度」），不必每次跑完整流程。现有 4 个扁平技能均为浅骨架（~10-16 行），深度远不足以支撑实质执业。

**如何应用**：规划时为每个主技能创建子技能目录 `skills/<主技能>/<子技能>/SKILL.md`，每个子技能有独立 SKILL.md 与 frontmatter；原 SKILL.md 改为编排入口。现有浅骨架内容无丢失迁移到子技能并深化。

### 冷启动访谈策略

**决策**：完整访谈流程 → 写入实践配置文件（criminal-compliance/CLAUDE.md）

- 冷启动访谈拆分为可独立触发的子技能，按部分收集身份与企业画像 / 合规立场与服务范围 / 风险与升级 / 执业偏好
- 访谈结果写入实践配置文件 `criminal-compliance/CLAUDE.md`（散文模板，**本阶段新建**，律师/法务/合规官可读可编辑），而非引入新文件格式
- 每个技能从该配置文件读取个性化设置（执业角色与企业画像、整体服务立场 事前合规/风险评估/合规不起诉/单位犯罪辩护、合规严格度、是否已涉案、刑事风险等级、升级矩阵 含已立案/强制措施/具体罪名风险强制升级、可用检察/公安/监管/第三方监督评估资源、地方涉案企业合规试点口径）
- 支持重新运行访谈、单独更新某一部分、直接编辑、git 版本控制

**为什么**：与 Phase 2 / 3 / 4 / 5 / 6 / 7 / 8 一致——刑事合规律师/合规官/法务希望技能按其服务立场（事前预防 vs 涉案应对、企业侧合规 vs 辩护）个性化，而非固定模板。同一企业的刑事风险从「建设合规体系预防」与「已涉案争取合规不起诉」视角侧重截然不同，服务立场是刑事合规工作的核心变量。**刑事合规直接关涉企业与个人的刑事责任、人身自由**，配置中须固化「合规预防与辩护立场」「不提供规避侦查/逃避责任方法」「涉刑强制升级刑事辩护律师」的护栏与强制升级规则。沿用 CLAUDE.md 散文真相来源避免双重真相来源。注意现有 cold-start-interview 末尾「生成 YAML 格式」表述须在 schema 中纠正为写入 criminal-compliance/CLAUDE.md。

**如何应用**：本阶段新建 criminal-compliance/CLAUDE.md（刑事合规语境，章节：我们是谁=执业角色与企业画像 / 谁在使用 / 可用集成 检察·公安·行业监管·第三方监督评估机制资源 / 合规与服务立场（事前合规体系建设/风险评估/合规不起诉/单位犯罪辩护 + 合规严格度 + 是否已涉案）/ 刑事风险校准 / 升级矩阵（已立案·强制措施·具体罪名风险·重大涉案强制升级刑事辩护律师）/ 文书风格 / 输出 / 共享护栏（合规预防与辩护立场护栏 / 不提供规避侦查或逃避责任方法护栏 / 律师审查护栏 / 涉刑强制升级护栏 / 地方涉案企业合规试点提示））；定义 practice-profile-schema（刑事合规版）记录「配置文件是什么、在哪、技能怎么读」与字段映射；访谈子技能填充 criminal-compliance/CLAUDE.md 对应章节。

### 技能结构和组织

**决策**：独立技能包（可独立触发）

- 每个子技能是完整的 SKILL.md，含 frontmatter（name/description/argument-hint）
- 子技能可单独触发（`/criminal-compliance:<子技能名>`），也可由编排入口顺序调用
- 技能之间通过共享的 `criminal-compliance/CLAUDE.md` 实践配置文件协调
- 共享引用脊柱放在 `criminal-compliance/skills/_shared/`

**为什么**：律师/合规官/法务希望技能模块化、按需调用，单点能力不被整流程绑架。

**如何应用**：编排入口 SKILL.md 保留跨子技能护栏（服务立场加载、临时模式、企业画像确认、涉案情形识别、合规预防与辩护立场护栏、不提供规避侦查方法护栏、涉刑强制升级），正文改为子技能编排顺序表，逐一指向子技能路径并说明可单独触发。

### 中国法律适配深度

**决策**：全面法律研究（刑事合规语境，成文法为主）

- 引用法律：刑法（单位犯罪总则部分 第30-31条、自首立功 第67-68条、缓刑 第72条 等总则锚点；**各具体罪名条号一律 `[待验证]`，描述构成要件**）、刑事诉讼法（不起诉 第177条、认罪认罚从宽 第15条 等）
- 引用规范性文件：涉案企业合规改革相关（最高检涉案企业合规改革试点工作、企业合规第三方监督评估机制管理规定 2021 等，**属规范性文件且快速演进，一律描述规则 + `[待验证]`，不硬写文号/条号细节**）、认罪认罚从宽制度相关指导意见
- 引用司法解释：相关罪名的具体司法解释（如虚开增值税专用发票、非法吸收公众存款等的定罪量刑标准，**一律描述规则 + `[待验证]`，不臆造数额/比例与条号**）

**关键纪律**：**不臆造条号。** 刑法总则（单位犯罪第30/31条、自首第67条、立功第68条、缓刑第72条）、刑事诉讼法（不起诉第177条、认罪认罚从宽第15条）有把握的锚点可在标「建议复核」前提下引用；但**各具体罪名条号（如虚开增值税专用发票罪、非法吸收公众存款罪、职务侵占罪、单位行贿罪、对非国家工作人员行贿罪、污染环境罪、重大责任事故罪等）一律标 `[待验证]`，以构成要件描述呈现，绝不臆造具体罪名条号与法定刑档**。具体罪名的定罪量刑数额标准（如虚开税额、非法吸收金额、贿赂数额）多来自司法解释且经多次修订，**一律描述规则 + `[待验证]`，不写裸数额裸条号**。**涉案企业合规试点、第三方监督评估机制管理规定、认罪认罚从宽指导意见属规范性文件且快速演进，一律描述规则并标 `[待验证]`，不硬写文号、发文日期、具体条款编号**。

**为什么**：刑事合规直接关涉企业与个人的刑事责任、罚金、人身自由，对条号与构成要件准确性要求极高；具体罪名条号、定罪量刑数额标准众多且易随司法解释修订变化，模型回忆具体罪名条号与数额标准风险极高，**必须以构成要件描述 + 待验证呈现，不得臆造具体罪名条号**。涉案企业合规改革仍处试点演进中，文号、机制细节、有效合规认定标准快速变化，硬写文号细节风险极高。错误条号或数额会直接误导刑事风险判断与辩护策略。律师/合规官/法务需要全面、可核验、明确标注待验证的法律研究支持。

**如何应用**：建立刑事合规版引用脊柱（legal-basis-conventions 改编，含「法条/司法解释/规范性文件」的来源分类与保守标注规则 + 「具体罪名条号一律待验证」纪律 + 「规范性文件快速演进标注」纪律 + criminal-law-citations 引用库覆盖 刑法总则单位犯罪/自首立功缓刑锚点 + 刑事诉讼法不起诉/认罪认罚锚点 + 涉案企业合规规范性文件描述 + 具体罪名构成要件占位结构），每个子技能输出含 `## 法律依据` 章节，按来源分类引用脊柱并标待验证。

### 刑事合规立场与护栏

**决策**：合规预防与辩护立场护栏 + 不提供规避侦查/逃避责任方法护栏 + 涉刑强制升级护栏（刑事合规特有，须贯穿全部技能）

- **合规预防与辩护立场护栏**：本插件**仅用于合规风险预防、合规体系建设、合规不起诉争取与刑事辩护**。所有输出立足于「帮助企业事前预防刑事风险、涉案后依法合规整改争取从宽」，不替企业作有罪无罪的最终结论（那是司法机关与辩护律师的职责）。
- **不提供规避侦查/逃避责任方法护栏**：**绝不提供任何规避侦查、伪造或毁灭证据、串供、对抗调查、转移资产逃避责任、帮助逃避法律追究的方法**。识别到此类需求时拒绝并提示：合规的唯一正道是依法整改、如实供述、认罪认罚争取从宽。
- **涉刑强制升级护栏**：识别到企业或个人**已被立案、被采取强制措施（拘留·逮捕·取保候审·监视居住）、面临侦查、可能构成具体罪名、重大涉案**等情形时，**强制升级给执业刑事辩护律师**，本插件输出仅作合规研究与初步分析，不替代辩护律师的专业判断与辩护工作。
- **律师审查护栏**：合规不起诉申请提交、合规整改报告定稿、涉案应对方案、向司法机关提交材料等重大法律后果步骤前，非律师角色须暂停并升级给执业刑事律师确认。

**为什么**：刑事合规兼具高法律后果（刑事责任·罚金·人身自由）、高敏感（涉案调查）、强专业（辩护需执业刑事律师）三重特殊性。模板化输出若被误用于规避侦查或逃避责任，将构成严重的执业伦理与法律风险；涉案应对若无执业刑事律师介入，可能损害当事人重大权益。必须将「合规预防与辩护立场」「拒绝规避侦查」「涉刑强制升级」固化为不可逾越的护栏。

**如何应用**：在 criminal-compliance/CLAUDE.md 设 `## 合规与服务立场` 章节与 `## 共享护栏` 下的 合规预防与辩护立场护栏 / 不提供规避侦查或逃避责任方法护栏 / 律师审查护栏 / 涉刑强制升级护栏；practice-profile-schema 字段映射含是否已涉案与强制升级；每个子技能「前置」与「错误处理」均体现涉案情形识别与强制升级，并在涉及调查应对、证据、供述等处显式声明合规边界；编排入口入口级护栏含涉案情形识别、合规预防与辩护立场护栏、不提供规避侦查方法护栏与涉刑强制升级。
