# 06-CONTEXT.md — 新增插件深化：知识产权

**Phase**: 6

**Date**: 2026-06-05

## Domain

本阶段将知识产权插件（ip-legal）从 v0.1.0 的 4 个扁平浅骨架深化为完整深层子技能集，沿用 Phase 2 商事合同 / Phase 3 诉讼仲裁 / Phase 4 劳动人事 / Phase 5 数据合规已验证的深化模式：将每个主技能拆分为多个可独立触发的子技能，新增 1 个全新主技能（知识产权清权 ip-clearance），建立全插件复用的知识产权法律引用脊柱（商标法 / 专利法 / 著作权法 2020修正 / 反不正当竞争法 + 相关司法解释 的 法条/司法解释/案例 分类 + 待验证标记），定义实践配置文件契约使所有技能从同一处个性化读取，并**新建** ip-legal/CLAUDE.md 实践配置模板。

当前 ip-legal 是 v0.1.0，仅有 4 个**扁平浅骨架**技能（每个 ~14-18 行），且**尚无 ip-legal/CLAUDE.md 实践配置模板**：

- **trademark-search（商标查询与近似分析）** — 现 ~18 行浅骨架，仅列「工作流」三步纲要，须深化拆分为 商标检索与查询 / 商标近似判断 / 类似商品与服务判定 / 在先权利与障碍分析 四个子技能。
- **patent-analysis（专利侵权初步分析）** — 现 ~18 行浅骨架，仅列三步纲要，须深化拆分为 技术特征比对与权利要求解读 / 全面覆盖原则 / 等同原则 / 现有技术抗辩与不侵权抗辩 四个子技能。
- **copyright-registration（著作权登记）** — 现 ~18 行浅骨架，仅列三步纲要，须深化拆分为 作品类型认定 / 权属判定 / 登记材料准备 / 登记流程 四个子技能。
- **cold-start-interview（冷启动访谈）** — 现 ~14 行三部分访谈（且末尾「生成 YAML 格式」表述须纠正），须拆为可独立触发的访谈子技能并填充 ip-legal/CLAUDE.md。

并按 ROADMAP 覆盖的 5 个能力面，**新增 1 个全新主技能**：

- **ip-clearance（知识产权清权 / clearance，新增）** — 商标清权检索 / 专利清权（FTO 自由实施初步分析）/ 著作权与字号域名清权 / 清权风险评估与清权报告，四个子技能 + 编排入口。多维清权检索（商标/专利/著作权/字号/域名）+ 风险评估 + 清权报告。

覆盖能力（对齐 ROADMAP Phase 6 五个能力面）：商标查询与近似分析（trademark-search）、专利侵权初步分析（patent-analysis）、著作权登记（copyright-registration）、知识产权 clearance（ip-clearance，新增）、冷启动访谈（cold-start-interview）。

## Decisions

### 技能深度和范围

**决策**：深层子技能拆分（沿用 Phase 2 / Phase 3 / Phase 4 / Phase 5 模式）

- 每个主技能拆分为 4 个可独立触发的子技能
- 每个子技能包含：目的、详细步骤、检查清单、输出模板、边界条件、错误处理、按来源分类的法律依据（每个子技能 150+ 行的实质内容，非占位骨架）
- 拆分与新增示例：
  - **trademark-search（商标查询与近似分析）** 拆分为：商标检索与查询（检索数据库选择 / 检索式构造 / 查询结果整理 / 国际分类 尼斯分类应用）/ 商标近似判断（音/形/义三要素比对 / 整体观察与隔离比对 / 显著部分对比 / 混淆可能性判断）/ 类似商品与服务判定（《类似商品和服务区分表》应用 / 跨类保护 / 商品服务关联性判断）/ 在先权利与障碍分析（在先注册商标 / 在先申请 / 驰名商标 / 在先字号著作权外观设计等在先权利 / 注册障碍评估）（4 子技能 + 编排入口）
  - **patent-analysis（专利侵权初步分析）** 拆分为：技术特征比对与权利要求解读（权利要求拆解为技术特征 / 独立权利要求 vs 从属权利要求 / 说明书与附图解释权利要求 / 被诉技术方案特征提取）/ 全面覆盖原则（字面侵权判定 / 技术特征逐一比对表 / 缺一不可原则）/ 等同原则（手段-功能-效果三要素 + 本领域普通技术人员无需创造性劳动可联想 / 等同侵权判定 / 禁止反悔与捐献原则限制）/ 现有技术抗辩与不侵权抗辩（现有技术抗辩 / 先用权抗辩 / 不侵权抗辩 / 专利权稳定性初判 无效风险）（4 子技能 + 编排入口）
  - **copyright-registration（著作权登记）** 拆分为：作品类型认定（作品独创性判断 / 法定作品类型 文字/美术/视听/计算机软件等 / 思想表达二分 / 不受保护对象）/ 权属判定（创作者原始取得 / 职务作品 / 委托作品 / 法人作品 / 合作作品 / 权利归属约定）/ 登记材料准备（申请主体证明 / 作品说明 / 权利归属证明 / 创作完成与发表证明 / 样本要求）/ 登记流程（中国版权保护中心登记 / 软件著作权登记 / 流程与周期 / 登记证书效力 登记非确权但具初步证据效力）（4 子技能 + 编排入口）
  - **ip-clearance（知识产权清权 / clearance，新增）** 拆分为：商标清权检索（拟用商标/标识的在先商标障碍检索 / 近似与类似判断 / 可注册性与可使用性初判）/ 专利清权 FTO 初步分析（拟实施技术方案的在先专利检索 / 自由实施 Freedom-to-Operate 初判 / 侵权风险与规避设计提示）/ 著作权与字号域名清权（作品/素材的著作权来源合规 / 企业字号在先权利 / 域名与商标冲突 / 不正当竞争风险）/ 清权风险评估与清权报告（多维清权风险分级 / 清权结论 可用/有风险/不可用 / 清权报告组装）（4 子技能 + 编排入口）
  - **cold-start-interview（冷启动访谈）** 拆分为：身份与团队（权利人画像 / 知识产权组合 商标/专利/著作权 / IP 律师/IP 经理/内部法务角色）/ 维权与申请立场（整体维权立场 进取维权/防御/平衡 / 商标策略 / 专利策略 / 著作权策略 / 清权严格度）/ 风险与升级（风险等级定义 / 升级矩阵 / 维权/诉讼/异议触发）/ 文书风格与检索数据库（文书风格 / 可用检索数据库与工具 商标网/专利检索/版权登记系统 / 行业特色）（4 子技能 + 编排入口）

**为什么**：知识产权律师、企业 IP 经理与内部法务需要详细、可操作、可单独触发的技能。知识产权场景中往往只需单点能力（如「只判这两个商标是否近似」「只做这个技术方案的全面覆盖比对」「只查这个拟用 logo 有没有在先商标障碍」「只认定这个作品属于哪类作品」），不必每次跑完整流程。现有 4 个扁平技能均为浅骨架（~14-18 行），深度远不足以支撑实质执业。

**如何应用**：规划时为每个主技能创建子技能目录 `skills/<主技能>/<子技能>/SKILL.md`，每个子技能有独立 SKILL.md 与 frontmatter；原 SKILL.md 改为编排入口。现有浅骨架内容无丢失迁移到子技能并深化。

### 冷启动访谈策略

**决策**：完整访谈流程 → 写入实践配置文件（ip-legal/CLAUDE.md）

- 冷启动访谈拆分为可独立触发的子技能，按部分收集权利人 / IP 经理 / 律师执业立场
- 访谈结果写入实践配置文件 `ip-legal/CLAUDE.md`（散文模板，**本阶段新建**，律师/IP 经理可读可编辑），而非引入新文件格式
- 每个技能从该配置文件读取个性化设置（权利人画像、IP 组合、整体维权立场、商标/专利/著作权策略、清权严格度、风险等级、升级矩阵、可用检索数据库、行业特色）
- 支持重新运行访谈、单独更新某一部分、直接编辑、git 版本控制

**为什么**：与 Phase 2 / 3 / 4 / 5 一致——律师/IP 经理希望技能按其组织的 IP 组合与维权立场个性化，而非固定模板。沿用 CLAUDE.md 散文真相来源避免双重真相来源。注意现有 cold-start-interview 末尾「生成 YAML 格式」表述须在 schema 中纠正为写入 ip-legal/CLAUDE.md。

**如何应用**：本阶段新建 ip-legal/CLAUDE.md（知识产权语境，章节：我们是谁=权利人画像与 IP 组合 / 谁在使用 / 可用集成 检索数据库 / 维权与申请立场（商标/专利/著作权策略 + 清权严格度）/ 风险校准 / 升级矩阵（维权/异议/诉讼触发）/ 文书风格 / 输出 / 共享护栏（行业识别 / 时效性触发 / IP 法 2020/2021 修正版本提示））；定义 practice-profile-schema（知识产权版）记录「配置文件是什么、在哪、技能怎么读」与字段映射；访谈子技能填充 ip-legal/CLAUDE.md 对应章节。

### 技能结构和组织

**决策**：独立技能包（可独立触发）

- 每个子技能是完整的 SKILL.md，含 frontmatter（name/description/argument-hint）
- 子技能可单独触发（`/ip-legal:<子技能名>`），也可由编排入口顺序调用
- 技能之间通过共享的 `ip-legal/CLAUDE.md` 实践配置文件协调
- 共享引用脊柱放在 `ip-legal/skills/_shared/`

**为什么**：律师/IP 经理希望技能模块化、按需调用，单点能力不被整流程绑架。

**如何应用**：编排入口 SKILL.md 保留跨子技能护栏（维权立场加载、临时模式、行业识别、清权严格度、权利人画像/IP 组合确认、目的地/特权检查），正文改为子技能编排顺序表，逐一指向子技能路径并说明可单独触发。

### 中国法律适配深度

**决策**：全面法律研究（知识产权语境，成文法为主）

- 引用法律：商标法、专利法、著作权法（2020修正，2021年6月1日施行）、反不正当竞争法
- 引用司法解释：商标授权确权行政案件审理若干问题的规定、专利侵权纠纷案件应用法律若干问题的解释（一）（二）、著作权民事纠纷案件适用法律若干问题的解释 等（**具体条号一律标 `[待验证]`，以描述规则呈现**）
- 引用部门规章/规范性文件：商标审查审理指南、专利审查指南、作品自愿登记办法、计算机软件著作权登记办法 等（`[待验证]`）
- 案例：最高院/最高院知识产权法庭指导性案例与典型案例作参考（**案号一律标 `[待验证]`**），不具判例法约束力
- 对不确定内容标 `[待验证]`，所有引用按来源标记（法条/司法解释/部门规章 + 案例/学说）+ 来源标签（[法宝]/[法规网站]/[网络搜索 — 需验证]/[模型知识 — 需验证]/[用户提供]）

**已核实/中度把握锚点（IP 法 2020/2021 修正后条号须保守，统一标「建议复核」）**：

> **重要纪律 — IP 法在 2020/2021 修正中重排条号，须极度保守。仅写有把握的条号，其余一律 `[待验证]`。** 以下中度把握锚点须标「已核实锚点（中度把握），建议复核」，并以「规则 + 条号建议复核」呈现：

- 专利权期限 = 专利法第42条（发明专利 20 年 / 实用新型 10 年 / 外观设计 15 年，均自申请日起算；外观设计 15 年系 2021 修正后，旧为 10 年）— 中度把握，建议复核
- 商标注册有效期 = 10 年，期满可续展（每次续展 10 年）— 规则稳定；具体条号建议复核
- 商标侵权类型 = 商标法第57条（七类侵权行为，含未经许可同一种商品使用相同/近似商标致混淆等）— 中度把握，建议复核
- 著作权保护期 = 自然人作品终生及死后 50 年；法人作品/视听作品自首次发表后 50 年（著作权法 2020 修正）— 规则稳定；具体条号建议复核
- 著作权登记 = 自愿登记原则（登记非确权，但可作初步证据）— 规则稳定；办法名称见下
- 专利侵权判定 = **全面覆盖原则 + 等同原则**（来自专利侵权司法解释，**描述规则即可，不写裸条号**）；等同 = 手段-功能-效果基本相同 + 本领域普通技术人员无需创造性劳动可联想 — 规则稳定（来自司法解释）
- 商标近似 / 类似商品 = 以相关公众一般注意力为标准、整体比对与隔离观察、混淆可能性（来自商标授权确权司法解释，**描述规则即可**）
- 作品独创性与思想表达二分 = 受保护的是表达不是思想（著作权法基本原理）— 规则稳定

**关键纪律**：**不臆造条号。IP 法 2020/2021 修正后条号重排，须极度保守。** 只有上述中度把握锚点可在标「建议复核」前提下引用其条号（专利法第42条、商标法第57条）；其余商标法/专利法/著作权法/反不正当竞争法的具体条号若不确定，**描述规则并标 `[待验证]`，绝不硬写**。司法解释的具体条号、案例的案号一律标 `[待验证]`，以描述规则呈现。专利侵权判定（全面覆盖+等同）、商标近似判断、类似商品判定来自司法解释与审查指南，**描述规则即可，不写裸条号**。

**为什么**：知识产权对条号准确性要求极高，且商标法/专利法/著作权法均经 2019/2020/2021 修正导致条号重排，模型回忆旧条号风险极高，错误条号会直接误导侵权判定、保护期计算、续展期限与清权结论。律师/IP 经理需要全面、可核验、明确标注待验证的法律研究支持。

**如何应用**：建立知识产权版引用脊柱（legal-basis-conventions 改编，含「司法解释/部门规章 审查指南/案例」的来源分类与「IP 法 2020/2021 修正版本提示 — 条号重排保守标注」规则 + ip-law-citations 引用库覆盖 商标法/专利法/著作权法/反不正当竞争法/相关司法解释），每个子技能输出含 `## 法律依据` 章节，按来源分类引用脊柱并标待验证。
