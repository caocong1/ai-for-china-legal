# 07-CONTEXT.md — 新增插件深化：建设工程

**Phase**: 7

## Domain

本阶段将建设工程插件（construction-legal）从 v0.1.0 的 4 个扁平浅骨架深化为完整深层子技能集，沿用 Phase 2 商事合同 / Phase 3 诉讼仲裁 / Phase 4 劳动人事 / Phase 5 数据合规 / Phase 6 知识产权 已验证的深化模式：将每个主技能拆分为多个可独立触发的子技能，新增 1 个全新主技能（商品房买卖合同审查 commercial-housing-review），建立全插件复用的建设工程法律引用脊柱（民法典合同编建设工程合同 第788-808条 / 建筑法 / 招标投标法 / 建设工程质量管理条例 / 最高法建设工程施工合同司法解释(一) / 商品房买卖合同司法解释 / 城市房地产管理法 的 法条/司法解释/部门规章 分类 + 待验证标记），定义实践配置文件契约使所有技能从同一处个性化读取，并**新建** construction-legal/CLAUDE.md 实践配置模板。

当前 construction-legal 是 v0.1.0，仅有 4 个**扁平浅骨架**技能（每个 ~14-18 行），且**尚无 construction-legal/CLAUDE.md 实践配置模板**：

- **contract-review（施工合同审查）** — 现 ~18 行浅骨架，仅列「工作流」三步纲要，须深化拆分为 主体资质与合同效力 / 工期与质量 / 价款与结算 / 违约与解除 四个子技能。
- **payment-dispute（工程款争议处理）** — 现 ~18 行浅骨架，仅列三步纲要，须深化拆分为 工程价款优先受偿权 / 结算依据与黑白合同 / 质量抗辩与反索赔 / 工程款利息与垫资 四个子技能。
- **bidding-compliance（建设工程招投标合规）** — 现 ~18 行浅骨架，仅列三步纲要，须深化拆分为 必须招标范围与方式 / 招标投标程序 / 串通投标与虚假招标 / 中标无效与异议投诉 四个子技能。
- **cold-start-interview（冷启动访谈）** — 现 ~14 行三部分访谈（且末尾「生成 YAML 格式」表述须纠正），须拆为可独立触发的访谈子技能并填充 construction-legal/CLAUDE.md。

并按 ROADMAP 覆盖的 5 个能力面，**新增 1 个全新主技能**：

- **commercial-housing-review（商品房买卖合同审查，新增）** — 预售许可与主体核查 / 定金与认购书 / 逾期交房与逾期办证违约 / 面积误差处理 / 贷款不成解除，四个子技能 + 编排入口（其中面积误差与逾期可合并为「交付与面积违约」一个子技能以保持 4 子技能结构）。

覆盖能力（对齐 ROADMAP Phase 7 五个能力面）：施工合同审查（contract-review）、工程款争议处理（payment-dispute）、商品房买卖合同审查（commercial-housing-review，新增）、建设工程招投标合规（bidding-compliance）、冷启动访谈（cold-start-interview）。

## Decisions

### 技能深度和范围

**决策**：深层子技能拆分（沿用 Phase 2 / Phase 3 / Phase 4 / Phase 5 / Phase 6 模式）

- 每个主技能拆分为 4 个可独立触发的子技能
- 每个子技能包含：目的、详细步骤、检查清单、输出模板、边界条件、错误处理、按来源分类的法律依据（每个子技能 150+ 行的实质内容，非占位骨架）
- 拆分与新增示例：
  - **contract-review（施工合同审查）** 拆分为：主体资质与合同效力（发包人/承包人资质 + 施工许可与必须招标 + 合同无效情形 转包/违法分包/无资质/未招标/黑白合同 + 无效合同折价补偿处理）/ 工期与质量（开竣工与工期顺延 + 竣工验收 + 质量标准与保修 + 工程质量责任）/ 价款与结算（合同计价方式 固定总价/单价/成本加酬金 + 工程量与变更签证 + 结算依据 + 进度款与质保金）/ 违约与解除（工期违约/质量违约 + 发包人/承包人法定解除 + 损失赔偿 + 已完工程处理）（4 子技能 + 编排入口）
  - **payment-dispute（工程款争议处理）** 拆分为：工程价款优先受偿权（优先受偿权成立要件 + 范围与行使期限 + 与抵押权/消费者购房人权利顺位）/ 结算依据与黑白合同（结算依据认定 + 备案中标合同结算 黑白合同 + 未签证变更与实际履行）/ 质量抗辩与反索赔（发包人质量抗辩 + 质量不合格的工程款处理 + 反索赔工期延误/质量缺陷）/ 工程款利息与垫资（欠付利息计付标准与起算 + 垫资及垫资利息约定效力）（4 子技能 + 编排入口）
  - **bidding-compliance（建设工程招投标合规）** 拆分为：必须招标范围与方式（必须招标的工程建设项目范围 + 公开招标/邀请招标 + 可不招标情形 + 规模标准）/ 招标投标程序（招标公告与资格审查 + 招标文件 + 投标与开标评标定标 + 中标通知书与合同签订）/ 串通投标与虚假招标（投标人之间串通 + 招标人与投标人串通 + 弄虚作假骗取中标 + 围标陪标识别）/ 中标无效与异议投诉（中标无效情形 + 异议与投诉处理 + 行政监督与法律责任）（4 子技能 + 编排入口）
  - **commercial-housing-review（商品房买卖合同审查 / 新增）** 拆分为：预售许可与主体核查（商品房预售许可证 + 五证核查 + 开发商主体资质 + 现售条件）/ 定金与认购书（认购书/定金/订金性质 + 定金罚则 + 认购书向正式合同转化 + 诚意金返还）/ 交付与违约（逾期交房违约责任 + 逾期办证违约责任 + 面积误差处理 3% 规则 + 质量与保修）/ 贷款解除与风险（按揭贷款不成的合同解除 + 退房退款 + 一房二卖与查封风险 + 网签备案）（4 子技能 + 编排入口）
  - **cold-start-interview（冷启动访谈）** 拆分为：身份与项目（主体画像 发包人/承包人/监理/购房人代理 + 项目类型 房建/市政/装饰/商品房买卖 + 团队角色）/ 审查立场（整体立场 发包人侧/承包人侧/中立/购房人侧 + 合同审查严格度 + 争议处理倾向）/ 风险与升级（风险等级定义 + 升级矩阵 + 诉讼/仲裁/优先受偿权行使触发）/ 文书风格与地方规定（文书风格 + 工程造价/鉴定机构 + 地方建设主管部门与地方性规定）（4 子技能 + 编排入口）

**为什么**：建设工程律师、发包人/承包人内部法务、监理与造价人员需要详细、可操作、可单独触发的技能。建设工程场景中往往只需单点能力（如「只判这份施工合同是否因转包无效」「只算这笔工程款的优先受偿权能不能行使、期限有没有过」「只看这个项目是否必须招标」「只审这份商品房买卖合同的逾期交房违约金」），不必每次跑完整流程。现有 4 个扁平技能均为浅骨架（~14-18 行），深度远不足以支撑实质执业。

**如何应用**：规划时为每个主技能创建子技能目录 `skills/<主技能>/<子技能>/SKILL.md`，每个子技能有独立 SKILL.md 与 frontmatter；原 SKILL.md 改为编排入口。现有浅骨架内容无丢失迁移到子技能并深化。

### 冷启动访谈策略

**决策**：完整访谈流程 → 写入实践配置文件（construction-legal/CLAUDE.md）

- 冷启动访谈拆分为可独立触发的子技能，按部分收集主体画像 / 项目类型 / 审查立场 / 执业偏好
- 访谈结果写入实践配置文件 `construction-legal/CLAUDE.md`（散文模板，**本阶段新建**，律师/法务可读可编辑），而非引入新文件格式
- 每个技能从该配置文件读取个性化设置（主体画像 发包人/承包人/监理/购房人侧、项目类型、审查立场、风险等级、升级矩阵、可用造价/鉴定资源、地方性规定）
- 支持重新运行访谈、单独更新某一部分、直接编辑、git 版本控制

**为什么**：与 Phase 2 / 3 / 4 / 5 / 6 一致——建设工程律师/法务希望技能按其代理立场（发包人侧 vs 承包人侧 vs 购房人侧）与项目类型个性化，而非固定模板。同一份施工合同从发包人视角审查与从承包人视角审查侧重点截然不同，立场是建设工程审查的核心变量。沿用 CLAUDE.md 散文真相来源避免双重真相来源。注意现有 cold-start-interview 末尾「生成 YAML 格式」表述须在 schema 中纠正为写入 construction-legal/CLAUDE.md。

**如何应用**：本阶段新建 construction-legal/CLAUDE.md（建设工程语境，章节：我们是谁=主体画像与项目组合 / 谁在使用 / 可用集成 造价鉴定资源 / 审查与争议立场（发包人/承包人/中立/购房人侧 + 各审查严格度）/ 风险校准 / 升级矩阵（诉讼/仲裁/优先受偿权行使触发）/ 文书风格 / 输出 / 共享护栏（项目类型识别 / 时效性触发 / 地方性规定提示））；定义 practice-profile-schema（建设工程版）记录「配置文件是什么、在哪、技能怎么读」与字段映射；访谈子技能填充 construction-legal/CLAUDE.md 对应章节。

### 技能结构和组织

**决策**：独立技能包（可独立触发）

- 每个子技能是完整的 SKILL.md，含 frontmatter（name/description/argument-hint）
- 子技能可单独触发（`/construction-legal:<子技能名>`），也可由编排入口顺序调用
- 技能之间通过共享的 `construction-legal/CLAUDE.md` 实践配置文件协调
- 共享引用脊柱放在 `construction-legal/skills/_shared/`

**为什么**：律师/法务希望技能模块化、按需调用，单点能力不被整流程绑架。

**如何应用**：编排入口 SKILL.md 保留跨子技能护栏（审查立场加载、临时模式、项目类型识别、主体画像确认、目的地/特权检查），正文改为子技能编排顺序表，逐一指向子技能路径并说明可单独触发。

### 中国法律适配深度

**决策**：全面法律研究（建设工程语境，成文法为主）

- 引用法律：民法典合同编（建设工程合同 第788-808 条；通则部分违约/解除/担保援引商事合同脊柱）、建筑法、招标投标法、城市房地产管理法
- 引用行政法规：建设工程质量管理条例、建设工程安全生产管理条例、招标投标法实施条例、城市房地产开发经营管理条例（`[待验证]` 具体条号）
- 引用司法解释：最高法建设工程施工合同司法解释(一)（黑白合同以备案中标合同结算、无效合同参照合同约定折价补偿、优先受偿权范围与期限、工程质量与价款抗辩等，**具体条号一律标 `[待验证]`，以描述规则呈现**）、商品房买卖合同司法解释（预售、定金、逾期交房办证、面积误差 3% 规则等，**具体条号一律标 `[待验证]`**）
- 引用部门规章/规范性文件：必须招标的工程建设项目规定、工程总承包/资质管理相关规章（`[待验证]`）
- 案例：最高院指导性案例与建设工程典型案例作参考（**案号一律标 `[待验证]`**），不具判例法约束力
- 对不确定内容标 `[待验证]`，所有引用按来源标记（法条/司法解释/部门规章 + 案例/学说）+ 来源标签（[法宝]/[法规网站]/[网络搜索 — 需验证]/[模型知识 — 需验证]/[用户提供]）

**已核实/中度把握锚点（须统一标「建议复核」）**：

> **重要纪律 — 民法典建设工程合同条号有把握的可引用并标建议复核，司法解释具体条号一律 `[待验证]`，描述规则呈现。** 以下中度把握锚点须标「已核实锚点（中度把握），建议复核」：

- 建设工程合同定义（承包人进行工程建设、发包人支付价款的合同；含勘察/设计/施工合同）= 民法典第788条 — 中度把握，建议复核
- 竣工验收（建设工程竣工经验收合格后方可交付使用；未经验收或验收不合格不得交付使用）= 民法典第799条 — 中度把握，建议复核
- 承包人工程价款优先受偿权（发包人未按约支付价款，承包人可催告后就该工程折价或拍卖价款优先受偿；不动产消费者购房人权利与垫资/工程款顺位等由司法解释细化）= 民法典第807条 — 中度把握，建议复核
- 建设工程合同章条文区间 = 民法典合同编第788-808条（分则典型合同之一）— 规则稳定；逐条具体条号建议复核
- 通则部分（违约责任第577/583/584条、违约金第585条第2款、定金第586/587/588条、法定解除第563条、情势变更第533条、保证第681/686条）援引商事合同脊柱 civil-code-contract-citations.md 已核实锚点 — 复用 Phase 2 已核实结论
- 黑白合同以备案中标合同结算、无效施工合同参照合同约定折价补偿（参照合同约定支付工程价款）= 最高法建设工程施工合同司法解释(一)（**描述规则即可，不写裸条号**）— 规则稳定（来自司法解释）
- 必须招标范围 = 招标投标法及招标投标法实施条例、必须招标的工程建设项目规定（**描述规则 + 规模标准，具体条号 `[待验证]`**）
- 商品房面积误差处理 = 商品房买卖合同司法解释面积误差比绝对值超出 3% 的处理规则（误差比 3% 以内据实结算、超出 3% 买受人可解除或多退少补含开发商双倍返还超 3% 部分价款的规则）（**描述规则，具体条号标 `[待验证]`**）— 规则稳定（来自司法解释）；条号待验证
- 商品房定金/认购、逾期交房办证违约、按揭贷款不成解除 = 商品房买卖合同司法解释（**描述规则，具体条号一律 `[待验证]`**）

**关键纪律**：**不臆造条号。** 民法典建设工程合同有把握的锚点（第788/799/807条、第788-808 区间）可在标「建议复核」前提下引用；通则部分复用商事脊柱已核实锚点（第585条第2款/第588条/第563条/第533条/第681条/第686条等，沿用 Phase 2 verified anchor list，**不得回退到已知错误如把 588 写成 585条第2款**）。**司法解释（建设工程施工合同司法解释(一)、商品房买卖合同司法解释）的具体条号一律标 `[待验证]`，以描述规则呈现**；行政法规（质量管理条例、招标投标法实施条例）具体条号 `[待验证]`；案例案号一律 `[待验证]`。黑白合同结算、无效合同折价补偿、优先受偿权范围期限、面积误差 3%、逾期违约金调整均来自司法解释，**描述规则即可，不写裸条号**。

**为什么**：建设工程争议金额大、对条号准确性要求高；建设工程施工合同司法解释经历(一)(二)整合与历次修订，模型回忆具体条号风险高；商品房买卖合同司法解释亦经修订。错误条号会直接误导优先受偿权行使期限、黑白合同结算、面积误差退补与逾期违约金计算。律师/法务需要全面、可核验、明确标注待验证的法律研究支持。

## 法律引用纪律（继承 MEMORY — 必须遵守）

模型对 民法典/法律 条号常「自信但有时错误」，`[待验证]` 标记必须保留，但错误数字仍是错误内容。Phase 2 曾出 6 处条号错误经对抗审查才发现。

- 作者**不要臆造具体条号**；不确定时**描述规则并标 `[待验证]`，不承诺数字**。
- 复用商事脊柱已核实锚点时**不得回退已知错误**：违约金与定金竞合=第588条（不是 585条第2款）；违约金调整=585条第2款；法定解除=563条；情势变更=533条；保证合同=681条/连带保证=686条；委托开发成果归属=859条。
- 建设工程司法解释（施工合同司法解释(一)、商品房买卖司法解释）具体条号风险高，**一律 `[待验证]`，描述规则呈现**。
- 本阶段产出后应进行对抗式 verify → fix → re-check（法律引用 / 实质 / 规范 / 集成 多镜头），fixer 可能引入新错误，须复核。
