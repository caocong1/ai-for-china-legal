## Domain

本阶段将婚姻家事插件（family-legal）从 v0.1.0 的 4 个扁平浅骨架深化为完整深层子技能集，沿用 Phase 2 商事合同 / Phase 3 诉讼仲裁 / Phase 4 劳动人事 / Phase 5 数据合规 / Phase 6 知识产权 / Phase 7 建设工程 已验证的深化模式：将每个主技能拆分为多个可独立触发的子技能，新增 1 个全新主技能（财产分割方案 property-division），建立全插件复用的婚姻家事法律引用脊柱（民法典婚姻家庭编 第1040-1118条 / 民法典继承编 第1119-1163条 / 妇女权益保障法 / 最高法婚姻家庭编司法解释(一) / 继承编司法解释 的 法条/司法解释/案例 分类 + 待验证标记），定义实践配置文件契约使所有技能从同一处个性化读取，并**新建** family-legal/CLAUDE.md 实践配置模板。

当前 family-legal 是 v0.1.0，仅有 4 个**扁平浅骨架**技能（每个 ~14-18 行），且**尚无 family-legal/CLAUDE.md 实践配置模板**：

- **divorce-agreement（离婚协议起草）** — 现 ~16 行浅骨架，仅列「工作流」三步纲要，须深化拆分为 财产分割条款 / 子女抚养与探望 / 债务处理 / 协议生效与登记 四个子技能。
- **inheritance（继承权分析）** — 现 ~16 行浅骨架，仅列三步纲要，须深化拆分为 法定继承顺序与份额 / 遗嘱继承效力 / 遗产范围与债务清偿 / 代位继承与转继承 四个子技能。
- **will-drafting（遗嘱起草）** — 现 ~16 行浅骨架，仅列三步纲要，须深化拆分为 遗嘱形式选择 / 遗嘱能力与意思表示 / 遗嘱内容与必留份 / 见证与生效 四个子技能。
- **cold-start-interview（冷启动访谈）** — 现 ~14 行三部分访谈（且末尾「生成 YAML 格式」表述须纠正），须拆为可独立触发的访谈子技能并填充 family-legal/CLAUDE.md。

并按 ROADMAP 覆盖的 5 个能力面，**新增 1 个全新主技能**：

- **property-division（财产分割方案，新增）** — 共同财产认定 / 特殊财产分割（房产·股权·公司·知识产权·投资）/ 分割原则与照顾（照顾子女·女方·无过错方·家务补偿·经济帮助）/ 隐藏转移财产救济（少分不分·再次分割），四个子技能 + 编排入口。

覆盖能力（对齐 ROADMAP Phase 8 五个能力面）：离婚协议起草（divorce-agreement）、遗嘱起草（will-drafting）、继承权分析（inheritance）、财产分割方案（property-division，新增）、冷启动访谈（cold-start-interview）。

## Decisions

### 技能深度和范围

**决策**：深层子技能拆分（沿用 Phase 2 / 3 / 4 / 5 / 6 / 7 模式）

- 每个主技能拆分为 4 个可独立触发的子技能
- 每个子技能包含：目的、详细步骤、检查清单、输出模板、边界条件、错误处理、按来源分类的法律依据（每个子技能 150+ 行的实质内容，非占位骨架）
- 拆分与新增示例：
  - **divorce-agreement（离婚协议起草）** 拆分为：财产分割条款（夫妻共同财产范围 第1062条 / 个人财产 第1063条 / 约定财产制 第1065条 / 房产·存款·股权·投资分割条款拟定 / 一方隐瞒财产的协议救济保留）/ 子女抚养与探望（抚养权归属 第1084条 / 抚养费标准与支付 / 探望权安排 第1086条 建议复核 / 变更抚养关系）/ 债务处理（夫妻共同债务认定 共债共签·家庭日常生活需要 / 个人债务 / 对外清偿与内部分担 / 协议债务条款的对外效力局限）/ 协议生效与登记（协议离婚冷静期30日 第1077条 / 民政登记生效 第1076条 / 协议反悔与可撤销 / 财产分割协议与离婚的关联效力）（4 子技能 + 编排入口）
  - **inheritance（继承权分析）** 拆分为：法定继承顺序与份额（法定继承开始 / 第一顺序 配偶·子女·父母 第1127条 / 第二顺序 兄弟姐妹·祖父母·外祖父母 / 同顺序均等与照顾酌情 第1130条 建议复核 / 丧偶儿媳女婿尽赡养义务的继承地位 第1129条 建议复核）/ 遗嘱继承效力（遗嘱继承优先于法定继承 / 遗嘱有效要件 / 数份遗嘱冲突与最后顺位规则 第1142条 建议复核 / 遗赠与遗赠扶养协议优先 第1158条 建议复核）/ 遗产范围与债务清偿（遗产范围 第1122条 / 夫妻共同财产先析产后继承 / 遗产债务清偿以遗产实际价值为限 第1161条 建议复核 / 无人继承遗产归属 第1160条 建议复核）/ 代位继承与转继承（代位继承 被继承人子女先于死亡由其晚辈直系血亲代位 第1128条 建议复核 / 转继承 继承人在遗产分割前死亡 / 代位与转继承区分 / 丧失继承权情形 第1125条 建议复核）（4 子技能 + 编排入口）
  - **will-drafting（遗嘱起草）** 拆分为：遗嘱形式选择（自书 第1134条 / 代书 第1135条 / 打印 第1136条 / 录音录像 第1137条 / 口头 第1138条 / 公证 第1139条；各形式要件与适用情形对照、公证遗嘱不再具优先效力的修订）/ 遗嘱能力与意思表示（完全民事行为能力要件 / 真实意思表示无欺诈胁迫 / 受欺诈胁迫所立遗嘱无效 / 伪造篡改遗嘱的效力 第1143条 建议复核）/ 遗嘱内容与必留份（遗产处分范围限于个人财产·先析产 / 必留份 缺乏劳动能力又无生活来源的继承人 第1141条 建议复核 / 附义务遗嘱 第1144条 建议复核 / 遗赠与遗嘱区分）/ 见证与生效（见证人资格与人数 第1140条 建议复核 / 不能作见证人情形 / 各形式生效要件 日期·签名·见证签名 / 遗嘱执行与遗产管理人 第1145条 建议复核）（4 子技能 + 编排入口）
  - **property-division（财产分割方案 / 新增）** 拆分为：共同财产认定（夫妻共同财产范围 第1062条 工资·生产经营·投资·知识产权收益等 / 个人财产 第1063条 婚前·人身损害赔偿·遗嘱赠与明确归一方等 / 约定财产制 第1065条 / 婚前财产婚后增值 自然增值vs主动增值）/ 特殊财产分割（房产 婚前一方购买·父母出资·按揭增值·加名 / 股权与公司份额分割 / 知识产权收益 / 股票基金等投资性资产 / 养老金保险）/ 分割原则与照顾（照顾子女·女方·无过错方权益原则 第1087条 建议复核 / 家务劳动补偿 第1088条 / 经济帮助 第1090条 / 离婚损害赔偿 重婚·同居·家暴·虐待遗弃 第1091条）/ 隐藏转移财产救济（一方隐藏转移变卖毁损挥霍共同财产 第1092条 建议复核 少分或不分 / 离婚后发现可再次分割 / 举证与财产保全）（4 子技能 + 编排入口）
  - **cold-start-interview（冷启动访谈）** 拆分为：身份与当事人画像（执业角色 律师/法务/当事人 + 服务对象立场 原告侧/被告侧/中立·一方代理 + 当事人婚姻继承情况 婚姻状态·子女·财产构成·有无家暴等敏感情形）/ 审查与代理立场（整体代理立场 离婚一方/继承某继承人/遗嘱人 + 协议审查严格度 + 诉讼倾向 协议优先/诉讼）/ 敏感度与升级（高度敏感情形识别 家暴·未成年人·当事人情绪危机 + 风险等级 + 升级矩阵 含强制升级 家暴·涉刑·重大财产·未成年人利益）/ 文书风格与地方规定（文书风格 + 当地法院·民政·公证资源 + 地方性规定 彩礼·房产·抚养费标准地区差异）（4 子技能 + 编排入口）

**为什么**：婚姻家事律师、企业/个人法务、当事人本人需要详细、可操作、可单独触发的技能。婚姻家事场景中往往只需单点能力（如「只拟这份离婚协议的财产分割条款」「只判这份打印遗嘱形式是否有效、见证人是否适格」「只算第一顺序法定继承人的份额、丧偶儿媳能不能继承」「只评估这套婚前按揭婚后还贷房产怎么分」），不必每次跑完整流程。现有 4 个扁平技能均为浅骨架（~14-18 行），深度远不足以支撑实质执业。

**如何应用**：规划时为每个主技能创建子技能目录 `skills/<主技能>/<子技能>/SKILL.md`，每个子技能有独立 SKILL.md 与 frontmatter；原 SKILL.md 改为编排入口。现有浅骨架内容无丢失迁移到子技能并深化。

### 冷启动访谈策略

**决策**：完整访谈流程 → 写入实践配置文件（family-legal/CLAUDE.md）

- 冷启动访谈拆分为可独立触发的子技能，按部分收集当事人画像 / 代理立场 / 敏感度与升级 / 执业偏好
- 访谈结果写入实践配置文件 `family-legal/CLAUDE.md`（散文模板，**本阶段新建**，律师/法务可读可编辑），而非引入新文件格式
- 每个技能从该配置文件读取个性化设置（执业角色与当事人画像、整体代理立场 离婚一方/某继承人/遗嘱人、协议审查严格度、敏感度与风险等级、升级矩阵 含家暴/未成年人强制升级、可用法院/民政/公证资源、地方性规定）
- 支持重新运行访谈、单独更新某一部分、直接编辑、git 版本控制

**为什么**：与 Phase 2 / 3 / 4 / 5 / 6 / 7 一致——婚姻家事律师/法务希望技能按其代理立场（离婚原告侧 vs 被告侧 vs 中立、继承中代理哪个继承人、遗嘱人立场）个性化，而非固定模板。同一份离婚财产分割从主张方与被主张方视角侧重截然不同，立场是婚姻家事工作的核心变量。**婚姻家事高度敏感**，配置中须固化当事人情绪、未成年人利益最大化、家暴等敏感情形的护栏与强制升级规则。沿用 CLAUDE.md 散文真相来源避免双重真相来源。注意现有 cold-start-interview 末尾「生成 YAML 格式」表述须在 schema 中纠正为写入 family-legal/CLAUDE.md。

**如何应用**：本阶段新建 family-legal/CLAUDE.md（婚姻家事语境，章节：我们是谁=执业角色与当事人画像 / 谁在使用 / 可用集成 法院·民政·公证·鉴定资源 / 审查与代理立场（离婚一方/某继承人/遗嘱人 + 协议审查严格度 + 诉讼倾向）/ 敏感度校准（家暴·未成年人·情绪危机识别）/ 风险校准 / 升级矩阵（家暴/涉刑/重大财产/未成年人利益强制升级）/ 文书风格 / 输出 / 共享护栏（当事人情绪护栏 / 未成年人利益最大化 / 律师审查护栏 / 地方性规定提示））；定义 practice-profile-schema（婚姻家事版）记录「配置文件是什么、在哪、技能怎么读」与字段映射；访谈子技能填充 family-legal/CLAUDE.md 对应章节。

### 技能结构和组织

**决策**：独立技能包（可独立触发）

- 每个子技能是完整的 SKILL.md，含 frontmatter（name/description/argument-hint）
- 子技能可单独触发（`/family-legal:<子技能名>`），也可由编排入口顺序调用
- 技能之间通过共享的 `family-legal/CLAUDE.md` 实践配置文件协调
- 共享引用脊柱放在 `family-legal/skills/_shared/`

**为什么**：律师/法务/当事人希望技能模块化、按需调用，单点能力不被整流程绑架。

**如何应用**：编排入口 SKILL.md 保留跨子技能护栏（代理立场加载、临时模式、当事人画像确认、敏感情形识别、当事人情绪护栏、未成年人利益护栏、目的地/特权检查），正文改为子技能编排顺序表，逐一指向子技能路径并说明可单独触发。

### 中国法律适配深度

**决策**：全面法律研究（婚姻家事语境，成文法为主）

- 引用法律：民法典婚姻家庭编（第1040-1118条）、民法典继承编（第1119-1163条）、妇女权益保障法、未成年人保护法、反家庭暴力法
- 引用司法解释：最高法关于适用民法典婚姻家庭编的解释(一)（夫妻共同财产/共同债务/抚养/离婚损害赔偿等，**具体条号一律标 `[待验证]`，以描述规则呈现**）、最高法关于适用民法典继承编的解释(一)（代位继承/必留份/遗嘱效力/遗产债务等，**具体条号一律标 `[待验证]`**）
- 引用案例：最高院指导性案例与婚姻家事典型案例作参考（**案号一律标 `[待验证]`**），不具判例法约束力
- 对不确定内容标 `[待验证]`，所有引用按来源标记（法条/司法解释 + 案例/学说）+ 来源标签（[法宝]/[法规网站]/[网络搜索 — 需验证]/[模型知识 — 需验证]/[用户提供]）

**关键纪律**：**不臆造条号。** 民法典婚姻家庭编（第1040-1118区间）、继承编（第1119-1163区间）有把握的锚点可在标「建议复核」前提下引用；离婚协议本质为合同，财产分割/债务/意思表示瑕疵可援引合同编通则锚点结构（解除/可撤销/欺诈胁迫），但**婚姻家事领域的具体条号属婚姻家庭编/继承编，不得错套合同编条号**。**司法解释（婚姻家庭编司法解释(一)、继承编司法解释(一)）的具体条号一律标 `[待验证]`，以描述规则呈现**；案例案号一律 `[待验证]`。共同债务认定（共债共签/家庭日常生活需要/超出部分债权人举证）、离婚损害赔偿适用细化、房产婚后还贷增值分割、彩礼返还、必留份与遗赠扶养顺位等规则多来自司法解释或区间内条文，**把握不足即描述规则 + 待验证，不写裸条号**。

**为什么**：婚姻家事直接关涉当事人财产、子女与人身安全，对条号准确性要求高；婚姻家庭编/继承编是 2021 年民法典施行后新编排的条号体系（取代原婚姻法/继承法），模型回忆旧法条号（如原婚姻法第17/18/32条、原继承法第10条）风险极高，**必须使用民法典新条号，不得回退旧法条号**。司法解释亦经民法典施行后修订整合。错误条号会直接误导财产分割、继承顺序与遗嘱效力判断。律师/法务/当事人需要全面、可核验、明确标注待验证的法律研究支持。

**如何应用**：建立婚姻家事版引用脊柱（legal-basis-conventions 改编，含「司法解释/案例」的来源分类与保守标注规则 + 「旧法条号→民法典新条号迁移警示」 + family-law-citations 引用库覆盖 民法典婚姻家庭编 第1040-1118条 + 继承编 第1119-1163条 + 妇女权益保障法 + 婚姻家庭编司法解释(一) + 继承编司法解释(一)），每个子技能输出含 `## 法律依据` 章节，按来源分类引用脊柱并标待验证。

### 婚姻家事敏感性与护栏

**决策**：高敏感度护栏 + 未成年人利益最大化 + 律师审查护栏（婚姻家事特有，须贯穿全部技能）

- **当事人情绪护栏**：婚姻家事当事人常处于情绪危机（离婚冲突、丧亲悲痛、家暴恐惧）。技能输出须以中立、克制、非煽动的措辞呈现，不替当事人作情绪化判断或激化对立；识别情绪危机信号时提示寻求专业支持。
- **未成年人利益最大化**：凡涉及子女抚养、探望、财产安排，须以「最有利于未成年子女」为首要原则（第1084条），不得仅按父母博弈结论处理；涉及未成年人重大利益须强制升级。
- **家暴/涉刑/人身安全识别与强制升级**：识别家庭暴力、虐待、遗弃、重婚、转移巨额财产、涉及刑事（重婚罪、虐待罪、遗弃罪、拒不执行）等情形时强制升级给律师，并优先提示人身安全（反家庭暴力法人身安全保护令路径）。
- **律师审查护栏**：协议离婚生效、遗嘱定稿、财产分割定案、发律师函/起诉等重大法律后果步骤前，非律师角色须暂停并升级给执业律师确认。

**为什么**：婚姻家事兼具高财产价值、高人身敏感、涉未成年人三重特殊性，是本项目所有领域中敏感度最高的之一。模板化或冷冰冰的输出可能激化当事人对立、损害未成年人利益、或在家暴情形下危及人身安全。

**如何应用**：在 family-legal/CLAUDE.md 设 `## 敏感度校准` 章节与 `## 共享护栏` 下的当事人情绪护栏/未成年人利益最大化/家暴识别与强制升级/律师审查护栏；practice-profile-schema 字段映射含敏感度与升级；每个子技能「前置」与「错误处理」均体现敏感情形识别与强制升级；编排入口入口级护栏含敏感情形识别与当事人情绪护栏。

## 法律引用纪律（继承 MEMORY — 必须遵守）

模型对 民法典/法律 条号常「自信但有时错误」，`[待验证]` 标记必须保留，但错误数字仍是错误内容。Phase 2 曾出 6 处条号错误经对抗审查才发现。

- 作者**不要臆造具体条号**；不确定时**描述规则并标 `[待验证]`，不承诺数字**。
- 婚姻家庭编/继承编是民法典施行后新编排的条号体系，**严禁回退旧法条号（原婚姻法第17/18/32条、原继承法第10条等），一律使用民法典婚姻家庭编 第1040-1118条 / 继承编 第1119-1163条 新条号**。
- 中度把握锚点（共同财产第1062条/个人财产第1063条/约定财产制第1065条/协议离婚第1076条/冷静期第1077条/诉讼离婚第1079条/子女抚养第1084条/家务补偿第1088条/经济帮助第1090条/损害赔偿第1091条/遗产范围第1122条/法定继承顺序第1127条/遗嘱形式第1134-1139条/必留份第1141条/遗赠扶养第1158条）标「建议复核」，作者把握不足的条号降级为「描述规则 + `[待验证]`」。
