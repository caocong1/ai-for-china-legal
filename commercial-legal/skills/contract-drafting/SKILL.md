# 合同起草

## 事项上下文

**事项上下文。** 检查实践级别 CLAUDE.md 中的 `## 事项工作区` 部分。如果未启用（默认），则使用实践级上下文。如果已启用且没有活跃事项，询问："这个起草属于哪个事项？运行 `/commercial-legal:matter-workspace switch <slug>` 或说 `practice-level`。" 加载活跃事项的 `matter.md` 获取事项级上下文。输出写入事项文件夹。

## 目的

根据用户的商业需求、风险偏好和中国大陆法律要求，起草结构完整、条款清晰、风险可控的商事合同。起草过程遵循用户的实践配置文件中的审查立场，预防常见法律风险，并提供法律依据引用。

## 工作流

### 第一步：需求收集

收集合同起草所需的关键信息：

| 问题 | 说明 |
|------|------|
| 合同类型 | 买卖合同 / 技术服务合同 / 租赁合同 / 合作协议 / 其他 |
| 我方立场 | 甲方（买方/委托方）/ 乙方（卖方/受托方） |
| 对方当事人 | 名称、类型（大公司/中小企业）、谈判地位 |
| 合同金额 | 总价/年费/计价方式 |
| 合同期限 | 期限长度、续约机制 |
| 核心商业条款 | 付款方式、交付标准、验收条件等 |
| 特殊要求 | 数据处理、知识产权、担保、竞业限制等 |

### 第二步：框架设计

根据合同类型和用户需求，设计合同框架：

1. **标准条款清单**（根据合同类型）
   - 定义条款
   - 合同标的
   - 价格和付款
   - 交付和验收
   - 质量保证
   - 违约责任
   - 知识产权
   - 保密条款
   - 不可抗力
   - 争议解决
   - 合同终止
   - 其他条款

2. **特别条款**（根据用户需求）
   - 数据处理条款（如涉及个人信息）
   - 知识产权归属条款（如委托开发）
   - 竞业限制条款（如特定行业）
   - 担保条款（如需要担保）

3. **法律依据引用**
   - 民法典合同编相关条款
   - 相关司法解释
   - 行业法规（如适用）

### 第三步：条款起草

逐个起草合同条款，遵循以下原则：

1. **语言清晰**：避免模糊表述，使用明确的法律术语
2. **风险预防**：预设可能的争议点，提前约定解决方案
3. **立场保护**：根据用户实践配置文件中的审查立场起草
4. **法律依据**：引用相关法律条文作为依据
5. **可执行性**：确保条款在法律上可执行

### 第四步：风险自查

起草完成后，进行风险自查：

- [ ] 是否包含所有标准条款？
- [ ] 是否体现用户的审查立场？
- [ ] 是否预防了常见法律风险？
- [ ] 是否引用了相关法律依据？
- [ ] 是否具有可执行性？
- [ ] 是否存在模糊表述？
- [ ] 是否存在交易破坏者条款？

### 第五步：输出生成

生成合同草案和相关文档：

```markdown
# [合同类型]草案

**合同编号：** [编号]
**起草日期：** [日期]
**我方立场：** 甲方 / 乙方
**对方当事人：** [名称]

---

## 合同正文

[合同条款全文]

---

## 起草说明

### 条款设计依据

| 条款 | 法律依据 | 设计说明 |
|------|----------|----------|
| [条款名称] | [法条引用] | [设计理由] |

### 风险提示

| 条款 | 风险类型 | 风险说明 | 建议 |
|------|----------|----------|------|
| [条款名称] | [风险类型] | [风险说明] | [建议] |

### 谈判要点

| 条款 | 我方立场 | 可接受备选 | 交易破坏者 |
|------|----------|------------|------------|
| [条款名称] | [立场] | [备选] | [是否] |

---

## 法律依据

[相关法律条文引用]
```

## 输出格式

**完整合同草案：** 包含合同正文、起草说明、风险提示和谈判要点。

**合同模板：** 可复用的合同模板，包含占位符和填写说明。

**条款库：** 常用条款的语言库，供后续合同起草参考。
