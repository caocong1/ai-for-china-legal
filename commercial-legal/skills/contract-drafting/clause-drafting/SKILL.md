# 条款起草（clause-drafting）

> **前置**：
> 1. 读取 `commercial-legal/CLAUDE.md`（实践配置文件），加载我方立场（审查立场、违约金立场、交易破坏者）。
> 2. 读取 `framework-design` 的框架草案（或从用户获取等效信息）。
> 3. 读取 `commercial-legal/skills/_shared/civil-code-contract-citations.md`（民法典合同编引用库）。
> 4. 读取 `commercial-legal/skills/_shared/legal-basis-conventions.md`（引用规范）。
> 本子技能可被 `contract-drafting/SKILL.md` 编排入口调用，也可通过 `/commercial-legal:clause-drafting` 单独触发。

---

## 目的

逐条起草合同正文，将框架设计转化为具体、可执行的合同语言。每个条款须满足五原则：
语言清晰、风险预防、立场保护、法律依据、可执行性。起草完成的条款须能直接用于谈判。

---

## 详细步骤

### 步骤 1：逐条起草原则确认

在开始起草每一章之前，对照五原则：

| 原则 | 说明 | 检查方式 |
|------|------|---------|
| **1. 语言清晰** | 避免模糊表述（「尽快」「适当」「合理」须替换为具体期限/标准） | 检查是否有不可量化的表述 |
| **2. 风险预防** | 预设可能的争议点，提前约定解决方案，不给对方留解释空间 | 对每个义务问：「对方不履行时怎么办？」 |
| **3. 立场保护** | 根据实践配置文件中的审查立场起草，偏向我方利益的合理保护 | 与「绝不接受」清单对照，确认无违背 |
| **4. 法律依据** | 约定条款须以相关法律条文为依据，识别强制性规定（不得约定排除） | 引用 `civil-code-contract-citations` 对应法条 |
| **5. 可执行性** | 条款须可被法院/仲裁机构强制执行（义务主体、义务内容、时间节点须明确） | 问：「这个条款能否直接向法院申请强制执行？」 |

### 步骤 2：核心义务条款起草

#### 2A. 合同标的条款

**起草要点**：
- 标的描述须精确（型号/规格/版本/数量/质量标准/交付形式）
- 买卖合同：明确所有权转移时点（支付货款时/交付时/其他）
- 技术服务合同：明确「工作成果」的边界（工作说明书作为附件）
- 常见模糊风险：「等」「及类似产品」「参照行业标准」—— 逐一替换为具体描述

#### 2B. 价格与付款条款

**起草要点**：
- 价格条款：固定价/浮动价（调整公式）/计时付款（计费方式）须明确
- 付款条件：里程碑/时间节点须具体，勿用「验收后支付」（需定义验收标准）
- 发票要求：发票类型（增值税专用发票/普通发票）、开票信息、税率
- 付款违约：逾期付款的利息计算标准（固定利率 vs 参照 LPR `[待验证]`）

**甲方立场（买方/委托方）**：争取分期付款、延后尾款、扣留质保金
**乙方立场（卖方/受托方）**：争取预付款占比更高、减少质保金或缩短扣留期

#### 2C. 交付与验收条款

**起草要点**：
- 交付时间：具体日历日（非工作日），里程碑节点可挂钩付款
- 验收标准：量化的验收测试/检验标准（非「符合行业标准」）
- 验收期限：收到后 X 个工作日内完成验收；逾期未验收视为验收通过（卖方立场）或视为拒绝验收（买方立场）
- 验收异议：书面通知 + 异议期限 + 补救期限
- 分批交付：明确每批独立验收还是整体验收

#### 2D. 违约责任条款（核心——需深度起草）

**起草要点**：

1. **违约情形枚举**（明确列举，避免「其他违约行为」兜底条款的不确定性）：
   - 延迟履行（每日/每周迟延违约金）
   - 质量不符合约定（退换货/修复/减价）
   - 付款违约（逾期利息计算）
   - 根本违约（解除权触发）

2. **违约金设计**（按民法典第585条）：
   - 违约金标准：以实际损失为基准参考，过高将被法院调整
   - 30% 经验值：约定违约金超出实际损失 30% 以上时，法院可予调整（实践参考，非法定比例）`[待验证 — 见下方法律依据]`
   - 建议违约金设计在合理范围内，同时约定「实际损失高于违约金时，赔偿实际损失」以充分保护
   - 定金与违约金并用：双方不可同时适用，需择一选择（《民法典》第588条）

3. **责任限制**（对照实践配置文件）：
   - 直接损失 vs 间接/后果性损失限制
   - 责任上限基数（如「过去 12 个月费用」须精确定义）
   - 例外情形（carveouts）：故意/重大过失、IP 侵权、保密违反、数据泄露、人身损害（见法条第506条）

4. **禁止约定的免责条款**（强制性规定，不得排除）：
   - 人身损害责任（《民法典》第506条第1款）
   - 故意或重大过失造成的财产损失（《民法典》第506条第2款）

**示范语言（甲方立场——技术服务合同违约责任）**：

```
第X条 违约责任

X.1 乙方延迟交付：乙方未能在约定日期完成交付的，应按合同价款的【0.05%】/日向甲方支付违约金，
    但每次延迟违约金累计不超过合同价款的【5%】；延迟超过【30】日的，甲方有权解除本合同并
    要求赔偿实际损失。

X.2 质量不符合约定：乙方交付的成果经验收不符合本合同约定标准的，应在甲方书面通知后
    【10】个工作日内修复或重新交付；超期未修复的，甲方有权要求退款并赔偿额外损失。

X.3 责任限制：任何一方在本合同下的总赔偿责任（违约金与损失赔偿合计）不超过
    [本合同总价款/受索赔前12个月乙方已收取的费用]；但以下情形不受本条限制：
    （一）因故意或重大过失造成的损失；（二）保密义务违反；（三）知识产权侵权；
    （四）个人信息保护法律违规造成的损失；（五）人身伤害。

X.4 违约金与实际损失：约定的违约金低于实际损失的，守约方有权主张赔偿实际损失。
    约定的违约金若明显过高，当事人可请求人民法院或仲裁机构予以调整（《民法典》第585条）。

X.5 定金与违约金不并用：本合同同时约定定金和违约金的，当事人只能选择适用其中之一。
```

#### 2E. 知识产权条款（若框架中含此章节）

**起草要点**：
- 委托开发：约定工作成果归甲方所有（须明确「委托开发」定性）
- 既有知识产权（背景 IP）：各方保留，仅授权使用
- 许可范围：被许可方不得超范围使用
- 改进归属：改进/派生物归属须事先约定

#### 2F. 格式条款（对方提供的标准合同）

**审查特别提示**：

若本次起草是基于对方（通常为大客户）提供的标准合同模板，须对每个条款检查：
- 是否存在不合理免除对方责任/加重我方责任的格式条款（《民法典》第497条）
- 争议条款按「不利于提供方原则」解释（《民法典》第498条）
- 对方未尽提示/说明义务的重要条款，可主张对方不生效（《民法典》第496条）

#### 2G. 争议解决条款

**起草要点**：
- 诉讼 vs 仲裁（两者不可同时约定）
- 诉讼：写明具体法院（「[城市]人民法院」而非「有管辖权的法院」）
- 仲裁：须写明仲裁机构（如 CIETAC、北京仲裁委、上海仲裁委）+ 仲裁规则 + 仲裁地
- 适用法律：中国大陆法律（默认）；涉外合同须明确约定

### 步骤 3：条款一致性检查

起草完所有条款后，检查：
- [ ] 定义条款：文中所有使用的专业术语在定义章节均有定义
- [ ] 数字一致：同一数字（如付款期限、通知期）在不同条款中保持一致
- [ ] 交叉引用：条款引用正确（如「第X条第X款」）
- [ ] 逻辑链：从义务违反 → 通知 → 救济机会 → 违约金/损失赔偿 → 终止，逻辑链完整

## 检查清单

**起草质量**：
- [ ] 所有必须条款均已起草（无空白）
- [ ] 无模糊表述（「尽快」「适当」「合理」均已替换为具体期限/标准）
- [ ] 违约金设计有法条依据，在合理范围内
- [ ] 责任限制条款已与实践配置文件「我方立场」对照
- [ ] 禁止免责条款（《民法典》第506条）未被约定排除

**立场保护**：
- [ ] 我方立场在关键条款（违约金、责任限制、IP、数据保护）中已充分体现
- [ ] 「绝不接受」清单中的条款未出现或已标记
- [ ] 我方「交易破坏者」风险已预防

**格式合规**：
- [ ] 条款编号连续，格式一致
- [ ] 无尾部空格，文件以换行结尾
- [ ] 法律依据引用按 法条/案例/学说 分类、标待验证

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 某条款无法用中文法律语言准确表达（如双方均采用英文模板） | 起草中文版并说明：「以下条款参照英文原文意图起草，建议律师确认中文版本是否准确传达原意」 |
| 用户要求约定违约金为合同价款的 200% | 提示风险：「该违约金明显高于合理范围，依据《民法典》第585条，有被法院调整的风险。建议调整至合同价款的 30%~50% 范围内」`[待验证]`，由用户决定是否调整 |
| 强制性法条与用户要求冲突（如用户要求完全排除人身损害责任） | 明确告知：「排除人身损害责任的条款依据《民法典》第506条第1款无效，不能写入合同。建议以限额赔偿代替」`[待验证]` |
| 起草中发现框架遗漏重要条款 | 提示：「发现框架中未包含[条款名称]，这可能导致[风险]。是否补充？」（Rule 1：补漏，继续任务） |
| 合同语言超出技术服务/买卖合同范畴（如需要大量公司法条款） | 说明范围限制，建议结合专业律师意见 |

| 错误场景 | 处理方式 |
|---------|---------|
| 框架草案缺失 | 告知：「未检测到 framework-design 的框架草案。我可以使用[合同类型]的默认框架，或请提供框架描述」 |

| 要求起草境外法律适用的条款 | 说明：「境外法律条款超出本插件能力范围，请咨询专业跨境律师」，其余条款正常起草 |

**第585条违约金实践注意事项（`[待验证]`）：**

> 关于「违约金超过实际损失 30%」的经验值：司法实践中，对于违约金明显过高的判断，
> 并无固定 30% 的法定比例。该比例来源于最高人民法院关于买卖合同的司法解释（2009年版
> 相关规则，2020年修正后适用情况有所变化）。现行《民法典合同编通则司法解释》（2023年）
> 对违约金调整标准有进一步规定，具体适用标准须核查最新司法解释全文。
