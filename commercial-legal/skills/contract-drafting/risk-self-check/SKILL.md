# 风险自查（risk-self-check）

> **前置**：
> 1. 读取 `commercial-legal/CLAUDE.md`（实践配置文件），加载「绝不接受」清单与「交易破坏者」条款。
> 2. 读取 `clause-drafting` 的合同草案（或从用户提供草案文本）。
> 3. 读取 `commercial-legal/skills/_shared/civil-code-contract-citations.md`（效力瑕疵检查依据）。
> 本子技能可被 `contract-drafting/SKILL.md` 编排入口调用，也可通过 `/commercial-legal:risk-self-check` 单独触发。

---

## 目的

在合同草案定稿前，进行系统性的最终质量把关。风险自查采用五维度可勾选检查清单，
确保每一个潜在问题在交给对方或签署前均被识别和处理。

---

## 详细步骤

### 步骤 1：结构完整性检查（维度一）

检查标准条款是否齐全（对照 `framework-design` 的框架清单）：

| 检查项 | 通过标准 | 状态 |
|-------|---------|-----|
| 合同标的条款 | 交付物/服务范围可量化，所有权转移时点明确 | ☐ |
| 价格与付款条款 | 计价方式、付款条件、发票要求均已明确 | ☐ |
| 交付与验收条款 | 验收标准量化，验收期限明确，逾期验收处理约定 | ☐ |
| 违约责任条款 | 违约情形枚举，违约金标准明确，责任限制已约定 | ☐ |
| 合同解除条款 | 解除权触发条件，解除通知程序，解除后果已约定 | ☐ |
| 不可抗力条款 | 不可抗力定义，通知义务，责任免除条件已约定 | ☐ |
| 争议解决条款 | 诉讼/仲裁选一，管辖/机构具体，适用法律明确 | ☐ |
| 一般条款 | 完整合同声明，修改程序，可分割条款，通知地址 | ☐ |

**对应框架中触发的特别条款**：

| 特别条款 | 是否已纳入草案 | 核心要素完整性 |
|---------|-------------|-------------|
| 数据处理条款 | ☐ 已纳入 / ☐ 不适用 | ☐ 委托处理定性/最小必要/安全措施/泄露通知 |
| 知识产权归属条款 | ☐ 已纳入 / ☐ 不适用 | ☐ 既有IP/新开发IP/改进归属 |
| 保密条款 | ☐ 已纳入 / ☐ 不适用 | ☐ 保密范围/期限/例外/违规后果 |
| 竞业限制条款 | ☐ 已纳入 / ☐ 不适用 | ☐ 范围/期限/违约责任 |
| 担保条款 | ☐ 已纳入 / ☐ 不适用 | ☐ 担保类型/主体资格核查 |

### 步骤 2：立场体现检查（维度二）

对照实践配置文件中我方立场（甲方/乙方），验证关键条款是否充分保护我方利益：

| 配置文件字段 | 合同草案对应条款 | 偏离状态 |
|-----------|-------------|--------|
| 责任限制标准立场（直接赔偿上限） | [草案中的责任限制条款] | ☐ 符合 / ☐ 偏离（说明） |
| 间接损失处理（排除/上限/不限）  | [草案中对间接损失的约定] | ☐ 符合 / ☐ 偏离（说明） |
| 基数定义 | [草案中的赔偿基数定义] | ☐ 已明确 / ☐ 含糊（标记） |
| 违约金标准立场 | [草案中的违约金条款] | ☐ 符合 / ☐ 偏离（说明） |
| 绝不接受条款 | [检查草案中是否有「绝不接受」的约定] | ☐ 无 / ☐ 发现（说明） |
| 交易破坏者 | [检查草案是否排除了「交易破坏者」风险] | ☐ 已排除 / ☐ 待处理（说明） |
| 管辖/争议解决偏好 | [草案争议解决条款] | ☐ 符合 / ☐ 偏离（说明） |

**操作规则**：
- 所有「偏离」和「发现」项须逐一出具风险说明（风险等级 + 建议）
- 「绝不接受」项为🔴阻断项——草案中出现须强制修改，不得提交

### 步骤 3：常见风险预防检查（维度三）

| 风险类别 | 检查内容 | 状态 |
|---------|---------|-----|
| 义务履行时间 | 是否存在「合理时间」「尽快」等不可量化表述？ | ☐ 无 / ☐ 有（列出） |
| 金额定义 | 责任限制基数是否有不明确的定义（见 §2D 基数分析）？ | ☐ 已明确 / ☐ 含糊（标记） |
| 通知程序 | 关键事件（违约、解除、索赔）通知程序是否明确？ | ☐ 已明确 / ☐ 缺失（标记） |
| 付款条件悬浮 | 付款条件是否取决于不确定事件（如「甲方满意时付款」）？ | ☐ 无 / ☐ 有（修改） |
| 验收条件悬浮 | 验收标准是否纯主观（如「甲方认为合格」）？ | ☐ 无 / ☐ 有（修改） |
| 自动续约陷阱 | 是否存在自动续约条款，且通知期不合理？ | ☐ 无 / ☐ 有（评估） |
| 税务安排 | 价格条款是否明确含税/不含税？增值税税率是否约定？ | ☐ 已明确 / ☐ 未明确（补充） |
| 减损义务 | 是否约定或提示守约方的减损义务（《民法典》第591条）？ | ☐ 已约定 / ☐ 未约定（提示） |
| 情势变更 | 合同期限 2 年以上是否考虑情势变更条款（《民法典》第533条）？ | ☐ 已评估 / ☐ 待评估 |

### 步骤 4：交易破坏者检查（维度四）

快速扫描合同草案，检查是否存在以下任何「交易破坏者」风险：

| 交易破坏者 | 检查内容 | 状态 |
|----------|---------|-----|
| 实践配置文件「交易破坏者」自定义项 | [从配置文件读取，逐项检查] | ☐ 未发现 / ☐ 发现 🔴 |
| 无限责任约定 | 是否存在不限额的赔偿承诺？ | ☐ 无 / ☐ 有 🔴 |
| 知识产权不保护 | 核心知识产权是否因条款设计流失？ | ☐ 安全 / ☐ 风险 🔴 |
| 管辖境外化 | 是否约定了境外法律管辖（违背实践配置偏好）？ | ☐ 无 / ☐ 有（标记） |
| 无限竞业或不合理竞业 | 竞业条款范围是否过宽导致效力瑕疵？ | ☐ 无 / ☐ 有（评估） |
| 数据安全责任被排除 | 数据泄露责任是否被错误纳入一般责任限制？ | ☐ 安全 / ☐ 有（修改） |

**🔴 交易破坏者规则**：发现任何🔴项，须立即标记并阻断流程，不得提交草案，直至问题解决或由律师明确授权。

### 步骤 5：效力瑕疵检查（维度五）

| 效力风险 | 检查内容 | 法律依据 | 状态 |
|---------|---------|---------|-----|
| 免责条款违法 | 是否有排除人身损害或重大过失财产损失的免责条款？ | 《民法典》第506条 `[待验证]` | ☐ 无 / ☐ 有 🔴 |
| 格式条款无效 | 对方模板中是否有不合理加重我方责任的格式条款？ | 《民法典》第497条 `[待验证]` | ☐ 无 / ☐ 有（标记） |
| 担保条款主体不适格 | 担保人是否具备合法担保资格？ | 《民法典》第683条 `[待验证]` | ☐ 已核查 / ☐ 待核查 |
| 违反效力性强制性规定 | 合同条款是否违反法律的效力性强制性规定？ | 《民法典》第153条 `[待验证]` | ☐ 无 / ☐ 待核查 |
| 违约金与定金并用 | 是否同时约定违约金和定金且未写明选择适用规则？ | 《民法典》第588条 `[待验证]` | ☐ 无 / ☐ 有（修改） |
| 仲裁/诉讼并存 | 是否同时约定仲裁和诉讼（两种救济途径不可并存）？ | — | ☐ 无 / ☐ 有（修改） |

## 检查清单汇总

**风险等级定义**：

| 等级 | 说明 | 处理要求 |
|-----|------|---------|
| 🔴 严重 | 可能导致合同无效、重大利益损失、阻断交易 | 必须修改，阻断提交 |
| 🟠 重要 | 存在显著法律或商业风险，应修改 | 强烈建议修改，律师确认 |
| 🟡 一般 | 存在潜在风险，可以接受但须知情 | 提示风险，律师决策 |
| 🟢 低 | 轻微改进建议 | 可选修改 |

**输出前须满足**：
- [ ] 维度一：结构完整性 — 所有 🔴 必须条款已勾选
- [ ] 维度二：立场体现 — 无「绝不接受」项出现
- [ ] 维度三：常见风险 — 所有模糊表述已替换或已知情标记
- [ ] 维度四：交易破坏者 — 无未解决的 🔴 项
- [ ] 维度五：效力瑕疵 — 无效条款已删除或修改

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 仅提供部分草案（如仅某几章） | 说明：「我仅能自查提供的部分，以下章节未提供，无法自查：[列出]。建议提供完整草案」 |
| 实践配置文件未配置（临时模式） | 在风险报告顶部标记 `[临时模式]`，立场检查使用通用默认值，不偏向任何一方 |
| 发现多处 🔴 项 | 优先处理效力瑕疵（无效条款）和「绝不接受」项，再处理其他 🔴 项 |
| 用户确认接受某 🔴 风险 | 在报告中标记：「律师已知情并接受此风险 — [日期]」，继续流程 |

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 合同草案未提供 | 询问：「请提供合同草案文本（可粘贴）以便进行风险自查」 |
| 草案语言为英文 | 告知：「本插件针对中国大陆法律，中文版本分析更准确。若为跨境合同，部分风险分析可能不适用，建议确认适用法律」 |
| 配置文件「绝不接受」字段为空/占位符 | 使用通用「绝不接受」清单（无限责任、排除人身损害、境外管辖）并标记 `[临时模式]` |
