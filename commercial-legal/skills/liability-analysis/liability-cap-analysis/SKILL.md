# 责任限制分析（liability-cap-analysis）

> **前置**：
> 1. 读取 `commercial-legal/CLAUDE.md`（实践配置文件），加载以下字段：
>    - `## 审查立场` → 对应立场（甲方/乙方）→ `#### 责任限制`（直接赔偿上限、间接损失处理、基数定义、接受的例外清单、可接受备选、绝不接受）
> 2. 读取 `commercial-legal/skills/_shared/civil-code-contract-citations.md`（民法典合同编引用库）
> 3. 读取 `commercial-legal/skills/_shared/legal-basis-conventions.md`（引用规范）
> 4. 分工说明：`contract-review/risk-identification` 负责识别责任限制条款的偏离风险（识别层）；
>    本子技能负责深度分析条款的实际含义与法律影响（分析层），两者互补不重叠。
> 本子技能可被 `liability-analysis/SKILL.md` 编排入口调用，也可通过 `/commercial-legal:liability-cap-analysis` 单独触发。

---

## 目的

对合同中的责任限制条款进行四维深度分析，使律师准确理解：
1. 哪些损失被限制、哪些不被限制
2. 责任上限的实际金额（基数定义是否清晰）
3. 例外清单是否合理保护我方关键利益
4. 整体方案与我方立场的符合程度及偏离风险

---

## 详细步骤

### 步骤 1：定位责任限制条款

在合同中定位所有与责任限制相关的条款：

| 条款类型 | 常见标题/位置 |
|---------|------------|
| 总体责任上限（cap） | 「责任限制」章节、「赔偿」章节 |
| 间接/后果性损失排除 | 通常与总体上限条款同条或紧邻 |
| 例外清单（carveouts） | 通常在责任限制条款末尾「但以下情形除外」 |
| 互相赔偿（indemnification） | 单独章节，可能优先于一般责任限制 |
| 保险要求 | 可能隐含责任分配安排 |

**完整定位清单**：
- [ ] 总体责任上限条款（cap） — 已定位 / 合同未约定（见边界条件）
- [ ] 间接损失排除条款 — 已定位 / 未约定
- [ ] 例外清单 — 已定位 / 未约定
- [ ] 互相赔偿条款 — 已定位 / 未约定
- [ ] 保险条款（隐含责任分配） — 已定位 / 未约定

### 步骤 2：维度一 — 直接损失 vs 间接损失

#### 2.1 分类分析

| 问题 | 分析 |
|------|------|
| 责任上限（cap）是否同时适用于直接损失和间接损失？ | [分析结果] |
| 间接/后果性损失是否被单独排除？ | [分析结果] |
| 合同是否定义了「直接损失」和「间接损失」？ | [分析结果] |
| 间接损失的定义是否与法律通常理解一致？ | [分析结果] |

#### 2.2 法律分析

**直接损失**（法律层面）：通常指违约直接造成的损失，包括：
- 已支付的款项（如预付款）
- 额外支出（如因违约产生的补救成本）
- 交付价值与约定价值的差额

**间接/后果性损失**（法律层面）：通常包括：
- 利润损失（利润损失是否为直接损失存在争议）`[待验证]`
- 机会损失
- 商誉损失
- 对第三方的赔偿

> ⚠️ **注意**：「间接损失」在中国合同法律中的定义与英美法中的「consequential damages」
> 并非完全对应。合同中的间接损失排除条款，需结合《民法典》第584条可预见性规则理解：
> 即使排除了「间接损失」，超出可预见范围的损失本就不在赔偿范围内。`[待验证]`

#### 2.3 责任结构图

```
损失类型
├── 直接损失  → [受 cap 限制 / 不受限制 / 按合同约定]
└── 间接损失  → [被完全排除 / 受 cap 限制 / 按合同约定]
    ├── 利润损失
    ├── 机会损失
    ├── 商誉损失
    └── 第三方索赔
```

### 步骤 3：维度二 — 基数定义

#### 3.1 逐字分析

逐字引用合同原文（责任上限基数部分），进行歧义分析：

**模板**：
> 合同原文：「[逐字引用合同语言]」

| 歧义维度 | 可能含义 A | 可能含义 B | 争议风险 |
|---------|---------|---------|---------|
| 「12个月费用」的计算基准 | 索赔前12个月（向前看） | 合同签署后首12个月（固定基准） | 争议时对我方影响 |
| 「已支付」还是「应支付」 | 实际到账金额 | 账单金额（含未付款） | [分析] |
| 「费用」的口径 | 含税总价 | 不含税净价 | [分析] |
| [其他歧义维度] | [含义A] | [含义B] | [分析] |

#### 3.2 基数不明确时的错误处理

当基数定义含糊或存在多种解读时，**必须**按以下格式标注并要求签署前确认：

> ⚠️ **基数定义不明确**
>
> 合同责任上限约定为：「[引用的确切合同语言]」
>
> 此表述存在歧义，可能含义包括：
> - 含义 A：[解读A] — 对我方影响：[分析]
> - 含义 B：[解读B] — 对我方影响：[分析]
>
> **风险等级：🟠 重要**
>
> **处理方式**：
> 1. 签署前与对方确认，明确约定计算方法（如「指索赔事件发生前12个自然月内，甲方已向乙方实际支付的款项之总额」）
> 2. 或通过补充协议明确
> 3. 保守策略：按对我方**最不利**的解读评估最坏风险

## 边界条件

| 场景 | 处理方式 |
|------|---------|
| 合同没有责任限制条款（无 cap） | 说明：「合同未约定责任限制，依据《民法典》第584条，赔偿范围为可预见损失，无金额上限。风险等级：🟠（需评估潜在无限赔偿风险）」 |
| 责任限制基数为 0 或象征性金额（如 ¥1） | 标记为🔴：「责任上限为象征性金额，实际等同于完全免责——依据《民法典》第506条，若因此导致对方无任何有效救济途径，可能被认定违背诚实信用原则」`[待验证]` |
| 例外清单异常宽泛（几乎所有责任均为例外） | 标记为🟠：「例外清单过宽将导致责任上限形同虚设——分析哪些例外对我方意义重大、哪些可谈判缩减」 |
| 互相赔偿条款与一般责任限制交互复杂 | 分别分析，说明互相赔偿是否受一般 cap 约束（通常互相赔偿是 carveout） |
| 实践配置文件未配置（临时模式） | 使用通用默认值（平衡型），在报告中标注 `[临时模式]` |

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 责任限制条款语言复杂，包含多层嵌套引用 | 先拆解结构，绘制「责任结构图」，再逐层分析 |
| 英文合同（外资方提供） | 翻译关键语言后分析，标注：「以下分析基于本律师对英文条款的理解，建议律师确认翻译准确性」 |
| 配置文件责任限制立场为占位符 | 标注 `[临时模式]`，使用以下通用标准：直接赔偿上限 = 合同总价；排除间接损失；接受故意/重大过失/IP/保密/人身损害例外 |
| 无法确认基数金额（未获取合同价格） | 提示：「请提供合同总价或过去12个月付款记录，以便量化最坏情形」 |
