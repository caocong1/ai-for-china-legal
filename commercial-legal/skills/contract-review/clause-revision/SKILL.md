# 条款修改建议子技能

> **迁移说明**：本子技能包含原 contract-review/SKILL.md 中「逐项对比」的建议修改块，以及完整的「修改粒度」段落（词<短语<句子<条款），和「如果对方不让步」的备选处理。
> 可被编排入口调用，也可单独触发：`/commercial-legal:clause-revision [合同]`

---

## 目的

基于风险识别结果（risk-identification 子技能输出），为每个偏离条款生成：
1. **具体替换语言** — 可直接粘贴到修订版合同，而非模糊的「建议修改」
2. **最小粒度修改** — 以尽可能小的修改实现审查立场，保留对方可接受的框架
3. **备选方案** — 对方不让步时的退而求其次选项（来自实践配置文件）

## 详细步骤

### 第一步：确认修改范围

若已有 risk-identification 输出，直接使用其风险条款列表作为修改范围。若单独触发，先运行风险识别（读取步骤见 risk-identification/SKILL.md）。

确认：
- 修改范围是「全部偏离条款」还是律师指定的特定条款
- 对方当事人类型（大公司 vs 中小企业，影响修改策略和预期）
- 当前谈判阶段（首轮 vs 最后一轮，影响粒度选择）

### 第二步：应用修改粒度原则

**以尽可能小的粒度修改。** 修订是谈判工具，不是重写。Wholesale 替换发出「我们扔掉了你的草稿」的信号；精细修订发出「我们有具体要求」，更容易被接受。

**修改粒度层级（从小到大）：**

| 粒度层级 | 适用场景 | 谈判信号 |
|---------|---------|---------|
| **词** | 单个词语不准确或有歧义 | 最精细，对方感知到被认真阅读 |
| **短语** | 需要补充修饰或替换表述 | 精确，显示具体诉求 |
| **子条款** | 需要重组逻辑或调整结构 | 中等，仍保留对方框架 |
| **句子** | 整句意思与立场冲突 | 较大改动，需说明理由 |
| **整条款** | 对方草稿与立场相差太远 | 最大改动，须主动说明：「我们替换了第X.X条，因为改动范围很大，很高兴带你过一遍差异」 |

**默认选择规则**：
1. 替换一个**词**优先于短语
2. 替换一个**短语**优先于句子
3. 重组一个**子条款**优先于替换句子
4. 替换一个**句子**优先于替换整条款
5. 仅当对方版本与立场相差太远，精细修改比新草稿更难阅读时，才替换整条款

**拿不准时，选更小的粒度。**

### 第三步：生成逐条修改建议

对每个需要修改的条款，生成完整修改建议块：

```markdown
### 修改建议：[第 X.X 条] [条款名称]

**风险等级：** 🔴/🟠/🟡/🟢 [来自风险识别]
**修改粒度：** 词 / 短语 / 子条款 / 句子 / 整条款

**原文：**
> "[合同中原文完整引用]"

**问题：** [一句话说明偏离类型和影响]

**建议修改（[粒度]级）：**
> "[具体替换语言 — 可直接粘贴，用删除线和加粗示意修改位置]"

**修改说明：** [简短说明为何这样修改，引用法条或审查立场]

**如果对方不让步：**
> [来自实践配置文件的备选方案；若无备选，说明「升级到 [人员]」或「可考虑接受，需标注风险」]

**法律依据：** [关键法条引用，来源标签 + 待验证标注，详见 legal-basis 子技能]
```

### 第四步：处理特殊条款类型

**格式条款修改**（对方提供标准合同时）：
- 按民法典第498条，格式条款有争议时作不利于提供方的解释，修改建议须明确
- 提示对方：本条按第498条解释对提供方不利，建议明确表述以避免争议
- 来源：`[模型知识 — 需验证]` `[待验证]`

**违约金/责任限制修改**：
- 按四维分析（来自 risk-identification）针对性修改
- 若需分别修改直接损失限制和间接损失排除，生成两个独立的修改建议
- 明确基数定义的替换语言

**免责条款修改**（涉及民法典第506条）：
- 若条款属于无效免责（人身损害或重大过失），说明「按民法典第506条，本条款无效，无需谈判」
- 若仍需修改以明确约定（如限制至法定最低标准），生成合规替换语言

### 第五步：汇总修改包

在逐条修改建议之后，汇总「修改包」供律师统一传递：

```markdown
## 修改包汇总

以下为本次审查建议的所有修改，已按条款顺序排列，可直接传递给对方。

| 条款 | 风险等级 | 修改粒度 | 摘要 |
|------|---------|---------|------|
| 第X.X条 | 🔴 | 句子级 | [一句话摘要] |
| 第X.X条 | 🟠 | 短语级 | [一句话摘要] |
| ...  | ... | ... | ... |

**传递说明**（根据实践配置文件 ## 文书风格 中的修订语气）：[建议的传递语气/方式]
```

## 检查清单

- [ ] 每个修改建议都有具体替换语言（不是模糊的「建议修改」）
- [ ] 修改粒度已选择最小可行粒度（不过度替换）
- [ ] 整条款替换时已附说明（「替换原因 + 愿意带对方过差异」）
- [ ] 每个修改建议都有「如果对方不让步」的处理（来自配置文件或说明需升级）
- [ ] 格式条款修改遵循第498条疑义解释规则
- [ ] 违约金/责任限制修改覆盖四维分析中识别的具体问题
- [ ] 第506条无效免责条款已标注「法定无效，无需谈判」
- [ ] 法律依据按来源分类标注（来源标签 + 待验证）
- [ ] 修改包汇总表列数一致（四列：条款/风险等级/修改粒度/摘要）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 仅指定审查特定条款 | 仅对指定条款生成修改建议，其他条款说明「未在本次范围内」 |
| 对方为大公司（标准合同，不会改） | 修改建议中增加「对方不让步时」的博弈策略（见 negotiation-points） |
| 交易破坏者条款 | 不生成「可接受备选」，仅生成坚持修改的语言 + 不签的说明 |
| 条款在本法律下法定无效（第506条） | 标注无效，说明「无需谈判，法定无效」，建议删除或替换为合规表述 |
| 合同语言非中文 | 修改建议提供中文，注明「建议对照原文语言同步修改，本建议为参考」 |
| 首轮谈判 vs 最后一轮 | 首轮优先推动所有修改；最后一轮优先保留 🔴🟠，放弃 🟡🟢 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 未先运行风险识别 | 「需要先完成风险识别才能生成修改建议。运行 risk-identification 子技能，或在此提供风险清单」 |
| 配置文件无备选方案 | 「配置文件中无该类条款的备选方案。建议：(1) 坚持标准立场；(2) 升级律师决定；(3) 更新配置文件记录本次决策」 |
| 修改语言与对方法律风格差异大 | 标注「本修改建议采用 [风格]，对方合同采用 [风格]，传递时建议适应对方格式」 |
| 法律依据查询无结果 | 遵循「无静默补充原则」，报告并询问下一步 |
