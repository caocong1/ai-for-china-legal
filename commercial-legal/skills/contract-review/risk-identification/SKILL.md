# 风险识别子技能

> **迁移说明**：本子技能包含原 contract-review/SKILL.md 中的「主体资格预检」「交易破坏者检查」「逐项对比」风险识别部分，以及「违约金/责任限制四维识别」和「管辖法律差异检查」中的识别部分。
> 可被编排入口调用，也可单独触发：`/commercial-legal:risk-identification [合同]`

---

## 目的

对照团队实践配置文件，从以下四个维度对合同逐条识别风险，输出按严重性分级的风险清单：

1. **主体资格风险** — 合同主体是否具备相应资格（担保权限、代表权限、外汇登记等）
2. **条款偏离风险** — 合同条款偏离审查立场的程度（交易破坏者/高风险/中风险/低风险）
3. **法律强制性风险** — 是否违反民法典强制性规定（如第506条免责无效、第497条格式条款无效）
4. **可执行性风险** — 在合同适用的管辖法律下，特定条款是否可执行

## 详细步骤

### 第一步：定位合同基本信息

快速扫描合同，建立定位表：

| 问题 | 回答 |
|------|------|
| 合同类型 | 买卖合同 / 技术服务合同 / 租赁合同 / 合作协议 / 其他 |
| 我方立场 | 甲方（买方/委托方）/ 乙方（卖方/受托方） |
| 对方当事人 | 名称 + 类型（大公司/中小企业/个人） |
| 合同金额 | 总价/年费（若未写明，见「合同金额处理」） |
| 合同期限 | 期限长度 + 续约机制 |
| 涉及数据处理 | 是（个人信息/重要数据）/ 否 |
| 涉及担保 | 保证/抵押/质押/无 |
| 适用法律/管辖 | 合同约定的管辖法律和争议解决方式 |

**合同金额处理**：若合同未写明金额（框架协议），**停下询问**：
> 本协议未写明合同金额，价格由订单/结算单确定。在路由升级前，需要预估金额。
> 选项：(1) 提供订单金额；(2) 告知是否超过升级阈值；(3) 保守路由到更高审批级别。
> 不得默默假设金额用于路由。

### 第二步：主体资格预检

**对照 `shared/references/subject-qualification-traps.md`**，若合同存在以下情形，**优先核查主体资格**：

| 情形 | 核查内容 | 依据 |
|------|---------|------|
| 担保条款/保证条款 | 担保人是否有权出具（融资担保公司、银行分支机构、上市公司等） | subject-qualification-traps.md |
| 上市公司为一方 | 对外担保是否有董事会/股东会决议 | 民法典第504条 `[待验证]` |
| 国企/事业单位为一方 | 是否有国资委审批或内部决策程序 | 国资监管规则 `[待验证]` |
| 银行分支机构为担保方 | 是否有总行授权文件 | 民法典第504/506条 `[待验证]` |
| 外商投资企业对外担保 | 是否有外汇局备案/登记 | 外汇管理条例 `[待验证]` |

**主体资格问题处理**：若发现问题，在风险清单顶部显著标记，并评估对合同效力的影响。

### 第三步：交易破坏者检查

优先扫描实践配置文件中「交易破坏者」条款（绝不可接受）。若存在，立即标记：

```markdown
## 存在交易破坏者

**第 [X.X] 条** 包含 [交易破坏者名称]。根据团队审查立场，这是硬性的「不」。

- 建议修改：[具体替代表述]
- 对方不让步：放弃，不签

详细风险识别见下文，但此问题不解决，其他识别结果参考意义有限。
```

### 第四步：逐条风险识别

对照实践配置文件，逐条扫描合同，对每个偏离生成识别块：

```markdown
### [第 X.X 条]：[问题名称]

**我方立场（引自实践配置文件）：** [具体立场]

**合同写的是：**
> "[原文引用]"

**偏离类型：** 缺失条款 | 弱于标准 | 弱于备选 | 非典型结构 | 不可接受

**法律风险：** 🔴 关键 | 🟠 高 | 🟡 中等 | 🟢 低
**商业摩擦：** 🔴 阻碍交易 | 🟠 拖慢交易 | 🟡 造成困惑 | 🟢 无感

**为什么重要：** [一到两句白话 — 若保持原样，业务会出什么问题]

**法律依据标签：** [相关法条来源标签，见法律依据子技能输出]
```

**风险等级校准：**

| 等级 | 法律风险含义 | 商业摩擦含义 |
|------|------------|------------|
| 🔴 关键 | 交易破坏者或绝不接受清单条款 | 阻碍交易进行 |
| 🟠 高 | 超出备选范围，强烈推动 | 拖慢谈判/交易 |
| 🟡 中等 | 在备选范围内但未达标准 | 造成日后执行困惑 |
| 🟢 低 | 明确容忍，或纯形式偏离 | 基本无感 |

### 第五步：违约金/责任限制四维识别

**针对责任限制/违约金条款，必须完成以下四个维度**（不得仅写「检查责任限制」）：

**维度 1 — 直接损失 vs 间接损失：**
- 限制是否适用于所有责任，还是仅直接损失？
- 明确陈述两种处理的不同立场

**维度 2 — 基数定义（逐字引用）：**
- 引用合同中「限制金额」的确切表述
- 判断属于哪种基数：(a) 索赔前12个月已付费用；(b) 当前12个月应付费用；(c) 合同总价；(d) 其他
- 若不明确，标记：「基数定义不明确 — [原文] — 可能意味着 [X] 或 [Y]，签署前确认」

**维度 3 — 例外交互：**
- 列举哪些在限制之外（例外），哪些在限制之内
- 评估被限制表面是否有意义：「限制覆盖 [一般违约]，数据泄露/IP/保密例外」

**维度 4 — 与审查立场的对应：**
- 对照配置文件中每个维度的立场（直接限制倍数、间接损失处理、例外清单、基数偏好）

### 第六步：管辖法律差异核查

**在按字面接受审查立场前，核查以下管辖差异：**

| 事项 | 民法典规定 | 核查要点 | 验证状态 |
|------|----------|---------|---------|
| 竞业限制 | 劳动合同法第23-24条（期限≤2年，需支付补偿） | 各地司法实践有差异 | `[管辖法律 — 需验证]` |
| 自动续约格式条款 | 民法典第496条对格式条款有特别规定 | 消费者合同有额外保护 | `[管辖法律 — 需验证]` |
| 责任排除 | 民法典第506条（人身损害/重大过失免责条款无效） | 强制性规定，不可约定排除 | `[管辖法律 — 需验证]` |
| 违约金调整 | 民法典第585条（可请求法院调整过高/过低违约金） | 各地法院裁量幅度不同 | `[管辖法律 — 需验证]` |
| 格式条款 | 民法典第496-498条（提示义务、无效情形、疑义解释） | `[模型知识 — 需验证]` | `[待验证]` |

若审查立场与合同适用法律的可执行性冲突，标记：
> 「你的审查立场倾向于 [X]，但合同受 [Y] 法律管辖，在那里 [X] 是 [不可执行/受限制]。`[管辖法律 — 需验证]`」

## 检查清单

在完成风险识别后，逐项确认：

- [ ] 定位表已完整填写（合同类型、我方立场、金额、期限等）
- [ ] 若有担保情形，主体资格已预检（对照 subject-qualification-traps.md）
- [ ] 交易破坏者优先检查并标注（若存在）
- [ ] 每个风险条款已生成「偏离类型 + 法律风险等级 + 商业摩擦等级」
- [ ] 风险等级已校准（不全是关键，对照配置文件）
- [ ] 责任限制条款已完成四维识别（直接/间接、基数、例外、立场对应）
- [ ] 管辖法律差异已核查（格式条款、违约金、免责条款）
- [ ] 所有法律依据已按来源分类标注（见 legal-basis 子技能）
- [ ] 合同金额若未写明，已停下询问，未默默假设

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 合同金额未写明 | 停下询问，不默默假设，不用于路由（见第一步） |
| 无实践配置文件 | 走临时模式（通用默认值，标 `[临时模式]`） |
| 合同语言非中文 | 标注「合同为 [语言] 版，风险识别基于中文翻译，建议对照原文复核 `[用户提供]`」 |
| 合同极短（<5条） | 识别可能条款缺失多，重点检查「缺失条款」 |
| 合同为框架协议 | 注明「框架协议，具体权利义务由订单/补充协议确定，本识别结果基于框架条款，订单签署前需复核」 |
| 涉及特殊管辖（经济特区/自贸区） | 标记 `[管辖 flagged — 需验证特别规定]`，提示特别立法权 |
| 担保主体资格存疑 | 在识别清单顶部置红，评估合同效力影响，建议法律意见 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 无法读取配置文件（路径错误） | 提示：「无法读取实践配置文件，说 `临时模式` 继续，或检查配置文件路径」 |
| 合同内容不完整（明显有遗漏） | 标注「合同文本似乎不完整，识别结果可能有遗漏，建议补充后重新运行」 |
| 条款格式特殊难以解析 | 「第X.X条格式特殊，无法自动解析，需人工审查」 |
| 法律依据查询无结果 | 遵循「无静默补充原则」（来自 commercial-legal/CLAUDE.md），报告并询问下一步，不自行填补 |
| 管辖法律不确定 | 标注 `[管辖法律 — 需验证]`，不按单一法律做结论性判断 |
