# 法律依据引用规范（法律技能中心版）

> 本文件定义 legal-builder-hub 插件所有技能统一使用的法律与技术依据引用规范。
> 本插件处于技术与法律的交叉领域，引用范围涵盖：数据安全法律、知识产权法律、信息安全技术标准、平台技术规范。
> 引用规范遵循项目级约定，复用五个标准来源标签。

---

## 一、来源分类

### 1. 法律（全国人大及其常委会制定）

指全国人民代表大会及其常务委员会制定的法律，与本插件相关的核心法律：
- 《中华人民共和国个人信息保护法》（PIPL，2021年11月1日施行）
- 《中华人民共和国数据安全法》（DSL，2021年9月1日施行）
- 《中华人民共和国网络安全法》（CSL，2017年6月1日施行）
- 《中华人民共和国著作权法》（2021年6月1日修正施行）
- 《中华人民共和国民法典》（2021年1月1日施行）

**格式要求（法律最小引用格式）：**

```
《[法律全名]》（[施行/修订年份]）第[X]条[第X款]  [来源标签]  [验证状态]
```

示例：
```
《中华人民共和国个人信息保护法》（2021年）第13条  `[模型知识 — 需验证]`  已核实锚点，建议复核
```

### 2. 行政法规（国务院制定）

与本插件相关的行政法规：
- 《关键信息基础设施安全保护条例》（2021年9月1日施行，国务院）
- 《网络数据安全管理条例》（施行状态须核实 `[待验证]`）
- 《计算机信息网络国际联网安全保护管理办法》（公安部）

**格式要求：**

```
《[行政法规全名]》（[施行年份]，国务院）第[X]条  `[来源标签]`  `[待验证]`
```

### 3. 部门规章与规范性文件

与本插件相关的部门规章：
- 国家互联网信息办公室相关规定（数据出境/算法/生成式 AI）
- 工业和信息化部相关规定（电信和互联网安全/软件产品管理）
- 公安部相关规定（网络安全等级保护/信息安全事件管理）
- 国家市场监督管理总局相关规定（软件产品质量/标准）

**格式要求：**

```
《[规章全名]》（[施行年份]，[制定机关]）[描述规则，不写条号]  `[模型知识 — 需验证]`  `[待验证]`
```

### 4. 国家标准（GB/T 系列）

与本插件相关的信息安全国家标准：
- 个人信息安全规范（GB/T 35273，推荐性标准 `[待验证]`）
- 网络安全等级保护基本要求（等保 2.0，GB/T 22239 `[待验证]`）
- 信息安全技术 软件产品安全测试规范 `[待验证]`
- 信息安全技术 移动互联网应用程序（App）收集个人信息基本规范 `[待验证]`

**国家标准核心纪律：**

> **国家标准编号（GB/T XXXXX-YYYY）、发布年份、推荐性/强制性区分，一律标 `[待验证]`。**

**格式要求：**

```
[标准名称] GB/T XXXXX-YYYY `[待验证]`（推荐性/强制性 `[待验证]`）  `[模型知识 — 需验证]`
```

### 5. 平台技术规范

本插件特有的平台技术规范引用：
- 技能清单规范（SKILL.md 格式规范）
- 插件元数据规范（plugin.json 格式规范）
- 安全审查标准（权限范围/数据处理/代码安全）
- 版本发布规范（语义化版本 SemVer）
- 市场清单规范（marketplace.json）

**格式要求：**

```
[平台规范名称] [版本号]  `[平台规范]`  [验证状态]
```

示例：
```
技能清单规范（SKILL.md 格式）v1.0  `[平台规范]`  当前适用版本
```

### 6. 案例与学说

技能安全与合规领域的案例和学说较少，引用时一律标 `[待验证]`。

## 二、来源标签表

| 标签 | 含义 | 使用场景 |
|------|------|----------|
| `[平台规范]` | 来自平台官方技术规范 | 技能清单/插件元数据/安全审查标准 |
| `[安全标准]` | 来自行业安全标准（OWASP/GB/T） | 安全扫描规则/权限评估标准 |
| `[网络搜索 — 需验证]` | 来自网络搜索 | 需对照原始来源核查 |
| `[模型知识 — 需验证]` | 来自训练数据回忆 | 风险较高，须优先核查 |
| `[用户提供]` | 来自用户提供的文件 | 技能代码/文档/安全声明 |

**标签使用规则：**
1. 每条引用行**必须且只能**附加一个来源标签
2. 禁止剥离或折叠标签
3. `[模型知识 — 需验证]` 来源在依赖前**必须**对照原始来源检查

---

## 三、待验证标记规则

| 触发情形 | 说明 | 建议验证途径 |
|---------|------|-------------|
| 来源标签为 `[模型知识 — 需验证]` | 模型回忆的条文号风险高 | 国家法律法规数据库 |
| 部门规章具体条号 | 条号不确定 | 制定机关官网 |
| 国家标准编号与版本 | GB/T 编号、年份须核实 | 全国标准信息公共服务平台 |
| 平台技术规范版本 | 规范可能更新 | 平台官方文档 |
| 用户口述的条款号或规则 | 可能记忆偏差 | 核查官方全文 |

---

## 四、成文法优先原则

1. **分析基础**：所有安全与合规结论须首先援引现行成文法
2. **不得以案例替代法条**：即使存在行政处罚决定，也须同时引用对应法条
3. **法条缺失时**：须明确说明「无直接成文法规定，参照以下标准/规范」

---

## 五、技术-法律交叉领域的特殊引用规则

### 安全审查中的法律引用

技能安全审查须同时引用：
- **技术依据**：安全扫描标准（OWASP Top 10、GB/T 信息安全标准）
- **法律依据**：PIPL 个人信息保护要求、DSL 数据安全要求

### 知识产权引用

技能内容涉及知识产权时：
- 法律条文引用《著作权法》相关条款
- 许可证引用须注明完整许可证文本来源
- 开源组件引用须注明版本和许可证兼容性
