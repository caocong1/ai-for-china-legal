# 兼容性检查（compatibility-check）

> **所属编排**：`legal-builder-hub/skills/installation/SKILL.md`（安装向导）
> **分工说明**：本子技能负责安装前的环境和依赖检查。安装执行由 `installation-execution` 处理；安装后配置由 `post-install-configuration` 处理；更新卸载由 `update-uninstall` 处理。
> **可单独触发**：`/legal-builder-hub:compatibility-check`

---

## 目的

在安装技能之前，全面检查目标技能对当前环境的兼容性要求，包括平台版本、依赖项版本、系统资源、安全审查状态，输出兼容性报告并给出是否可以安装的结论。

---

## 前置

1. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：安装前兼容性检查（强制/推荐/关闭）、是否允许安装未审查技能、安全审查偏好
2. **获取目标技能信息** — 技能名称、版本、SKILL.md 元数据、plugin.json
3. **检测当前环境** — 平台版本、已安装技能列表、系统资源
4. 本子技能产出是兼容性报告，**不执行安装操作**

---

## 检查项目

### 1. 平台版本检查

| 检查项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 平台最低版本 | 技能要求的最低平台版本 | 当前版本 ≥ 要求版本 → ✅ |
| 平台最高版本 | 技能声明的兼容最高版本 | 当前版本 ≤ 最高版本 → ✅ |
| API 兼容性 | 技能使用的 API 在当前版本是否可用 | 所有 API 均可用 → ✅ |

### 2. 依赖项检查

| 检查项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 依赖技能 | 技能声明依赖的其他技能是否已安装 | 所有依赖已安装且版本满足 → ✅ |
| 版本冲突 | 依赖项的版本要求是否与其他技能冲突 | 无冲突 → ✅ |
| 循环依赖 | 是否存在依赖循环 | 无循环 → ✅ |

### 3. 系统资源检查

| 检查项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 磁盘空间 | 安装目标位置是否有足够空间 | 剩余空间 > 技能大小 × 2 → ✅ |
| 内存要求 | 系统内存是否满足技能运行需求 | 可用内存 ≥ 要求内存 → ✅ |

### 4. 安全审查状态检查

| 检查项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 安全审查 | 技能是否已通过安全审查 | 已审查通过 → ✅ |
| 审查时效 | 安全审查报告是否在有效期内 | 审查后技能代码未变更 → ✅ |
| 审查级别 | 审查级别是否满足用户配置要求 | 审查级别 ≥ 用户配置级别 → ✅ |

### 5. 许可证兼容性检查

| 检查项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 许可证类型 | 技能许可证是否与用户接受范围兼容 | 用户接受该许可证 → ✅ |
| 依赖许可证 | 依赖项的许可证是否与主技能兼容 | 无许可证冲突 → ✅ |

```markdown

**目标技能**：[技能名称] v[版本号]
**检查日期**：[日期]
**检查环境**：[平台版本/系统信息摘要]

### 检查结果

| 检查类别 | 检查项 | 结果 | 说明 |
|---------|-------|------|------|
| 平台版本 | 最低版本 [X.X] | ✅ / ❌ | [说明] |
| 平台版本 | API 兼容性 | ✅ / ❌ | [说明] |
| 依赖项 | [依赖名称] [版本要求] | ✅ / ❌ | [已安装版本/未安装] |
| 依赖项 | 版本冲突检查 | ✅ / ❌ | [冲突详情] |
| 系统资源 | 磁盘空间 | ✅ / ❌ | [需要/可用] |
| 安全审查 | 审查状态 | ✅ / ❌ | [状态/日期] |
| 许可证 | 许可证类型 [类型] | ✅ / ❌ | [兼容性] |

### 综合结论

**结论**：[✅ 兼容 — 可以安装 / ⚠️ 部分兼容 — 需处理后安装 / ❌ 不兼容 — 无法安装]

**阻塞项**（须解决后方可安装）：
- [阻塞项清单]

**警告项**（不阻止安装但建议处理）：
- [警告项清单]

### 建议操作

**兼容时**：
1. 继续安装 → `/legal-builder-hub:installation-execution`

**部分兼容时**：
1. 处理阻塞项后重新检查
2. 确认接受警告项后继续安装

**不兼容时**：
1. 升级平台版本
2. 安装缺失依赖（→ 依赖解析 `/legal-builder-hub:dependency-resolution`）
3. 选择兼容的替代技能（→ `/legal-builder-hub:skill-search-browse`）
```

## 检查模式

| 用户配置 | 检查行为 |
|---------|---------|
| 强制 | 任何 ❌ 项均阻止安装；必须解决所有阻塞项 |
| 推荐 | ❌ 项发出警告，用户确认后可继续；阻塞项仍建议解决 |
| 关闭 | 跳过兼容性检查（不推荐，标注 `[风险 — 未进行兼容性检查]`） |

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 目标技能无元数据 | 无法执行兼容性检查；建议开发者补充 SKILL.md |
| 依赖项不可用 | 标注依赖缺失；推荐替代方案或依赖安装步骤 |
| 安全审查未通过 | 阻止安装；建议先完成安全审查（→ `/legal-builder-hub:security-review`） |
| 环境检测失败 | 提示用户手动提供环境信息；使用保守默认值 |

| 法律依据 | 核心规则 | 适用场景 |
|---------|---------|---------|
