# 安装执行（installation-execution）

> **所属编排**：`legal-builder-hub/skills/installation/SKILL.md`（安装向导）
> **分工说明**：本子技能负责实际的安装操作。兼容性检查由 `compatibility-check` 前置处理；安装后配置由 `post-install-configuration` 处理；更新卸载由 `update-uninstall` 处理。
> **可单独触发**：`/legal-builder-hub:installation-execution`
>
> **硬性规则**：未通过安全审查的技能不得安装。此规则不可绕过。

---

## 目的

在通过兼容性检查后，执行技能的实际安装操作：创建回滚点、下载或复制技能文件、注册到本地环境、验证安装完整性、记录完整安装日志。

---

## 前置

1. **兼容性检查通过** — 须先完成 `compatibility-check` 并确认为 ✅ 兼容或 ⚠️ 部分兼容（用户已确认）
2. **安全审查通过** — 须确认技能已通过安全审查，结论为「通过」或「有条件通过（已修复）」
3. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：默认安装位置、安装日志风格
4. 用户确认安装 — 展示安装摘要，获得用户明确确认

---

## 安装流程

### 步骤一：创建回滚点

```markdown
### 回滚点

**创建时间**：[时间戳]
**回滚点标识**：[唯一标识]
**当前环境快照**：
- 已安装技能清单：[N] 个
- 配置文件状态：[已备份至 [路径]]
- 安装目录状态：[已备份至 [路径]]
```

### 步骤二：获取技能文件

| 安装来源 | 获取方式 |
|---------|---------|
| 远程注册表 | 从注册表地址下载技能包（须验证完整性校验和） |
| 本地路径 | 从本地路径复制技能文件 |
| Git 仓库 | 从 Git 仓库克隆或检出指定版本 |

**完整性验证**：

- 从注册表下载时，须验证 SHA-256 校验和
- 校验和与注册表记录不一致时，**终止安装**并告警

### 步骤三：安装技能文件

```markdown
### 安装目标

**安装位置**：[用户配置的默认位置 / 用户指定位置]
**安装方式**：[复制 / 符号链接 / 注册引用]
```

安装操作：
1. 创建技能目录（如 `[安装位置]/[技能名称]/`）
2. 复制/链接技能文件（SKILL.md、子技能文件、共享文件）
3. 设置文件权限（只读，防止意外修改）

### 步骤四：注册到本地环境

1. 在本地技能注册表中添加技能记录
2. 注册技能名称与路径的映射
3. 注册斜杠命令路由（`/legal-builder-hub:[skill-name]`）
4. 注册依赖关系

### 步骤五：验证安装

| 验证项 | 检查内容 | 判定标准 |
|-------|---------|---------|
| 文件完整性 | 所有必需文件是否存在 | 所有文件存在 → ✅ |
| 元数据可读 | SKILL.md 是否可正确解析 | 解析成功 → ✅ |
| 命令注册 | 斜杠命令是否可用 | 命令路由正确 → ✅ |
| 依赖满足 | 依赖项是否可正确解析 | 所有依赖可用 → ✅ |

### 步骤六：记录安装日志

```markdown
## 安装日志

**技能名称**：[名称]
**版本**：[版本号]
**安装时间**：[时间戳]
**安装来源**：[注册表地址/本地路径]
**安装位置**：[安装路径]
**安全审查**：[审查编号/日期/结论]
**兼容性检查**：[检查日期/结论]
**回滚点**：[回滚点标识]

### 安装步骤记录

| 步骤 | 操作 | 状态 | 时间 |
|------|------|------|------|
| 1 | 创建回滚点 | ✅ | [时间] |
| 2 | 获取技能文件 | ✅ | [时间] |
| 3 | 验证完整性 | ✅ | [时间] |
| 4 | 安装技能文件 | ✅ | [时间] |
| 5 | 注册到本地环境 | ✅ | [时间] |
| 6 | 验证安装 | ✅ | [时间] |

**安装结论**：[✅ 安装成功 / ❌ 安装失败 — 已回滚]
```

---

## 安装失败处理

| 失败阶段 | 处理方式 |
|---------|---------|
| 文件获取失败 | 终止安装；无需回滚（未修改环境） |
| 完整性验证失败 | 终止安装；删除已下载文件；告警 |
| 文件安装失败 | 自动回滚到回滚点；记录失败原因 |
| 注册失败 | 删除已安装文件；回滚注册表变更 |
| 安装验证失败 | 保留文件但标注安装异常；建议排查 |

## 安装确认摘要

安装前须向用户展示确认摘要：

```markdown
## 安装确认

**技能**：[名称] v[版本]
**来源**：[注册表/本地路径]
**安装位置**：[目标路径]
**安全审查**：[状态]
**兼容性**：[状态]
**依赖项**：[新增安装 N 个 / 已满足]
**磁盘空间**：需要 [大小]，可用 [大小]

⚠️ **安装即接受许可证条款**：[许可证类型]

**确认安装？** [是 / 否]
```

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 安装位置已有同名技能 | 提示版本冲突；选择覆盖安装/取消/指定其他位置 |
| 安装过程中断 | 自动回滚；提示用户重试 |
| 权限不足 | 提示用户提升权限或更改安装位置 |
| 安全审查报告过期 | 终止安装；建议重新审查 |

## 法律依据

| 法律依据 | 核心规则 | 适用场景 |
|---------|---------|---------|
| 《个人信息保护法》（2021年）第13条 | 安装数据处理的合法性基础 | 安装日志记录 |
| 《著作权法》（2021年修正） | 安装即接受许可证条款 | 许可证确认 |
| 《网络安全法》（2017年）第21条 | 软件安装须与系统等保相适应 | 安装环境检查 |
