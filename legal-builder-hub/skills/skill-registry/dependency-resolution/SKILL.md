# 依赖解析（dependency-resolution）

> **所属编排**：`legal-builder-hub/skills/skill-registry/SKILL.md`（技能注册表）
> **分工说明**：本子技能负责依赖关系的解析和冲突检测。清单校验由 `skill-manifest-management` 处理；版本管理由 `version-release-management` 处理；注册表操作由 `registry-search-publish` 处理。
> **可单独触发**：`/legal-builder-hub:dependency-resolution`

---

## 目的

解析技能的依赖关系树，检测版本冲突和循环依赖，验证依赖项的兼容性和许可证一致性，生成依赖报告和可视化依赖图。确保技能的依赖关系完整、可解析、无冲突。

---

## 前置

1. **获取技能元数据** — plugin.json 中的依赖声明
2. **连接注册表** — 获取依赖项的版本信息
3. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：注册表类型
4. 本子技能产出是依赖解析报告，**不修改依赖项**

---

## 依赖声明规范

### 依赖类型

| 依赖类型 | 说明 | 示例 |
|---------|------|------|
| 技能依赖 | 依赖其他技能的功能 | 依赖 `contract-review` 技能的合同解析能力 |
| 库依赖 | 依赖第三方库或工具 | 依赖特定 Python 库或 Node.js 包 |
| 平台依赖 | 依赖特定平台版本 | 需要平台 API v2.0+ |
| 数据依赖 | 依赖特定数据源 | 依赖法律法规数据库连接 |

### 版本声明格式

```
[技能/库名称]: [版本范围]
```

| 版本范围语法 | 含义 | 示例 |
|------------|------|------|
| `^1.2.3` | 兼容 1.2.3 及以上（<2.0.0） | `contract-review: ^1.0.0` |
| `~1.2.3` | 兼容 1.2.x 及以上（<1.3.0） | `legal-search: ~2.1.0` |
| `>=1.0.0 <2.0.0` | 明确范围 | `data-tools: >=1.0.0 <2.0.0` |
| `*` | 任意版本（不推荐） | `utils: *` |

**规范**：依赖版本须使用范围声明，不推荐使用精确版本锁定。

## 解析流程

### 步骤一：依赖树构建

```markdown
### 依赖树

**根技能**：[名称] v[版本]

```
[根技能] v[版本]
├── [依赖A] ^1.0.0 → 解析为 v1.2.3
│   ├── [子依赖A1] ^1.0.0 → 解析为 v1.0.5
│   └── [子依赖A2] ~2.1.0 → 解析为 v2.1.8
├── [依赖B] ^2.0.0 → 解析为 v2.3.1
│   └── [子依赖A1] ^1.2.0 → 解析为 v1.2.3（与依赖A共享）
└── [依赖C] ^1.0.0 → 解析为 v1.1.0
```

**依赖总数**：[N] 个直接依赖，[M] 个传递依赖
```

### 步骤二：版本冲突检测

```markdown
### 冲突检测

| 依赖项 | 要求来源 | 版本要求 | 解析结果 | 冲突状态 |
|-------|---------|---------|---------|---------|
| [子依赖A1] | 依赖A | ^1.0.0 | v1.2.3 | ✅ |
| [子依赖A1] | 依赖B | ^1.2.0 | v1.2.3 | ✅（兼容） |

**冲突**：[无冲突 / 存在冲突]
```

### 步骤三：循环依赖检测

```markdown
### 循环依赖检测

**检测结果**：[✅ 无循环 / ❌ 检测到循环依赖]

[若存在循环]：
- [技能A] → [技能B] → [技能C] → [技能A]（循环！）
```

### 步骤四：许可证兼容性检查

```markdown
### 许可证兼容性

| 依赖项 | 许可证 | 与主技能兼容性 | 结论 |
|-------|-------|-------------|------|
| [依赖A] | MIT | ✅ 兼容 | ✅ |
| [依赖B] | GPL | ⚠️ 传染性 | 须确认主技能许可证 |
| [依赖C] | Apache 2.0 | ✅ 兼容 | ✅ |
```

### 步骤五：依赖安全状态

```markdown
### 依赖安全状态

| 依赖项 | 版本 | 已知漏洞 | 安全状态 |
|-------|------|---------|---------|
| [依赖A] | v1.2.3 | 无 | ✅ |
| [依赖B] | v2.3.1 | CVE-XXXX-YYYY（低） | ⚠️ |
```

## 冲突解决策略

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| 最高版本 | 选择满足所有约束的最高版本 | 约束可兼容时 |
| 最小集合 | 选择最少数量的版本满足所有约束 | 约束较紧时 |
| 用户选择 | 展示冲突详情，由用户决定 | 自动解决失败时 |

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 依赖项不在注册表中 | 标注缺失；建议手动指定路径 |
| 版本约束不可满足 | 报告冲突详情；建议调整版本约束 |
| 依赖树过深（>5 层） | 警告性能风险；建议扁平化 |
| 注册表不可达 | 使用本地缓存；标注缓存可能过期 |
