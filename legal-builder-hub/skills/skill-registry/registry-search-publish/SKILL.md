# 注册表查询与发布（registry-search-publish）

> **所属编排**：`legal-builder-hub/skills/skill-registry/SKILL.md`（技能注册表）
> **分工说明**：本子技能负责注册表的查询和发布操作。清单校验由 `skill-manifest-management` 处理；版本管理由 `version-release-management` 处理；依赖解析由 `dependency-resolution` 处理。
> **可单独触发**：`/legal-builder-hub:registry-search-publish`

---

## 目的

提供技能注册表的查询和发布功能：在注册表中搜索技能的元数据和版本信息；将新版本发布到注册表；管理发布状态和审核流程。

---

## 前置

1. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：注册表类型、注册表地址、发布审核流程、用户角色
2. **连接注册表** — 根据注册表类型建立连接（本地/远程）
3. 本子技能可修改注册表数据（发布操作时）

---

## 查询功能

### 查询维度

| 查询维度 | 说明 | 示例 |
|---------|------|------|
| 按名称 | 精确或模糊匹配技能名称 | `contract-review` |
| 按领域 | 按法律领域分类查询 | 数据合规领域的所有技能 |
| 按版本 | 查询特定版本信息 | `contract-review@1.2.0` |
| 按安全状态 | 按安全审查状态筛选 | 仅显示已审查通过的技能 |
| 按许可证 | 按许可证类型筛选 | MIT 许可证的技能 |
| 按作者 | 按开发者查询 | 某开发者发布的所有技能 |
| 按依赖 | 查询依赖某技能的其他技能 | 哪些技能依赖 `legal-search` |

### 查询结果格式

```markdown
## 注册表查询结果

**注册表**：[本地/组织/公共 — 地址]
**查询条件**：[条件描述]
**匹配数量**：[N] 个技能

| 技能名称 | 最新版本 | 安全状态 | 许可证 | 发布时间 |
|---------|---------|---------|-------|---------|
| [名称] | [版本] | [状态] | [许可证] | [日期] |
```

## 发布功能

### 发布前检查

在执行发布前，须确认以下条件：

```markdown
### 发布前检查

- [ ] SKILL.md 通过规范校验（`skill-manifest-management`）
- [ ] 版本号符合 SemVer 规范（`version-release-management`）
- [ ] 安全审查已通过（`security-review`）
- [ ] 依赖关系可解析（`dependency-resolution`）
- [ ] 变更日志已生成
- [ ] 用户有发布权限
```

### 发布流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 上传技能包 | 将技能文件上传到注册表 |
| 2 | 注册元数据 | 将技能元数据写入注册表索引 |
| 3 | 安全状态关联 | 关联安全审查报告编号 |
| 4 | 审核（如需要） | 按发布审核流程处理 |
| 5 | 发布确认 | 发布成功通知 |

### 发布审核

| 审核流程 | 行为 | 通过条件 |
|---------|------|---------|
| 自动审核 | 通过前检查后自动发布 | 所有前检查项通过 |
| 人工审核 | 提交后等待审核员审批 | 审核员批准 |
| 社区审核 | 发布到社区公示期 | 公示期无异议 |

### 发布结果

```markdown
## 发布结果

**技能**：[名称] v[版本号]
**发布时间**：[时间戳]
**注册表**：[注册表名称/地址]

| 项目 | 状态 |
|------|------|
| 技能包上传 | ✅ / ❌ |
| 元数据注册 | ✅ / ❌ |
| 安全状态关联 | ✅ / ❌ |
| 审核状态 | [自动通过 / 待审核 / 已批准] |
| 发布状态 | [✅ 已发布 / ⏳ 待审核 / ❌ 发布失败] |

**注册表记录**：
- 查询地址：[地址]
- 安装命令：`/legal-builder-hub:installation-execution [名称]@[版本]`
```

## 发布状态管理

### 状态流转

```
开发中 → 待审核 → 审核中 → [已批准 / 已拒绝]
                                    ↓
                              [已发布 / 重新提交]
已发布 → 已撤销（发现问题时）
已发布 → 已废弃（不再维护时）
```

### 状态查询

```markdown
### 发布状态

| 技能 | 版本 | 状态 | 更新日期 | 说明 |
|------|------|------|---------|------|
| [名称] | [版本] | [状态] | [日期] | [说明] |
```

## 撤销与废弃

### 版本撤销

- 标注版本为「已撤销」
- 记录撤销原因
- 通知已安装用户
- 不影响历史审计记录

### 技能废弃

- 标注技能为「已废弃」
- 记录废弃原因和建议替代方案
- 搜索结果中标注废弃状态
- 不再接受新安装（已安装的可继续使用）

## 注册表类型对比

| 特性 | 本地注册表 | 组织内部注册表 | 公共社区注册表 |
|------|----------|-------------|-------------|
| 访问范围 | 仅本地 | 组织内部 | 公开访问 |
| 审核要求 | 无 | 组织自定义 | 社区审核 |
| 安全审查 | 建议但非强制 | 通常要求 | 强制要求 |
| 版本管理 | 自主管理 | 组织管理 | 平台管理 |

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 注册表不可达 | 提示检查网络和配置；提供离线操作建议 |
| 版本号已存在 | 阻止发布；建议使用更高版本号 |
| 发布权限不足 | 提示联系注册表管理员获取权限 |
| 审核被拒绝 | 展示拒绝原因；提供修改建议 |
| 技能名称已被占用 | 提示名称冲突；建议使用不同名称 |
