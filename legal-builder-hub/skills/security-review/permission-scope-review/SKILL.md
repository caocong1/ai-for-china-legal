# 权限范围审查（permission-scope-review）

> **所属编排**：`legal-builder-hub/skills/security-review/SKILL.md`（安全审查）
> **分工说明**：本子技能负责权限范围的合规性审查。静态分析由 `static-analysis-scan` 处理；隐私评估由 `data-privacy-assessment` 处理；报告认证由 `review-report-certification` 处理。
> **可单独触发**：`/legal-builder-hub:permission-scope-review`

---

## 目的

审查技能声明的权限范围是否与其实际功能需求匹配（最小权限原则），检测是否请求了超出功能所需的权限，检查是否触碰用户配置的敏感权限零容忍列表，输出权限审查报告。

---

## 前置

1. **获取技能声明** — 从 SKILL.md 提取技能的功能声明
2. **获取权限声明** — 从技能元数据提取请求的权限清单
3. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：敏感权限零容忍列表、安全审查级别
4. 本子技能产出是权限审查报告，**不修改技能代码**

---

## 权限分类体系

### 权限级别

| 权限级别 | 说明 | 审查要求 |
|---------|------|---------|
| **基础权限** | 读取技能自身文件、执行基本计算 | 默认允许 |
| **文件权限** | 读写用户文件系统 | 须说明具体路径和原因 |
| **网络权限** | 访问外部网络/服务 | 须说明目标地址和目的 |
| **系统权限** | 执行系统命令、访问系统资源 | 须充分理由，默认拒绝 |
| **数据权限** | 访问用户数据（对话/文件/案件） | 须说明数据类型和处理目的 |
| **敏感权限** | 访问敏感信息（身份/财务/法律文件） | 须单独声明，用户逐一审批 |

### 常见权限清单

| 权限标识 | 权限名称 | 风险级别 | 典型使用场景 |
|---------|---------|---------|------------|
| `file:read` | 读取文件 | 基础 | 读取技能配置文件 |
| `file:write` | 写入文件 | 中等 | 生成输出文件 |
| `file:read-user` | 读取用户文件 | 🟠 高 | 读取用户提供的法律文件 |
| `file:write-system` | 写入系统文件 | 🔴 关键 | 修改系统配置 |
| `network:outbound` | 网络出站 | 🟠 高 | 调用外部 API |
| `network:outbound-legal` | 向外部发送法律数据 | 🔴 关键 | 将案件数据发送至外部服务 |
| `system:execute` | 执行系统命令 | 🔴 关键 | 运行外部程序 |
| `data:conversation` | 访问对话内容 | 🟠 高 | 读取用户与 AI 的对话 |
| `data:case-files` | 访问案件文件 | 🔴 关键 | 读取案件相关文档 |
| `data:personal-info` | 访问个人信息 | 🔴 关键 | 读取用户身份信息 |

## 审查流程

### 步骤一：权限清单提取

```markdown
### 技能权限声明

**技能**：[名称] v[版本]

| 权限标识 | 请求原因（技能声明） | 权限级别 |
|---------|-------------------|---------|
| [权限] | [原因] | [级别] |
```

### 步骤二：最小权限原则检查

将每个请求的权限与技能声明的功能进行对照：

```markdown
### 权限必要性分析

| 请求权限 | 对应功能 | 是否必要 | 分析说明 |
|---------|---------|---------|---------|
| `file:read-user` | 读取用户提供的合同文件 | ✅ 必要 | 合同审查功能需要读取文件 |
| `network:outbound` | 调用法规数据库 API | ✅ 必要 | 法规检索功能需要网络访问 |
| `network:outbound-legal` | 向境外服务器发送分析结果 | ❌ 超出需要 | 功能描述中无跨境需求说明 |
| `system:execute` | [未说明] | ❌ 超出需要 | 功能不需要系统命令执行 |
```

### 步骤三：零容忍权限检查

对照用户配置的敏感权限零容忍列表：

```markdown
### 零容忍权限检查

**用户零容忍列表**：[从 CLAUDE.md 读取]

| 零容忍权限 | 技能是否请求 | 结论 |
|-----------|------------|------|
| [权限A] | 是 | 🔴 触碰零容忍 — 不可安装 |
| [权限B] | 否 | ✅ 未触碰 |
```

### 步骤四：权限风险综合评估

```markdown
### 权限风险评估

| 风险等级 | 发现数量 |
|---------|---------|
| 🔴 关键（零容忍触碰/严重越权） | [N] |
| 🟠 高（不必要的高风险权限） | [N] |
| 🟡 中等（权限声明不完整） | [N] |
| 🟢 低（权限范围合理） | [N] |
```

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 技能未声明权限清单 | 标注为 🟠 高风险；建议开发者补充权限声明 |
| 权限描述模糊 | 标注为 🟡 中等；要求开发者明确权限使用场景 |
| 运行时动态请求权限 | 标注为 🟠 高；须在声明中列出所有可能的权限请求 |
| 权限随配置变化 | 标注各配置下的权限变化范围 |
