# 静态分析与代码扫描（static-analysis-scan）

> **所属编排**：`legal-builder-hub/skills/security-review/SKILL.md`（安全审查）
> **分工说明**：本子技能负责代码层面的静态安全分析。权限审查由 `permission-scope-review` 处理；隐私评估由 `data-privacy-assessment` 处理；报告认证由 `review-report-certification` 处理。
> **可单独触发**：`/legal-builder-hub:static-analysis-scan`

---

## 目的

对技能源代码和配置文件进行静态安全分析，不运行代码即可发现潜在安全隐患：恶意代码模式检测、硬编码密钥和凭证扫描、不安全函数调用识别、已知漏洞依赖项检查、代码注入风险评估。

---

## 前置

1. **获取技能源代码** — 技能代码路径或 Git 仓库地址
2. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：安全审查级别、敏感权限零容忍列表
3. **加载扫描规则库** — OWASP Top 10、平台安全规则、自定义规则
4. 本子技能产出是安全扫描报告，**不修改源代码**

---

## 扫描维度

### 1. 恶意代码模式检测

| 检测类别 | 检测内容 | 风险等级 |
|---------|---------|---------|
| 后门/木马 | 远程代码执行、反向连接、隐蔽通信通道 | 🔴 关键 |
| 数据窃取 | 未经授权的数据收集和发送、隐蔽数据编码 | 🔴 关键 |
| 系统破坏 | 文件系统破坏、注册表修改、系统进程操控 | 🔴 关键 |
| 权限提升 | 未授权的权限请求、安全绕过尝试 | 🔴 关键 |

### 2. 硬编码密钥与凭证扫描

| 检测类别 | 检测模式 | 风险等级 |
|---------|---------|---------|
| API 密钥 | 硬编码的 API 密钥、Token、Secret | 🔴 关键 |
| 数据库凭证 | 硬编码的数据库连接串、用户名/密码 | 🔴 关键 |
| 加密密钥 | 硬编码的加密密钥、私钥文件 | 🔴 关键 |
| 服务凭证 | 硬编码的云服务凭证、OAuth 客户端密钥 | 🟠 高 |

### 3. 不安全函数调用

| 检测类别 | 检测内容 | 风险等级 |
|---------|---------|---------|
| 代码注入 | `eval()`、动态代码执行、模板注入 | 🟠 高 |
| 命令注入 | 不安全的系统命令执行、shell 调用 | 🟠 高 |
| SQL 注入 | 未参数化的 SQL 查询 | 🟠 高 |
| 路径遍历 | 不安全的文件路径拼接 | 🟡 中等 |
| 不安全的反序列化 | 未经验证的反序列化操作 | 🟡 中等 |

### 4. 依赖项漏洞检查

| 检测类别 | 检测内容 | 风险等级 |
|---------|---------|---------|
| 已知严重漏洞 | 依赖项存在 CVE 记录的严重漏洞 | 🔴 关键 / 🟠 高 |
| 过时依赖 | 依赖项版本过旧，可能存在未修复漏洞 | 🟡 中等 |
| 废弃依赖 | 依赖项已被废弃，不再维护 | 🟡 中等 |
| 许可证风险 | 依赖项许可证与主项目不兼容 | 🟡 中等 |

### 5. 数据泄露风险

| 检测类别 | 检测内容 | 风险等级 |
|---------|---------|---------|
| 日志泄露 | 在日志中输出敏感信息 | 🟠 高 |
| 错误信息泄露 | 错误消息中包含敏感信息 | 🟡 中等 |
| 注释泄露 | 代码注释中包含敏感信息 | 🟡 中等 |
| 临时文件泄露 | 临时文件未及时清理 | 🟡 中等 |

## 扫描规则配置

| 规则集 | 说明 | 默认启用 |
|-------|------|---------|
| OWASP Top 10 | Web 应用安全风险 | 是 |
| 平台安全规则 | 技能清单特有的安全规则 | 是 |
| 密钥扫描 | 硬编码凭证检测 | 是 |
| 依赖漏洞 | 已知 CVE 检测 | 是 |
| 自定义规则 | 用户定义的安全规则 | 否 |

---

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 代码混淆/压缩 | 标注无法分析的代码段；建议提供源代码版本 |
| 扫描超时 | 优先扫描核心文件；标注未扫描部分 |
| 误报处理 | 标注置信度；允许用户标记为误报 |
| 加密文件 | 标注无法分析加密内容 |

## 扫描规则库更新

扫描规则库须定期更新以覆盖新发现的漏洞模式和攻击手法：

- **OWASP Top 10 更新**：每年 OWASP 发布更新时同步更新规则库
- **CVE 数据库同步**：定期同步国家信息安全漏洞库（CNNVD）和国际 CVE 数据库
- **平台规则更新**：随平台技术规范版本迭代更新平台安全检查规则
- **自定义规则维护**：用户定义的自定义规则须定期审查有效性
