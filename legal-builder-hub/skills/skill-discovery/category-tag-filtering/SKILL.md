# 分类·标签·筛选（category-tag-filtering）

> **所属编排**：`legal-builder-hub/skills/skill-discovery/SKILL.md`（技能发现）
> **分工说明**：本子技能负责构建筛选条件和分类体系。搜索执行由 `skill-search-browse` 处理；排序推荐由 `recommendation-ranking` 处理；详情由 `skill-detail-evaluation` 处理。
> **可单独触发**：`/legal-builder-hub:category-tag-filtering`

---

## 目的

提供多维度的技能分类和筛选体系，帮助用户通过法律领域、能力类型、安全等级、许可证类型、兼容性等维度精确缩小搜索范围，快速定位符合需求的技能。

---

## 前置

1. **读取 `legal-builder-hub/CLAUDE.md`** — 获取：关注的法律领域、关注的能力类型、是否接受未审查技能
2. **读取注册表分类体系** — 获取当前可用的分类标签

---

## 分类体系

### 维度一：法律领域

| 一级分类 | 二级分类（示例） |
|---------|----------------|
| 商事合同 | 买卖合同、技术服务合同、租赁合同、特许经营合同 |
| 诉讼仲裁 | 民事诉讼、商事仲裁、行政复议、执行 |
| 劳动人事 | 劳动合同、竞业限制、裁员方案、劳动争议 |
| 数据合规 | 个人信息保护、数据出境、网络安全等保、数据安全 |
| 知识产权 | 商标、专利、著作权、不正当竞争 |
| 监管合规 | 行业监管、行政处罚、合规体系建设 |
| AI 治理 | 算法备案、深度合成、生成式 AI 合规 |
| 建设工程 | 施工合同、工程结算、工期索赔、质量保修 |
| 婚姻家事 | 离婚协议、遗嘱、继承、婚前财产协议 |
| 刑事合规 | 企业刑事风险、合规不起诉、单位犯罪预防 |
| 法考培训 | 客观题训练、主观题写作、案例研习 |
| 法律援助 | 案件 intake、法律咨询、案件转介 |

### 维度二：能力类型

| 能力类型 | 说明 | 示例 |
|---------|------|------|
| 文书审查 | 审查法律文件并出具审查意见 | 合同审查、协议审查 |
| 文书起草 | 起草法律文件和文书 | 合同起草、协议起草、诉状起草 |
| 法规检索 | 检索法律法规条文 | 法条查询、案例检索 |
| 案例分析 | 分析法律案例 | 案例比对、裁判分析 |
| 合规检查 | 检查合规状态并出具报告 | PIA 生成、等保检查 |
| 期限管理 | 管理法律期限和时效 | 诉讼时效、履约期限 |
| 风险评估 | 评估法律风险 | 合规风险评估、诉讼风险 |
| 培训训练 | 提供培训和练习 | 法考刷题、写作练习 |

### 维度三：安全等级

| 安全等级 | 说明 | 筛选含义 |
|---------|------|---------|
| 🟢 已审查通过 | 通过完整安全审查，无重大发现 | 可安全安装 |
| 🟡 有条件通过 | 存在中等风险，修复后可安装 | 须确认修复情况 |
| 🔴 未通过 | 存在高/关键风险 | 不建议安装 |
| ⚪ 未审查 | 尚未进行安全审查 | 安装前须审查 |

### 维度四：许可证类型

| 许可证 | 使用限制 | 适合场景 |
|-------|---------|---------|
| MIT | 最宽松，几乎无限制 | 所有场景 |
| Apache 2.0 | 宽松，含专利授权 | 所有场景（含商业用途） |
| GPL | 衍生作品须同样开源 | 开源项目 |
| LGPL | 链接使用可不开源 | 库类技能 |
| 自定义 | 须查看具体条款 | 须逐案审查 |

### 维度五：兼容性

| 兼容性状态 | 说明 |
|-----------|------|
| ✅ 兼容当前平台 | 与当前系统环境完全兼容 |
| ⚠️ 需确认 | 与部分环境配置存在兼容性风险 |
| ❌ 不兼容 | 与当前系统环境不兼容 |

## 筛选条件构建流程

### 步骤一：确定筛选维度

根据用户需求，确定要使用的筛选维度：

```
请选择筛选维度（可多选）：
□ 法律领域：[选择一个或多个领域]
□ 能力类型：[选择一个或多个类型]
□ 安全等级：[选择最低可接受等级]
□ 许可证类型：[选择可接受的许可证]
□ 兼容性：[选择兼容性要求]
```

### 步骤二：设置筛选条件

```markdown
## 当前筛选条件

| 维度 | 条件 | 匹配数量 |
|------|------|---------|
| 法律领域 | 数据合规 | [N] 个技能 |
| 能力类型 | 文书审查 | [N] 个技能 |
| 安全等级 | 🟢 已审查通过 | [N] 个技能 |
| 许可证 | MIT / Apache 2.0 | [N] 个技能 |
| 兼容性 | ✅ 兼容 | [N] 个技能 |

**组合筛选结果**：[M] 个技能
```

### 步骤三：调整与确认

- 如果结果过多：建议增加筛选维度或缩小范围
- 如果结果过少：建议放宽某些条件
- 确认筛选条件后，传递给 `skill-search-browse` 执行搜索

## 智能筛选建议

根据用户画像提供智能筛选建议：

| 用户配置 | 默认筛选建议 |
|---------|------------|
| 关注的法律领域 | 自动勾选相关领域 |
| 关注的能力类型 | 自动勾选相关能力类型 |
| 是否接受未审查技能 | 控制安全等级筛选条件 |
| 保守型偏好 | 安全等级默认 🟢 已审查通过 |
| 进取型偏好 | 安全等级默认包含 ⚪ 未审查 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 筛选结果为空 | 建议放宽条件；显示各维度的匹配数量帮助用户调整 |
| 用户无法确定筛选条件 | 根据用户画像智能推荐默认筛选 |
| 分类标签不存在用户需要的值 | 记录用户需求标签；建议向注册表提交新分类标签 |
