## 风险校准

> 技能安全与合规风险等级定义（作为安全审查输出中的风险标注参考）

| 级别 | 含义 | 技能安全典型触发情形 |
|------|------|-------------------|
| 🔴 关键 | 存在严重安全隐患或违规风险，禁止安装 | 技能包含恶意代码或后门；技能未经授权收集用户数据；技能请求过度权限（如读取全部文件系统）；技能违反 PIPL 收集个人信息 |
| 🟠 高 | 有较大安全风险，需修复后方可安装 | 技能权限范围超出其声明功能所需；技能未提供隐私政策或数据处理说明；依赖项存在已知漏洞 |
| 🟡 中等 | 存在改进空间，风险可控但建议优化 | 技能缺少安全文档；依赖项版本过旧但无已知漏洞；技能描述不完整 |
| 🟢 低 | 基本安全，风险可接受 | 技能通过完整安全审查；权限范围合理；有明确的隐私政策和数据处理说明 |

## 升级矩阵

### 角色权限表

| 角色 | 可独立处理事项 | 需升级条件 |
|------|--------------|-----------|
| 技能使用者 | 搜索浏览技能、安装已审查通过的技能、更新已安装技能 | 安装未经审查的技能；修改安全审查配置 |
| 技能开发者 | 开发技能、提交技能到注册表、修改自己的技能 | 发布包含敏感权限的技能；修改已发布技能的安全声明 |
| 平台管理员 | 审核技能安全审查报告、管理注册表、制定安全策略 | 批准有争议的安全审查结论；修改平台级安全策略 |
| 安全审核员 | 执行完整安全审查、出具审查报告、签发安全认证 | 审查涉及自身开发技能的报告（利益冲突） |

### 自动升级触发条件

无论角色，以下情况必须自动升级：

- **安全审查发现 🔴 关键风险**：技能包含恶意代码、未授权数据收集或过度权限请求
- **技能请求访问敏感数据**：技能请求访问法律文件、客户信息、案件数据等敏感内容
- **技能包含网络外传能力**：技能声明或代码中发现向外部服务器发送数据的能力
- **依赖项存在严重漏洞**：技能依赖的第三方库存在已知严重安全漏洞
- **技能违反平台安全策略**：技能行为与平台声明的安全策略不一致

## 共享护栏

### 安全优先原则

> **未经安全审查的技能不得安装。** 这是不可违反的硬性规则。

- 所有技能在安装前必须通过 security-review 模块的安全审查
- 安全审查发现 🔴 关键风险时，安装流程自动终止
- 即使开发者声明技能安全，也必须经过独立审查确认
- 安全审查报告须保留至少 2 年备查

### 数据合规要求

技能涉及数据处理时，须同时符合以下法律要求：

| 法律 | 核心要求 | 适用场景 |
|------|---------|---------|
| 《个人信息保护法》（PIPL） | 处理个人信息须有合法性基础（第13条，已核实）；敏感信息须单独同意（第29条，已核实） | 技能处理用户个人信息 |
| 《数据安全法》（DSL） | 数据分类分级保护（第21条，已核实）；重要数据安全评估 | 技能处理重要数据 |
| 《网络安全法》（CSL） | 网络安全等级保护（第21条，已核实） | 技能运行在网络环境中 |

### 来源标签规范

| 标签 | 含义 |
|------|------|
| `[平台规范]` | 来自平台官方技术规范和 API 文档 |
| `[安全标准]` | 来自行业安全标准（如 OWASP、GB/T 信息安全标准） |
| `[网络搜索 — 需验证]` | 来自网络搜索，需对照原始来源检查 |
| `[模型知识 — 需验证]` | 来自训练数据回忆，须核实 |
| `[用户提供]` | 来自用户提供的技能代码或文档 |

### 知识产权提示

- 社区贡献的技能须明确声明许可证类型（如 MIT、Apache 2.0、GPL 等）
- 技能引用的法律条文属于公共领域，但技能的编排、注释、分析模型可能受著作权保护
- 技能开发者须保证不侵犯第三方知识产权
