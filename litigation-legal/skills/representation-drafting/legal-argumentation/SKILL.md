# 法律论证与法条适用

> **研究闸门**：除非用户明确豁免联网检索，必须先确认 `shared/research-gate/SKILL.md` 已运行，研究底稿已完成，再进入本子技能。特别是法条核验与指导性案例检索须经研究闸门完成。
>
> **单独触发说明**：本子技能可通过 `/litigation-legal:legal-argumentation` 单独触发，只运行法律论证，无需重跑整个代理词起草流程。

## 目的

完成代理词的法律论证核心工作：

1. **识别适用法律**：确定请求权基础（原告）或抗辩法律构成（被告）
2. **要件分析**：逐一检验法律构成要件是否满足
3. **法律涵摄**：将已证明的案件事实涵摄至法律构成要件
4. **案例支持**：引用最高人民法院指导性案例佐证裁判思路（参考，非判例法）
5. **法律依据清单**：按法条/案例/学说分类输出完整法律依据

**前提**：本子技能依赖 `dispute-focus`（争议焦点）和 `fact-argumentation`（事实论证）的产出；如直接触发，需先向律师确认核心争议焦点、已证明的事实以及当事人的诉讼请求。

**引用纪律**（硬性要求）：

## 步骤一：识别请求权基础

### 1.1 原告方：识别请求权基础

如我方为原告，识别本案诉讼请求所依据的请求权基础：

| 请求权类型 | 常见请求权基础 | 关键法律规则 |
|---------|------------|------------|
| 合同违约 | 违约方承担违约责任（含继续履行/赔偿损失/支付违约金）| 《民法典》合同编相关条文 `[条号待验证]` |
| 侵权赔偿 | 侵权行为人承担侵权责任（含过错责任/无过错责任）| 《民法典》侵权责任编相关条文 `[条号待验证]` |
| 不当得利返还 | 无法律根据取得不当利益 | 《民法典》不当得利相关条文 `[条号待验证]` |
| 合同无效/可撤销 | 合同不成立、无效或可撤销及其后果 | 《民法典》总则编/合同编相关条文 `[条号待验证]` |
| 其他（按案件类型）| [由律师确认具体请求权基础] | [对应法条 `[待验证]`] |

> **边界条件**：存在多个并用请求权时（如同时主张违约和侵权），应分别列出，并说明我方主张的优先顺序和理由。

### 1.2 被告方：识别抗辩法律构成

如我方为被告，识别可用的抗辩：

| 抗辩类型 | 法律依据 | 适用条件 |
|---------|---------|---------|
| 实体抗辩 | 请求权要件不满足（如过错不存在、损害与行为无因果关系）| 攻击对方请求权构成要件 |
| 程序性抗辩 | 管辖权异议、诉讼时效抗辩、起诉资格问题 | 须在答辩期间提出；时效抗辩须主动提出（法院不主动适用）|
| 仲裁协议抗辩 | 存在有效仲裁协议，排除法院管辖 | 《仲裁法》或裁或审原则，具体条号待验证 |
| 减损与贡献过错 | 原告未尽减损义务；原告存在过错 | 《民法典》相关条文 `[条号待验证]` |

## 步骤二：要件分析

### 2.1 确定法律构成要件

**原告**——以违约责任为例（其他类型参照执行）：

```
请求权基础：要求被告承担违约责任

要件：
(1) 合同成立且有效
(2) 被告存在违约行为（不履行/迟延履行/不适当履行）
(3) 原告已履行对待给付义务（如有）
(4) 损害及其范围（如主张赔偿损失）
(5) 违约行为与损害之间存在因果关系（如主张赔偿）
```

**被告**——以时效抗辩为例：

```
抗辩构成：原告诉讼时效已届满

要件：
(1) 时效期间已届满（一般3年=《民法典》第188条，已核实）
(2) 在时效期间内无中止/中断事由
(3) 被告主动提出时效抗辩（时效抗辩须当事人主动提出，法院不主动适用）
```

### 2.2 逐要件检验（涵摄）

对每个要件，完成「要件 → 相关事实 → 证据 → 是否满足」的检验：

| 要件编号 | 要件内容 | 相关事实（案件中对应情况）| 关键证据 | 满足状态 |
|---------|---------|---------------------|---------|---------|
| 要件(1) | [要件描述] | [案件中的具体事实] | 证据 [N] | 满足 / 不满足 / 存疑 |
| 要件(2) | [要件描述] | [案件中的具体事实] | 证据 [M] | 满足 / 不满足 / 存疑 |
| 要件(3) | [要件描述] | [案件中的具体事实] | [证据或无需证据]| 满足 / 不满足 / 存疑 |

**对「存疑」要件的处理**：说明存疑原因，提示证明方向，标注「[需律师确认事实/证据]」。

## 步骤三：法律论证段落起草

### 3.1 论证结构

法律论证遵循「要件 → 事实涵摄 → 结论」三段论结构：

```
[大前提（法律规则）]：根据《[法律名称]》第[X]条[第X款] `[待验证]`，……（法律规则表述）

[小前提（案件事实涵摄）]：本案中，……（将已证明的事实涵摄至法律要件）

[结论]：因此，[被告构成违约/原告请求权成立/抗辩成立等]。
```

### 3.2 各争议焦点的法律论证

对每个争议焦点，起草独立的法律论证段落。段落格式：

**（一）关于[争议焦点一]的法律分析**

[大前提：适用法条]

[小前提：事实涵摄]

[结论：我方法律主张]

**（二）关于[争议焦点二]的法律分析**

（同上）

### 3.3 指导性案例支持

如有与本案相关的最高人民法院指导性案例，附于对应焦点的法律分析之后：

> **[指导案例第X号] `[待验证]`**：[裁判要点摘要]。
>
> **裁判意义**：[对本案的借鉴意义]。
>
> **中国法律体系说明**：中国采成文法体系，本指导案例为参考，**不具有判例法约束力**，实际效力以法条为准。

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 存在多个并用请求权 | 逐一分析每个请求权的构成，说明并用逻辑和主张优先级 |
| 某要件的事实存疑 | 列出最有利和最不利情形，标注「[存疑 — 需律师确认事实]」，提出补充证据方向 |
| 适用特殊诉讼时效 | 识别特殊时效法律依据（不确信条号则标 `[待验证]`），与 limitation-monitoring 协作分析 |
| 争议涉及仲裁条款有效性 | 引用仲裁法关于仲裁协议独立性和或裁或审原则（条号待验证），明确若协议无效则法院有管辖权 |
| 涉及法律适用争议（如新旧法适用）| 分析新旧法适用规则，标注法律争议存在，建议律师确认 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 不确定适用的实体法条号 | 描述法律规则，标注「具体条号待验证」；不臆造条号 |
| 无法找到相关指导性案例 | 不捏造案例；说明「暂无可引用的最高院指导性案例，本焦点依法条论证」 |
| 对方法律论证有新的法律依据（庭审中提出）| 提示律师需即时回应；标注「[对方新法律主张 — 需律师即时评估]」 |
| 法律要件分析涉及高度专业判断 | 提供分析框架，标注「[最终法律判断须由律师确认]」 |
