# 代理词装配

> **研究闸门**：代理词装配须在所有前序子技能（dispute-focus / fact-argumentation / legal-argumentation）完成并经律师确认后执行。联网检索（研究闸门）须在此步骤之前完成。
>
> **单独触发说明**：本子技能可通过 `/litigation-legal:representation-assembly` 单独触发。直接触发时需提供：争议焦点对照表、事实论证段落、法律论证段落和代理请求清单。

## 目的

将前序子技能产出的各部分装配为完整、可交付的代理词文书，并生成配套的庭审工具：

1. **代理词全文**：开头（代理人身份/代理权限/简要意见）→ 事实部分 → 法律部分 → 代理请求/理由总结
2. **辩论提纲**：庭审辩论要点，按争议焦点排序，含应急应对提示
3. **法庭陈述**：开庭陈述（精简版）和结案陈词（总结式）
4. **交付前检查**：特权标题、研究闸门完成、引用验证状态、格式合规

## 前置读取

执行本子技能前，读取以下资源：

1. `litigation-legal/CLAUDE.md` — 读取代理词风格、工作产物标题（律师/非律师角色）、工作产物存放位置、是否需要案例引用/法条引用
2. `litigation-legal/skills/_shared/legal-basis-conventions.md` — 引用规范（用于交付前检查）

   - `dispute-focus` 产出：争议焦点对照表
   - `fact-argumentation` 产出：事实论证段落 + 事实-证据对应表
   - `legal-argumentation` 产出：法律论证段落 + 法律依据清单
4. `shared/references/document-structures.md`（如有）— 代理意见结构模板

**临时模式**：如 `litigation-legal/CLAUDE.md` 未配置，使用临时模式默认值，输出标记 `[临时模式 — 配置你的档案以获得定制输出]`。

## 步骤二：装配代理词全文

### 2.1 完整代理词结构

```
[文书标题]
[特权标题（如适用）]

审判长、审判员（或：仲裁庭）：

根据《中华人民共和国民事诉讼法》的相关规定，[代理人名称] 接受 [委托人名称] 的委托，担任本案的诉讼代理人。就本案发表如下代理意见：

一、代理人身份及代理权限
二、本案事实部分（整合 fact-argumentation 产出）
三、法律分析部分（整合 legal-argumentation 产出）
四、代理意见与请求

代理人：[签名]
日期：[日期]
```

### 2.2 各部分装配细则

**开头部分**：

- 代理人身份声明：律师姓名、执业证号（如有）、律所名称
- 代理权限声明：委托授权书编号（如有）、代理范围（一审/二审/全程）
- 简要代理意见：1-2 句概括我方核心法律立场（悬念式结构，引导法官关注重点）

**事实部分**：

- 直接整合 `fact-argumentation` 产出的完整事实概述段落
- 保留事实-证据对应标注（证据序号）
- 未质证证据的标注须保留

**法律部分**：

- 直接整合 `legal-argumentation` 产出的法律论证段落
- 保留法律依据（条文引用 + 来源标签 + 待验证标注）
- 指导性案例标注「参考，不具判例法约束力」须保留

**代理请求/理由总结**：

```
综上所述，[我方/被代理人] 认为，[核心法律立场总结，1-3 句]。

为维护 [委托人] 的合法权益，恳请法院（或仲裁庭）依法：

1. [请求一：具体请求内容]
2. [请求二：具体请求内容（如有）]
3. 本案诉讼费用由 [对方] 承担。

此致

[受理法院/仲裁机构名称]

代理人：[姓名]
日期：[日期]
```

## 步骤三：生成辩论提纲

辩论提纲供庭审法庭辩论阶段使用，结构化记录每个争议焦点的辩论要点：

```markdown
## 辩论提纲

**案件**：[案件名称]
**庭审日期**：[日期]

---

### 焦点一：[焦点名称]

**我方核心主张**：[1-2 句]
**关键证据**：证据 [N]（[名称]）
**法律依据**：《[法律]》第[X]条 `[待验证]`
**对方预期反驳**：[预测对方辩论方向]
**应对策略**：[如何应对]

---

### 焦点二：[焦点名称]
（同上格式）

---

## 关键事实快查表

| 事实 | 证据序号 | 一句话摘要 |
|------|---------|-----------|
| [事实一] | 证据 [N] | [摘要] |
| [事实二] | 证据 [M] | [摘要] |
```

## 步骤四：生成法庭陈述

### 4.1 开庭陈述（如适用）

开庭陈述为庭审第一环节，通常限时（3-5 分钟），须精简：

```
审判长、审判员，原被告各方：

本案代理人 [姓名] 代表 [委托人] 出庭。

本案核心争点在于：[1-2 个最关键争议焦点，一句话描述]。

我方的基本立场是：[核心法律立场，2-3 句]。

主要事实和证据已在书面代理词中详述，请法院依法审查。

谢谢！
```

### 4.2 结案陈词

结案陈词（最终陈述）为庭审最后环节，总结强化我方立场：

```
审判长、审判员：

经过今天的庭审，我方进一步确认：

[庭审强化点一：本次庭审证明了什么，1-2 句]
[庭审强化点二：对方论据为何不成立，1-2 句]

综上，我方请求法院依法支持/驳回[对方请求]：[重申核心请求]。

感谢法院的审理。
```

## 步骤五：交付前检查

交付前必须完成以下检查清单：

### 5.1 特权标题检查

| 检查项 | 通过条件 | 状态 |
|-------|---------|------|
| 工作产物标题是否正确 | 律师角色：「律师工作产物 — 受律师-当事人特权保护 / 保密」；非律师角色：「研究笔记 — 非法律建议，供律师审查」| [ ] |
| 交付目的地是否在特权圈内 | 法庭提交/内部使用 → 保留特权标题；发送对方/外部 → 提供脱敏版本 | [ ] |

### 5.2 研究闸门完成检查

| 检查项 | 通过条件 | 状态 |
|-------|---------|------|
| 研究闸门已运行 | `shared/research-gate/SKILL.md` 已完成，研究底稿已存档 | [ ] |
| 法条已核验 | 所有引用条文已通过研究闸门核查，或明确标注 `[待验证]` | [ ] |

### 5.3 引用验证状态检查

| 检查项 | 通过条件 | 状态 |
|-------|---------|------|
| 所有不确信条号均标 `[待验证]` | 无遗漏，无臆造条号 | [ ] |
| 所有案例引用均标 `[待验证]` | 案号/指导案例编号均有验证标注 | [ ] |
| 指导性案例均标注「参考，不具判例法约束力」| 说明已附于每处案例引用之后 | [ ] |

### 5.4 格式合规检查

| 检查项 | 通过条件 | 状态 |
|-------|---------|------|
| 代理词结构完整 | 开头/事实/法律/代理请求四部分均有 | [ ] |
| 未经质证证据标注 | 「待质证」标注未遗漏 | [ ] |
| 事实-证据对应表附后 | 表格列数一致，格式规范 | [ ] |
| 法律依据清单附后 | 按法条/案例/学说三类分组 | [ ] |

## 下一步决策树

代理词交付后，提示律师选择下一步：

1. **确认并提交** — 律师确认代理词内容，准备庭审提交
2. **修改调整** — 律师有修改意见，指定具体修改方向
3. **补充研究** — 需要进一步法律研究或证据核查
4. **庭审准备** — 进入庭审辩论准备（使用辩论提纲作为基础）
5. **时效监控** — 如案件涉及时效问题，运行 `/litigation-legal:limitation-monitoring`

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 前序子技能产出不完整 | 标注「[依赖前序子技能产出未就绪：待 dispute-focus/fact-argumentation/legal-argumentation 完成后装配]」，不自行补全 |
| 仲裁案件 | 将「审判长、审判员」替换为「仲裁庭」；代理词结构相同，注意仲裁特有程序（如庭审不对外公开）|
| 二审代理词 | 开头须说明一审情况和上诉理由；事实和法律部分聚焦一审有争议的事项，不重复一审无争议事实 |
| 非律师角色 | 工作产物标题改为「研究笔记 — 非法律建议，供律师审查」；在每个法律结论处提示「此为研究草案，须经律师确认方可使用」|

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 代理请求与事实/法律论证不一致 | 识别不一致之处，向律师反馈，等待确认后再装配，不自行调整 |
| 律师要求在代理词中加入过激措辞或人身攻击 | 解释代理词应保持客观，说明不当措辞可能影响法官观感，提供替代表达 |
| 文书格式与法院要求不符 | 如律师提供了法院的格式要求，优先遵从；否则采用本模板标准格式 |
