# 证据保全与种子文档

> 引用规范：`litigation-legal/skills/_shared/legal-basis-conventions.md`
> 本子技能可被 `litigation-legal/skills/matter-intake/SKILL.md` 编排入口调用，
> 也可通过 `/litigation-legal:evidence-preservation` 单独触发。

## 目的

在案件录入阶段完成三项关键安排：

1. **外部律师**：记录外部律师/律所的委托状态和联系方式，确保团队协作无盲区
2. **内部负责人**：识别并记录需参与本案的内部利益相关者
3. **证据保全**：确认证据保全通知是否已发出；如未发出且案件为活跃诉讼或合理预期诉讼，触发紧急标记并提供发出选项

**目标**：在录入阶段就锁定关键联系人和证据保全状态，避免后续因联系方式缺失或证据灭失造成的被动局面。

## 前置读取

执行本子技能前，读取以下文件：

1. `litigation-legal/CLAUDE.md` — 读取「外部律师合作」「证据保全」「输出/工作产物标题」章节
2. `litigation-legal/skills/_shared/practice-profile-schema.md` — 外部律师和证据保全字段映射
3. 上游子技能产出（如在 matter-intake 编排入口下调用）：`case-identification` 的风险评级（中/高/极高 → 无外部律师时标记）

## 步骤一：外部律师安排

### 1.1 从配置文件读取外部律师策略

从 `litigation-legal/CLAUDE.md` → `## 外部律师合作` 读取：
- **合作律所**：首选律所列表或选择标准
- **预算审批流程**：谁批准、多少金额以内可自行决定

### 1.2 录入外部律师信息

| 字段 | 说明 | 状态 |
|------|------|------|
| **律所名称** | 全称（如：金杜律师事务所）| 必填（如已确定）|
| **主办合伙人** | 姓名 + 职务 | 必填（如已确定）|
| **主办合伙人邮箱** | `/oc-status` 用于起草外部律师状态请求 | 必填（如已确定）|
| **聘用协议状态** | 已签 / 待签 / 无 | 必填 |
| **预算授权** | 授权金额 + 审批人姓名 | 如有填写 |
| **种子文档机会** | 「聘用协议路径，如果已签」| 提示 |

### 1.3 风险触发标记

**如果风险综合评级为「中等或更高」（来自 case-identification）且没有分配外部律师：**

→ 标记 `[外部律师未分配 — 风险评级 [级别]，建议评估是否需要外部律师介入]`

此标记在 `matter.md` 的「外部律师」章节中保持可见，直到律师：
- 分配了外部律师，或
- 明确记录「不需要外部律师：[理由]」

---

## 步骤二：内部负责人

### 2.1 从配置文件读取全景

从 `litigation-legal/CLAUDE.md` → `## 我们是谁` 了解律所/法务团队结构，识别本案涉及的内部利益相关者。

### 2.2 内部负责人清单

| 角色 | 触发条件 | 记录内容 |
|------|---------|---------|
| **业务负责人** | 所有案件 | 姓名 + 联系方式 |
| **HR 合伙人** | 劳动争议案件 | 姓名 + 联系方式 |
| **公关联系人** | 有声誉风险的案件（风险评级高/极高 + 媒体关注可能）| 姓名 + 联系方式 |
| **CISO / 数据合规** | 涉及数据泄露、网络安全、个人信息保护的案件 | 姓名 + 联系方式 |
| **财务负责人** | 重大性为「已计提准备金」或「已披露」的案件 | 姓名 + 联系方式 |
| **其他** | 案件特定需要 | 角色说明 + 联系方式 |

---

## 步骤三：证据保全

### 3.1 从配置文件读取默认保全策略

从 `litigation-legal/CLAUDE.md` → `## 证据保全` 读取：
- **默认保全触发方式**：自动（每案自动发出）/ 逐案决定（人工评估后触发）
  - 若为「自动」：在步骤 3.2 确认保全状态后，如保全通知尚未发出，直接提示律师立即起草发出（无需额外确认触发）
  - 若为「逐案决定」：在步骤 3.2 确认状态后，仅在律师明确请求时才提示发出；否则仅标记「保全通知待触发」
  - 若字段为 `[PLACEHOLDER]` 或缺失：默认按「逐案决定」处理，提示律师评估是否需要发出
- **默认保全范围**：如「相关业务部门所有与争议相关的文件、邮件、聊天记录」
- **默认刷新周期**：通常为 6 个月
- **保全通知模板位置**：模板路径（如有）

### 3.2 确认证据保全状态

询问律师以下信息：

| 字段 | 说明 |
|------|------|
| **保全通知是否已发出** | 是 / 否 / 不适用（如仲裁尚未启动）|
| 如**已发出**：发出日期 | YYYY-MM-DD |
| 如**已发出**：保全范围 | 涉及的文件类型/系统/时间段 |
| 如**已发出**：保管人 | 收到保全通知的人员姓名列表 |
| 如**已发出**：下次刷新日期 | 默认为发出日 + 6 个月（可调整）|
| 如**未发出**：原因说明 | [律师提供原因]|

### 3.3 未发出保全通知的触发规则

**如果保全通知未发出，且满足以下任一条件：**
- 案件为活跃诉讼/仲裁（已有起诉状或仲裁申请送达）
- 案件为合理预期诉讼（风险评级中等以上 + 有法律威胁信号）

→ 标记 `[证据保全紧急 — 保全通知未发出，合理预期诉讼]`，并提示：

> 建议在录入完成后立即运行 `/litigation-legal:legal-hold [slug] --issue` 发出证据保全通知。
> 可使用配置文件中的默认保全范围作为起点，根据本案情况定制。

### 3.4 刷新提醒规则

如果已发出保全通知，记录「下次刷新日期」（默认 6 个月后）。在每次 `/matter-update` 中，当距刷新日期 ≤ 30 日时提醒律师刷新保全通知。

## 步骤四：种子文档登记

在每个步骤中，识别并提示相关的种子文档：

| 步骤 | 种子文档机会 | 提示语 |
|------|------------|--------|
| 外部律师 | 聘用协议 | 「聘用协议路径，如果已签，请提供或说明存放位置。」|
| 案件来源 | 起诉状/仲裁申请/律师函 | 「如果你有 initiating 文档，附上或分享路径，能让录入更精确。」|
| 证据保全 | 保全通知 | 「保全通知，如果已发出，请提供路径或文本。」|

**种子文档登记格式：**

```markdown
| 文档 | 路径/指针 |
|-----|---------|
| [聘用协议] | [路径或「尚未签署」] |
| [起诉状] | [路径或「尚未共享」] |
| [证据保全通知] | [路径或「尚未发出」] |
```

## 检查清单

- [ ] 外部律师信息已录入（或明确标记「未分配」并说明理由）
- [ ] 如风险中等以上且无外部律师：已触发标记
- [ ] 内部负责人清单已完成（至少包括业务负责人）
- [ ] HR/公关/CISO 按触发条件检查是否需要纳入
- [ ] 证据保全状态已确认（已发出 / 未发出 / 不适用）
- [ ] 如已发出：范围/保管人/下次刷新日期已记录
- [ ] 如未发出且活跃诉讼/合理预期：已触发紧急标记并提供发出选项
- [ ] 种子文档机会已在各步骤提示
- [ ] 已登记可提供的种子文档路径（或记录「尚未提供」）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 法律援助案件（无外部律师预算）| 记录「不适用外部律师」，不触发风险标记 |
| 外部律师已委托但聘用协议尚未签署 | 记录为「待签」，提示律师跟进 |
| 证据已部分灭失（如员工已删除邮件）| 在 matter.md「待决问题」中显著标记，建议律师评估补救措施 |
| 远程/异地诉讼（无法亲自出庭）| 在内部负责人中标记「需要当地代理律师」|
| 配置文件证据保全章节为 `[PLACEHOLDER]` | 使用临时默认范围：「相关业务部门所有与争议相关的文件、邮件、聊天记录」，刷新周期默认 6 个月 |

---

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 律师不提供外部律师联系方式 | 记录「联系方式待补充」，在后续 `/matter-update` 中提示填写 |
| 案件来源文档无法提供 | 记录「尚未共享」，不阻止继续录入 |
| 证据保全范围不明确 | 使用配置文件默认范围，并在 matter.md 中标记「范围需律师确认」|
