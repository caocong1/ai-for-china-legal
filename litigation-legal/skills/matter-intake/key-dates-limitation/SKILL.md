# 关键日期与诉讼时效起算初判

> 引用规范：`litigation-legal/skills/_shared/legal-basis-conventions.md`
> 引用库：`litigation-legal/skills/_shared/civil-procedure-citations.md`（重点：D 节民法典时效 + A.5 答辩/举证期）
> 本子技能可被 `litigation-legal/skills/matter-intake/SKILL.md` 编排入口调用，
> 也可通过 `/litigation-legal:key-dates-limitation` 单独触发。

## 目的

记录案件的关键程序性期限，并对诉讼时效作出初步判断（非深度时效分析）：

1. **关键日期记录**：答辩期限、举证期限、下次听证/庭审日期、监管期限，确保律师不遗漏紧迫截止日
2. **诉讼时效起算初判**：识别案由对应的时效期间（一般3年=《民法典》第188条），初步判断起算点，标记是否临近届满
3. **深度分析路径提示**：时效中止/中断事由分析属于深度任务，本技能仅作初判并提示运行 03-03 的 `limitation-monitoring` 子技能

**本子技能不作出时效结论**——起算点不清时一律标记 `[需律师确认]` 并列出待查事实。

## 步骤一：关键程序性日期

### 1.1 答辩期限

**已核实规则**：普通程序下，被告应于收到起诉状副本之日起**十五日**内提出答辩状；举证期限不得少于**十五日**。（《民事诉讼法》2023修正，具体条号待验证）

收集以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **起诉状/仲裁申请送达日期** | 我方收到的日期 | 2026-04-20 |
| **答辩期限届满日** | 送达日 + 15 日（普通程序）/ 30 日（仲裁，具体规则见各仲裁机构规则 `[待验证]`）| 2026-05-05 |
| **距今天数** | 自动计算 | 如：还有 7 日 |
| **是否已提交答辩状** | 是 / 否 / 不适用 | — |

**紧急提示条件**：距答辩期限届满 ≤ 5 个工作日时，标记 `[紧急 — 答辩期限临近]`。

### 1.2 举证期限

| 字段 | 说明 |
|------|------|
| **法院/仲裁指定的举证期限** | 从开庭通知书或庭前会议确认 |
| **举证截止日期** | 具体日期 YYYY-MM-DD |
| **证据提交状态** | 已全部提交 / 部分提交 / 尚未提交 |

> **重要**：逾期举证的后果为法院一般不予认可（《证据规定》2020修正，条号待验证）。逾期须能说明正当理由。

### 1.3 下次听证/庭审

| 字段 | 说明 |
|------|------|
| **下次听证/庭审日期** | YYYY-MM-DD HH:MM（如已排期）|
| **庭审形式** | 开庭审理 / 书面审理 / 视频庭审 |
| **庭审地点** | 法院/仲裁机构名称 + 地址 |

### 1.4 监管期限（如有）

如案件涉及监管调查通知或行政程序，记录：

| 字段 | 说明 |
|------|------|
| **监管机构** | 名称 |
| **期限类型** | 答复期限 / 材料提交期限 / 其他 |
| **期限届满日期** | YYYY-MM-DD |

### 1.5 诉讼时效截止（初判，见步骤二）

在步骤二完成后填写。

## 步骤二：诉讼时效起算初判

> **纪律**：本步骤作初步判断，不下最终结论。起算点不清时不推断结论，一律标 `[需律师确认]` 并列出待查事实。深度时效分析（中止/中断事由、特殊时效、最长期间）提示运行 `limitation-monitoring` 子技能。

### 2.1 识别案由对应的时效期间

**已核实锚点：一般诉讼时效期间 = 3 年（《民法典》第188条）**

| 时效类型 | 规则 | 来源 | 状态 |
|---------|------|------|------|
| **一般诉讼时效：3 年** | 向人民法院请求保护民事权利的诉讼时效期间为三年 | 《民法典》（2020年）第188条 | **已核实** |
| 特殊时效（如票据纠纷、国际货物买卖等）| 依各专门法律确定，通常为 1 年或 4 年 `[待验证]` | 各专门法律 `[模型知识 — 需验证]` | `[待验证]` |
| **最长权利保护期间：20 年** | 自权利受到损害之日起超过二十年的，法院不予保护 | 《民法典》（2020年）第188条第2款 `[待验证]` | `[待验证]` |

**判断本案适用的时效期间：**

- 一般合同纠纷/侵权/劳动（超出劳动仲裁部分）→ 一般 3 年时效
- 票据纠纷、海事纠纷、国际货物买卖等 → 核查专门法律 `[待验证]`，标记特殊时效
- 如不确定本案案由适用的时效期间 → 标记 `[时效期间待确认 — 需律师核查]`

### 2.2 初判时效起算点

**已核实规则**：时效期间自权利人**知道或者应当知道**权利受到损害以及义务人之日起计算（《民法典》第188条）。

收集以下信息，尝试初判起算点：

| 信息项 | 说明 |
|-------|------|
| **权利受损的事件** | 违约发生日、侵权行为日、债务到期日等 |
| **我方知悉日期** | 何时知道或应当知道权利受损及义务人 |
| **合同履行期限** | 债务约定的履行期限届满日（如有）|
| **最后催告日期** | 如有书面催告，最后一次催告日期 |

**初判起算点规则：**

| 情形 | 初判起算点 |
|------|-----------|
| 有明确违约日期 | 知悉违约之日（通常为违约日当日或合理知悉后）|
| 债务有明确到期日 | 到期日起 |
| 分期履行债务 | 最后一期履行期届满之日 `[待验证]` |
| 未约定履行期限的债务 | 催告后合理期限届满之日 `[待验证]` |
| 连续侵权 | 侵权行为终止之日（争议，标 `[需律师确认]`）|

> **不确定时**：不推断，直接标记 `[起算点不确定 — 需律师确认]` 并列出待查清的关键事实。

### 2.3 计算初步时效届满日

```
初步时效届满日 = 起算日 + 3 年（或对应特殊时效期间）
```

| 时效状态 | 标记 |
|---------|------|
| 距届满 > 1 年 | 常规监控 |
| 距届满 6 个月 ~ 1 年 | `[时效预警 — 距届满不足1年，建议启动时效监控]` |
| 距届满 < 6 个月 | `[时效紧急 — 距届满不足6个月，需立即评估起诉/仲裁必要性]` |
| 已届满（疑似）| `[时效疑似届满 — 需律师立即核查时效中止/中断事由]` |
| 起算点不清 | `[时效起算点不确定 — 需律师确认后方可计算]` |

### 2.4 时效中止/中断事由（初步标记）

本技能**不分析**时效中止/中断事由——该工作属于深度时效分析任务。初判阶段仅询问：

- 是否有书面催告（如有，记录最近一次日期）？
- 是否有起诉或申请仲裁的记录（如有，记录日期）？
- 是否有义务人书面承认债务的文件？

以上任一情形存在，标记 `[可能存在时效中断事由 — 建议运行 limitation-monitoring 深度分析]`。

## 步骤三：深度分析路径提示

如果存在以下任一情形，向律师提示运行深度时效分析：

- 时效距届满 < 1 年
- 案件涉及多个权利，各权利起算点可能不同
- 有疑似时效中止/中断事由
- 适用的时效期间本身不确定（特殊时效）
- 最长权利保护期间（20 年）可能已过

**提示内容：**

> 本次录入完成后，建议运行 `/litigation-legal:limitation-monitoring` 进行深度时效分析，包含：时效期间识别 / 起算点精确判定 / 中止中断事由完整分析 / 监控台账与预警机制建立。

## 检查清单

- [ ] 起诉状/申请送达日期已记录（如适用）
- [ ] 答辩期限届满日已计算（送达日 + 15 日，普通程序）
- [ ] 距答辩期限天数已计算，紧急情形（≤5工作日）已标记
- [ ] 举证截止日期已记录（如已知）
- [ ] 下次庭审日期已记录（如已排期）
- [ ] 监管期限已记录（如有）
- [ ] 案由对应时效期间已识别（一般3年=第188条，或标注特殊时效）
- [ ] 初判起算点已尝试确认，不确定时标记 `[需律师确认]`
- [ ] 初步时效届满日已计算（或标注「起算点待确认」）
- [ ] 时效状态已标记（常规/预警/紧急/疑似届满/起算点不确定）
- [ ] 时效中止/中断初步询问已完成
- [ ] 深度分析路径（limitation-monitoring）已按需提示

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 起算点有争议（对方可能主张不同起算点）| 列出双方可能的起算点，不下单方面结论，标记 `[起算点争议 — 需律师评估]` |
| 案件涉及多个独立权利（如多笔货款）| 逐笔计算时效，不合并处理 |
| 仲裁程序时效规则与诉讼不同 | 仲裁案件时效同样适用民法典，但仲裁申请中断时效规则与起诉相同 `[待验证]` |
| 当事人口述时效期间与民法典不符 | 触发法律事实合理性检查（见 practice-profile-schema.md 规则 2），标记 `[前提 flagged — 需验证]` |
| 案件处于诉前阶段，无送达日期 | 记录「尚未送达」，答辩期限字段为「待定」，重点关注时效起算初判 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 律师无法提供送达日期 | 记录「送达日期待确认」，标记 `[紧急 — 答辩期限需尽快确认]` |
| 时效期间涉及境外法律 | 标记 `[管辖 flagged — 涉外时效，超出中国大陆法律范围]`，不适用民法典第188条 |
| 对方已提出时效抗辩 | 记录时效抗辩存在，标记为高优先级，提示运行 limitation-monitoring 深度分析 |
