# 案件识别与分级

> 引用规范：`litigation-legal/skills/_shared/legal-basis-conventions.md`
> 引用库：`litigation-legal/skills/_shared/civil-procedure-citations.md`
> 本子技能可被 `litigation-legal/skills/matter-intake/SKILL.md` 编排入口调用，
> 也可通过 `/litigation-legal:case-identification` 单独触发。

## 目的

完成案件的基础识别并建立风险全景：确定「这是什么案件、我方是谁、在哪里打」，再对照实践配置文件的风险校准矩阵给出综合风险评级和重大性结论，为后续所有子技能提供共享的案件坐标。

**前置要求**：`subject-conflict-precheck` 子技能须已完成（利益冲突门槛已过）。

## 步骤一：案件识别

### 1.1 基础案件信息

收集以下字段（带 * 为必填，其余可补充）：

| 字段 | 说明 | 要求 |
|------|------|------|
| **案件名称*** | 律师通常引用的案件名称（如「甲公司诉我公司合同纠纷 2026」）| 应能唯一标识案件 |
| **对方当事人*** | 主要对方当事人全称；多方并列时逐一列出 | 包含关联公司 |
| **案件类型*** | 见下方分类 | 从列表选择或组合 |
| **我方角色*** | 见角色确认规则 | 不得默默假设 |
| **管辖机构*** | 见管辖判断流程 | 须区分法院/仲裁 |

**案件类型分类：**

```
合同纠纷 | 劳动争议 | 知识产权 | 侵权 | 公司治理 | 房地产 | 建设工程 |
股权/并购 | 金融/证券 | 破产/重整 | 刑事（刑附民）| 行政 | 仲裁 | 其他
```

### 1.2 我方角色确认（绝不默默假设）

按以下规则从 `litigation-legal/CLAUDE.md` → `## 谁在使用` → `角色倾向` 确认：

| 配置文件角色倾向 | 处理规则 |
|--------------|---------|
| `原告` | 预填角色为「原告/申请人」，向律师确认本案是否一致 |
| `被告` | 预填角色为「被告/被申请人」，向律师确认本案是否一致 |
| `两者 — 默认 X` | 按默认方 X 预填，询问律师本案是否与默认一致 |
| `因案而异` 或 `[PLACEHOLDER]` | 直接询问律师，不预填 |

**角色驱动下游技能的方向：**
- **原告/申请人立场**：风险分级导向案件价值、胜诉概率、代理经济（费用/胜算）
- **被告/被申请人立场**：风险分级导向敞口/准备金/保险理赔/和解阈值

### 1.3 管辖判断流程

按以下链条逐步确认管辖：

```
第一步：是否有有效仲裁协议？
  → 有效仲裁协议 → 仲裁管辖，排除法院管辖（「或裁或审」原则）
  → 无仲裁协议 → 进入法院管辖判断

第二步：确定级别管辖（法院）
  → 参考 civil-procedure-citations.md A.2 级别管辖
  → 注意：具体标的额划分以当地高院级别管辖规定为准

第三步：确定地域管辖
  → 合同纠纷：被告住所地 or 合同履行地 [条号待验证]
  → 侵权纠纷：侵权行为地 or 被告住所地 [条号待验证]
  → 检查是否有协议管辖（须书面约定，且在法定范围内）

第四步：检查专属管辖
  → 不动产 → 不动产所在地（不可协议变更）[条号待验证]
  → 港口作业 → 港口所在地 [条号待验证]

第五步：检查专门法院管辖
  → 海事纠纷 → 海事法院
  → 重大知识产权 → 知识产权法院
  → 特定金融纠纷（上海）→ 金融法院
  → 互联网相关纠纷 → 互联网法院
```

**管辖识别结果格式：**

```
管辖类型：[法院 / 仲裁]
  如为法院：[级别] — [具体法院名称/城市]
  如为仲裁：[仲裁机构名称]（CIETAC / BAC / SHIAC / SCIA / 其他）
管辖依据：[说明为何如此判断，有仲裁协议须说明内容]
异常标记：[管辖 flagged — 需验证管辖权] / 无异常
```

### 1.4 案件来源

案件是怎么来的？从以下选项选择：

```
起诉状送达 | 仲裁申请送达 | 律师函 | 监管调查通知 |
内部举报 | 诉前威胁 | 其他
```

**种子文档机会：** 「如果你有 initiating 文档（起诉状、仲裁申请、律师函），附上或分享路径，能让录入更精确。」

## 步骤二：风险分级

### 2.1 对照实践配置文件校准

从 `litigation-legal/CLAUDE.md` → `## 风险校准` 读取：
- **严重程度定义**：各级别（极高/高/中/低）对应的金额阈值和业务影响描述
- **可能性定义**：各级别（高/中/低）对应的概率区间

> **重要纪律**：如果实践配置文件的风险校准字段为 `[PLACEHOLDER]`（未配置），**不得假装精确**。使用用户直觉评级，并注明「风险校准未配置，评级为用户估计值」。

### 2.2 风险分级维度

| 维度 | 选项 | 判断依据 |
|------|------|---------|
| **严重程度** | 极高 / 高 / 中 / 低 | 对照配置文件严重程度定义，考虑：金融敞口金额、业务连续性影响、声誉风险、刑事责任可能性 |
| **可能性** | 高（>70%）/ 中（30%-70%）/ 低（<30%）| 对照配置文件可能性定义，考虑：证据情况、法律依据清晰度、司法实践倾向 |
| **综合风险评级** | 极高 / 高 / 中 / 低 | 按矩阵计算（见下）|

**风险综合评级矩阵：**

| 严重程度↓ \ 可能性→ | 高 | 中 | 低 |
|--------------------|----|----|-----|
| **极高** | 极高 | 极高 | 高 |
| **高** | 极高 | 高 | 中 |
| **中** | 高 | 中 | 低 |
| **低** | 中 | 低 | 低 |

### 2.3 敞口量化

| 字段 | 说明 |
|------|------|
| **金钱敞口范围** | 最佳估计范围（如 ¥200万–¥500万）；含利息/罚款/费用 |
| **非金钱敞口** | 禁令风险 / 同意令 / 经营许可吊销 / 声誉/媒体 / 先例影响 / 刑事风险 |

## 步骤三：重大性评估

对照 `litigation-legal/CLAUDE.md` → `## 风险校准` → `### 重大性阈值`：

| 重大性级别 | 触发条件 | 后续行动 |
|---------|---------|---------|
| **已计提准备金** | 敞口下限 ≥ 准备金阈值 | 通知财务，确认准备金金额 |
| **已披露** | 敞口下限 ≥ 披露阈值 | 确认披露备案和脚注位置 |
| **仅监控** | 敞口在监控阈值以上 | 纳入组合监控，无需立即准备金/披露 |
| **无** | 低于监控阈值 | 正常录入 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 仲裁协议效力存疑（如合同无效但可能包含仲裁条款）| 标记 `[管辖 flagged — 仲裁协议效力待定]`，提示运行 subject-conflict-precheck 确认仲裁独立性规则 |
| 管辖地不明确（合同无明确管辖约定）| 列出可选管辖地，标记最可能选项，告知律师须结合被告住所地和合同履行地确认 |
| 多方当事人涉及不同司法管辖 | 识别主要法律关系，以主要管辖为准，标记次要管辖问题 |
| 配置文件风险校准薄（含 PLACEHOLDER）| 使用用户直觉估计，注明「风险校准未配置 — 评级为用户估计值」，不假装精确 |
| 案件处于极早期（诉前阶段，无起诉状）| 以现有信息录入，可能性/敞口标记为「初步估计，待起诉状确认」|

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 我方角色仍不明确（律师未选择）| 不允许继续；明确说明角色未确认时无法进行风险分级（因驱动方向相反）|
| 管辖涉及境外/港澳台 | 标记 `[管辖 flagged — 涉外管辖，超出中国大陆法律范围]`，提示律师专项分析 |
| 标的额不明确 | 使用「金额不明」，不猜测；在敞口字段记录「待确认」|
