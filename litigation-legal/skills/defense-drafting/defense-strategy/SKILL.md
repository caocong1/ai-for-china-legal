# 答辩策略选择

> 引用规范：`litigation-legal/skills/_shared/legal-basis-conventions.md`
> 引用库：`litigation-legal/skills/_shared/civil-procedure-citations.md`（重点：A节民诉法管辖/答辩期 + B节仲裁法 + D节时效）
> 本子技能可被 `litigation-legal/skills/defense-drafting/SKILL.md` 编排入口调用，
> 也可通过 `/litigation-legal:defense-strategy` 单独触发。

---

## 目的

根据对方起诉状（或仲裁申请书）及我方案件事实，系统决策答辩路线，为后续子技能（管辖权异议、实体抗辩、程序时效抗辩、答辩状装配）提供策略框架。

**本子技能不起草文书** — 只输出策略决策表和路线优先级。文书起草由对应子技能完成。

---

## 前置读取

执行本子技能前，读取以下文件：

1. **研究闸门**：除非用户明确豁免联网检索，先运行 `shared/research-gate/SKILL.md`，完成研究底稿后再进入本子技能
2. `litigation-legal/skills/_shared/civil-procedure-citations.md` — A节（管辖/答辩期）+ B节（仲裁法）+ D节（民法典时效）
3. `litigation-legal/skills/_shared/legal-basis-conventions.md` — 引用规范
4. `litigation-legal/CLAUDE.md` — 读取 `## 谁在使用 → 角色倾向`（原告/被告/因案而异）+ `## 和解策略`（和解倾向/授权阈值）
5. 对方起诉状或仲裁申请书（用户提供）

**实践配置文件加载规则**：若 `litigation-legal/CLAUDE.md` 仍有 PLACEHOLDER，提示用户运行 `/litigation-legal:cold-start-interview` 或说「临时模式」使用通用默认值。

## 步骤一：案件基本信息核查

收集以下信息，缺失时停下索取（**不推断**）：

| 字段 | 说明 | 状态 |
|------|------|------|
| 当事人双方 | 我方（答辩人）/ 对方（原告/申请人）的名称、法律性质 | [ ] |
| 程序类型 | 诉讼（哪级哪地法院）/ 仲裁（哪家仲裁机构）| [ ] |
| 案由 | 合同纠纷/侵权/劳动/其他 | [ ] |
| 对方起诉状/申请书 | 已收到 / 未收到（**未收到时不能进入策略决策**）| [ ] |
| 答辩期限截止日 | 送达日 + 15 日（普通程序）或仲裁规则约定期限 `[待验证]` | [ ] |
| 我方基本事实立场 | 对方主张哪些，我方基本认可或否认 | [ ] |

> **硬停止条件**：未收到起诉状/仲裁申请时，必须停下：「尚未收到对方起诉状，无法进行答辩策略分析。请在收到后再触发本子技能。」

## 步骤三：答辩路线决策

### 3.1 仲裁场景特殊识别（优先判断）

> **重要**：仲裁案件须先识别仲裁协议有效性与或裁或审原则，避免错误提管辖权异议。

| 检查项 | 结论 | 备注 |
|--------|------|------|
| 是否存在有效仲裁协议？| [是/否/存疑] | 仲裁协议须具备：请求仲裁意思表示 + 仲裁事项 + 选定仲裁委员会 `[待验证]` |
| 仲裁协议是否独立有效？| [是/否] | 合同无效不影响仲裁协议效力（仲裁协议独立性原则）`[待验证]` |
| 是否为一裁终局程序？| [是/否] | 仲裁裁决终局，不得向法院另行起诉 `[待验证]` |
| 仲裁案件是否可提管辖异议？| 仲裁案件异议路径为：仲裁管辖权异议/仲裁协议效力异议，不同于法院管辖权异议 `[待验证]` | — |

### 3.2 程序抗辩路线评估

| 抗辩类型 | 可行性 | 说明 | 优先级 |
|---------|--------|------|--------|
| **管辖权异议** | [高/中/低/不适用] | 级别管辖是否有问题？地域管辖是否有争议？是否有专属管辖？约定管辖是否有效？| [高/中/低] |
| **诉讼时效抗辩** | [高/中/低/不适用] | 对方权利是否可能超过3年时效期间？起算点如何认定？ | [高/中/低] |
| **主体不适格** | [高/中/低/不适用] | 原告是否与本案有直接利害关系？被告主体是否准确？ | [高/中/低] |
| **重复起诉** | [高/中/低/不适用] | 是否存在已有生效判决/裁决的同一争议？ | [高/中/低] |
| **前置程序未履行** | [高/中/低/不适用] | 是否须经劳动仲裁、行政复议等前置程序？ | [高/中/低] |

> **时效抗辩特别提示**：时效抗辩须由当事人主动提出，法院不主动适用。一旦错过提出时机，即视为放弃时效利益，不可恢复。请在答辩阶段即明确是否主张。

### 3.3 实体答辩路线决策（逐请求分析）

针对步骤二中每项诉讼请求，决策实体答辩路线：

| 序号 | 对方请求 | 承认/否认/抗辩 | 核心抗辩理由 | 证据基础 |
|------|---------|---------------|------------|---------|
| 1 | [请求] | [承认 / 否认 / 部分承认] | [如：请求权基础不成立；合同无效；已履行；违约金过高] | [现有证据支撑度] |
| N | ... | ... | ... | ... |

**实体抗辩路线说明**：

- **承认**：对方事实与法律主张均正确，考虑和解
- **否认**：对方事实不实，或法律定性错误
- **抗辩**：承认基础事实，但提出抵消、豁免、减轻责任的事由（如履行抗辩、同时履行抗辩、不安抗辩、情势变更等）`[待验证]`

### 3.4 反诉评估

| 评估项 | 结论 |
|--------|------|
| 是否有对对方的独立权利主张？| [是/否] |
| 反诉请求具体内容 | [如有] |
| 反诉与本诉是否基于同一法律关系或可以合并审理？`[待验证]` | [是/否] |
| 反诉提出时机（须在一审答辩期届满前 `[待验证]`） | [是否在期限内] |
| 反诉胜算评估 | [高/中/低] |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 仲裁案件误提法院管辖权异议 | 标记错误：仲裁案件应提仲裁管辖权异议，不适用民诉法管辖权异议程序，引导至正确路径 |
| 尚未收到对方起诉状 | 硬停止，等待收到后再触发 |
| 管辖权异议与实体答辩同时需要 | 提示管辖权异议优先（须在答辩期内），但应同时准备实体答辩状（万一异议被驳回）|
| 对方请求存在明显超诉讼时效情形 | 立即标记 `[时效抗辩 — 须主动提出]`，提示触发 procedural-limitation-defense |
| 案件事实复杂，无法初步判断策略路线 | 拆解确定项与不确定项，对不确定项标记 `[需律师确认]`，不推断 |
| 反诉需要进一步研究 | 标记反诉待评估，提示收集相关证据和法律依据后再决策 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 配置文件缺失/仍为PLACEHOLDER | 提示运行 `/litigation-legal:cold-start-interview` 或说「临时模式」|
| 律师提供的事实与起诉状不一致 | 标记不一致点，提示「以下事实与起诉状陈述存在差异，请确认」，不自行选择 |
| 研究闸门运行失败 | 标记 `[研究闸门未完成 — 以下策略分析基于已知信息，建议验证法律依据]` |
