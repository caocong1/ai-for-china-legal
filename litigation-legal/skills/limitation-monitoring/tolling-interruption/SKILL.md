# 中止与中断事由分析

> **与 matter-intake/key-dates-limitation 的分工**：`matter-intake/key-dates-limitation` 仅作初步询问（是否有催款函/起诉记录），提示可能存在中断事由。本子技能完成**完整的中止中断事由分析**：确认事由类型、分析法律后果、重新计算届满日。
>
> **单独触发说明**：本子技能可通过 `/litigation-legal:tolling-interruption` 单独触发。

## 目的

分析本案诉讼时效的中止和中断事由，准确计算经中止/中断调整后的时效届满日：

1. **区分中止与中断**：明确两者法律后果的根本差异
2. **识别本案事由**：收集并评估所有可能的中止/中断事由及其证据
3. **计算调整后届满日**：根据中止/中断事由重新计算实际届满日
4. **举证风险提示**：识别事由证据不足时的举证风险

## 步骤一：明确中止与中断的法律后果区别

> **关键纪律**：中止和中断是两种完全不同的法律制度，法律后果根本不同，不可混淆。

### 1.1 诉讼时效中止（tolling）

**法律规则**（《民法典》第194条，条号 `[待验证]`）：

| 要素 | 内容 |
|------|------|
| **触发条件** | 在时效期间**最后六个月内**，因以下障碍不能行使请求权：（1）不可抗力；（2）权利人为无/限制民事行为能力人且无法定代理人；（3）继承开始后未确定继承人或遗产管理人；（4）权利人被义务人控制；（5）其他障碍 `[待验证]` |
| **时间要求** | 障碍须发生在时效期间最后六个月内（若早于最后六个月，不构成中止）|
| **法律后果** | 障碍消除后，时效期间**继续计算**（不归零，只是暂停）；剩余时间不足六个月的，延长至六个月 |
| **计算方式** | 时效届满日 = 原届满日 + 中止期间（障碍持续时长）|

**中止事由验证要点**：

- 障碍发生时点：是否在时效最后六个月内？（必须）
- 障碍性质：是否属于法定障碍类型？
- 障碍消除时点：何时恢复行使权利的能力？
- 证据：不可抗力证明、行为能力状态文件等

### 1.2 诉讼时效中断（interruption）

**法律规则**（《民法典》第195条，条号 `[待验证]`）：

| 要素 | 内容 |
|------|------|
| **触发事由** | （一）权利人向义务人提出履行请求（如：催款函、发函要求赔偿等）；（二）义务人同意履行义务（如：还款承诺、对账确认）；（三）权利人提起诉讼或申请仲裁；（四）与提起诉讼或申请仲裁具有同等效力的其他情形 `[待验证]` |
| **法律后果** | 时效期间**重新起算**（**归零，从头开始计算完整的时效期间**）|
| **计算方式** | 新的时效届满日 = 中断事由发生日 + [适用时效期间（3年或其他）] |
| **多次中断** | 如有多次中断事由，以**最后一次中断事由发生日**为起算点，重新计算 |

**中断事由验证要点**：

| 事由类型 | 证据要求 | 注意事项 |
|---------|---------|---------|
| 发出催款函/要求履行的函件 | 函件原件 + 送达证明（快递面单/签收回执/公证送达）| 须有证据证明送达；电子催告须证明对方收到 |
| 义务人书面承诺履行 | 还款承诺函、对账确认函、债务确认书 | 须明确确认债务存在及义务人身份 |
| 起诉或申请仲裁 | 起诉状副本 + 法院受理通知 / 仲裁申请书 + 受案通知 | 提起诉讼/仲裁本身即发生中断效力，无需等待判决 |
| 对账单/月结确认 | 双方签署的对账单 | 签署日为中断日 |
| 微信/邮件催款 | 完整的电子数据（含时间戳 + 对方已读/回复记录）| 须符合电子证据认定要求，公证保全更有力 |

## 步骤二：识别本案中止/中断事由

### 2.1 收集可能的事由

向律师逐一询问（或从事项工作区读取）：

**催款类（中断）**：
- 是否向义务人发出过书面催款函/律师函/要求履行通知？
- 如有，最近一次是何时（YYYY-MM-DD）？有无送达证明？

**对方确认类（中断）**：
- 义务人是否曾书面承认债务/承诺还款？
- 如有，日期和形式？（书面/微信/邮件）

**诉讼/仲裁类（中断）**：
- 此前是否就同一权利提起过诉讼或申请过仲裁？
- 如有，提起日期？结果如何（撤案/判决/未结）？

**其他同等效力行为（中断）** `[待验证]`：
- 是否有公证发函或其他正式通知行为？

**不可抗力等（中止）**：
- 是否有不可抗力或其他法定障碍，且发生在时效最后六个月内？

### 2.2 事由分析与证据评估

| 事由序号 | 事由类型 | 事由描述 | 发生日 | 证据状态 | 中止/中断 |
|---------|---------|---------|-------|---------|---------|
| 1 | [催款函] | [第X次催款函] | YYYY-MM-DD | [有/无/不完整] | [中断] |
| 2 | [对方承认] | [对方签字对账确认单] | YYYY-MM-DD | [有书面证据] | [中断] |
| 3 | [不可抗力] | [如有] | YYYY-MM-DD | [有/无] | [中止] |

## 步骤三：计算调整后的时效届满日

### 3.1 中断后的重新计算

```
有效最后中断事由发生日：YYYY-MM-DD
调整后时效届满日 = 最后中断日 + [时效期间]
```

示例：
```
初始起算日：2020-01-15
一般时效：3年 → 初步届满日：2023-01-15
中断事由：2021-08-10 发出催款函（有送达证明）
  → 中断后重新起算：2021-08-10
  → 调整后届满日：2024-08-10（距今[X]天）
```

### 3.2 中止后的计算

```
中止开始日：YYYY-MM-DD（须在时效最后6个月内）
中止结束日：YYYY-MM-DD
中止期间：[X]天
调整后时效届满日 = 原届满日 + 中止期间
```

### 3.3 多次中断的处理

若存在多次中断事由，按以下规则：

1. 逐一列出所有中断事由及日期
2. 以**最后一次有效中断**为新起算日
3. 中断事由有效性须证明（送达证明/对方确认书等）

| 中断序号 | 中断日 | 证据强度 | 有效性评估 |
|---------|-------|---------|---------|
| 第1次中断 | YYYY-MM-DD | 强（公证送达）| 有效 |
| 第2次中断 | YYYY-MM-DD | 弱（微信，无回复）| `[有效性存疑 — 需律师评估]` |
| 最后一次有效中断 | YYYY-MM-DD | [强/中/弱] | — |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 中断证据存在（如：微信催款），但形式有争议 | 评估证据可信度等级，标注「[证据效力存疑 — 建议补充公证保全]」|
| 不可抗力发生但不在时效最后六个月内 | 不构成中止；说明「不可抗力发生于时效最后六个月之前，不产生中止效力」|
| 义务人对中断事由有异议（如：否认收到催款函）| 标注争议，提示我方须举证送达事实 |
| 时效可能已届满但存在中断事由 | 重点核查中断事由有效性，标注「[时效疑似届满，但存在中断事由——需律师紧急核查]」`[紧急]` |

---

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 律师混淆中止和中断（说「中断了但不用从头算」）| 纠正：「中断导致时效重新起算（归零重算），区别于中止（继续计算剩余时间），请确认您指的是哪种情形」 |
| 无法提供催款函送达证明 | 标注「[中断事由有效性存疑 — 无送达证明：催款函是否实际到达对方无法证明，存在时效届满风险]」 |
