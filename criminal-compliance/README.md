# criminal-compliance

刑事合规 AI 插件——五大模块二十个子技能，规则基于刑法（单位犯罪定义第30条/双罚制第31条/自首第67条/立功第68条/缓刑第72条）与刑事诉讼法（不起诉第177条/认罪认罚从宽第15条）。具体罪名条号一律 `[条号待验证]`，涉案企业合规试点及第三方监督评估机制规范性文件快速演进，引用时描述规则并标 `[待验证]`。

## 模块与子技能

### 1. 冷启动访谈 `/criminal-compliance:cold-start-interview`

初始化刑事合规实践配置档案（`criminal-compliance/CLAUDE.md`），顺序调用四个访谈子技能：

- **`/criminal-compliance:identity-enterprise-profile`** — 采集执业角色（律师/合规官/法务/当事人本人）、服务对象与企业画像（行业·规模·业务模式·是否涉外·历史涉案情况·重点刑事风险领域）、可用集成资源（检察/公安/监管/第三方监督评估）
- **`/criminal-compliance:compliance-stance-scope`** — 配置整体服务立场（事前刑事风险评估/合规体系建设/涉案企业合规不起诉/单位犯罪预防/单位犯罪辩护研究）、合规严格度与是否已涉案字段（直接驱动涉刑强制升级护栏）
- **`/criminal-compliance:risk-escalation`** — 配置刑事风险等级定义（🔴关键/🟠高/🟡中等/🟢低）、角色权限表、六条强制升级触发条件与涉案应对合规边界
- **`/criminal-compliance:style-regulatory-resources`** — 配置文书风格偏好（刑事风险评估报告/合规制度/合规不起诉申请/辩护研究）与地方涉案企业合规试点口径

### 2. 刑事风险评估 `/criminal-compliance:risk-assessment`

编排四个子技能，全面梳理企业刑事合规风险敞口：

- **`/criminal-compliance:industry-crime-identification`** — 识别企业所在行业高发刑事罪名，按税收/金融/商业贿赂/职务/环境安全/扰乱市场秩序等类别梳理构成要件，结合行业监管特点与行刑衔接路径，条号一律 `[条号待验证]`
- **`/criminal-compliance:position-conduct-risk`** — 分析关键岗位（法定代表人/财务/采购/销售/安全生产等）日常行为风险点，识别单位犯罪双罚制风险（刑法第30条单位犯罪定义/第31条双罚制，建议复核）
- **`/criminal-compliance:risk-level-likelihood`** — 综合评估各风险点等级与现实化可能性，结合已涉案状态（已涉案强制升级刑事辩护律师）、合规控制现状、监管执法趋势，输出风险优先级排序
- **`/criminal-compliance:risk-list-control-output`** — 整合评估结论，输出结构化风险清单（风险点/罪名构成要件描述/等级/控制建议），数额标准描述规则 `[待验证]`，不硬写裸数额

### 3. 合规不起诉方案 `/criminal-compliance:non-prosecution`

编排四个子技能，协助已被追诉企业争取检察机关不起诉处理（须律师主导）：

- **`/criminal-compliance:applicability-assessment`** — 评估企业是否满足涉案企业合规不起诉适用条件（认罪认罚/合规整改可行性/适用罪名范围），结合刑诉法第177条酌定相对不起诉与认罪认罚从宽第15条（建议复核），涉案企业合规规范性文件快速演进标 `[待验证]`
- **`/criminal-compliance:compliance-plan`** — 制定针对已被追诉具体问题的涉案专项合规整改计划（事后整改），整改方案须可验证、真实客观，须律师主导，有效合规认定标准 `[待验证]`
- **`/criminal-compliance:third-party-monitoring`** — 配合企业合规第三方监督评估机制（管理委员会统筹/评估组织具体负责），梳理考察期内合规整改进度，准备评估所需材料；第三方评估机制规范性文件快速演进，须核实现行版本 `[待验证]`
- **`/criminal-compliance:application-hearing`** — 协助起草合规不起诉申请书与拟不起诉公开听证材料，结合认罪认罚/自首（第67条建议复核）/立功（第68条建议复核）/合规整改从宽情节，须执业刑事律师审核签署

### 4. 单位犯罪预防 `/criminal-compliance:corporate-crime`

编排四个子技能，防范单位犯罪构成与直接负责主管人员个人刑事责任：

- **`/criminal-compliance:unit-crime-constitution`** — 分析单位犯罪构成要件（刑法第30条单位犯罪定义/第31条双罚制，建议复核），识别单位犯罪与自然人犯罪的区分标准（单位名义/单位利益/违法所得归单位），梳理直接负责主管人员与其他直接责任人员认定规则（司法解释 `[待验证]`）
- **`/criminal-compliance:common-unit-crime-types`** — 梳理高发单位犯罪类型（商业贿赂/虚开增值税专用发票/非法集资/职务侵占/挪用资金/逃税/串通投标等），以构成要件描述呈现各罪名单位犯罪适用性，条号一律 `[条号待验证]`，数额标准描述规则 `[待验证]`
- **`/criminal-compliance:internal-control-prevention`** — 设计针对单位犯罪构成要件的内部控制与防范机制（关键决策审批/双签制度/异常交易监控/合规审批节点/高管授权边界），结合三重一大决策制度，防范直接负责主管人员刑事责任风险
- **`/criminal-compliance:incident-response`** — 涉案应对研究（已面临单位犯罪追诉），分析认罪认罚/自首/立功/合规整改从宽情节，须执业刑事辩护律师主导；绝不提供规避侦查/伪造证据/串供方法

### 5. 刑事合规体系建设 `/criminal-compliance:compliance-system-building`

编排四个子技能，建设事前预防型刑事合规体系（与模块三涉案后整改互补）：

- **`/criminal-compliance:compliance-risk-identification`** — 从业务/岗位/区域/关联方四维度全面梳理刑事合规风险，构建合规风险地图（重点领域/风险敞口/优先级），具体罪名构成要件描述+条号 `[待验证]`
- **`/criminal-compliance:policy-process-design`** — 制定合规政策与行为准则（Tone at the Top）、重点领域专项合规制度（反商业贿赂/反垄断/数据/安全生产/财税）、合规审批与禁止性清单，制度须律师审查
- **`/criminal-compliance:organization-training`** — 设计合规组织架构（首席合规官/合规部门独立性/业务合规员）、三道防线职责分工、分层分类培训计划（高管/业务岗/全员）与合规考核问责，有效合规组织保障要点 `[待验证]`
- **`/criminal-compliance:effectiveness-evaluation`** — 建立合规体系有效性评估指标（制度健全性/执行有效性/监测发现能力/响应整改能力）、定期审计与自查计划、问题整改闭环管理，有效合规衔接合规不起诉的积极作用（认定标准规范性文件快速演进，描述规则 `[待验证]`）

## 实践配置文件与冷启动流程

所有技能从 `criminal-compliance/CLAUDE.md` 读取实践配置，包括服务立场、企业画像、风险校准、升级矩阵、文书风格和监管资源。首次使用前运行：

```bash
/criminal-compliance:cold-start-interview
```

访谈完成后结果填入 `criminal-compliance/CLAUDE.md`，后续可直接编辑该文件或只更新某章节：

```bash
# 只更新合规立场（如服务事项从事前预防转为涉案应对）
/criminal-compliance:compliance-stance-scope

# 只更新风险升级矩阵配置
/criminal-compliance:risk-escalation
```

配置文件字段契约见 `criminal-compliance/skills/_shared/`。

## 核心护栏

刑事合规事项高度敏感，以下护栏**凌驾于服务立场与合规严格度之上**，任何配置不得覆盖：

| 情形 | 处理 |
|------|------|
| 企业或相关人员已被立案（公安或检察机关） | 立即强制升级执业刑事辩护律师 → 输出标 `[强制升级 — 涉刑情形]` |
| 被采取强制措施（拘留/逮捕/取保候审/监视居住） | 立即强制升级执业刑事辩护律师 |
| 已收到侦查传唤/询问通知 | 立即强制升级执业刑事辩护律师 |
| 可能构成具体罪名或重大涉案 | 暂停常规分析 → 强制升级 |
| 识别到规避侦查/证据干扰/串供/资产转移需求 | 拒绝 → 输出标 `[合规边界 — 拒绝]` |
| 非律师角色向检察/公安提交材料前 | 暂停 → 须执业刑事律师审核确认 |
| 合规不起诉申请定稿/具结书签署前 | 须执业刑事律师在场 |

## 法律依据

- **刑法**：第30条（单位犯罪定义）、第31条（双罚制）、第67条（自首）、第68条（立功）、第72条（缓刑）；具体罪名条号一律 `[条号待验证]`
- **刑事诉讼法**：第177条（不起诉：法定/酌定/存疑）、第15条（认罪认罚从宽）（建议复核现行版本）
- **涉案企业合规试点/第三方监督评估机制（2021年起）**：属规范性文件，快速演进，本插件描述规则 `[待验证]`，须通过最高人民检察院官网（https://www.spp.gov.cn）核实现行版本
- **行业监管规章（行刑衔接）**：税务/市场监管/证监/药监/生态环境/应急管理/网信办，具体移送标准 `[待验证]`
- **最高人民法院/最高人民检察院刑事司法解释**：具体数额标准与罪名认定规则快速演进，一律 `[待验证]`
