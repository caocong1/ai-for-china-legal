# 合规有效性评估与整改 — 刑事合规体系建设第四步

## 目的

本子技能服务于**事前合规体系建设**立场。合规制度建立、组织搭建、培训落实后，需要持续评估合规体系的实际有效性，并构建整改闭环，防止合规体系沦为「纸面合规」。本子技能负责：

1. **合规体系有效性评估指标** — 从制度健全性、执行有效性、监测发现能力、响应整改能力四个维度构建评估框架
2. **定期审计与自查** — 合规审计计划、专项检查、自查自纠的制度化安排
3. **问题整改闭环** — 发现问题→整改方案→执行整改→验证整改→复盘改进的完整闭环
4. **有效合规衔接合规不起诉** — 明确事前有效合规对涉案后争取合规不起诉的积极作用

> **分工说明：本子技能 = 事前合规体系有效性评估（事前预防）；non-prosecution/compliance-plan = 涉案后专项合规整改计划（涉案应对）。两者互补**：
> - 本子技能：企业尚未涉案，评估合规体系是否真正有效运行
> - `compliance-plan`：企业已被追诉，制定专项合规整改计划以争取合规不起诉（须律师主导）
> - **衔接点**：事前建立有效合规体系、留存合规有效性评估记录，是涉案后第三方监督评估「有效合规」认定的重要基础，但有效合规的具体认定标准属于规范性文件，快速演进，须核实现行版本 `[待验证 — 规范性文件快速演进，须核实现行版本]`

**可独立触发**：本子技能可由合规官/法务/律师单独调用（`/criminal-compliance:effectiveness-evaluation`），也可作为 `compliance-system-building` 编排入口调用链的第四步。

## 详细步骤

### 步骤一：建立合规体系有效性评估框架

**四维度评估框架：**

有效合规不等于形式合规——须从以下四个维度综合评估：

| 评估维度 | 核心问题 | 评估方法 |
|---------|---------|---------|
| **制度健全性** | 合规制度体系是否完整、合理、可操作？ | 制度文件审查；与行业最佳实践对标 |
| **执行有效性** | 合规制度是否真正被执行？员工是否知悉并遵守？ | 访谈、问卷、案例测试；合规审批记录检查；培训记录核查 |
| **监测发现能力** | 合规体系是否能有效发现违规行为？举报机制是否有效运转？ | 举报案例统计；内审发现率；合规自查完成情况 |
| **响应整改能力** | 发现违规后是否能快速响应、有效整改、防止再发？ | 整改闭环完成率；同类问题再发率；整改时效 |

---

### 步骤二：合规有效性评估指标体系

**制度健全性指标：**

| 指标 | 评估标准 | 评估方法 |
|-----|---------|---------|
| 合规政策完整性 | 是否有合规总体政策 + 行为准则 + 各重点领域专项制度 | 制度目录审查 |
| 制度覆盖范围 | 是否覆盖企业全部高风险业务领域（依合规风险地图） | 风险地图 vs 制度覆盖对照 |
| 禁止性清单 | 是否包含明确的刑事红线禁止性行为清单 | 制度文本审查 |
| 制度可落地性 | 操作规程是否具体、员工是否理解 | 访谈抽查 |
| 制度更新及时性 | 制度是否随法规变化（刑法修正案/司法解释 `[待验证]`）及时更新 | 制度版本记录 |

**执行有效性指标：**

| 指标 | 评估标准 | 评估方法 |
|-----|---------|---------|
| 合规培训完成率 | 年度培训完成率目标（如 95% 以上） | 培训记录系统 |
| 合规测验通过率 | 各层次测验通过率 | 培训记录系统 |
| 合规审批执行率 | 关键节点（礼品娱乐/顾问协议/三重一大）合规审批完成率 | 审批记录抽查 |
| 行为准则知悉率 | 全员签署《行为准则知悉确认书》比例 | HR记录 |
| 高风险岗位合规测试通过率 | 场景测试中合规决策正确率 | 情景测试结果 |

**监测发现能力指标：**

| 指标 | 评估标准 | 评估方法 |
|-----|---------|---------|
| 举报渠道使用率 | 年度举报数量（过低可能说明举报文化不健全，过高需分析原因） | 举报记录 |
| 举报处理及时性 | 举报接收后 [N] 工作日内回应比例 | 举报处理记录 |
| 内审合规发现率 | 每次内审发现的合规问题数量与严重程度 | 内审报告 |
| 合规数据监控指标 | 是否建立财务/费用/交易的异常监控指标（如大额现金/单次费用超限预警） | 监控系统检查 |
| 自查自纠质量 | 业务部门自查发现的问题与上级复查发现的问题比率 | 自查与抽查结果对比 |

**响应整改能力指标：**

| 指标 | 评估标准 | 评估方法 |
|-----|---------|---------|
| 违规事件响应及时性 | 发现违规后 [N] 工作日内启动调查的比例 | 调查启动记录 |
| 整改完成率 | 合规审计/自查发现问题的整改按时完成率 | 整改台账 |
| 同类问题再发率 | 同类违规在整改后12个月内再发生率 | 违规记录追踪 |
| 根因分析质量 | 整改方案是否针对违规根因（制度缺失/人员认知/流程漏洞/管理层授意） | 整改报告审查 |
| 整改验证有效性 | 整改措施实施后是否经独立验证（非自我宣称） | 内审验证记录 |

### 步骤三：定期审计与自查

**合规审计计划（年度）：**

| 审计类型 | 主体 | 频次 | 覆盖范围 |
|---------|------|------|---------|
| 全面合规审计 | 内审部门（第三道防线）| 每年1次 | 合规体系整体有效性评估；覆盖所有重点合规领域 |
| 专项合规审计 | 内审部门 + 合规部门 | 半年1次（针对高风险领域）| 反商业贿赂/税收合规/安全生产/数据合规等单一领域深度审计 |
| 日常合规监控 | 合规部门（第二道防线）| 持续 | 合规审批合规性检查；费用报销合规性抽查；异常交易监控 |
| 业务自查 | 各业务部门（第一道防线）| 季度 | 本部门合规政策执行情况自查 |
| 外部合规评估 | 外部律师/合规顾问 | 每年1-2次（视需要）| 独立评估合规体系的完整性与有效性；提供外部视角 |

**合规审计工作流程：**

```
审计计划 → 审计前准备（文件清单/访谈安排）→ 现场审计（文件审查/访谈/测试）
→ 审计发现整理 → 草稿审计报告 → 被审计部门确认 → 最终审计报告
→ 整改方案制定 → 合规委员会审议 → 整改执行监督 → 整改验证 → 闭环
```

**自查自纠机制（各业务部门季度自查）：**

| 自查要素 | 内容 |
|---------|------|
| 自查清单 | 合规部门制定标准化自查清单（覆盖本部门主要合规风险点）|
| 自查执行 | 业务合规员组织本部门按清单自查；完成情况填报系统 |
| 问题上报 | 发现问题或疑似违规，立即上报合规部门（不得自行处置和隐瞒） |
| 自查记录 | 自查结果保存记录，供内审部门抽查验证 |

### 步骤四：问题整改闭环

**整改闭环管理流程：**

| 阶段 | 内容 | 责任主体 | 时限 |
|-----|------|---------|------|
| **发现** | 通过审计/自查/举报/外部监管发现合规问题 | 发现主体（内审/合规部门/业务部门） | 发现后 [N] 工作日内报告 |
| **定性** | 评估违规的严重程度（一般/较重/严重）；识别是否涉及刑事风险 | 合规部门 + 法律部门 | [N] 工作日内完成定性 |
| **根因分析** | 分析违规的根本原因（制度缺失/人员认知/流程漏洞/管理层因素） | 合规部门 + 内审部门 | 定性后 [N] 工作日内 |
| **整改方案** | 制定针对根因的整改措施（制度修订/培训加强/流程优化/人员问责） | 责任部门 + 合规部门 | 根因分析后 [N] 工作日内提交方案 |
| **整改执行** | 按方案执行整改措施；过程记录留档 | 责任部门 | 按整改方案时间表 |
| **整改验证** | 由内审部门或合规部门独立验证整改措施是否有效落实 | 内审部门/合规部门 | 整改执行完毕后 [N] 工作日内 |
| **复盘改进** | 系统性分析同类问题，改进合规体系（制度/流程/培训） | 合规部门 | 验证后纳入年度合规报告 |
| **闭环确认** | 合规委员会确认整改完毕，关闭整改台账条目 | 合规委员会/首席合规官 | 验证通过后 |

**整改严重程度分级处理：**

| 严重程度 | 判断标准 | 处理方式 |
|---------|---------|---------|
| 一般问题 | 操作规程未遵守；合规审批漏填；培训未参加 | 合规部门内部处理；整改台账记录 |
| 较重问题 | 违反禁止性清单但未达刑事程度；合规审批实质性缺失 | 合规委员会知悉；责任人问责；制度/流程专项整改 |
| 严重问题 | 可能构成刑事犯罪的行为；重大违规；管理层授意违规 | 立即触发 `[强制升级 — 涉刑情形]`；律师介入内部调查；合规委员会/董事会报告；评估是否向司法机关报告 |

### 步骤五：有效合规衔接合规不起诉

> **核心说明（硬性要求）**：有效合规是涉案企业争取合规不起诉/从宽处罚的重要因素，但**有效合规的具体认定标准属于规范性文件，快速演进，一律描述规则并标 `[待验证 — 规范性文件快速演进，须核实现行版本]`，不硬写文号、认定标准具体条款、认定流程细节**。

**事前有效合规体系对涉案企业合规不起诉的积极作用（描述规则，待验证）：**

涉案企业合规改革相关规范性文件和第三方监督评估机制相关规定（规则描述，须核实现行版本 `[待验证]`）表明，企业在案发前已建立有效的合规管理体系，是考察企业合规整改可信度的重要背景。具体体现在：

1. **合规体系完整性**：制度、组织、培训、评估四位一体的合规体系存在，说明企业有合规预防意识和能力（规范性文件 `[待验证]`）
2. **合规记录完备性**：审计报告、自查记录、培训签到、整改台账等合规记录，是合规体系实质运行的证明材料（规范性文件 `[待验证]`）
3. **案发风险隔离**：已建立合规体系且留有记录，有助于证明企业对违规行为已采取预防措施，违规系部分人员个人行为而非单位决策，可能影响单位犯罪认定（须结合案情，由律师评估 `[待验证]`）
4. **第三方评估认可**：第三方监督评估机制相关规定将考察企业合规整改的真实性与有效性，事前完善的合规体系是可信整改的基础（规范性文件 `[待验证]`）

> **重要说明**：上述分析为规则描述，有效合规认定的具体标准、认定程序、第三方评估的具体评估项目，均属规范性文件范畴，快速演进。**任何具体认定事项须在律师指导下，通过最高检官网（https://www.spp.gov.cn）核实现行版本。**

**路由提示**：
- 尚未涉案（事前预防）：继续完善合规体系有效性评估，保存合规记录
- 已涉案：立即触发 `[强制升级 — 涉刑情形]`，联系执业刑事辩护律师，并结合 `non-prosecution` 技能评估合规不起诉适用性

## 检查清单

- [ ] 四维度评估框架明确（制度健全性/执行有效性/监测发现能力/响应整改能力）

- [ ] 年度合规审计计划已制定（全面审计/专项审计/日常监控/业务自查频次）

- [ ] 整改严重程度分级处理机制已建立，严重问题触发涉刑强制升级
- [ ] 有效合规与合规不起诉的衔接已以描述规则方式呈现，并标 `[待验证]`
- [ ] **有效合规认定标准未硬写文号、未写具体条款编号**（规范性文件快速演进）

| 边界情形 | 处理方式 |

| 有效性的客观衡量困难（定性指标多、数据难量化） | 建议建立基准线（第一次评估作为基准），后续评估与基准对比；同时采用访谈、观察等定性方法补充 |
| 纸面合规 vs 实质合规（制度完备但执行形同虚设） | 访谈+现场测试+案例测验是识别纸面合规的有效方式；访谈须随机抽样，不仅访谈合规部门人员 |
| 评估独立性不足（自查报告美化） | 强调第三道防线（内审）的独立评估；必要时引入外部律师/合规顾问的独立评估 |
| 企业发现重大违规但不想上报 | 提示：隐瞒重大违规可能加重刑事风险（妨害调查等）；合规的正道是依法整改；触发 `[强制升级 — 涉刑情形]`评估 |
| 涉案后要求本子技能出具「有效合规证明」 | 提示：涉案后有效合规评估须由第三方监督评估机制（规范性文件 `[待验证]`）认定，而非本子技能；立即 `[强制升级 — 涉刑情形]`路由 `non-prosecution` |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 识别到已涉案情形 | 立即触发 `[强制升级 — 涉刑情形]`；提示涉案后有效合规评估须在律师与第三方监督评估机构指导下进行；路由 `non-prosecution` |
| 用户要求以「合规评估」名义整理规避侦查的证据处理方案 | 拒绝，标 `[合规边界 — 拒绝]`；提示合规评估的目的是改进合规体系，绝不用于规避侦查 |
| 用户要求提供「有效合规」认定的文号或具体标准细节 | 提示：有效合规认定标准属规范性文件，快速演进，不硬写文号；建议通过最高检官网核实现行版本；标 `[待验证 — 规范性文件快速演进]` |
| 合规审计发现严重问题（可能触及刑事追诉门槛） | 立即识别为潜在涉刑情形，触发强制升级；律师介入内部调查；在律师指导下决定后续处置 |
| 评估报告作为外部证明使用 | 提示：评估报告是否向外部提交（如检察机关）须在律师指导下决定，直接提交可能作为证据使用 |
