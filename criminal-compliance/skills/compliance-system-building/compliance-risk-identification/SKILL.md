# 合规风险识别 — 刑事合规体系建设第一步

## 目的

本子技能服务于**事前合规体系建设**立场。与 `risk-assessment`（面向单次、事件触发的刑事风险评估）不同，本子技能的目标是：

1. 从**业务、岗位、区域、关联方**四个维度，系统梳理企业全面刑事合规风险底图
2. 构建企业专属**合规风险地图**（领域 × 风险点 × 风险敞口 × 优先级）
3. 确定**重点合规领域**，为后续合规制度与流程设计、组织培训、有效性评估提供输入

> **与 risk-assessment 的分工说明**：
> - `risk-assessment` 面向单次刑事风险评估：输入特定行为或事件，输出该行为的刑事风险等级与控制建议，聚焦「此事/此行为是否触发刑事追诉风险」
> - 本子技能（compliance-risk-identification）面向**合规体系建设**：输入企业全貌（行业、业务、岗位、区域、关联方），输出企业合规风险底图与重点领域清单，聚焦「哪些领域最需要建立制度、流程与培训」
> - 两者互补：本子技能产出的风险地图可作为 `risk-assessment` 各次评估的背景地图；`risk-assessment` 的评估结论可更新、丰富本子技能的风险地图

**可独立触发**：本子技能可由合规官/法务/律师单独调用（`/criminal-compliance:compliance-risk-identification`），也可作为 `compliance-system-building` 编排入口调用链的第一步。

## 前置

**读取 `criminal-compliance/CLAUDE.md`：**

1. 读取 `## 合规与服务立场` — 整体服务立场须为「事前合规体系建设」；若显示「已立案或被采取强制措施」，立即触发强制升级并停止执行
2. 读取 `## 我们是谁` — 行业、规模、业务模式、重点刑事风险领域
3. 读取 `## 合规与服务立场` > `### 合规严格度` — 保守型/平衡型/务实型，影响风险识别深度与红线标记密度
4. 读取 `## 合规与服务立场` > `### 是否已涉案` — **任何已涉案迹象一律强制升级**
5. 读取 `## 升级矩阵` — 确认强制升级触发条件
6. 引用 `_shared/legal-basis-conventions.md` — 引用规范（具体罪名条号一律待验证，规范性文件描述规则）
7. 引用 `_shared/criminal-law-citations.md` — 罪名构成要件库（C节七类罪名）

**配置缺失时**：以临时模式继续，默认「事前合规体系建设立场 · 律师角色 · 保守型严格度」，输出标 `[临时模式 — 请运行冷启动访谈完成配置]`。

**涉案情形识别**：执行过程中发现任何已立案、强制措施、侦查传唤、可能构成具体罪名迹象，立即中止并输出 `[强制升级 — 涉刑情形]`，提示联系执业刑事辩护律师。

## 详细步骤

### 步骤一：企业全貌梳理

**目标**：从 `criminal-compliance/CLAUDE.md` 读取企业画像，确认风险识别的基础输入。

| 输入维度 | 读取字段 | 刑事合规风险相关性 |
|---------|---------|-----------------|
| 行业 | `## 我们是谁` > 行业 | 行业高发罪名（税收/商业贿赂/金融/安全生产/环境等） |
| 规模 | `## 我们是谁` > 规模 | 规模越大，风险敞口越广；上市公司有证券合规风险 |
| 业务模式 | `## 我们是谁` > 业务模式 | 制造+出口涉走私/虚开；金融销售涉非法集资/内幕交易；工程涉商业贿赂/串标 |
| 是否涉外 | `## 我们是谁` > 是否涉外 | 跨境业务涉走私、外汇违规行刑衔接 |
| 历史涉案情况 | `## 我们是谁` > 历史涉案情况 | 曾受行政处罚/刑事案件是重点审查领域 |
| 重点刑事风险领域 | `## 我们是谁` > 重点刑事风险领域 | 已识别的高关注领域 |

**确认步骤**：向用户回显企业全貌摘要，确认准确性后开始风险识别：

> 以下为企业全貌摘要（来自 criminal-compliance/CLAUDE.md）：
> - 行业：[X]  规模：[X]  业务模式：[X]  是否涉外：[X]
> - 历史涉案情况：[X]  已知重点刑事风险领域：[X]
>
> 如有更新，请告知，我将重新识别。

### 步骤二：业务维度风险识别

**目标**：逐一梳理企业各业务线/业务环节的刑事合规风险点。

**识别方法**：以业务流程（如采购→生产→销售→财务→对外融资）为主线，逐环节检索高发刑事风险。

| 业务环节 | 典型刑事合规风险点 | 涉及罪名类型（构成要件描述，条号待验证） |
|---------|----------------|---------------------------------|
| 采购与供应链 | 向采购人员行贿以获取订单/供应资格；发票虚开配合采购套现 | 单位行贿罪（为谋取不正当利益给予国家/非国家工作人员财物，`[条号待验证]`）；虚开增值税专用发票罪（`[条号待验证]`） |
| 销售与营销 | 给予客户商业贿赂以获得合同；压低价格结合虚开发票；医药代表渠道费用 | 对非国家工作人员行贿罪（`[条号待验证]`）；非法经营罪（`[条号待验证]`） |
| 财务与资金 | 挪用公司资金；职务侵占；资金混同；非法吸收公众存款 | 挪用资金罪（`[条号待验证]`）；职务侵占罪（`[条号待验证]`）；非法吸收公众存款罪（`[条号待验证]`） |
| 税务申报 | 虚开增值税专用发票；逃税；骗取出口退税 | 虚开增值税专用发票罪/逃税罪/骗取出口退税罪（`[条号待验证]`） |
| 生产与安全生产 | 违规生产、忽略安全隐患、强制工人违规作业 | 重大责任事故罪/重大劳动安全事故罪（`[条号待验证]`） |
| 对外融资 | 向不特定社会公众融资无资质；以非法占有目的集资 | 非法吸收公众存款罪/集资诈骗罪（`[条号待验证]`） |
| 出口与进口 | 申报价格低于实际；伪报品名规避关税 | 走私普通货物物品罪（偷逃应缴税额达一定标准，依司法解释 `[待验证]`，`[条号待验证]`） |
| 环境与废弃物处置 | 私排废水废气；非法处置危险废物 | 污染环境罪（违反国家规定严重污染环境，`[条号待验证]`） |
| 招投标 | 串通投标报价；与招标方串通；提供虚假资质 | 串通投标罪（`[条号待验证]`）；合同诈骗罪（`[条号待验证]`） |
| 知识产权 | 仿冒注册商标；生产销售伪劣产品 | 假冒注册商标罪（`[条号待验证]`） |

**用户输入**：请逐一确认企业实际开展的业务环节，并补充不在上表中的特殊业务：

> 上述业务环节中，企业实际开展的有哪些？是否有以上未覆盖的业务？（如：数据跨境、医疗器械、金融产品代销、政府采购等）

### 步骤三：岗位维度风险识别

**目标**：识别各关键岗位/岗位类别的刑事风险敞口，确定高风险岗位清单。

**识别方法**：以岗位职责和日常决策权限为基础，识别易触发刑事风险的岗位行为模式。

| 岗位类别 | 高风险行为模式 | 刑事风险类型（构成要件描述，条号待验证） | 单位犯罪双罚制关联 |
|---------|--------------|---------------------------------------|----------------|
| 法定代表人/CEO | 决策授权商业贿赂；签署虚假申报；批准不合规融资 | 单位犯罪「直接负责的主管人员」`[条号待验证]`，刑法第31条 `[建议复核]` | 高度关联——决策层直接负刑事责任 |
| 财务总监/CFO | 主导虚开发票；指示挪用资金；指挥逃税申报 | 虚开/逃税/挪用资金罪「其他直接责任人员」`[条号待验证]` | 关联——执行层财务决策 |
| 采购总监/经理 | 接受供应商贿赂；安排回扣返点 | 非国家工作人员受贿罪（`[条号待验证]`）；职务侵占相关 | 关联——采购岗位权力寻租 |
| 销售总监/代表 | 行贿客户关键人员；虚开发票配合销售返利 | 对非国家工作人员行贿罪（`[条号待验证]`）；虚开增值税专用发票罪（`[条号待验证]`） | 关联 |
| 安全生产负责人 | 违规指挥作业；忽视事故隐患 | 重大责任事故罪（`[条号待验证]`）；重大劳动安全事故罪（`[条号待验证]`） | 直接责任人员风险 |
| 环保负责人 | 违规排放；伪造环保记录 | 污染环境罪（`[条号待验证]`） | 直接责任人员风险 |
| 项目经理/工程部门 | 串通投标；建设期违规操作 | 串通投标罪（`[条号待验证]`）；重大责任事故罪（`[条号待验证]`） | 关联 |
| 对外融资负责人 | 向不特定公众募资；以欺骗方式集资 | 非法吸收公众存款罪/集资诈骗罪（`[条号待验证]`） | 关联 |

**输出**：高风险岗位清单（根据企业实际情况定制化）：

| 🟠 高 | [待填入] | [待填入] |

### 步骤四：区域维度风险识别

**目标**：识别企业经营区域的地方监管特点与合规风险差异。

| 区域维度 | 识别要点 |
|---------|---------|
| 地方监管执法强度 | 各地税务稽查频次、安监执法力度、环保整治行动 |
| 地方司法实践差异 | 各地检察机关对涉案企业合规试点的参与程度、适用口径 `[待验证 — 规范性文件快速演进]` |
| 跨地区运营风险 | 多省市经营的税务注册风险、不同地方的合规认定标准差异 |
| 涉外地区特殊风险 | 自贸区、跨境电商、保税区的海关与外汇合规风险 |

> **地方涉案企业合规试点提示**：
> 各地检察机关对涉案企业合规的参与程度与适用口径不同，涉及特定地区时标 `[地方 flagged — 需核实地方涉案企业合规试点口径与地方检察机关实践]`。

### 步骤五：关联方维度风险识别

**目标**：识别通过供应商、客户、中介、代理、股东等关联方传导的刑事合规风险。

| 关联方类型 | 刑事风险传导路径 | 风险描述 |
|---------|--------------|---------|
| 供应商 | 供应商行贿企业采购人员；虚开发票给企业 | 受贿/虚开风险传导至企业 |
| 销售中介/代理商 | 代理商代企业行贿；企业通过中介支付不当利益 | 行贿罪共同犯罪风险 |
| 股东/投资方 | 股东指令企业非法融资；关联交易掏空企业 | 非法集资/职务侵占风险 |
| 政府对接人员 | 向政府官员行贿；项目审批中的权钱交易 | 单位行贿罪（`[条号待验证]`） |
| 合作方/联营体 | 联营中的串通投标；合作中的非法分包 | 串通投标罪（`[条号待验证]`） |
| 境外关联公司 | 跨境转移利润规避税收；境外账户隐匿资产 | 逃税罪/走私相关（`[条号待验证]`） |

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 企业规模小（20人以下），风险识别全面性与成本矛盾 | 重点识别 TOP3 高风险领域，制度设计简化，注明成本约束 |
| 新型业务模式（如元宇宙/数字资产/平台经济），现有罪名对应不明确 | 描述行为模式与构成要件的相似性，标 `[新型业务 — 需法律研究]`，提示咨询专业律师 |
| 行刑衔接风险（行政处罚移送刑事追诉标准） | 描述行刑衔接规则，移送标准标 `[待验证]`；提示用户确认监管部门已有行政处理记录时须升级律师 |
| 企业同时存在多个高优先级领域，无法排序 | 依以下原则排序：1. 已有行政处罚记录的领域优先；2. 涉金额大/人数多的领域优先；3. 法定代表人直接参与的领域优先 |
| 用户要求评估「是否会被追诉」 | 提示不作定罪结论，仅描述构成要件特征并标记风险等级；作有罪无罪结论是司法机关与辩护律师的职责 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 识别到已发生风险事件（已行政处罚/事故/被举报/内部违规记录） | 立即触发强制升级评估：若涉及刑事追诉门槛，标 `[强制升级 — 涉刑情形]`；提示咨询执业刑事辩护律师 |
| 用户陈述涉案迹象（已立案/强制措施/被传唤） | 立即触发 `[强制升级 — 涉刑情形]`，停止风险识别，提示联系律师 |
| 用户要求提供规避监管或规避侦查的方法 | 拒绝，标 `[合规边界 — 拒绝]`，提示合规的正道是依法整改而非规避 |
| 企业画像信息严重不足（行业/业务均未提供） | 提示必须信息，提供通用风险识别框架，等待用户补充 |
| 具体罪名条号信息由用户口述 | 标 `[待验证]`，提示以构成要件描述方式记录，建议用户通过国家法律法规数据库核实 |

## 法律依据

### 一、法律（中度把握锚点，建议复核）

**单位犯罪** — 作为风险识别的主体维度分析基础：

- 《中华人民共和国刑法》**第30条** — 单位犯罪定义（公司、企业、事业单位等实施的危害社会行为，法律规定为单位犯罪的，应当负刑事责任）`[模型知识 — 需验证]` 已核实锚点（中度把握），建议复核
- 《中华人民共和国刑法》**第31条** — 单位犯罪双罚制（对单位判处罚金，同时对直接负责的主管人员和其他直接责任人员判处刑罚）`[模型知识 — 需验证]` 已核实锚点（中度把握），建议复核

> 合规风险识别的岗位维度须重点分析单位犯罪双罚制的责任人员范围，识别「直接负责的主管人员」（决策授权的高管）与「其他直接责任人员」（执行层）的风险敞口。

**刑事诉讼法** — 风险识别需了解追诉门槛：

- 《中华人民共和国刑事诉讼法》**第177条** — 不起诉（酌定相对不起诉是涉案企业合规不起诉的主要路径）`[模型知识 — 需验证]` 已核实锚点（中度把握），建议复核
