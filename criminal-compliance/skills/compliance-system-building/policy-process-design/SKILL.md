# 合规制度与流程设计 — 刑事合规体系建设第二步

## 目的

本子技能服务于**事前合规体系建设**立场。在 `compliance-risk-identification` 产出合规风险地图后，本子技能负责将风险识别结论转化为可落地的**制度体系**：

1. **合规政策与行为准则** — 顶层定调，明确企业对刑事合规的承诺
2. **重点领域专项合规制度** — 针对反商业贿赂、反垄断、数据合规、安全生产、财税合规等高风险领域制定专项制度
3. **合规审批与禁止性清单** — 关键决策节点的合规审批机制与绝对红线清单
4. **合规控制点嵌入业务流程** — 将合规要求内化至日常业务操作，形成制衡

> **与其他子技能的关系**：
> - 前序：`compliance-risk-identification` — 提供重点合规领域与风险地图（制度设计的输入）
> - 并行：`organization-training` — 制度须有组织架构保障与培训配套才能落地
> - 后续：`effectiveness-evaluation` — 评估本子技能制定的制度是否有效执行
> - **与 non-prosecution/compliance-plan 的分工**：本子技能 = 事前合规体系制度层建设；`compliance-plan` = 涉案后专项合规整改计划（针对已被追诉的具体问题），两者互补不重复

**可独立触发**：本子技能可由合规官/法务/律师单独调用（`/criminal-compliance:policy-process-design`），也可作为 `compliance-system-building` 编排入口调用链的第二步。

## 详细步骤

### 步骤一：确定制度设计输入

**从 `compliance-risk-identification` 读取输入**（或由用户提供）：

| 输入项 | 来源 | 说明 |
|-------|------|------|
| 合规风险地图 | `compliance-risk-identification` 输出 / 用户提供 | 确定重点合规领域与优先级 |
| 高风险岗位清单 | `compliance-risk-identification` 输出 / 用户提供 | 确定合规审批节点与禁止性行为清单覆盖范围 |
| 行业监管要求 | `## 我们是谁` > 行业 + 用户补充 | 确定是否有特定行业合规强制性要求（如金融监管合规、医药合规）|
| 单位规模与结构 | `## 我们是谁` > 规模 | 大型企业须体系化制度，中小企业须简化且可落地 |

---

### 步骤二：制定合规政策与行为准则

**目标**：制定企业合规管理的顶层文件，明确对刑事合规的承诺与整体要求。

**合规制度体系层级表：**

| 层级 | 文件名称 | 主要内容 | 适用范围 | 审批权限 |
|-----|---------|---------|---------|---------|
| 第一层：总纲 | 《合规管理总体政策》 | 合规承诺、定义、适用范围、总体原则、合规监管架构 | 全体员工及适用第三方 | 董事会/最高决策层批准 |
| 第二层：行为准则 | 《员工行为准则》 | 日常行为要求、利益冲突、保密、礼品娱乐规定、合规报告渠道 | 全体员工 | CEO/总经理批准 |
| 第三层：专项制度 | 《反商业贿赂制度》《税收合规制度》等 | 各重点合规领域的具体规则、审批程序、禁止性行为 | 相关业务部门 | 合规委员会/董事会批准 |
| 第四层：操作规程 | 《礼品娱乐申请程序》《差旅合规报销规程》等 | 具体操作步骤、表格、审批流 | 执行层 | 合规部门/业务部门负责人批准 |

**合规政策核心内容要素：**

1. **合规承诺（Tone at the Top）**：由法定代表人/CEO签署，明确企业对刑事合规零容忍的立场
2. **适用范围**：全体员工 + 适用第三方（供应商/代理商/合作方须签署合规承诺书）
3. **举报渠道**：设立独立、保密的合规举报渠道（举报热线/邮箱），明确不打击报复政策
4. **违规后果**：明确违规行为的处罚标准（纪律处分 → 解除劳动合同 → 移送司法机关）
5. **法律合规义务**：明确企业适用的中国现行法律法规框架（刑法第30/31条双罚制含义，建议复核）

### 步骤三：制定重点领域专项合规制度

根据合规风险地图确定的优先领域，制定以下专项制度（根据企业实际选择适用领域）：

#### 3.1 反商业贿赂专项合规制度

**涉及罪名**（构成要件描述，条号待验证）：
- 单位行贿罪：单位为谋取不正当利益给予国家工作人员财物 `[条号待验证]`
- 对非国家工作人员行贿罪：为谋取不正当商业利益给予公司企业工作人员财物 `[条号待验证]`

**专项制度核心内容：**

| 制度要素 | 内容要点 |
|---------|---------|
| 适用范围 | 所有员工、代理商、销售顾问、联合体成员 |
| 礼品娱乐规定 | 礼品价值上限（企业自行设定）、禁止现金/有价证券、节假日礼品限制、申报与审批要求 |
| 商业招待规定 | 合理商业目的要求、费用标准、审批权限分级、记录保存要求 |
| 顾问与代理协议 | 尽职调查、合理报酬标准、付款审批、合同条款合规条件 |
| 利益冲突处理 | 员工持有竞争方股权、配偶任职相关单位、接受回扣的识别与处理 |
| 政府官员接触规定 | 接触政府官员的审批、记录保存、「两节」期间特别限制 |
| 举报与调查 | 合规举报保密保护、内部调查程序、与执法机关配合的律师审查要求 |

#### 3.2 反垄断（含串通投标）专项合规制度

**涉及罪名**（构成要件描述，条号待验证）：
- 串通投标罪：投标人相互串通投标报价损害招标方/其他投标人利益；投标人与招标方串通 `[条号待验证]`

**专项制度核心内容：**

| 制度要素 | 内容要点 |
|---------|---------|
| 价格与市场行为 | 禁止与竞争对手交流价格、分配客户/市场/地区，参加行业协会须合规审查 |
| 招投标合规 | 独立报价要求、投标前禁止与其他投标方接触、中标或落标的合规记录 |
| 市场支配行为 | 具有市场支配地位的企业须特别注意定价、搭售、独占合同的合规要求 |
| 与竞争对手的合法接触 | 行业协会活动的合规边界、接受法律顾问建议的记录 |

#### 3.3 数据合规专项制度

**涉及罪名**（构成要件描述，条号待验证）：
- 侵犯公民个人信息罪：违反国家有关规定向他人出售或提供公民个人信息，情节严重 `[条号待验证]`

**专项制度核心内容：**

| 制度要素 | 内容要点 |
|---------|---------|
| 数据分类与保护 | 个人信息/敏感个人信息/企业数据分级，各级保护要求 |
| 数据收集与处理 | 最小必要原则、目的限制、同意要求 |
| 数据安全 | 访问控制、加密要求、数据泄露应急响应 |
| 数据跨境 | 跨境数据传输的合规要求（须核实数据出境安全评估规则 `[待验证]`） |
| 个人信息主体权利 | 查阅、更正、删除请求的处理流程 |

#### 3.4 安全生产专项合规制度

**涉及罪名**（构成要件描述，条号待验证）：
- 重大责任事故罪：生产作业中违反安全管理规定造成重大伤亡事故（过失）`[条号待验证]`
- 重大劳动安全事故罪：安全生产设施条件不符国家规定经提出后仍不整改造成严重后果 `[条号待验证]`

**专项制度核心内容：**

| 制度要素 | 内容要点 |
|---------|---------|
| 安全生产责任体系 | 主要负责人、分管安全负责人、班组长的安全责任清单 |
| 安全生产规程 | 特种作业操作规程、危险作业许可制度、安全操作规程 |
| 安全检查与隐患排查 | 定期安全检查计划、隐患登记台账、整改闭环要求 |
| 事故报告与应急响应 | 事故报告路径（不得迟报、谎报、漏报）、应急预案启动条件 |
| 劳动防护 | 个人防护用品配备标准、禁止违规指挥作业 |

#### 3.5 财税合规专项制度

**涉及罪名**（构成要件描述，条号待验证）：
- 虚开增值税专用发票罪：以虚开等方式违反增值税专用发票管理 `[条号待验证]`
- 逃税罪：采取欺骗隐瞒手段虚假申报或不申报逃避税款 `[条号待验证]`

**专项制度核心内容：**

| 制度要素 | 内容要点 |
|---------|---------|
| 发票管理 | 增值税专用发票开具与取得的合规要求；禁止虚开；供应商发票资质审核 |
| 税务申报 | 税务申报的审批流程、税务师/会计师审核要求、申报记录存档 |
| 财务记录 | 账实一致要求；禁止双套账；财务记录保存年限 |
| 关联交易 | 关联交易的公允价值定价要求；关联交易审批与披露 |
| 纳税激励合规 | 税收优惠申请的合法性审查；享受激励的证明文件保存 |

### 步骤四：制定合规审批与禁止性清单

**目标**：将关键高风险决策节点纳入合规审批，制定绝对红线禁止性行为清单。

**合规审批体系（分级审批）：**

| 审批类型 | 触发条件 | 审批层级 | 所需文件 |
|---------|---------|---------|---------|
| 一般礼品娱乐 | 单次礼品价值超过企业设定下限 | 直接主管 + 合规专员 | 礼品申报表 |
| 顾问/代理协议 | 签署含佣金/顾问费的协议 | 合规部门 + 法务 | 尽职调查报告 + 合同审查 |
| 政府官员接触 | 接待政府官员活动 | 合规部门审批 | 接触记录表 |
| 三重一大决策 | 重大决策、重大项目、重大资金安排、重要人员任免 | 集体决策 + 合规部门见证 | 集体决策记录 |
| 关联交易 | 与关联方发生任何商业往来 | 合规委员会 + 独立董事（如适用） | 公允价值评估 |
| 境外付款 | 向境外账户付款超过设定金额 | 财务负责人 + 合规部门 | 业务合理性说明 + 合同 |
| 现金交易 | 单笔现金交易超过设定金额 | 财务负责人 | 书面说明 |

**禁止性行为清单（刑事合规红线）：**

以下行为属于刑事合规绝对禁止事项，无论何种商业目的均不得从事：

| 类别 | 禁止行为 | 涉及刑事风险 |
|-----|---------|------------|
| 行贿 | 以任何形式（现金/礼品/购物卡/报销/旅游/就业机会）向国家工作人员或商业对象提供不正当利益 | 单位行贿罪/对非国家工作人员行贿罪（`[条号待验证]`） |
| 发票虚开 | 开具与实际交易不符的增值税专用发票；让他人为自己虚开；介绍他人虚开 | 虚开增值税专用发票罪（`[条号待验证]`） |
| 串通投标 | 与其他投标方事先沟通报价；与招标方串通操纵结果 | 串通投标罪（`[条号待验证]`） |
| 双套账 | 设立账外账、黑白两本账、伪造财务记录 | 逃税罪/职务侵占罪（`[条号待验证]`） |
| 非法集资 | 未经金融监管批准向不特定公众融资并承诺回报 | 非法吸收公众存款罪/集资诈骗罪（`[条号待验证]`） |
| 证据销毁 | 在任何调查、诉讼、监管检查启动后销毁、篡改、隐匿相关文件 | 妨害证据罪（构成要件描述，`[条号待验证]`） |
| 规避侦查 | 任何以规避侦查、对抗调查为目的的行为 | `[合规边界 — 拒绝]` |

### 步骤五：合规控制点嵌入业务流程

**目标**：将合规要求嵌入日常业务流程，形成事前合规检查机制。

**业务流程合规控制点示意：**

| 业务环节 | 合规控制点 | 控制方式 | 责任岗位 |
|---------|----------|---------|---------|
| 供应商选择 | 反商业贿赂条款、税务合规资质 | 供应商准入尽职调查 + 合规承诺书 | 采购部门 + 合规专员 |
| 合同签署 | 合规条款审查（反腐败、保密、数据合规） | 合同法律合规审查单 | 法务/合规部门 |
| 差旅报销 | 礼品娱乐报销合规性检查 | 报销系统合规字段 + 合规专员抽查 | 财务 + 合规专员 |
| 三重一大决策 | 集体决策记录、合规影响评估 | 合规影响评估表 + 集体审批签字 | 合规委员会 |
| 投标准备 | 串通投标风险检查 | 投标合规审批单 | 业务部门 + 合规专员 |
| 税务申报 | 发票合规性、申报数据核查 | 税务申报前合规审查 | 财务负责人 + 税务顾问 |
| 安全生产 | 安全检查与隐患排查 | 定期安全检查计划 + 隐患整改台账 | 安全生产负责人 |
| 对外融资 | 融资方式合法性审查 | 融资方案合规审批 | 法务/合规部门 + 管理层 |

## 检查清单

- [ ] 合规制度体系层级明确（总纲 → 行为准则 → 专项制度 → 操作规程）
- [ ] 合规政策含法定代表人/CEO的合规承诺（Tone at the Top）
- [ ] 重点领域专项制度覆盖企业实际高风险领域（依合规风险地图定制）
- [ ] 禁止性行为清单涵盖主要刑事红线，措辞清晰、无歧义
- [ ] 合规审批体系分级明确，关键节点（三重一大/顾问协议/政府官员接触）均有合规审批
- [ ] 合规控制点已嵌入主要业务流程（采购/销售/财务/投标/安全生产）
- [ ] 制度设计未包含任何以规避侦查为目的的内容（合规边界护栏）
- [ ] 制度须与现有公司管理体系（人力资源/财务/业务）整合，避免重复冲突
- [ ] 中小企业：制度已适度简化，确保可落地执行
- [ ] 所有涉及具体罪名的条号均标 `[条号待验证]`，以构成要件描述呈现
- [ ] 规范性文件（涉案企业合规相关文件）描述规则并标 `[待验证]`

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 制度形式化（制度文本存在但无人执行） | 提示：制度的可落地性须结合组织设计与培训考核保障，需结合 `organization-training` 子技能 |
| 制度与现有管理体系冲突（如现有采购流程与合规审批冲突） | 识别冲突节点，建议在制度设计中整合而非另起炉灶，保持单一权威流程 |
| 中小企业（20人以下）制度简化需求 | 合并制度层级，制定一份综合《员工合规手册》代替多份专项制度；重点保留禁止性清单与关键审批 |
| 制度须法律审查 | 在输出中明确标注：本方案为草案，需执业律师审查，涉及对外提交的制度须律师审核 |
| 涉及行业特许经营合规要求（如金融监管、药品监管） | 提示行业监管特定合规要求（行刑衔接相关）须结合监管法规专项审查，建议咨询行业法律专家 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 用户要求设计「如何规避证据留存」的合规程序 | 拒绝，标 `[合规边界 — 拒绝]`，提示制度设计绝不以规避侦查为目的 |
| 识别到已涉案情形 | 立即触发 `[强制升级 — 涉刑情形]`，提示本子技能聚焦事前合规体系建设，涉案后须转入 `non-prosecution` 路径并在律师指导下实施 |
| 制度设计须依托行业专业知识（如医药合规/金融合规细节） | 提供通用框架，提示行业专项合规须结合行业法律专家意见，本子技能聚焦刑事合规风险维度 |
| 用户要求「制度上看起来合规即可，不想真正执行」 | 提示：形式合规无法有效抵御刑事追诉，有效合规须实质执行；合规有效性评估将核查实质执行情况（可参考涉案企业合规相关规范性文件对有效合规的要求 `[待验证]`） |
