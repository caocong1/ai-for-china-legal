# 刑事合规体系建设 — 编排入口

> 本技能编排四个可独立触发的合规体系建设子技能。律师/合规官/法务可运行完整流程，也可单独触发任一子技能。

---

## 目的

**刑事合规体系建设**是企业事前预防刑事风险的系统性工程，涵盖从风险识别到制度设计、组织培训到有效性评估的完整链条。本编排入口将四个子技能整合为一个有序的体系建设流程，同时允许各子技能独立运行。

**整体流程**：合规风险识别 → 合规制度与流程设计 → 合规组织与培训 → 合规有效性评估与整改

## 入口级护栏

### 1. 企业画像与服务立场确认

在开始任何子技能前，向用户确认：

> 在开始刑事合规体系建设前，请确认：
> 1. **企业画像**：行业 / 规模 / 主要业务模式 / 是否涉外
> 2. **服务立场**：事前合规体系建设（预防立场）/ 混合立场（含涉案应对）
> 3. **是否已涉案**：纯预防 / 有苗头线索 / 已立案或被采取强制措施

### 2. 涉案情形识别与强制升级

**识别到以下任一情形，立即停止体系建设编排，触发强制升级：**

| 情形 | 处置 |
|-----|------|
| 企业或相关人员已被立案 | `[强制升级 — 已立案]` 停止，联系执业刑事辩护律师，调用 `/criminal-compliance:non-prosecution` |
| 被采取强制措施（拘留/逮捕/取保/监视居住） | `[强制升级 — 强制措施]` 停止，立即联系执业刑事辩护律师 |
| 已收到侦查传唤/询问通知 | `[强制升级 — 侦查传唤]` 停止，联系律师 |
| 可能构成具体罪名（明确迹象） | `[强制升级 — 具体罪名风险]` 停止，联系律师 |
| 重大涉案情形 | `[强制升级 — 重大涉案]` 停止，联系律师 |

> **已涉案时**：本插件的输出仅作合规研究背景资料，不替代执业刑事辩护律师的辩护工作。请立即联系执业刑事辩护律师。

### 3. 合规预防与辩护立场护栏

- 本技能用于**合规风险预防与体系建设**，不替企业作有罪无罪的最终结论
- 不得以「合规」为名行规避侦查、伪造毁灭证据、串供、转移资产之实
- **识别到上述需求时，一律拒绝，标 `[合规边界 — 拒绝]`**

### 4. 不提供规避侦查或逃避责任方法护栏

以下需求无论以何种形式提出，一律拒绝：
- 合规制度设计中加入「应对突击检查」的证据清理规程
- 合规评估报告中包含「哪些材料不宜留存」的建议
- 任何实质上对抗调查的「合规」建议

拒绝后标 `[合规边界 — 拒绝]`，并提示：合规的正道是依法整改、如实供述。

### 5. 与 risk-assessment 衔接说明

本技能与 `risk-assessment` 的分工：

| 技能 | 定位 | 触发时机 |
|-----|------|---------|
| `risk-assessment` | 单次刑事风险评估：评估特定行为的刑事风险等级 | 需要评估「某具体行为是否有刑事风险」时触发 |
| `compliance-system-building`（本技能）| 体系化合规建设：全面风险底图 + 制度 + 组织 + 评估 | 需要「建立或完善企业刑事合规管理体系」时触发 |
| 两者衔接 | `risk-assessment` 评估结论可作为合规风险地图的输入；体系建设中的风险识别成果可完善 `risk-assessment` 的背景地图 | — |

### 6. 与 non-prosecution 衔接说明

| 技能 | 定位 |
|-----|------|
| `compliance-system-building`（本技能）| 事前合规体系建设（预防立场，企业尚未涉案）|
| `non-prosecution` + `compliance-plan` | 涉案后合规整改计划（涉案应对，须律师主导）|
| 衔接点 | 事前建立的有效合规体系是涉案后争取合规不起诉的重要基础（有效合规认定标准 `[待验证]`） |

### 7. 目的地/特权检查

输出合规体系文件前，确认目的地：

- **内部使用（律师工作产物/内部合规文件）**：加注「律师工作产物 — 受律师-当事人特权保护」
- **对外提交（如向检察机关提交合规报告）**：须在律师指导下决定是否提交及内容；直接提交可能作为证据使用

## 运行说明

**完整体系建设流程（推荐）**：依次运行步骤1→2→3→4，建立完整刑事合规体系。

**单独子技能触发**：律师/合规官/法务可单独触发任一子技能：
- 只需评估风险底图：`/criminal-compliance:compliance-risk-identification`
- 只需制度设计：`/criminal-compliance:policy-process-design`
- 只需组织培训方案：`/criminal-compliance:organization-training`
- 只需有效性评估：`/criminal-compliance:effectiveness-evaluation`

**与 risk-assessment 衔接路径**：先运行 `/criminal-compliance:risk-assessment` 评估单次事件风险，再运行本编排更新合规风险地图。
