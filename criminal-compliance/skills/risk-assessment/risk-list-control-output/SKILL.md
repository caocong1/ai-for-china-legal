# 风险清单与控制建议输出（risk-list-control-output）

> **可单独触发**：本子技能可由律师/合规官/法务单独触发（`/criminal-compliance:risk-list-control-output`），也可由 risk-assessment 编排入口调用。
>
> **与其他子技能分工**：本子技能是 risk-assessment 的汇总输出节点，整合行业风险（industry-crime-identification）、岗位风险（position-conduct-risk）与等级评估（risk-level-likelihood）的成果；也可单独基于用户提供的风险信息产出。
>
> **合规边界**：本子技能的输出用于合规改进，绝不包含规避侦查或逃避责任的内容。识别到此类需求一律拒绝并标 `[合规边界 — 拒绝]`。

---

## 目的

整合 risk-assessment 各子技能的分析成果，输出结构化的「刑事风险清单」与「控制建议」，并生成完整的「刑事风险评估报告」——涵盖风险点、对应罪名（构成要件描述）、涉及岗位与业务环节、可能性、影响程度、综合风险等级、控制建议、责任部门与整改时限。

报告格式须适应不同目的地（企业内部合规使用 vs 面向检察机关的涉案企业合规不起诉配合材料）。

---

## 前置

### 读取实践配置

技能启动时读取 `criminal-compliance/CLAUDE.md`：

1. **服务立场**（`## 合规与服务立场` > 整体服务立场）：
   - **事前预防立场**：输出完整风险地图，风险清单覆盖全部识别到的风险点，控制建议侧重事前防范制度建设
   - **涉案应对立场**：输出聚焦已涉嫌问题与整改建议，控制建议衔接合规不起诉申请，须与律师主导的合规计划协调
2. **是否已涉案**（`## 合规与服务立场` > `### 是否已涉案`）：
   - **已立案或被采取强制措施**：立即触发 `[强制升级 — 涉刑情形]`，输出仅作辩护律师参考材料
3. **文书风格**（`## 文书风格`）：读取报告风格偏好
4. **工作产物标题**（`## 输出`）：读取律师/非律师标题格式
5. **目的地/特权检查**（`## 共享护栏` > 目的地/特权检查）：确认报告是内部合规使用还是面向检察机关/第三方评估组织

若配置缺失，进入临时模式（事前预防立场、律师角色、保守合规严格度）。

### 涉刑强制升级检查

如「是否已涉案」为「已立案或被采取强制措施」，立即输出 `[强制升级 — 涉刑情形]`，提示联系执业刑事辩护律师，本子技能输出仅作律师参考背景材料。

## 详细步骤

### 步骤一：汇集风险评估成果

| 来源 | 内容 | 若未完成前置子技能 |
|-----|------|-----------------|
| industry-crime-identification 输出 | 行业高发罪名类别与触发情形 | 要求用户提供行业与罪名类别信息，或先触发该子技能 |
| position-conduct-risk 输出 | 岗位行为风险点与责任人员范围 | 要求用户提供岗位风险信息，或先触发该子技能 |
| risk-level-likelihood 输出 | 风险等级（🔴🟠🟡🟢）与是否已现实化分层 | 要求用户提供评级，或先触发该子技能 |

若前置子技能输出已提供，直接整合；若未提供，询问补充后继续。

### 步骤二：构建刑事风险清单

刑事风险清单是本子技能的核心输出，须覆盖以下完整列：

| 列名 | 内容说明 |
|-----|---------|
| 风险点编号 | R-001、R-002… |
| 风险点描述 | 具体业务行为或环节 |
| 对应罪名 | 罪名类别 + 构成要件要素描述，`[条号待验证]` |
| 涉及岗位 | 涉及的具体岗位 |
| 涉及业务环节 | 采购/销售/财务/审批/外部合作等 |
| 可能性等级 | 极高/高/中/低 |
| 影响程度等级 | 极高/高/中/低 |
| 综合风险等级 | 🔴/🟠/🟡/🟢 |
| 是否已现实化 | 纯风险/苗头/已有线索/已立案 |
| 控制建议 | 事前防范措施（见步骤三） |
| 责任部门 | 整改主责部门 |
| 整改时限 | 立即/30天内/90天内/持续改进 |
| 验收标准 | 整改完成的可验证标准 |

### 步骤三：控制建议制定

对每个风险点制定控制建议，按分级分类输出：

#### 🔴 关键风险控制建议（须立即处置）

| 控制措施类型 | 具体建议 |
|------------|---------|
| **立即停止** | 识别到可能构成罪名的具体行为，立即停止，不得继续 |
| **律师介入** | 立即联系执业刑事辩护律师评估已发生行为的法律性质 |
| **证据保全** | 在律师指导下，保全相关合规努力证据（审批记录、培训记录等） |
| **上报机制** | 触发内部强制上报程序，避免后续共同犯罪风险 |

#### 🟠 高风险控制建议（优先整改）

| 控制措施类型 | 具体建议 |
|------------|---------|
| **制度建设** | 制定或修订针对该风险点的专项合规制度（如反腐败制度、发票管理制度） |
| **审批流程** | 在涉及风险行为的关键节点增设审批或合规审查环节 |
| **禁止性清单** | 明确列示禁止行为，向相关岗位人员书面告知 |
| **留痕要求** | 规定相关业务的记录与存档要求（合同、发票、审批单、往来函件） |
| **培训教育** | 对相关岗位人员开展针对性合规培训，记录培训考核结果 |

#### 🟡 中等风险控制建议（纳入合规改进计划）

| 控制措施类型 | 具体建议 |
|------------|---------|
| **制度完善** | 完善现有制度中存在的漏洞或不明确之处 |
| **内控强化** | 在现有制度基础上增加复核或抽查机制 |
| **监测预警** | 建立风险指标监测机制，定期检查合规执行情况 |

#### 🟢 低风险控制建议（维持监控）

- 维持现有控制措施，定期审查有效性
- 在合规年度审查时纳入常规检查

### 步骤四：责任部门与整改时限分配

| 风险等级 | 建议整改时限 | 主责部门 | 监督部门 |
|---------|------------|---------|---------|
| 🔴 关键 | 立即（发现当日） | CEO/法定代表人直接负责 | 法律部/合规部（律师主导） |
| 🟠 高 | 30天内完成初步整改，90天内完成全面整改 | 分管副总/业务负责人 | 合规部，须律师参与审核 |
| 🟡 中等 | 90天内纳入合规改进计划并启动 | 相关部门负责人 | 合规部 |
| 🟢 低 | 下一周期合规审查时复核 | 相关岗位负责人 | 合规部 |

### 步骤五：报告目的地与特权/脱敏处理

根据报告的使用目的，区分输出版本：

| 目的地 | 版本类型 | 特权/保密说明 |
|-------|---------|------------|
| **企业内部合规使用** | 完整版（含律师工作产物特权标注） | 律师-当事人特权保护，不得对外披露 |
| **向检察机关提交（涉案企业合规）** | 脱敏版（仅包含整改措施与成果，不包含内部法律意见） | **律师-当事人特权可能失效**，须律师指导脱敏，不包含自证其罪的内容 |
| **向第三方监督评估组织提交** | 整改评估版（聚焦整改措施的可验证性与实施进度） | 须律师指导，明确提交内容范围 |

**关键提示**：报告不得包含任何规避侦查内容（`[合规边界 — 拒绝]`）；面向检察机关/第三方监督评估的版本须律师审核定稿。

## 边界条件

| 情形 | 处置方式 |
|------|---------|
| 风险清单超大规模（风险点超30个） | 分层优化：汇总表先行，重点呈现关键和高风险项，中低风险项附录 |
| 报告面向检察机关（涉案应对） | 强制律师审核，脱敏处理（不含内部法律意见），明确提示特权失效 |
| 报告不得包含规避侦查内容 | 合规边界检查：逐条检查控制建议是否含规避侦查成分，如有立即删除并标 `[合规边界 — 拒绝]` |
| 控制建议落地难（内部阻力大） | 建议升级管理层背书；纳入合规体系建设子技能的组织支持设计 |
| 风险清单动态更新需求 | 说明风险清单须定期更新（建议季度或年度），并在重大业务变化时触发重新评估 |
| 已涉案情形 | 立即强制升级，报告仅作律师参考，不得直接提交检察机关 |

## 错误处理

| 错误情形 | 处置方式 | 输出标记 |
|---------|---------|---------|
| 识别到已涉案（立案/强制措施） | 立即强制升级，报告降级为辩护律师参考背景材料 | `[强制升级 — 涉刑情形]` |
| 识别到规避侦查内容需求 | 拒绝，提示合规正道，不将此类建议写入报告 | `[合规边界 — 拒绝]` |
| 前置子技能输出不完整 | 要求补充缺失信息，或以用户提供信息为基础继续，标注输入不完整 | `[信息不足 — 报告基于不完整输入]` |
| 报告定稿前非律师角色要求提交 | 暂停，强制要求律师审核，标注「非律师角色须律师审核后方可对外提交」 | `[律师审查护栏 — 须律师审核]` |
| 涉案应对报告误用于内部作自证材料 | 提示报告属于内部律师工作产物，不得在未经律师确认的情况下对外披露 | `[特权检查 — 须律师确认披露范围]` |
