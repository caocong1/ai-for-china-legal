# 行业高发罪名识别（industry-crime-identification）

> **可单独触发**：本子技能可由律师/合规官/法务单独触发（`/criminal-compliance:industry-crime-identification`），也可由 risk-assessment 编排入口调用。
>
> **与其他子技能分工**：本子技能专注于「行业维度」的高发罪名识别与构成要件速查；岗位维度风险点见 position-conduct-risk；风险等级评估见 risk-level-likelihood；风险清单与控制建议输出见 risk-list-control-output。
>
> **合规边界**：本子技能仅用于合规风险识别与预防，绝不提供规避侦查或逃避责任的方法。识别到用户需求涉及上述禁止情形，一律拒绝并标 `[合规边界 — 拒绝]`。

---

## 目的

按企业所属行业，系统识别高发刑事风险点与对应罪名类别，提供行业刑事风险地图与各罪名构成要件速查表，帮助律师/合规官/法务快速定位风险敞口。

**关键纪律**：各具体罪名条号一律标 `[条号待验证]`，以构成要件描述呈现，绝不臆造具体罪名条号与法定刑档。定罪量刑数额一律描述规则 + `[待验证]`，不写裸数额。

## 前置

### 读取实践配置

技能启动时读取 `criminal-compliance/CLAUDE.md`：

1. **执业角色与企业画像**（`## 我们是谁`）：提取「行业」「业务模式」「重点刑事风险领域」字段
2. **服务立场**（`## 合规与服务立场` > 整体服务立场）：
   - **事前预防立场**：全面扫描行业所有高发罪名，输出完整风险地图
   - **涉案应对立场**：聚焦已涉嫌罪名的构成要件比对，识别哪些行为已满足构成要件
3. **是否已涉案**（`## 合规与服务立场` > `### 是否已涉案`）：
   - **已立案或被采取强制措施**：立即触发 `[强制升级 — 涉刑情形]`，不继续本子技能分析
4. **合规严格度**（`## 合规与服务立场` > `### 合规严格度`）：影响风险标红阈值

若配置缺失或含 `[PLACEHOLDER]`，进入临时模式（事前预防立场、律师角色、合规严格度保守）。

### 涉刑强制升级检查

在进入任何实质分析前：如识别到企业或相关人员**已被立案、被采取强制措施、可能构成具体罪名（明确迹象）、重大涉案**，立即输出：

> `[强制升级 — 涉刑情形]` 识别到涉刑情形，本子技能仅提供初步合规背景分析，须立即联系执业刑事辩护律师，由律师主导后续应对。

## 详细步骤

### 步骤一：确认行业与业务模式

| 确认项 | 来源 | 若未配置 |
|-------|------|---------|
| 主要行业（如制造、金融、医药、建设） | CLAUDE.md `## 我们是谁` | 询问用户：「请描述企业主要业务」 |
| 核心业务模式（如自营生产+出口、经纪+理财产品销售） | CLAUDE.md `## 我们是谁` | 询问用户 |
| 是否涉外（跨境业务） | CLAUDE.md `## 我们是谁` | 询问用户 |
| 历史涉案/行政处罚情况 | CLAUDE.md `## 我们是谁` | 询问用户 |

**行业范围确认后，按步骤二检索对应高发罪名类别。**

### 步骤二：行业刑事风险点对照

按企业所属行业，从下表中提取高发罪名类别（**所有具体罪名条号一律 `[条号待验证]`，以构成要件描述呈现**）：

| 行业 | 高发罪名类别 | 典型触发情形 | 主体（单位/自然人） |
|------|------------|------------|------------------|
| 制造/贸易 | 虚开增值税专用发票罪 `[条号待验证]`、逃税罪 `[条号待验证]`、假冒注册商标罪 `[条号待验证]`、走私普通货物物品罪 `[条号待验证]`、重大责任事故罪 `[条号待验证]` | 取得或开具虚开发票、品牌仿冒、逃避海关监管、安全事故 | 单位+自然人 |
| 金融/投资 | 非法吸收公众存款罪 `[条号待验证]`、集资诈骗罪 `[条号待验证]`、操纵证券期货市场罪 `[条号待验证]`、内幕交易罪 `[条号待验证]` | 未经批准公众募资、操纵市场价格、利用内幕信息交易 | 单位+自然人 |
| 医药/器械 | 对非国家工作人员行贿罪 `[条号待验证]`、单位行贿罪 `[条号待验证]`、生产销售伪劣产品相关罪名 `[条号待验证]` | 医药代表给付回扣、生产不合规药品器械 | 单位+自然人 |
| 房地产/建设 | 串通投标罪 `[条号待验证]`、单位行贿罪 `[条号待验证]`、对非国家工作人员行贿罪 `[条号待验证]`、重大责任事故罪 `[条号待验证]` | 招投标环节串通、与政府部门人员行贿、施工安全事故 | 单位+自然人 |
| 环保敏感行业 | 污染环境罪 `[条号待验证]`（化工/能源/制造） | 违规排放有毒有害物质、倾倒危险废物 | 单位+自然人 |
| 安全生产高危行业 | 重大责任事故罪 `[条号待验证]`、重大劳动安全事故罪 `[条号待验证]`（矿山/建筑/危化品） | 违反安全规定酿成重大事故 | 自然人（直接责任人） |
| 互联网/科技 | 非法经营罪 `[条号待验证]`、侵犯公民个人信息罪 `[条号待验证]` | 无资质从事特许经营业务、违规收集使用个人数据 | 单位+自然人 |
| 混合/跨行业 | 结合各行业风险点综合识别 | — | — |

**注**：上表为行业高发罪名类别识别指引，不构成对企业是否构成犯罪的判断。具体认定须结合企业实际情况与执行律师专业意见。

### 步骤三：各高发罪名构成要件速查

对步骤二中识别的行业高发罪名，逐一描述构成要件（**条号一律 `[条号待验证]`**）。以下为示例框架，具体罪名参照 `criminal-compliance/skills/_shared/criminal-law-citations.md`（C节）：

| 维度 | 说明 |
|------|------|
| 罪名（条号待验证） | XX罪 `[条号待验证]` |
| 保护法益 | 国家XX制度/XX秩序/XX权利 |
| 客观行为 | 具体行为描述（无须条号，以行为特征呈现） |
| 主体 | 单位犯罪（双罚制适用）/ 自然人 / 单位与自然人均可 |
| 主观要件 | 故意/过失 + 是否有特殊目的 |
| 定罪量刑数额标准 | 依相关司法解释，数额/情节达一定标准，`[待验证]` |
| 行业触发特征 | 本行业典型触发该罪名的业务场景与行为模式 |
| 单位犯罪说明 | 是否适用双罚制（刑法第31条，建议复核） |

### 步骤四：服务立场驱动分析侧重

根据从 CLAUDE.md 读取的服务立场，调整输出重点：

**事前预防立场**：
- 全面列出本行业所有高发罪名类别
- 重点输出：哪些业务环节存在触发风险，如何在制度与流程层面预防
- 为后续 risk-level-likelihood 与 risk-list-control-output 子技能提供输入

**涉案应对立场**：
- 聚焦已涉嫌或关注的具体罪名类别
- 重点输出：企业实际行为与该罪名构成要件的比对（哪些要件已满足，哪些存在疑问）
- 须明确标注：比对结论仅为初步分析，最终认定须执业刑事辩护律师判断，`[强制升级 — 涉案应对]`

### 步骤五：行业单位犯罪风险说明

- 识别企业所在行业中常见的单位犯罪形态
- 说明单位犯罪认定要件（刑法第30条 — 已核实锚点，中度把握，建议复核）
- 说明双罚制（刑法第31条 — 已核实锚点，中度把握，建议复核）对单位与直接负责主管人员、其他直接责任人员的影响

## 边界条件

| 情形 | 处置方式 |
|------|---------|
| 单位犯罪与自然人犯罪区分不清（以单位名义但实为个人获益） | 说明认定逻辑（单位意志vs个人决定，违法所得归属），标 `[待验证 — 须律师核实单位犯罪认定]` |
| 一行多罪（同一行为可能触发多个罪名） | 逐一列出可能的罪名类别，说明竞合逻辑，标 `[待验证 — 罪数认定须律师核实]` |
| 新型业务模式罪名定性不明（互联网新业务、数字资产等） | 标 `[待验证 — 新型业务模式刑事定性尚存争议，须律师专项研究]` |
| 行业跨境业务（涉外元素，两国法律均需关注） | 说明中国刑法的属地管辖与属人管辖，境外法律风险超出本插件范围 |
| 已涉案（任何涉案迹象） | 立即触发强制升级，不继续本子技能分析 |
| 用户要求提供具体罪名条号 | 拒绝臆造，说明条号一律 `[待验证]`，以构成要件描述呈现 |
