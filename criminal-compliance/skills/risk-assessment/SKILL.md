# risk-assessment — 刑事风险评估编排入口

## 目的与整体流程概览

刑事风险评估帮助企业（及其律师/合规官/法务）系统识别刑事风险敞口、评估风险等级、输出风险清单与控制建议，支撑事前合规预防与涉案应对合规整改。

**整体评估流程（顺序调用，也可单独触发任一子技能）**：

```
行业高发罪名识别 → 岗位与行为风险点 → 风险等级与可能性评估 → 风险清单与控制建议输出
```

### 读取 criminal-compliance/CLAUDE.md

在启动任何子技能前，读取 `criminal-compliance/CLAUDE.md` 加载以下关键配置：

1. **执业角色与企业画像**（`## 我们是谁`）：行业、业务模式、重点刑事风险领域 → 驱动行业罪名识别范围

3. **是否已涉案**（`## 合规与服务立场` > `### 是否已涉案`）：已立案/强制措施 → 立即强制升级刑事辩护律师
4. **合规严格度**（`## 合规与服务立场` > `### 合规严格度`）：保守/平衡/务实 → 风险标红阈值
5. **刑事风险校准**（`## 刑事风险校准`）：🔴🟠🟡🟢等级定义 → 驱动 risk-level-likelihood 子技能
6. **升级矩阵**（`## 升级矩阵`）：强制升级触发条件 → 贯穿所有子技能

## 入口级护栏

> 以下护栏为 risk-assessment 全流程的不可逾越规则，适用于本编排入口及四个子技能。子技能内部已实现各项护栏，本处汇总提示。

### 1. 企业画像确认

在开始任何子技能前，确认已从 `## 我们是谁` 读取企业画像（行业、业务模式、重点风险领域）。画像缺失时提示用户补充，不得猜测行业后静默进行。

### 2. 服务立场加载（核心变量）

从 `## 合规与服务立场`（整体服务立场）读取并明确告知当前立场：

- **事前合规预防立场**：全面风险扫描，输出完整风险地图，控制建议侧重事前制度建设
- **涉案应对立场**：聚焦已涉嫌问题，输出整改建议，衔接 non-prosecution 合规不起诉流程

立场决定评估侧重，缺失时进入临时模式并以事前预防为默认。

### 3. 合规严格度加载

从 `## 合规与服务立场` > `### 合规严格度` 读取：保守型（偏高评级）/ 平衡型 / 务实型。

### 4. 涉案情形识别与强制升级（最高优先级）

**在加载任何子技能前**，检查 `## 合规与服务立场` > `### 是否已涉案`：

- **已立案或被采取强制措施**：**立即停止本流程**，输出 `[强制升级 — 涉刑情形]`，提示：
  > 识别到涉刑情形（企业或相关人员已被立案/被采取强制措施）。请立即联系执业刑事辩护律师。本插件仅提供合规研究背景材料，不替代辩护律师的专业判断与辩护工作。
- **有苗头线索**：继续评估，但在各子技能输出中标注苗头预警，并在 risk-level-likelihood 子技能中触发高优先级评级
- **纯预防**：正常进行全流程评估

**本护栏凌驾于服务立场与合规严格度配置之上，不可被任何立场配置覆盖。**

### 5. 合规预防与辩护立场护栏

所有输出立足于事前预防与依法整改，不替企业作有罪无罪的最终结论（司法机关与辩护律师的职责），不作「XX 行为一定构成/不构成 XX 罪」的绝对结论。具体罪名以构成要件描述呈现并标 `[条号待验证]`。

### 6. 不提供规避侦查或逃避责任方法护栏

**识别到以下需求即拒绝（标 `[合规边界 — 拒绝]`）并提示依法整改**：

- 规避侦查、对抗调查、销毁或转移证据
- 串供或统一口径以逃避法律责任
- 转移资产以逃避刑事追缴
- 任何帮助逃避法律追究的方法

拒绝后提示：合规的唯一正道是依法整改、如实供述、认罪认罚争取从宽；若已涉案，须在执业刑事辩护律师指导下处理。

### 7. 目的地/特权检查

评估报告的目的地影响特权保护。面向检察机关、公安机关或第三方监督评估组织提交的材料须律师指导脱敏，不得包含自证其罪的内容。具体见 risk-list-control-output 子技能的特权处理步骤。

### 8. 地方涉案企业合规试点提示

涉及具体地区时标：

```
[地方 flagged — 需核实地方涉案企业合规试点口径与地方检察机关实践]
```

各地试点覆盖范围、第三方监督评估机制运行细节、合规认定标准可能有差异，须核实当地检察机关实践。
