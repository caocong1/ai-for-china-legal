# 风险与升级矩阵访谈 — 冷启动访谈第三部分

## 目的

本访谈子技能采集企业刑事合规工作的**风险等级定义体系与强制升级规则**，包括：刑事风险等级定义（🔴关键/🟠高/🟡中等/🟢低）、各角色的权限边界、自动/强制升级触发条件（已立案/强制措施/具体罪名风险/重大涉案须强制升级刑事辩护律师）、以及涉案应对的合规边界（绝不规避侦查/伪造毁灭证据/逃避责任）。

采集结果写入 `criminal-compliance/CLAUDE.md` 以下章节：
- `## 刑事风险校准`（风险等级定义与触发情形）
- `## 升级矩阵`（角色权限表 + 强制升级触发条件 + 非律师重大步骤升级表）
- `## 共享护栏`（涉刑强制升级护栏 + 合规边界护栏）

**可独立触发**：本子技能可单独调用（`/criminal-compliance:risk-escalation`）以单独更新风险校准章节（如风险等级定义调整），也可作为 `cold-start-interview` 编排入口的第三步顺序调用。

> **核心护栏提示**：本子技能采集的「强制升级触发条件」写入后，将凌驾于所有服务立场与合规严格度配置之上。任何情形下，识别到已立案/强制措施/可能构成具体罪名/重大涉案/涉及人身自由，一律触发强制升级给执业刑事辩护律师。

## 前置

**读取 `criminal-compliance/CLAUDE.md`：**

1. 读取 `## 刑事风险校准` — 确认是否已有定义（询问是否覆盖）
2. 读取 `## 合规与服务立场` > `### 是否已涉案` — **若已涉案，立即触发强制升级，仍可完成本子技能的风险定义采集以供后续技能使用**
3. 读取 `## 我们是谁` > 执业角色 — 角色权限表须匹配实际执业角色
4. 参照 `_shared/practice-profile-schema.md` 第五节「涉案情形与强制升级契约」

**涉案情形提示**：若已涉案已在 `compliance-stance-scope` 中确认，本子技能优先回显：

> `[强制升级 — 涉刑情形]` — 已涉案情形已记录，本子技能将帮助完善风险矩阵配置供律师参考，但不替代执业刑事辩护律师的辩护工作。

## 访谈问卷

### H 组：刑事风险等级定义

> 风险等级定义将嵌入所有刑事合规子技能的输出标注体系。

**H1.** 请确认以下**刑事风险等级**的定义，可调整各级触发条件：

```
默认定义（请确认或修改）：

🔴 关键 — 可能构成具体罪名、已立案或被采取强制措施；须立即处理，强制升级刑事辩护律师
   典型触发：已被立案/强制措施；有证据显示可能构成具体罪名；虚开发票规模大；非法集资已向公众募资；高管被调查

🟠 高 — 重大刑事风险敞口，高发罪名典型行为模式，尚未现实化但风险极大
   典型触发：采购销售存在商业贿赂行为模式；财务存在异常挪用迹象；安全生产事故频发；税收合规严重缺失

🟡 中等 — 合规制度缺失或不完善，一般风险点未有效控制
   典型触发：缺乏书面反腐败政策；合规培训未落实；审批流程缺失

🟢 低 — 已有合规控制措施且有效执行，风险可接受
   典型触发：已建立书面合规制度并有效执行；定期审计无重大发现

是否接受以上默认定义？（接受/需要调整，请说明需调整的级别与触发条件）
```

**H2.** 企业是否有**特别关注的刑事风险领域**，需要在风险等级触发时特别说明？

```
例如：
- 税收合规：任何虚开疑点即标🔴
- 商业贿赂：礼品价值超过 [X] 元即标🟠
- 安全生产：有隐患未整改即标🟠
（请结合合规严格度配置，保守型通常整体标高，务实型通常关注明显高风险）
```

```
|------|----------|-----------|

```

### J 组：强制升级触发条件

> **强制升级触发条件凌驾于服务立场与合规严格度之上，不可被任何配置覆盖。**

**J1.** 请确认以下**强制升级触发条件**（通常无需修改，仅需用户明确知悉）：

```
无论当前服务立场与风险等级，以下情形必须强制升级给执业刑事辩护律师：

1. 企业或相关人员已被立案（公安或检察机关立案）→ [强制升级 — 已立案]
2. 被采取强制措施（拘留/逮捕/取保候审/监视居住）→ [强制升级 — 强制措施]
3. 已收到侦查传唤/询问通知（公安/检察机关发出）→ [强制升级 — 侦查传唤]
4. 有明确迹象可能构成具体罪名（行为模式符合构成要件描述）→ [强制升级 — 具体罪名风险]
5. 重大涉案情形（涉及多名高管、金额巨大、社会影响大）→ [强制升级 — 重大涉案]
6. 涉及人身自由风险（任何可能导致相关人员被限制人身自由的情形）→ [强制升级 — 人身自由风险]

是否理解并确认上述强制升级规则？（确认/有疑问，请说明）
```

**J2.** 对于「有苗头线索」情形，希望设定哪些**预警提示条件**？（不触发强制升级，仅提示提高警惕）

```
例如：
- 已收到行政监管部门的调查通知 → 提示：密切关注，建议律师提前介入评估
- 内部发现可能触及刑事追诉门槛的违规行为 → 提示：立即内部调查，律师评估是否须升级
- 苗头持续超过 [X] 天未处理 → 提示：重新评估是否已升级为立案情形
```

### K 组：涉案应对合规边界

> 合规边界护栏是刑事合规工作的绝对底线，不可被任何立场或配置覆盖。

**K1.** 请确认以下**涉案应对合规边界**（这是本插件的硬性护栏，无法修改，仅需用户知悉）：

```
本插件绝不提供以下任何内容（合规边界——不可谈判）：

1. 规避侦查、对抗调查的具体方法或建议 → [合规边界 — 拒绝]
2. 伪造、毁灭、隐匿证据的建议 → [合规边界 — 拒绝]
3. 串供、统一口径以逃避刑事责任的方案 → [合规边界 — 拒绝]
4. 转移资产、隐匿财产以逃避刑事追缴的操作 → [合规边界 — 拒绝]
5. 帮助相关人员出逃或逃避法律追究的方法 → [合规边界 — 拒绝]
6. 任何以「合规」为名实质上干扰侦查的行为建议 → [合规边界 — 拒绝]

涉刑强制升级情形下的正确路径：
- 立即联系执业刑事辩护律师
- 在律师指导下，了解申请取保候审、阅卷权、辩护权等合法权利
- 如实供述、认罪认罚争取从宽（须在律师见证下进行）
- 积极合规整改，争取合规不起诉（须在律师主导下进行）

是否理解并确认上述合规边界？（确认）
```

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 用户要求修改合规边界护栏（如希望去除「不提供规避侦查方法」限制）| 拒绝修改，解释合规边界是系统固定护栏，不可被任何配置覆盖；这是本插件的核心安全机制 |
| 用户要求修改强制升级触发条件（如希望降低强制升级门槛）| 仅接受用户**提高**警惕（扩大强制升级范围），不接受降低强制升级门槛 |
| 角色权限表与实际情况不符（如合规官独立处理范围须扩大）| 按用户需求合理调整，但提示：扩大非律师角色的独立处理范围可能增加法律风险 |
| 已涉案情形下运行本子技能 | 优先触发强制升级提示，仍可完成风险矩阵配置以供律师参考 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 识别到已立案/强制措施（无论在何步骤）| 立即触发 `[强制升级 — 涉刑情形]`，提示联系执业刑事辩护律师 |
| 用户提出规避侦查需求 | 拒绝，标 `[合规边界 — 拒绝]`，提示正确路径：联系律师 + 如实供述 + 认罪认罚 |
| 写入 CLAUDE.md 失败 | 将采集结果以文本块返回，提示用户手动填入 |
| 用户不愿确认合规边界 | 明确说明：合规边界是本插件的运行前提条件，用户理解才可继续使用 |
