# 合规立场与服务范围访谈 — 冷启动访谈第二部分

## 目的

本访谈子技能采集刑事合规工作的**核心服务立场配置**，包括整体服务立场（事前合规预防 vs 涉案后应对）、合规严格度（保守/平衡/务实）以及**是否已涉案**（这是刑事合规工作最关键的状态字段，直接驱动涉刑强制升级规则）。

采集结果写入 `criminal-compliance/CLAUDE.md` 的以下章节：
- `## 合规与服务立场`（整体服务立场）
- `## 合规与服务立场` > `### 合规严格度`
- `## 合规与服务立场` > `### 是否已涉案`

**可独立触发**：本子技能可单独调用（`/criminal-compliance:compliance-stance-scope`）以单独更新立场章节（如服务立场发生变化），也可作为 `cold-start-interview` 编排入口的第二步顺序调用。

> **重要说明**：**「是否已涉案」是本子技能的核心采集字段**。涉案状态一经确认为「已立案或被采取强制措施」，本子技能将立即触发强制升级护栏，并在写入后同步触发所有技能的强制升级规则。

---

## 前置

**读取 `criminal-compliance/CLAUDE.md`：**

1. 读取 `## 合规与服务立场` — 确认是否已有配置（询问是否覆盖）
2. 读取 `## 我们是谁` — 确认执业角色与服务事项（与本访谈立场采集一致性检查）
3. 参照 `_shared/practice-profile-schema.md` 第四节「服务立场选择」确认采集范围

**涉案情形识别（最高优先级）**：

> **在开始立场采集前，请先确认**：企业或相关人员是否已被立案、被采取强制措施（拘留/逮捕/取保候审/监视居住）、已收到侦查传唤/询问通知，或有明确信息显示可能被立案？
>
> 若有上述情形，我将立即提示 `[强制升级 — 涉刑情形]` 并提供后续操作建议。

## 访谈问卷

### D 组：整体服务立场采集

> **服务立场是刑事合规工作最核心的变量**。同一企业从「事前预防」与「涉案应对」视角，工作侧重、法律依据与输出形式截然不同。

**D1.** 本次刑事合规工作的**整体服务立场**是什么？

```
选项（单选，可加备注）：
1. 事前刑事风险评估 — 全面识别企业刑事风险敞口，输出风险地图与控制建议（无现实刑事风险）
2. 事前合规体系建设 — 以预防刑事风险为目标设计合规制度、流程、培训、组织架构
3. 涉案企业合规不起诉 — 企业已被追诉，争取通过合规整改获得检察机关不起诉处理（须律师主导）
4. 单位犯罪预防 — 聚焦防范单位犯罪构成（刑法第30/31条双罚制，建议复核），尤重高管责任
5. 单位犯罪辩护研究 — 企业或个人已面临单位犯罪追诉，辩护策略研究（须执业刑事律师主导）
6. 混合多类型 — 同时涉及上述多类型（请说明主次）

追问：若选「3」「5」，请确认「是否已涉案」字段（D3）——这些立场通常伴随已涉案情形，须触发强制升级评估。
```

**D2.** 对于事前合规工作，服务侧重是哪一方面？

```
选项（仅事前立场适用）：
1. 侧重合规风险识别（摸清企业刑事风险底图）
2. 侧重制度与流程设计（建立合规规程）
3. 侧重组织与培训（搭建合规团队与培训体系）
4. 侧重有效性评估（评估现有合规体系是否真正有效）
5. 全面体系建设（四个方面均需推进）
```

---

### E 组：是否已涉案（最关键字段）

> **此字段直接驱动涉刑强制升级规则，凌驾于服务立场与合规严格度配置之上。**

**E1.** 企业或相关人员目前的**涉案状态**是什么？

```
选项（单选）：
1. 纯预防——无任何现实刑事风险或涉案迹象（正常事前合规工作）
2. 有苗头线索——有被监管部门关注的迹象，尚未立案（如：税务稽查、环保检查、市场监管调查）
3. 已立案或被采取强制措施 → 立即触发强制升级（见下方护栏）
```

**E2.** 如选「2.有苗头线索」，请说明苗头的性质：

```
选项（可多选）：
a. 已收到行政监管部门的调查通知/询问函
b. 内部发现可能触及刑事追诉门槛的违规行为
c. 同行业发生相关刑事案件，企业存在类似风险暴露
d. 外部举报或投诉，可能被监管部门跟进
e. 其他（请说明）

→ 苗头线索情形提示：提高风险警惕，建议律师介入评估，密切关注是否升级为立案情形
```

**E3.** 如选「3.已立案或被采取强制措施」，请确认具体情形：

```
（强制升级触发，立即执行以下处置）：
a. 企业已被立案（公安或检察机关）
b. 相关人员被采取强制措施（拘留/逮捕/取保候审/监视居住）
c. 企业已收到侦查询问通知、调查传票
d. 有明确信息显示可能被立案

处置：
→ 输出 [强制升级 — 涉刑情形]，提示立即联系执业刑事辩护律师
→ 仍可写入涉案状态供后续 non-prosecution 技能读取
→ 路由 /criminal-compliance:non-prosecution 评估合规不起诉适用性
```

### F 组：合规严格度采集

> 合规严格度影响各子技能的风险标记强度（🔴🟠🟡）与建议措辞。

**F1.** 在合规风险处置上，倾向哪种**严格度**？

```
选项（单选）：
1. 保守型 — 一切疑点从严标红并升级，宁可过度谨慎也不放过风险
   适用：已有历史违规记录 / 行业监管严格 / 不起诉申请阶段 / 合规严格文化要求
2. 平衡型 — 重大刑事风险（可能触及具体罪名）标红，一般风险点标橙，优先合规整改
   适用：日常合规体系建设 / 预防立场 / 中等规模企业通用
3. 务实型 — 仅拦截明显高风险（有清晰刑事追诉迹象），一般风险关注可行控制措施
   适用：资源有限的小型企业 / 风险已全面识别后的持续监控阶段

→ 默认值（如未提供）：保守型（合规工作的安全基准）
```

**F2.** 以下情形的严格度是否有特别要求？（可选填）

```
a. 商业贿赂/礼品娱乐：保守型（零容忍）/ 平衡型 / 务实型
b. 税收合规：保守型 / 平衡型 / 务实型
c. 安全生产：保守型 / 平衡型 / 务实型
d. 数据合规：保守型 / 平衡型 / 务实型
e. 其他（请说明）
```

### G 组：服务立场补充说明

**G1.** 服务立场说明（已在 D1 选择后需进一步确认的事项）：

```
若选「事前合规体系建设」：
- 是否已有现有合规制度？（是，需要评估改进 / 否，从零开始建立）
- 合规体系建设的主要驱动因素：董事会要求 / 监管要求 / 主动预防 / 商业合规认证要求

若选「涉案企业合规不起诉」：
- 是否已有辩护律师？（是 / 否，目前依赖本插件初步了解）
- 检察机关是否已启动合规考察程序？（是 / 否 / 不确定）`[待验证]`

若选「单位犯罪辩护研究」：
- 是否已有辩护律师主导？（是 / 否）→ 若否，强制升级提示
```

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 服务立场选择「涉案不起诉」但是否已涉案选「纯预防」 | 提示矛盾：「涉案不起诉」立场通常对应已涉案情形，请确认实际状态 |
| 是否已涉案字段用户回答含糊（如「可能」「也许」） | 请求明确确认；保守处理：含糊回答默认触发涉案情形评估，提示联系律师 |
| 合规不起诉立场但无辩护律师 | 提示：合规不起诉申请须在执业刑事律师主导下进行，建议立即委托律师，本插件提供背景研究 |
| 服务立场中途变更（如预防立场变为涉案应对） | 可重新运行本子技能更新「是否已涉案」字段；立场变更将影响所有后续技能行为 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 确认已立案或被采取强制措施 | 立即触发 `[强制升级 — 涉刑情形]`；提示联系执业刑事辩护律师；路由 `/criminal-compliance:non-prosecution` |
| 用户提供具体罪名条号或数额 | 标 `[待验证]`，提示以构成要件描述方式记录，建议核实 |
| 写入 CLAUDE.md 失败 | 将采集结果以文本块返回，提示用户手动填入 |
| 服务立场与 D1 与现有配置冲突 | 提示冲突，询问是否覆盖现有立场配置 |
