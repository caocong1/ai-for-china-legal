# 身份与企业画像访谈 — 冷启动访谈第一部分

## 目的

本访谈子技能采集刑事合规工作的**基础身份信息与企业画像**，包括执业角色（律师/合规官/法务/当事人本人）、服务对象（企业侧合规预防/涉案企业辩护）、企业行业与规模与业务模式与历史涉案情况与重点刑事风险领域，以及可用的检察/公安/监管/第三方监督评估资源。

采集结果写入 `criminal-compliance/CLAUDE.md` 的以下章节：
- `## 我们是谁`（执业角色 + 企业画像）
- `## 谁在使用`（使用角色 + 律师联系人）
- `## 可用集成`（检察/公安/监管/第三方资源）

**可独立触发**：本子技能可单独调用（`/criminal-compliance:identity-enterprise-profile`）以单独更新画像章节，也可作为 `cold-start-interview` 编排入口的第一步顺序调用。

---

## 前置

**读取/创建 `criminal-compliance/CLAUDE.md`：**

1. 检查文件是否存在：
   - 若存在：读取 `## 我们是谁` 章节，确认是否含 `[PLACEHOLDER]`（需要更新）或已有内容（询问是否覆盖）
   - 若不存在：告知用户将创建文件并填充章节
2. 确认 `practice-profile-schema.md` 字段映射契约：本访谈采集的字段对应 `## 我们是谁` / `## 谁在使用` / `## 可用集成`
3. 参照 `_shared/practice-profile-schema.md` 第二节字段映射表确认采集范围

**涉案情形识别（优先检查）**：在访谈第一个问题前，先确认：

> 在开始画像采集前，请告知：企业或相关人员是否**已被立案、被采取强制措施（拘留/逮捕/取保候审/监视居住）、已收到侦查传唤/询问通知**？
>
> 若有上述情形，我将立即提示升级至执业刑事辩护律师，并暂停访谈。

## 访谈问卷

### A 组：执业角色采集

> 以下问题帮助确定您的执业角色，影响后续所有技能的输出定性、特权标题选择与升级路由。

**A1.** 您在本次刑事合规工作中的**主要角色**是什么？

```
选项（单选）：
1. 刑事合规律师（律所执业律师，承接本案）
2. 企业合规官（企业内部合规管理负责人）
3. 企业法务（企业内部法律事务部门）
4. 当事人本人（有执业律师可咨询）
5. 当事人本人（目前无执业律师，寻求初步了解）
6. 非律师，但有律师联系人（如财务总监、CEO等）
7. 其他（请说明）

→ 默认值（如未提供）：律师（最高权限，降低误升级风险）
```

**A2.** 您的**主要服务事项**是什么？（可多选）

```
选项：
1. 事前刑事风险评估（识别企业刑事合规风险敞口）
2. 事前合规体系建设（预防机制设计）
3. 涉案企业合规不起诉（企业已被追诉，争取从宽）
4. 单位犯罪预防（防范单位犯罪构成与双罚制责任）
5. 单位犯罪辩护研究（已面临追诉，须律师主导）
6. 混合多类型（同时涉及上述多类型）
```

**A3.** 如果您是律师，您的律师事务所名称或业务部门是什么？（选填，用于输出标题与特权标注）

---

### B 组：企业画像采集

> 以下问题采集企业基本信息，用于识别行业高发罪名、合规重点领域与监管资源。

**B1.** 服务的**企业/组织名称**是什么？（可匿名，如「某制造企业」）

**B2.** 企业所在**行业**是什么？

```
参考选项（请选择最接近的）：
- 制造业（含食品、电子、化工、机械）
- 金融投资（含银行、证券、私募、P2P）
- 医药健康（含药企、医疗器械、医院）
- 房地产与建设工程（含开发、施工、监理）
- 能源化工（含石油、矿产、电力）
- 互联网与数据（含平台、数据服务、AI）
- 贸易与物流（含进出口、供应链）
- 国有企业（涉职务犯罪合规风险）
- 政府相关单位（事业单位）
- 其他（请说明）
```

**B3.** 企业**规模**如何？（员工人数 + 年营业额）

```
参考：
- 小型（50人以下，年营业额1亿元以下）
- 中型（50-500人，年营业额1-10亿元）
- 大型（500人以上，年营业额10亿元以上）
- 上市公司（A股/港股/美股，请注明）
- 国有企业（央企/地方国企，请注明）
```

**B4.** 企业**主要业务模式**是什么？（如：制造+出口 / 金融理财产品销售 / 工程建设承包 / 医药代表+医院渠道 / 互联网平台数据业务）

**B5.** 企业是否有**涉外业务**？

```
选项：
1. 是（跨境贸易/境外投资/外资股东）
2. 否
3. 部分涉外（如部分出口业务）
```

**B6.** 企业是否有**历史涉案记录**？

```
选项：
1. 无历史涉案
2. 曾受行政处罚（请说明领域：税务/市场监管/环保/安全生产/其他）
3. 曾有刑事案件涉及（请说明情况）
4. 不确定/待评估
```

**B7.** 企业目前**已知的重点刑事风险领域**有哪些？（可多选，依行业自行判断）

```
选项：
1. 税收合规（虚开增值税专用发票/逃税风险）
2. 反商业贿赂（采购/销售回扣/行贿政府官员风险）
3. 金融融资合规（非法集资/非法吸收公众存款风险）
4. 数据与个人信息合规（侵犯公民个人信息风险）
5. 安全生产合规（重大责任事故风险）
6. 环境合规（污染环境罪风险）
7. 反垄断与招投标合规（串通投标/垄断风险）
8. 单位职务类风险（职务侵占/挪用资金）
9. 暂不清楚，需要全面梳理
10. 其他（请说明）

→ 若选「暂不清楚」，建议完成画像后运行 `/criminal-compliance:compliance-risk-identification`
```

**B8.** 企业的**主要合规痛点**是什么？（可简述，帮助确定后续技能侧重）

**B9.** 负责刑事合规工作的**团队规模**？（如：刑事合规律师X人 + 合规官X人 / 法务部X人）

---

### C 组：可用集成与资源采集

> 以下问题采集检察/公安/监管/第三方资源的可用性，影响合规不起诉路径与监管对接选择。

**C1.** 是否已与**当地人民检察院**建立联系？（如有，请提供基本信息）

```
选项：
1. 已确认联系（请填写：XX区/市人民检察院，联系人/联系方式）
2. 未确认，需要建立联系
3. 暂不适用（纯预防立场，暂无需对接检察机关）
```

**C2.** 是否已与**公安经济犯罪侦查部门**建立联系？

```
选项：
1. 已确认（请填写：XX市公安局经侦支队/总队）
2. 未确认
3. 不适用
```

**C3.** 企业涉及的**行业监管部门**有哪些？

```
参考选项（可多选）：
- 税务机关（税务合规/逃税行刑衔接）
- 市场监督管理局（竞争/假冒伪劣行刑衔接）
- 中国证监会（证券市场行刑衔接）
- 国家药品监督管理局（药品医疗器械行刑衔接）
- 生态环境局（污染环境罪行刑衔接）
- 应急管理局（安全生产行刑衔接）
- 国家互联网信息办公室（数据合规行刑衔接）
- 其他（请说明）
```

**C4.** 是否了解**当地第三方监督评估机制管理委员会**的情况？

```
选项：
1. 已知管委会联系方式（请填写）
2. 尚未接触，可能需要
3. 不适用（纯预防，尚未涉案）
```

**C5.** 是否已委托**司法鉴定/会计审计机构**？

```
选项：
1. 已委托（请填写机构名称）
2. 未委托，可根据需要委托
3. 不适用
```

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 用户不愿提供企业名称 | 接受匿名（如「某制造企业」），不强制要求 |
| 访谈过程中识别已涉案迹象 | 立即中止画像采集，触发 `[强制升级 — 涉刑情形]` |
| B6 历史涉案：已有刑事案件 | 提示：涉案企业的合规工作须在律师指导下进行；询问是否已有律师，提示运行 `compliance-stance-scope` 采集涉案状态 |
| 用户直接提供企业画像文件 | 解析文件内容填充各字段，向用户确认关键字段后写入 |

---

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 识别到已涉案情形 | 立即触发 `[强制升级 — 涉刑情形]`，停止访谈，提示联系执业刑事辩护律师 |
| 用户提供具体罪名条号 | 标 `[待验证]`，提示以构成要件描述方式记录，建议通过国家法律法规数据库核实 |
| 写入 CLAUDE.md 失败 | 将采集结果以文本块形式返回，提示用户手动填入 `criminal-compliance/CLAUDE.md` 对应章节 |
| 用户中途退出访谈 | 保存已采集内容，提示可以单独重新运行本子技能或继续后续子技能 |
