# 文书风格与监管资源访谈 — 冷启动访谈第四部分

## 目的

本访谈子技能采集刑事合规工作的**文书风格偏好与监管资源配置**，包括刑事风险评估报告风格、合规制度文本风格、合规不起诉申请与整改报告风格、辩护研究风格，以及检察/公安/行业监管/第三方监督评估的本地资源信息与地方涉案企业合规试点的口径偏好。

采集结果写入 `criminal-compliance/CLAUDE.md` 以下章节：
- `## 文书风格`（各类文书风格偏好）
- `## 输出`（工作产物标题格式）
- `## 共享护栏` > `### 地方涉案企业合规试点提示`（地方试点口径与偏好）

**可独立触发**：本子技能可单独调用（`/criminal-compliance:style-regulatory-resources`）以单独更新文书风格配置（如新增业务类型），也可作为 `cold-start-interview` 编排入口的第四步顺序调用。

---

## 前置

**读取 `criminal-compliance/CLAUDE.md`：**

1. 读取 `## 文书风格` — 确认是否已有配置（询问是否覆盖）
2. 读取 `## 我们是谁` > 执业角色（影响工作产物标题选择：律师角色 vs 非律师角色）
3. 读取 `## 合规与服务立场` > 整体服务立场（影响主要文书类型的侧重）
4. 读取 `## 可用集成` — 确认检察/公安/监管机构已采集情况（本子技能补充地方试点口径）

**涉案情形检查**：读取 `## 合规与服务立场` > `### 是否已涉案`：
- 若已涉案：优先采集合规不起诉申请与整改报告风格，并重点采集检察机关与第三方监督评估的本地口径

## 访谈问卷

### L 组：文书风格采集

> 文书风格影响所有子技能的输出格式、措辞风格、详细程度与引用注释密度。

**L1.** 刑事风险评估报告的**风格偏好**是什么？

```
选项（可多选，请说明偏好组合）：
1. 客观、风险导向、可操作 — 逐项列示风险点+罪名构成要件对照+风险等级+控制建议；避免主观认定是否构成犯罪
2. 执行摘要优先 — 首先呈现关键风险红旗，详细分析置后；适合高管阅读
3. 监管合规框架 — 结合行政监管要求（行刑衔接）与刑事风险，一体呈现
4. 法律审计风格 — 详细条文引用（构成要件描述+条号待验证）+充分说明理由+法律依据分类
5. 简洁实用型 — 去掉冗余说明，直接列表输出「风险点/等级/建议」，适合律师内部使用

默认（如未提供）：客观、风险导向、可操作；逐项列示；构成要件描述+条号待验证
```

**L2.** 合规制度文本（合规政策/行为准则/专项制度）的**风格偏好**是什么？

```
选项（可多选）：
1. 规范、可落地 — 制度条文明确、操作指引具体、禁止性行为清单详细；含审批权限与责任追究
2. 简洁实用型 — 减少说明性文字，突出操作要求，适合中小企业
3. 对标行业最佳实践 — 参考上市公司或行业领头企业合规制度结构，体现专业性
4. 中英双语（如涉外企业）
5. 法律技术型 — 条文严谨，含法律依据注释（适合律所内部工作产物）

默认（如未提供）：规范、可落地；制度条文明确；含审批权限与责任追究
```

**L3.** 合规不起诉申请与整改报告的**风格偏好**是什么？

```
选项（可多选）：
1. 务实、可验证、配合检察机关 — 完整记录整改措施+进度+可验证成果；真实客观，不夸大整改成效
2. 结构化呈现 — 按整改领域分类（制度/培训/人员/流程），分别陈述整改情况
3. 突出「有效合规」要素 — 强调第三方评估认可的整改维度（组织保障/制度健全/执行有效，待验证 `[待验证]`）
4. 与认罪认罚从宽制度衔接 — 在报告中体现认罪认罚情节与整改互补关系

默认（如未提供）：务实、可验证；真实客观；配合检察机关
```

**L4.** 辩护研究文书的**风格偏好**是什么？

```
选项（可多选）：
1. 严谨、以构成要件为框架 — 区分单位犯罪与个人犯罪；梳理直接负责主管人员与其他直接责任人员的责任边界
2. 量刑情节导向 — 重点分析自首/立功/认罪认罚/合规整改等从宽情节（中度把握锚点标建议复核）
3. 证据清单型 — 梳理现有证据对各构成要件的支撑或缺失
4. 辩护策略研究型 — 提供多种辩护角度分析（须律师主导，本插件仅提供研究参考）

默认（如未提供）：严谨；以构成要件为分析框架；须标注「需律师主导」
```

**N2.** 是否已了解**当地检察机关对涉案企业合规的具体口径**？

```
选项：
```

## 边界条件

| 边界情形 | 处理方式 |
|---------|---------|
| 用户要求辩护研究文书中不标注「须律师主导」 | 拒绝删除该字样，解释：辩护研究须律师主导是法律执业规范要求，也是本插件的合规边界 |
| 地方涉案企业合规试点口径信息不全 | 提示标 `[地方 flagged — 需核实]`，建议联系当地检察机关了解具体口径；不臆造地方口径 |
| 用户要求不添加审查者注释块 | 审查者注释块为必要输出格式（含待验证标注），不可完全删除；可简化非关键字段 |
| 多个经营地区（跨省业务）| 列出主要地区，各地标 `[地方 flagged]`，提示分别核实各地口径差异 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 识别到已涉案情形 | 触发 `[强制升级 — 涉刑情形]`；重点采集合规不起诉申请风格与地方检察机关口径 |
| 用户要求删除「具体罪名条号待验证」的审查者注释 | 拒绝，这是本插件的核心法律准确性护栏，不可删除 |
| 写入 CLAUDE.md 失败 | 将采集结果以文本块返回，提示用户手动填入 |
| 地方试点口径用户主动提供的具体文号 | 标 `[待验证 — 规范性文件快速演进]`，提示文号快速演进须核实现行版本 |
