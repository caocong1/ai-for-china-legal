# 刑事合规实践配置文件契约（Practice Profile Schema）

> 本文件定义 criminal-compliance 插件的实践配置文件：**是什么、在哪、技能怎么读**。
> 刑事合规版改编自 family-legal/skills/_shared/practice-profile-schema.md，结合刑事合规语境特殊化（合规预防与辩护立场护栏、不提供规避侦查方法护栏、涉刑强制升级契约、合规边界契约）。

---

## 一、配置文件是什么 / 真相来源

**唯一真相来源：`criminal-compliance/CLAUDE.md`（散文模板）**

- 本阶段（09-01）**首次新建**，由 `/criminal-compliance:cold-start-interview` 冷启动访谈填充和更新
- 律师/合规官/法务可直接编辑该文件，无需重新运行访谈
- 所有 criminal-compliance 技能从 `criminal-compliance/CLAUDE.md` 读取实践配置，不从其他来源读取

**架构决策（不引入冲突新格式）**：

> 本插件沿用 CLAUDE.md 散文模板作为单一真相来源，不引入独立 JSON 或 YAML 格式的配置文件。理由：
> 1. 散文模板对刑事合规律师/合规官/法务/当事人更直观、可读，直接编辑体验好
> 2. 与 commercial-legal / construction-legal / employment-legal / family-legal 等插件架构一致，维护成本低
> 3. 避免 CLAUDE.md 与独立 YAML/JSON 文件的双重真相来源冲突

**纠正现有访谈表述**：

现有 `criminal-compliance/skills/cold-start-interview/SKILL.md` 末尾「生成 刑事合规 实践配置文件（YAML 格式）」的表述应理解为**填充 `criminal-compliance/CLAUDE.md` 对应章节**，而非另存独立 YAML 文件。技能和用户都应将访谈结果直接写入 CLAUDE.md，不得创建单独的 `practice-profile.yaml` 等文件。

## 三、技能读取契约

### 3.1 标准读取流程

每个 criminal-compliance 技能在执行实质性工作前，**必须**按以下顺序读取配置：

1. **读取 `criminal-compliance/CLAUDE.md`**
2. **检查关键字段是否含 `[PLACEHOLDER]`**：
   - `## 我们是谁` 的「执业角色」字段
   - `## 合规与服务立场` 的「整体服务立场」字段
   - `## 合规与服务立场` > `### 是否已涉案` 字段（刑事合规特别关键）
   - `## 谁在使用` 的「角色」字段
3. **据此选择运行模式**（正常模式 or 临时模式，见下节）
4. **检查是否已涉案**：如识别到已涉案情形，无论其他配置，立即触发强制升级

### 3.2 配置缺失/含占位符时的标准提示

当 `criminal-compliance/CLAUDE.md` 不存在或关键字段含 `[PLACEHOLDER]` 时，技能输出以下标准提示：

```
> **配置提示**：未检测到完整的刑事合规实践配置（criminal-compliance/CLAUDE.md 中关键字段含占位符或文件不存在）。
>
> 如需个性化配置，请运行 `/criminal-compliance:cold-start-interview` 完成冷启动访谈，将结果填入 criminal-compliance/CLAUDE.md。
>
> 或者，告诉我你的服务立场（事前预防/涉案应对）和是否已涉案状态，我将以「临时模式」继续当前任务。
>
> **临时模式安全警示**：临时模式下，涉刑强制升级护栏与合规边界护栏默认最高。如存在已立案、强制措施、可能构成具体罪名或重大涉案情形，请立即联系执业刑事辩护律师，不要继续依赖本工具处理涉案事项。
```

### 3.3 临时模式默认值表

| 配置字段 | 临时模式默认值 | 说明 |
|---------|-------------|------|
| 整体服务立场 | 事前刑事风险评估（预防立场） | 最安全的默认值；涉案应对须用户明确说明 |
| 执业角色 | 律师（最高权限） | 降低误升级风险；非律师须用户明确说明 |
| 合规严格度 | 保守型 | 临时模式下所有疑点从严标红 |
| 是否已涉案 | **按用户陈述逐案判断** | **不默认「无涉案」——临时模式下一旦识别任何涉案迹象即强制升级，不等待配置确认** |
| 行业刑事风险领域 | 按企业实际陈述逐案判断 | 不预设行业 |
| 刑事风险校准等级 | 保守标注（偏高不偏低） | 临时模式下未知情形偏保守 |

临时模式下所有输出标 `[临时模式 — 请运行冷启动访谈完成配置]`。

## 四、服务立场选择

> **服务立场是刑事合规工作最核心的变量。** 缺失时须显式提示并默认事前合规预防。

### 4.1 服务立场与审查侧重

| 服务立场 | 审查侧重 | 风险标记侧重 | 主要输出 |
|---------|---------|------------|---------|
| **事前刑事风险评估** | 全面识别刑事风险敞口，风险地图 | 识别可能触发刑事追诉的行为模式，构成要件比对 | 刑事风险评估报告、风险清单、控制建议 |
| **事前合规体系建设** | 预防机制设计，制度制衡，流程嵌入 | 制度缺失与流程漏洞，合规控制有效性 | 合规制度草案、风险控制清单、培训方案 |
| **涉案企业合规不起诉** | 合规整改可行性，与检察机关协商，第三方评估配合 | 整改方案的可验证性，认罪认罚条件 | 合规整改计划、不起诉申请、整改报告 |
| **单位犯罪预防** | 单位犯罪构成要件，双罚制责任，内控防范 | 直接负责主管人员与其他直接责任人员的责任风险 | 单位犯罪预防分析、内控强化建议 |
| **单位犯罪辩护研究** | 犯罪构成要件比对，责任切割，量刑情节 | 认罪认罚从宽条件，自首立功情节 | 辩护研究报告（须律师主导） |

### 4.2 服务立场缺失时的处理

技能在工作前确认服务立场（从 `## 合规与服务立场` 读取）：
- 若含 `[PLACEHOLDER]` 或未配置，须显式提示「当前服务立场未配置，将以临时模式（事前合规预防立场）继续，如需针对涉案场景，请运行冷启动访谈或直接告知立场」
- 不得静默假定服务立场
- 同时确认「是否已涉案」字段，若显示「已立案或被采取强制措施」，无论服务立场如何，一律触发强制升级

### 4.3 护栏不可被立场覆盖

无论服务立场配置为何，以下护栏**不可被任何立场覆盖**：

- 合规预防与辩护立场护栏（不替企业作有罪无罪最终结论）
- 不提供规避侦查或逃避责任方法护栏（一律拒绝，标 `[合规边界 — 拒绝]`）
- 涉刑强制升级护栏（识别已涉案情形一律强制升级，标 `[强制升级 — 涉刑情形]`）
- 律师审查护栏（重大法律后果步骤前非律师须升级）

## 五、涉案情形与强制升级契约（刑事合规特有）

### 5.1 强制升级触发情形

技能识别到以下任一情形时，无论服务立场与风险等级，**一律强制升级执业刑事辩护律师**：

| 情形 | 识别方式 | 输出标记 |
|------|---------|---------|
| 企业或相关人员已被立案 | 用户陈述、材料记录 | `[强制升级 — 已立案]` |
| 被采取强制措施（拘留/逮捕/取保/监视居住） | 用户陈述、材料 | `[强制升级 — 强制措施]` |
| 已收到侦查传唤/询问通知 | 用户陈述、文书材料 | `[强制升级 — 侦查传唤]` |
| 可能构成具体罪名（明确风险迹象） | 用户陈述的行为模式符合构成要件描述 | `[强制升级 — 具体罪名风险]` |
| 重大涉案情形（多名高管/金额巨大） | 用户陈述、案情描述 | `[强制升级 — 重大涉案]` |
| 涉及人身自由风险 | 任何可能导致相关人员被限制人身自由的情形 | `[强制升级 — 人身自由风险]` |

### 5.2 强制升级后的输出范围

触发强制升级后，技能输出须：
1. **明确标注 `[强制升级 — 涉刑情形]`**
2. **提示立即联系执业刑事辩护律师**，本插件不替代辩护工作
3. **可提供**：初步合规分析（作为辩护律师参考的背景资料）、合规整改研究（供律师指导下实施）
4. **不得提供**：辩护策略的最终判断、认罪认罚具结书起草（须在律师见证下由当事人自愿完成）、向司法机关提交任何材料的最终建议

### 5.3 强制升级契约的优先级

**本契约凌驾于服务立场配置之上**，且本插件不替代辩护律师的专业判断与辩护工作。即使配置中服务立场为「单位犯罪辩护研究」，技能仍须在强制升级后提示律师主导，不得自行作出辩护策略的最终判断。

## 六、合规边界契约（刑事合规特有）

### 6.1 合规边界触发情形

技能识别到用户需求涉及以下情形时，**一律拒绝并提示**：

| 需求类型 | 识别关键词/情形 | 处置方式 |
|---------|--------------|---------|
| 规避侦查、对抗调查 | 「不想被查到」「如何应对突击检查」「怎么隐藏」 | 拒绝，标 `[合规边界 — 拒绝]` |
| 伪造、毁灭、隐匿证据 | 「删除记录」「销毁文件」「修改账目」 | 拒绝，标 `[合规边界 — 拒绝]` |
| 串供、统一口径 | 「和相关人员对好口径」「统一陈述」 | 拒绝，标 `[合规边界 — 拒绝]` |
| 转移资产逃避追缴 | 「转移财产」「隐匿账户」「境外转移」 | 拒绝，标 `[合规边界 — 拒绝]` |
| 帮助相关人员逃逸 | 「出境」「躲避」「不出庭」 | 拒绝，标 `[合规边界 — 拒绝]` |

### 6.2 合规边界拒绝后的提示

拒绝后提示：

> **合规边界**：刑事合规的核心是依法整改、如实供述、配合调查、认罪认罚争取从宽，而非规避法律追究。上述请求不在本插件的服务范围内。
>
> 如您正面临刑事追诉，请立即联系执业刑事辩护律师，由律师在合法范围内保障您的权利（如申请取保候审、阅卷权、辩护权等），而非采取任何可能加重罪责的规避行动。
>
> 输出标 `[合规边界 — 拒绝]`。

### 6.3 合规边界契约的优先级

**本契约不可被任何立场或配置覆盖**，包括「单位犯罪辩护研究」立场。合规边界与合规预防立场护栏共同构成本插件不可逾越的底线。

## 七、重跑 / 编辑 / 版本控制

### 7.1 重跑冷启动访谈

- **完整重跑**：运行 `/criminal-compliance:cold-start-interview` 全部子技能，覆盖 `criminal-compliance/CLAUDE.md` 全部章节
- **单独更新某部分**：运行对应子技能（如「合规立场与服务范围」子技能），仅更新 `## 合规与服务立场` 章节

### 7.2 直接编辑

律师/合规官/法务可直接编辑 `criminal-compliance/CLAUDE.md`，无需重新运行访谈。编辑时须保持章节结构（`##` 标题）不变，避免破坏技能的章节定位读取。

## 八、法律事实合理性检查

当用户口述具体罪名条号、定罪量刑数额、期限（合规考察期、认罪认罚时限）等时，技能在写入配置前须进行合理性检查：

### 8.1 具体罪名条号

- **处理原则**：用户口述的具体罪名条号须标 `[待验证]`，以构成要件描述方式记录，不将裸条号写入配置
- **检查提示**：「你提到 XX罪 第XX条，刑法分则具体罪名条号须以现行刑法核实（刑法含多次修正案），建议以构成要件描述方式记录，标 `[待验证]`」

### 8.2 定罪量刑数额标准

- **处理原则**：数额标准来自司法解释，且经多次修订，须描述规则 + `[待验证]`，不写裸数额
- **检查提示**：「你提到 XX数额，该数额标准来自司法解释且经多次修订，须以「依相关司法解释，数额达一定标准」描述并标 `[待验证]`，不建议直接写入裸数额」

### 8.3 涉案企业合规规范性文件文号

- **处理原则**：文号快速演进，须描述规则 + `[待验证 — 规范性文件快速演进]`，不写具体文号
- **检查提示**：「涉案企业合规试点相关文件的文号快速演进，建议以规则描述方式记录，标 `[待验证]`，不建议写入具体文号」

### 8.4 矛盾与疑点澄清

- 用户口述的行为描述与典型构成要件不符时，标 `[前提 flagged — 需验证]`，提示澄清
- 用户口述「已涉案」但未明确说明涉案程度时，询问确认以触发正确的升级路由
