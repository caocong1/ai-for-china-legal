# 清权风险评估与清权报告（clearance-risk-report）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

本子技能是 ip-clearance 多维清权流程的**装配收口**，负责：

1. **多维清权风险分级**：汇总 trademark-clearance / patent-fto / copyright-tradename-domain-clearance 三个子技能的发现，按商标/专利/著作权/字号/域名各维度分别评级（🔴🟠🟡🟢）
2. **清权结论**：综合各维度最高风险，给出三档总结论：**✅ 可用 / ⚠ 有风险（可附条件使用或规避后使用）/ ❌ 不可用（建议放弃或重新设计）**
3. **清权报告组装**：按标准报告结构组装最终清权报告（清权对象/检索范围/各维度发现/风险分级/清权结论/规避建议/检索局限免责）

**清权结论决策原则**：各维度最高风险等级决定总结论（按清权严格度调整门槛）；任一维度🔴即触发全局❌；任一维度🟠触发全局⚠（附条件）；全部维度🟡/🟢触发✅（保守模式下🟡也触发⚠）。

**本子技能可由律师/IP 经理单独触发：`/ip-legal:clearance-risk-report`**

**本子技能也可由 ip-clearance 编排入口在所有上游清权子技能完成后调用。**

## 前置

### 读取实践配置

在开始清权报告组装前，读取 `ip-legal/CLAUDE.md`：

- **清权严格度**（`## 维权与申请立场` → `### 清权严格度`）：调整清权结论门槛（严格/平衡/宽松）
- **整体维权立场**（`## 维权与申请立场`）：驱动总结论的保守/宽松倾向
- **升级矩阵**（`## 升级矩阵`）：识别高风险维度是否需要自动升级律师处理
- **文书风格**（`## 文书风格` → 清权报告风格）：决定清权报告格式（结构化表格/叙述/摘要+附件）
- **执业角色**（`## 谁在使用`）：决定工作产物标题（律师工作产物/研究笔记）

**临时模式（配置缺失/含占位符时）：**

> 未检测到知识产权实践档案（或关键字段含占位符）。将使用临时模式默认值：清权严格度=保守（🟡中等风险也触发⚠）、整体立场=平衡型。输出标记 `[临时模式]`。

### 与其他子技能的分工

| 子技能 | 分工说明 |
|-------|---------|
| trademark-clearance | 提供商标维度清权发现（可注册性/可使用性初判/商标清权结论） |
| patent-fto | 提供专利维度 FTO 发现（落入保护范围风险/规避设计方向） |
| copyright-tradename-domain-clearance | 提供著作权/字号/域名维度清权发现 |
| **本技能（clearance-risk-report）** | 汇总所有维度发现，分维度评级，给出综合清权结论，组装清权报告 |

**单独触发场景**：律师/IP 经理已有各维度清权发现（来自其他渠道），直接提供给本技能组装报告并作综合风险评估。

## 详细步骤

### 第一步：收集上游清权输入

| 输入来源 | 必须/可选 | 内容要求 |
|---------|---------|---------|
| trademark-clearance 子技能输出 | 必须（若清权范围含商标） | 商标清权结论（✅/⚠/❌）+ 关键障碍商标清单 + 可使用性初判 |
| patent-fto 子技能输出 | 必须（若清权范围含专利） | FTO 初判结论（✅/⚠/❌）+ 高风险障碍专利清单 + 规避方向 |
| copyright-tradename-domain-clearance 子技能输出 | 必须（若清权范围含著作权/字号/域名）| 各维度清权结论（✅/⚠/❌）+ 具体障碍事项 |
| 用户直接提供的清权信息 | 备选（未运行上游子技能时）| 各维度清权发现摘要，本技能在此基础上评估 |

**若上游输入不完整**：明确说明哪些维度信息缺失，不得自行填补空白，且须在报告中声明评估范围受限。

### 第二步：多维清权风险分级

**分维度风险评级（四色评级系统）：**

**评级定义（与 `ip-legal/CLAUDE.md` → `## 风险校准` 对应）：**

| 等级 | 颜色 | 含义 | 清权场景典型情形 |
|-----|------|------|--------------|
| 🔴 | 关键/不可用 | 明显清权障碍，继续使用极大可能构成侵权或被追究 | 与有效注册商标相同；核心专利权利要求全覆盖且无规避可能；著作权完全复制且无授权 |
| 🟠 | 高风险/有障碍 | 有较大清权风险，须采取行动才能使用 | 商标高度近似且商品类似；专利等同侵权可能性大但有规避可能；字号与知名字号高度近似 |
| 🟡 | 中等风险/需关注 | 存在一定风险，建议进一步评估或采取保守措施 | 商标有一定近似但混淆可能性存疑；专利边缘侵权风险；开源协议有传染性风险 |
| 🟢 | 低风险/可接受 | 风险可接受或基本无障碍 | 商标检索无相同/高度近似结果；专利 FTO 未发现实质障碍；著作权来源合规 |

**多维清权风险汇总表：**

| 清权维度 | 关键发现 | 风险等级 | 是否已有规避方案 |
|---------|---------|---------|--------------|
| 商标（可注册性）| [关键障碍商标或「无障碍」] | [🔴/🟠/🟡/🟢] | [是/否/不适用] |
| 商标（可使用性）| [侵权风险商标或「无风险」] | [🔴/🟠/🟡/🟢] | [是/否/不适用] |
| 专利 FTO | [高风险障碍专利或「无障碍」] | [🔴/🟠/🟡/🟢] | [有规避方向/无规避方向] |
| 著作权（素材来源）| [问题素材或「来源合规」] | [🔴/🟠/🟡/🟢] | [是/否/不适用] |
| 企业字号 | [冲突字号或「无冲突」] | [🔴/🟠/🟡/🟢] | [是/否/不适用] |
| 域名 | [冲突商标/可用性或「无冲突」] | [🔴/🟠/🟡/🟢] | [是/否/不适用] |

### 第三步：清权结论决策矩阵

**清权结论根据各维度最高风险等级决定，按清权严格度调整（从 `ip-legal/CLAUDE.md` 读取）：**

| 清权严格度 | ✅ 可用条件 | ⚠ 有风险条件 | ❌ 不可用条件 |
|---------|-----------|-----------|------------|
| **严格** | 全部维度 🟢 | 任一维度 🟡 以上 | 任一维度 🟠 以上 |
| **平衡** | 全部维度 🟡/🟢 | 任一维度 🟠 | 任一维度 🔴 |
| **宽松** | 全部维度 🟡/🟢（可接受 🟠 须备注）| 任一维度 🔴 且有规避方案 | 任一维度 🔴 且无规避方案 |

**清权结论逻辑说明：**

> 清权结论的三档（✅/⚠/❌）并非对所有维度取平均，而是**取最高风险维度**决定总结论。类比木桶原则：一块短板决定整体清权结论。唯一例外是：若最高风险维度存在明确规避方案（如商标可修改设计绕开/专利有规避设计），可在⚠结论中说明「规避后可用」，但须说明规避方案的可行性评估。

### 第四步：规避或整改建议

对于⚠/❌结论，提供具体建议：

| 维度 | 障碍性质 | 规避/整改建议 | 难度评估 |
|-----|---------|------------|--------|
| 商标（注册） | [障碍说明] | [如：放弃申请该标识 / 修改显著部分绕开近似 / 等待障碍商标到期/撤销 / 申请异议清除障碍] | [低/中/高] |
| 商标（使用）| [障碍说明] | [如：修改标识设计 / 取得被侵权方授权许可 / 更改商品类别 / 律师评估后再用] | [低/中/高] |
| 专利 FTO | [障碍说明] | [如：规避设计（修改特征X）/ 取得专利许可 / 提无效宣告请求 / 重新设计方案] | [低/中/高] |
| 著作权素材 | [障碍说明] | [如：更换有授权的素材 / 取得著作权人授权 / 使用原创内容] | [低/中/高] |
| 字号 | [障碍说明] | [如：修改字号方案 / 增加独特性元素区分 / 律师评估不正当竞争风险] | [低/中/高] |
| 域名 | [障碍说明] | [如：更换域名核心词 / 评估是否构成恶意抢注] | [低/中/高] |

### 第五步：升级提示（高风险维度）

依据 `ip-legal/CLAUDE.md` → `## 升级矩阵`，以下情形须自动升级：

- 任何🔴关键风险维度 → 须律师主导处理
- 拟继续使用/实施前需进一步法律意见 → 须律师确认
- ⚠结论中的规避方案需专业评估可行性（商标/专利设计规避须律师+工程师）

## 边界条件

| 情形 | 处理方式 |
|-----|---------|
| 仅部分维度有上游输入 | 在报告中明确说明哪些维度已检索/评估，哪些维度未覆盖，清权结论限于已覆盖范围 |
| 多维度均为🔴关键风险 | 总结论为❌不可用，提供分维度规避建议（不同维度可能有不同修改方向） |
| ⚠结论中规避方案可行性不明 | 标注「规避方案有效性须专业评估」，不得凭猜测确认规避有效 |
| 清权严格度不明（配置缺失）| 使用临时模式「保守」（🟡以上均触发⚠），标 `[临时模式]` |
| 用户希望接受某维度风险 | 可记录用户决策，但须标注「权利人已知悉该风险并决定接受」，不作风险消除处理 |
| 清权结论产生分歧（维度间相互影响）| 明确说明各维度结论及其相互关系，给出综合建议，不作强制结论 |

## 错误处理

| 错误类型 | 错误信息 | 处理建议 |
|---------|---------|---------|
| 上游输入缺失 | 未提供任何维度清权发现 | 要求用户提供清权信息，或引导用户先运行上游清权子技能 |
| 维度输入不完整 | 部分维度无发现 | 报告缺失维度，在清权结论中声明「X 维度未评估」 |
| 清权结论矛盾 | 不同维度信息相互矛盾 | 说明矛盾原因，提出两种可能解读，不强行给出单一结论 |
| 配置文件缺失/含占位符 | ip-legal/CLAUDE.md 关键字段为 PLACEHOLDER | 切换临时模式，输出标记 `[临时模式]` |
