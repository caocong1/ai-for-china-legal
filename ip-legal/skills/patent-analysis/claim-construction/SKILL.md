# 技术特征比对与权利要求解读（claim-construction）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

对涉案专利的权利要求进行系统解读，将独立权利要求拆解为若干技术特征，同时提取被诉技术方案（被诉产品或方法）的对应技术特征，制作技术特征逐一比对表，为后续全面覆盖原则判断（all-elements-rule）与等同原则判断（doctrine-of-equivalents）奠定基础。

本子技能是 all-elements-rule 与 doctrine-of-equivalents 两个子技能的**前置步骤**，其输出的技术特征比对表将直接被下游子技能引用。

**本子技能可由律师/IP 经理/专利代理师单独触发：`/ip-legal:claim-construction`**

---

| 子技能 | 分工说明 |
|-------|---------|

## 详细步骤

### 第一步：专利基本信息收集

在开始权利要求解读前，确认以下专利基本信息。若信息不全，停止推进并告知用户需要补充哪些信息。

| 信息项 | 必填/选填 | 说明 |
|-------|---------|------|
| 专利号 | 必填 | 中国专利号格式：CN + 年份 + 序号 + 类型后缀 |
| 专利类型 | 必填 | 发明专利 / 实用新型专利 / 外观设计专利（三种类型比对方法不同） |
| 权利要求书 | 必填（发明/实用新型） | 完整权利要求书全文 |
| 说明书及附图 | 建议提供 | 用于解释权利要求，尤其功能性特征情形 |
| 专利权人 | 必填 | 当前登记的权利人（注意是否发生转让） |
| 法律状态 | 必填 | 有效（年费正常）/ 年费欠费风险 / 已失效 / 无效宣告程序中 |
| 申请日 | 必填 | 自申请日起算专利权期限（见专利法第42条，建议复核） |
| 授权日 | 选填 | 若申请日已过多年仍在审查，注意期限补偿问题 `[待验证]` |
| 外观设计图片/照片 | 必填（外观设计） | 外观设计专利以图片/照片确定保护范围 |

**专利法律状态核实提示**：年费缴纳状态须在专利检索系统中当日查询，过期专利权已终止，不构成有效侵权标的（建议通过国家知识产权局专利检索系统 https://pss-system.cponline.cnipa.gov.cn 核实）。

### 第二步：专利类型判断与比对方法选择

不同专利类型采用不同的保护范围确定方法：

| 专利类型 | 保护范围确定方式 | 比对方法 |
|---------|---------------|---------|
| 发明专利 | 以权利要求内容为准，说明书及附图可用于解释权利要求（描述规则，不写裸条号） | 技术特征逐一比对（全面覆盖 + 等同） |
| 实用新型专利 | 同发明专利，以权利要求为准 | 技术特征逐一比对（全面覆盖 + 等同） |
| 外观设计专利 | 以图片或照片中表示的该产品的外观设计为准，简要说明可用于解释（描述规则，不写裸条号） | 整体视觉效果比对，以相关消费者视觉判断为主；与发明/实用新型的技术特征比对方法根本不同 |

**外观设计专利特别说明**：外观设计不适用技术特征逐一比对方法，应以**整体视觉效果**是否相同或近似为判断标准，考量设计特征（形状、图案、色彩）对整体视觉效果的影响程度，相关消费者（而非本领域普通技术人员）为判断主体。本文件后续步骤主要针对发明专利和实用新型，若涉及外观设计，在步骤中单独说明。

### 第三步：独立权利要求选择与解读

**独立权利要求确定最大保护范围**：

> **权利要求解读规则（来自专利侵权司法解释，描述规则，不写裸条号）**：
> 发明或实用新型专利权的保护范围以权利要求的内容为准，说明书及附图可用于解释权利要求。解读时以本领域普通技术人员的理解为标准，结合说明书和附图解释权利要求中的技术术语。可参考审查历史文档（审查意见答复、修改历史）。

**独立权利要求 vs 从属权利要求选择策略：**

| 类型 | 定义 | 保护范围 | 侵权分析策略 |
|-----|------|---------|------------|
| 独立权利要求 | 自身完整描述发明技术方案，不引用其他权利要求 | 最宽（技术特征最少）| 优先分析独立权利要求——若构成侵权，从属权利要求一并侵权 |
| 从属权利要求 | 引用并附加特征于被引用的权利要求之上 | 较窄（技术特征更多）| 独立权利要求不侵权时，无需再分析从属权利要求 |

**功能性特征的解读（特别注意）**：

功能性特征（以实现功能/效果来限定技术特征的权利要求撰写方式）的解读规则：
- 功能性特征的保护范围通常限定在说明书中所描述的实现该功能的具体方式，以及本领域普通技术人员无需创造性劳动即可想到的等同方式（描述规则，不写裸条号）
- 若被诉方案以不同方式实现相同功能，须额外分析是否构成等同

### 第四步：权利要求拆解为技术特征

将选定的独立权利要求逐一拆解为单独的技术特征（A+B+C+D 格式）：

**拆解原则：**
- 每个技术特征应是权利要求中独立描述的、可以单独识别的最小技术单元
- 连词「并且」、「其中」、「所述」通常标志特征之间的分隔
- 功能性特征单独列出，并备注需结合说明书解读

**权利要求技术特征拆解表（模板）：**

| 特征编号 | 技术特征描述（原文） | 特征类型 | 解读说明 |
|--------|------------------|---------|---------|
| A | [来自权利要求的技术特征原文] | 结构特征 / 方法步骤 / 功能特征 | [说明书或本领域惯常理解] |
| B | [来自权利要求的技术特征原文] | 结构特征 / 方法步骤 / 功能特征 | [说明书或本领域惯常理解] |
| C | [来自权利要求的技术特征原文] | 结构特征 / 方法步骤 / 功能特征 | [说明书或本领域惯常理解] |
| D | [来自权利要求的技术特征原文] | 结构特征 / 方法步骤 / 功能特征 | [说明书或本领域惯常理解] |

**多个独立权利要求处理**：若专利有多个独立权利要求，按保护范围从宽到窄列出，优先分析最宽的独立权利要求。

### 第五步：被诉技术方案技术特征提取

提取被诉产品/方法的对应技术特征，与权利要求技术特征逐一对应：

**信息来源（按优先级）：**
1. 用户提供的产品样品、产品说明书、技术手册
2. 公开信息（官网、电商平台、产品宣传材料）
3. 拆解/检测报告（若有）
4. 若以上均无，须告知用户：无法基于描述性信息作出确定性技术特征提取，建议取样送检

**被诉技术方案特征提取表（模板）：**

| 对应权利要求特征编号 | 被诉技术方案特征描述 | 特征来源 | 不确定性说明 |
|-----------------|-----------------|---------|------------|
| 对应 A | [被诉方案对应特征描述] | [产品说明书/样品/公开信息] | [如有不确定性，标注] |
| 对应 B | [被诉方案对应特征描述] | [产品说明书/样品/公开信息] | [如有不确定性，标注] |
| 对应 C | [被诉方案对应特征描述] | [产品说明书/样品/公开信息] | [如有不确定性，标注] |
| 对应 D | [被诉方案对应特征描述] | [产品说明书/样品/公开信息] | [如有不确定性，标注] |

### 第六步：制作技术特征综合比对表

汇总权利要求技术特征与被诉方案技术特征，形成供 all-elements-rule 与 doctrine-of-equivalents 使用的综合比对表：

| 特征编号 | 权利要求技术特征 | 被诉方案对应特征 | 初步对应关系 |
|--------|--------------|--------------|-----------|
| A | [权利要求特征 A] | [被诉方案对应特征 a] | 相同 / 不同（需等同分析）/ 缺少 |
| B | [权利要求特征 B] | [被诉方案对应特征 b] | 相同 / 不同（需等同分析）/ 缺少 |
| C | [权利要求特征 C] | [被诉方案对应特征 c] | 相同 / 不同（需等同分析）/ 缺少 |
| D | [权利要求特征 D] | [被诉方案对应特征 d] | 相同 / 不同（需等同分析）/ 缺少 |

**比对表说明**：
- 「相同」：被诉方案技术特征与权利要求技术特征字面相同
- 「不同（需等同分析）」：字面不同，但可能等同，需交由 doctrine-of-equivalents 判断
- 「缺少」：被诉方案中找不到对应于该权利要求技术特征的技术特征

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 外观设计专利 | 切换至整体视觉效果比对方法（以相关消费者为判断主体，不适用技术特征逐一比对） |
| 功能性特征（以功能/效果限定的权利要求用语） | 结合说明书实施方式解读，保护范围通常限于说明书中具体描述的实现方式；描述规则，不写裸条号 |
| 被诉方案信息不完整 | 停止提取，告知用户需补充材料（样品/说明书/检测报告）；不得凭想象补全 |
| 专利已过期 | 告知专利权已终止，不构成有效侵权标的，分析可以停止或转为历史侵权分析 |
| 专利处于无效宣告程序中 | 标注专利稳定性风险；现阶段专利权仍有效，但无效宣告结果可能影响侵权结论 |
| 多个独立权利要求 | 逐一分析，从保护范围最宽的开始；若最宽者不侵权，较窄者也不侵权 |
| 专利转让/质押情形 | 提示核实当前权利人（国家知识产权局登记系统）；权利人身份影响诉讼资格 |
| 方法专利 + 产品被诉 | 注意方法专利与产品专利的侵权认定差异；方法专利可以产品被告（若该产品系依专利方法直接制造）`[待验证具体规则]` |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 用户仅提供专利号，无权利要求书 | 提示用户通过国家知识产权局专利检索系统（https://pss-system.cponline.cnipa.gov.cn）获取完整权利要求书；不得凭记忆推断权利要求内容 |
| 权利要求书不完整或存在技术术语不明 | 标注不明确处，建议查阅说明书对应描述；仍无法理解的术语，建议咨询本领域技术专家 |
| 被诉方案信息仅有品牌/型号，无技术细节 | 建议取样、获取说明书或委托专业检测机构；不得基于商标/型号猜测技术特征 |
| 外观设计专利使用了技术特征比对逻辑 | 纠正为整体视觉效果比对；提示外观设计不适用全面覆盖+等同原则体系 |
| 权利要求中有不清楚/范围不明确的技术特征 | 标注待进一步解释，建议通过说明书或审查历史文档澄清；必要时提示本技术特征存在解释争议 |
