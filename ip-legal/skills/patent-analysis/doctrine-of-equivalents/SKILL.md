# 等同原则（doctrine-of-equivalents）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

当 all-elements-rule 判断出被诉技术方案的技术特征与权利要求技术特征**字面不相同**时，运用等同原则（doctrine of equivalents）判断该技术特征是否构成**等同特征**。等同侵权是专利侵权判断的第二层分析，防止被告通过技术外形的细微调整逃避专利保护范围。

同时，本技能须检查**禁止反悔原则**与**捐献原则**两项对等同原则的重要限制——若被主张的等同技术方案被禁止反悔原则或捐献原则所限制，等同原则不适用。

**本子技能可由律师/IP 经理/专利代理师单独触发：`/ip-legal:doctrine-of-equivalents`**

## 详细步骤

### 第一步：确认等同分析对象

明确哪些技术特征进入等同分析：

| 确认项 | 说明 |
|-------|------|
| 等同分析特征列表 | 来自 all-elements-rule 标注「字面不同（需等同分析）」的特征编号 |
| 每对技术特征的权利要求侧描述 | 来自权利要求书（结合说明书解读） |
| 每对技术特征的被诉方案侧描述 | 来自被诉产品/方法技术信息 |
| 说明书中是否有记载该技术方案 | 用于捐献原则检查 |
| 审查历史文档是否有对该技术特征的修改/意见陈述 | 用于禁止反悔原则检查 |

**重要前提确认**：等同分析的**时间基准**为**侵权行为发生时**（而非专利申请日），即以侵权发生时的本领域技术水平判断是否构成等同（描述规则，不写裸条号）。

### 第二步：等同特征三要素判断

**等同原则核心判断规则（来自专利侵权司法解释，描述规则，不写裸条号）**：

> 被诉侵权技术方案的对应特征与权利要求的技术特征，以**基本相同的手段**，实现**基本相同的功能**，达到**基本相同的效果**，且**本领域普通技术人员在侵权行为发生时无需经过创造性劳动即可联想到**，则认定构成等同特征，等同侵权成立。

**三要素判断（逐一分析每对字面不同的技术特征）：**

**要素一：手段（means）基本相同**

| 判断维度 | 说明 |
|---------|------|
| 手段的含义 | 实现技术效果所采用的具体技术途径、实现方式或结构 |
| 基本相同的标准 | 不要求完全相同，技术本质相同或高度类似即可；在本领域惯常技术视野下属于同类手段 |
| 判断要点 | 是否属于本领域中已知的可替代手段？替换是否改变了技术本质？ |

**要素二：功能（function）基本相同**

| 判断维度 | 说明 |
|---------|------|
| 功能的含义 | 该技术特征在整体技术方案中所起到的作用和贡献 |
| 基本相同的标准 | 在技术方案的整体框架下，两者实现的功能是否相同或高度类似 |
| 判断要点 | 功能是否是该技术特征对技术方案整体作出的核心贡献？若功能不同则通常不构成等同 |

**要素三：效果（result）基本相同**

| 判断维度 | 说明 |
|---------|------|
| 效果的含义 | 实施该技术特征最终所产生的技术效果或性能 |
| 基本相同的标准 | 技术效果相同或相近，包括性能指标、解决的技术问题等 |
| 判断要点 | 两者是否解决了相同的技术问题，达到了相近的技术效果？ |

**本领域普通技术人员「无需创造性劳动即可联想到」：**

| 判断维度 | 说明 |
|---------|------|
| 判断主体 | 本领域普通技术人员（假想中掌握本领域公知技术、但不具有创造性能力的技术人员） |
| 「无需创造性劳动」的含义 | 从权利要求技术特征联想到被诉方案技术特征是常规替换，不需要技术创新；如需创造性劳动才能想到，则不构成等同 |
| 时间基准 | 侵权行为发生时（注意：专利申请日之后的技术进步可能影响本领域普通技术人员的认知水平） |

**等同三要素判断表（每对字面不同的特征各有一张）：**

| 要素 | 权利要求技术特征侧 | 被诉方案技术特征侧 | 基本相同？ | 说明 |
|-----|----------------|----------------|---------|------|
| 手段 | [权利要求侧的手段描述] | [被诉方案侧的手段描述] | 是/否/存疑 | [分析说明] |
| 功能 | [权利要求侧的功能描述] | [被诉方案侧的功能描述] | 是/否/存疑 | [分析说明] |
| 效果 | [权利要求侧的效果描述] | [被诉方案侧的效果描述] | 是/否/存疑 | [分析说明] |
| **本领域普通技术人员可联想到** | — | — | 是/否/存疑 | [分析说明] |
| **等同特征初步结论** | — | — | 构成等同 / 不构成等同 / 存疑 | — |

### 第三步：禁止反悔原则检查

**禁止反悔原则规则（来自专利侵权司法解释，描述规则，不写裸条号）**：

> 专利权人在专利申请或无效宣告程序中，通过修改权利要求或意见陈述明确放弃的技术方案，在侵权诉讼中**不得通过等同原则重新将其纳入保护范围**。禁止反悔原则是对等同原则的重要限制。

**禁止反悔检查步骤：**

1. 获取涉案专利的审查历史文档（专利局审查意见答复、权利要求修改历史、无效宣告程序中的意见陈述）
2. 核查专利权人是否在申请/无效程序中针对等同分析所涉及的技术特征或技术方案作出过明确放弃或限缩
3. 若存在明确放弃/限缩，且被诉方案的技术特征属于被放弃的范围，则禁止反悔原则适用——等同原则在此被限制，不得主张等同

**禁止反悔判断表：**

| 检查项 | 结论 | 说明 |
|-------|------|------|
| 审查/无效程序中是否有权利要求修改 | 有/无 | [列出修改内容] |
| 修改是否涉及当前等同分析的技术特征 | 是/否 | [具体分析] |
| 意见陈述是否对该技术特征范围作出限制性陈述 | 是/否 | [引用相关陈述] |
| **禁止反悔原则是否适用** | **适用/不适用** | [结论理由] |

**若禁止反悔原则适用**：等同原则在此特征上受限，不能通过等同将被放弃的技术方案纳入保护范围；直接得出该特征不构成等同侵权。

### 第四步：捐献原则检查

**捐献原则规则（来自专利侵权司法解释，描述规则，不写裸条号）**：

> 说明书中记载了某种技术方案，但**未写入权利要求**的，视为捐献给公众，不受专利保护，也不能通过等同原则将其纳入保护范围。捐献原则是防止通过等同原则超越权利要求保护范围的另一重要限制。

**捐献原则检查步骤：**

1. 核查被诉方案的技术特征是否在说明书中有具体描述
2. 若说明书中有描述，核查该技术方案是否被写入了任何权利要求
3. 若说明书中有描述但未写入权利要求，则该技术方案被视为捐献给公众，专利权人不得通过等同原则主张对其的保护

**捐献原则判断表：**

| 检查项 | 结论 | 说明 |
|-------|------|------|
| 被诉方案技术特征是否在说明书中有具体记载 | 是/否/不确定 | [说明书相关段落] |
| 该技术方案是否被写入任何权利要求 | 是（写入了第X项）/ 否 | [说明] |
| **捐献原则是否适用** | **适用/不适用** | [结论理由] |

**若捐献原则适用**：等同原则在此技术方案上受限，专利权人不得主张等同；直接得出该特征不构成等同侵权。

### 第五步：等同侵权综合结论

汇总三要素判断结果与禁止反悔/捐献原则限制，得出等同侵权结论：

**等同侵权综合结论判定矩阵：**

| 步骤 | 结果 | 对应结论 |
|-----|------|---------|
| 三要素均基本相同且可联想到 + 禁止反悔不适用 + 捐献原则不适用 | 全部满足 | **构成等同特征，等同侵权成立** |
| 三要素均满足，但禁止反悔原则适用 | 被限制 | **等同原则受禁止反悔原则限制，不构成等同侵权** |
| 三要素均满足，但捐献原则适用 | 被限制 | **等同原则受捐献原则限制，不构成等同侵权** |
| 任一要素不满足（手段/功能/效果之一不基本相同）| 不满足 | **不构成等同特征** |
| 本领域普通技术人员须经创造性劳动才能联想到 | 不满足 | **不构成等同特征（须创造性劳动）** |

**等同侵权风险等级：**

| 等级 | 条件 |
|-----|------|
| 🔴 等同侵权高风险 | 三要素均明显满足，无禁止反悔/捐献限制 |
| 🟠 等同侵权较高风险 | 三要素基本满足，禁止反悔/捐献适用存疑 |
| 🟡 等同侵权风险中等 | 部分要素明显满足，部分要素存疑 |
| 🟢 等同侵权风险低 | 有要素明显不满足，或禁止反悔/捐献原则明确适用 |

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 等同分析的时间基准（申请日 vs 侵权日） | 等同原则适用侵权行为发生时的本领域技术水平为判断基准；专利申请日后出现的技术可能被纳入本领域普通技术人员可联想到的范围 |
| 功能性特征的等同判断 | 功能性特征的等同判断有特殊规则：保护范围已经通过说明书限定，等同仅限于说明书中描述的等同方式（与一般技术特征等同规则不同，须额外保守）（描述规则，不写裸条号） |
| 审查历史文档不可获取 | 标注「禁止反悔检查因审查历史不可获得而不完整，建议向专利权人索取或通过国家知识产权局档案室获取」；不得凭猜测作出禁止反悔结论 |
| 说明书信息不完整 | 捐献原则检查不完整，标注「需进一步核查说明书全文」 |
| 被诉方案与权利要求技术特征差异极大 | 三要素之一明显不满足时，无需继续分析其余要素，直接得出不构成等同特征的结论 |
| 无效宣告程序中产生的意见陈述 | 无效宣告程序中作出的技术特征限制陈述，同样可触发禁止反悔原则（描述规则，不写裸条号） |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 对字面相同的技术特征进行等同分析 | 纠正：等同分析仅针对字面不同的特征，字面相同的特征已由 all-elements-rule 处理 |
| 三要素笼统认定（未逐一分析） | 要求逐一分析手段、功能、效果三个要素，不接受笼统结论 |
| 忽略禁止反悔/捐献原则检查 | 补充禁止反悔与捐献原则检查步骤，任何等同分析结论在此两项检查完成前均标注「不完整」 |
| 以申请日（而非侵权日）为等同判断基准 | 纠正时间基准为侵权行为发生时 |
