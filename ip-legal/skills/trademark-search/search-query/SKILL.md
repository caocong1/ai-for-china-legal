# 商标检索与查询（search-query）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

对拟申请注册或拟使用的商标进行系统性检索，查明是否存在在先相同或近似商标，为后续近似判断（similarity-assessment）、类似商品判定（goods-services-classification）、在先权利障碍分析（prior-rights-obstacles）提供准确的检索数据基础。

**本子技能是商标查询与近似分析流程的第一步，也可由律师/IP 经理单独触发：`/ip-legal:search-query`**

---

## 前置

### 读取实践配置

在开始检索前，读取 `ip-legal/CLAUDE.md`：

- **检索数据库可用性**（`## 可用集成`）：确认中国商标网/商业数据库/字号检索系统是否可用
- **IP 组合概况**（`## 我们是谁` → 商标）：了解权利人现有商标类别与地域，判断检索范围
- **商标策略**（`## 维权与申请立场` → `### 商标策略`）：了解注册防御范围，确定检索类别广度
- **执业角色**（`## 谁在使用`）：决定工作产物标题格式

若配置含 `[PLACEHOLDER]` 或文件不存在，使用临时模式默认值（见 practice-profile-schema.md）并标注 `[临时模式]`。

### 与其他子技能的分工

| 子技能 | 分工说明 |
|-------|---------|
| **本技能（search-query）** | 检索查询，整理检索结果清单，不作近似判断结论 |
| similarity-assessment | 对本技能找到的近似商标进行音/形/义三维度近似判断与混淆可能性评估 |
| goods-services-classification | 判定相关商品/服务是否类似，配合本技能结果使用 |
| prior-rights-obstacles | 汇总在先权利障碍，综合三个子技能结果作出障碍结论 |

## 详细步骤

### 第一步：厘清检索目标

收集以下信息（若用户未提供，须询问确认）：

| 信息项 | 说明 | 示例 |
|-------|------|------|
| 商标类型 | 文字商标/图形商标/组合商标（文字+图形）/其他 | 文字商标「云鹏」 |
| 商标文字内容 | 完整文字、大写/小写、中文/英文/中英文组合 | 「云鹏 YUNPENG」 |
| 图形要素描述 | 图形主要视觉元素、颜色（颜色通常不影响近似检索但影响直观判断） | 蓝色飞鸟图形 |
| 申请人信息 | 公司/个人名称（用于识别同一主体的关联商标） | [申请人名称] |
| 拟申请/使用类别 | 尼斯分类 45 类中的类别（若不明确，须先协助判断） | 第 9 类（软件）、第 35 类（广告服务） |
| 拟用商品/服务描述 | 具体商品/服务名称（用于确定精确的类似群组） | 「计算机软件（已录制）」「在线广告服务」 |
| 检索地域范围 | 中国大陆（必须）；港澳台/海外（如权利人有布局需求） | 中国大陆 |
| 申请人同族商标 | 同一申请人是否有相关商标（了解申请人整体布局） | 如有，提供核心商标信息 |

**如信息不完整，停止检索，先请用户补充以上信息。**

### 第二步：确定检索类别

根据拟使用商品/服务，结合《类似商品和服务区分表》（以下简称「区分表」）确定检索类别：

**类别确定决策流程：**

1. **精确类别**：拟使用的具体商品/服务所属的尼斯分类类别（必须检索）
2. **关联类别**：根据商品服务关联性，参照区分表确定可能构成类似商品的相关类别（建议检索）
3. **防御类别**：根据权利人商标策略（全类防御 vs 核心类别），确定是否检索其他类别
4. **驰名商标情形**：若权利人已有驰名商标认定，跨类检索范围可能更广（交由 prior-rights-obstacles 处理）

**常见类别关联参照（示例，须结合区分表核实）：**

| 权利人业务 | 精确类别（示例） | 常见关联类别（示例） |
|-----------|--------------|---------------|
| 软件/互联网 | 第 9 类、第 42 类 | 第 35 类（广告/商业服务）、第 38 类（通信服务） |
| 服装/鞋类 | 第 25 类 | 第 18 类（皮具/箱包）、第 35 类（零售服务）|
| 食品饮料 | 第 29 类、第 30 类、第 32 类 | 第 35 类（零售服务） |
| 医疗/保健 | 第 5 类、第 10 类 | 第 44 类（医疗服务）、第 3 类（化妆品）|

> **注意**：以上为示例，实际关联类别须结合具体商品服务内容与区分表确认，不得机械套用。

### 第三步：选择检索数据库

根据 `ip-legal/CLAUDE.md` → `## 可用集成` 确认可用数据库，按以下优先级选择：

**主要检索数据库：**

| 数据库 | 类型 | 适用情形 | 可用性确认 |
|-------|------|---------|---------|
| 中国商标网（sbj.cnipa.gov.cn） | 官方免费 | 基础检索，实时数据（申请/注册/公告状态） | 从 `## 可用集成` 读取 |
| 商业商标数据库（如智慧芽/商标局授权商业数据库等） | 商业付费 | 批量检索/历史数据/近似检索辅助 | 从 `## 可用集成` 读取 |

**辅助检索数据库（视需要）：**

| 数据库 | 适用情形 |
|-------|---------|
| 国家企业信用信息公示系统（gsxt.gov.cn） | 字号检索（在先字号权利核查，交由 prior-rights-obstacles 处理） |
| 域名 WHOIS 查询工具 | 域名冲突核查（交由 prior-rights-obstacles 处理） |

**若官方商标网不可用**：提示用户手动访问 https://sbj.cnipa.gov.cn 进行商标查询，并将结果粘贴提供。

### 第四步：构造检索式

根据商标类型，采用不同检索策略：

**文字商标检索策略（按优先级）：**

| 检索策略 | 说明 | 示例（以「云鹏」为例） |
|---------|------|----------------|
| 精确检索 | 输入完整商标文字 | 检索词：「云鹏」 |
| 谐音检索 | 检索发音相同/相近的词 | 检索词：「云朋」「运鹏」（声调差异/谐音字） |
| 近音检索 | 检索声母/韵母相近的词 | 根据音节分解，检索同音近音字 |
| 字形检索 | 检索字形相近的汉字 | 「云」→「云」（繁体）；「鹏」→「朋」「鹏」（相近字形） |
| 含义检索 | 检索含义相同/相近的词 | 「云」→「云朵」「白云」；「鹏」→「大鹏」「鸿鹏」 |
| 拼音/英文检索 | 检索对应拼音或英文翻译 | 检索词：「YUNPENG」「CLOUD ROC」 |

**图形商标检索策略：**

| 检索策略 | 说明 | 注意事项 |
|---------|------|---------|
| 维也纳分类编码检索 | 按图形要素对应的维也纳分类编码检索 | 维也纳分类编码须准确识别图形类别（需人工判断） |
| 描述性检索 | 按图形描述词检索（如「飞鸟」「山峰」） | 结合文字描述检索 |
| 视觉相似检索 | 商业数据库提供的图形相似度检索（AI 辅助） | 须通过商业数据库操作，系统准确性有限 |

> **图形商标检索说明**：图形商标的精确检索依赖维也纳分类编码的准确性，人工识别可能存在偏差，图形检索结果需人工核实。建议同时进行视觉比对。

**组合商标检索策略：**

分别对文字要素和图形要素进行独立检索，再综合判断：
1. 检索文字要素（完整文字商标 + 文字近似检索）
2. 检索图形要素（维也纳分类编码检索）
3. 检索组合形式（整体形式）

### 第五步：执行检索并整理结果

**检索时间节点说明：**

商标状态随时变化，检索结果以**检索当日状态**为准。检索结果需注明检索日期，后续分析基于检索日的数据。

**结果筛选与分类：**

| 状态类别 | 说明 | 处置建议 |
|---------|------|---------|
| **有效注册商标** | 已注册且在有效期内 | 须重点分析，可能构成注册障碍 |
| **申请中/待审商标** | 已提交申请但尚未注册 | 注意「申请在先」原则，须评估 |
| **初审公告期商标** | 已通过初审，3 个月公告期内 `[待验证]` | 可提异议，时限紧迫 |
| **无效/撤销/注销商标** | 已无效、撤三成功或注销 | 一般不构成障碍，但需确认状态 |
| **驰名商标认定记录** | 经行政或司法认定的驰名商标 | 跨类保护风险，交由 prior-rights-obstacles 处理 |

## 检查清单

在完成检索后，按以下清单验证检索质量：

- [ ] 商标类型（文字/图形/组合）已明确，并针对各要素分别检索
- [ ] 精确类别已检索
- [ ] 关联类别已检索（结合商标策略中的注册防御范围确定范围）
- [ ] 文字商标已进行音近/形近/义近检索（至少覆盖三维度）
- [ ] 图形商标已进行维也纳分类编码检索（如适用）
- [ ] 官方商标网检索已完成（记录检索日期）
- [ ] 商业数据库检索已完成（如可用）
- [ ] 检索结果已按状态分类（有效/申请中/公告期/已失效）
- [ ] 同一申请人关联商标已标注
- [ ] 检索日期已记录（结论时效性依据）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 图形商标维也纳编码不准确 | 说明图形检索的局限性，建议人工辅助核实；结果仅供参考，不得单独作为「无近似」结论依据 |
| 商业数据库不可用 | 仅使用官方商标网；说明检索可能不够全面，建议用户使用商业数据库进行补充检索 |
| 跨类检索范围过大 | 结合商标策略（注册防御范围）确定检索边界；若权利人有驰名商标，跨类检索交由 prior-rights-obstacles 处理 |
| 检索结果数量过多（如超过100条） | 按「高度近似优先」筛选，提供最相关的前20-30条，说明筛选逻辑 |
| 用户提供的商标信息不完整 | 停止检索，明确列出需要补充的信息项，等待用户补充 |
| 拟检索商标尚未确定（多方案比选） | 对每个方案分别检索，输出各方案的检索结果对比表 |
| 海外商标检索 | 明确说明本技能仅处理中国大陆商标检索，海外检索须使用WIPO/EUIPO/USPTO等境外检索系统 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 商标网无法访问 | 提示用户手动访问 https://sbj.cnipa.gov.cn，将检索结果以截图或文字形式提供；不得跳过检索作出「无障碍」结论 |
| 检索词含特殊字符/繁体/异体字 | 同时检索简体、繁体、异体字形式；说明字形差异对近似判断的影响 |
| 申请人名称有多种写法 | 对所有已知写法/关联主体分别检索，防止遗漏关联商标 |
| 检索结果状态显示「异常」或不明 | 不确定的状态标注 `[待验证 — 须核实商标当前有效状态]`，建议通过商标局咨询确认 |
| 商标类别判断存疑 | 提示用户确认具体商品/服务描述，结合区分表核实类别；不得在类别未确定的情况下单独作出检索结论 |
