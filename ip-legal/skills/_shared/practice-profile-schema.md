# 实践配置文件契约（知识产权版 Practice Profile Schema）

> 本文件定义 ip-legal 插件的实践配置文件契约：字段定义、存储位置、技能读取规则，以及与 `ip-legal/CLAUDE.md` 实践配置模板的映射关系。
>
> **所有知识产权技能通过本契约读取个性化配置。** 冷启动访谈子技能按本契约填充配置文件。

---

## 一、配置文件是什么

### 真相来源

实践配置文件的唯一真相来源是 **`ip-legal/CLAUDE.md`**（散文形式）。该文件：

- 由 `/ip-legal:cold-start-interview` 冷启动访谈生成和更新
- 可由律师/IP 经理直接编辑（无需重新运行访谈）
- 为所有 ip-legal 技能提供个性化设置的读取来源
- 本阶段（06-01）**首次新建**，此前插件无实践配置文件

### 架构决策

**决策：不引入与现有体系冲突的新配置文件格式。**

背景：现有 `ip-legal/skills/cold-start-interview/SKILL.md` 末尾有「生成知识产权实践配置文件（YAML 格式）」的表述。**此表述应理解/纠正为**：冷启动访谈的结果应填充/写入 `ip-legal/CLAUDE.md` 对应章节，而非另存独立 YAML 文件。

**选择理由：**

1. `ip-legal/CLAUDE.md` 是律师/IP 经理可直接阅读和编辑的人类友好格式，散文胜于 YAML/JSON 用于表达「维权立场」与「策略考虑」
2. Claude 的实践配置文件读取本质是文本理解，不需要机器解析的结构化格式
3. 引入独立 YAML 文件会产生双重真相来源，维护成本高且容易产生不一致
4. 沿用 commercial-legal / data-compliance 已验证的 CLAUDE.md 单一真相来源模式

**结论：** `ip-legal/CLAUDE.md` 是唯一真相来源。冷启动访谈填充 `ip-legal/CLAUDE.md` 的占位符字段，不另存 YAML 文件。

## 三、技能读取契约

### 3.1 技能读取时机

**在执行任何实质性工作之前**，每个技能（或其编排入口）必须：

1. 读取 `ip-legal/CLAUDE.md`
2. 检查关键字段是否已填充（不含 `[PLACEHOLDER]`）
3. 根据检查结果选择运行模式

**关键字段清单（配置读取优先级顺序）：**

| 优先级 | 关键字段 | 所在章节 |
|--------|---------|---------|
| 必须读取 | 整体维权立场 | `## 维权与申请立场`（顶层） |
| 必须读取 | 执业角色 | `## 谁在使用` → 角色 |
| 必须读取（商标类技能） | 商标策略（含近似容忍度） | `## 维权与申请立场` → `### 商标策略` |
| 必须读取（专利类技能） | FTO 清权严格度、专利类型偏好 | `## 维权与申请立场` → `### 专利策略` |
| 必须读取（著作权类技能） | 著作权策略（含登记立场） | `## 维权与申请立场` → `### 著作权策略` |
| 必须读取（清权类技能） | 清权严格度（含清权范围与结论门槛） | `## 维权与申请立场` → `### 清权严格度` |
| 建议读取 | IP 组合概况（商标/专利/著作权数量与类别） | `## 我们是谁` → IP 组合概况 |
| 建议读取 | 可用检索数据库 | `## 可用集成` |
| 建议读取 | 风险校准（各等级定义） | `## 风险校准` |
| 建议读取 | 升级矩阵 | `## 升级矩阵` |

### 3.2 配置缺失或含占位符时的处理

当技能检测到 `ip-legal/CLAUDE.md` 缺失或关键字段含 `[PLACEHOLDER]` 时，**不得静默使用猜测值**，须按以下逻辑处理：

**标准提示词：**

> 我注意到你还没有配置知识产权实践档案——这是让我根据你的权利人画像、IP 组合与维权立场来定制输出的关键。
>
> **两个选择：**
> - 运行 `/ip-legal:cold-start-interview`（约 5 分钟）来配置你的档案，我会根据你的真实 IP 策略进行工作。
> - 说 **「临时模式」**，我会使用通用默认值并在每个输出上标记 `[临时模式 — 配置你的档案以获得定制输出]`。

**临时模式默认值：**

| 字段 | 临时模式默认值 |
|-----|-------------|
| 整体维权立场 | 平衡型 |
| 执业角色 | 律师 |
| 商标近似容忍度 | 平衡（高度近似不容忍） |
| 商标异议倾向 | 被动异议 |
| 清权严格度 | 保守（按保守标准标记常见障碍） |
| 清权范围 | 仅商标（默认商标清权） |
| 专利 FTO 严格度 | 平衡 |
| 著作权登记立场 | 选择性登记 |
| 风险等级 | 按标准 🔴🟠🟡🟢 四级分类 |
| 商标近似判断标准 | 相关公众一般注意力，逐案整体判断（按司法解释规则，不写裸条号） |
| 专利侵权标准 | 全面覆盖原则 + 等同原则，逐一比对（按司法解释规则，不写裸条号） |
| 著作权标准 | 独创性 + 思想表达二分（基本原理） |

### 3.3 维权立场/清权严格度立场选择

对于每次 IP 分析，技能须在开始实质工作前确认维权立场与清权严格度：

1. **从 `ip-legal/CLAUDE.md` 直接读取**：优先从 `## 维权与申请立场`（顶层整体维权立场）读取
2. **逐级细化**：整体立场 → 对应 IP 类型策略（商标/专利/著作权）→ 清权严格度
3. **不明确时确认**：若整体立场与具体策略存在冲突，向律师/IP 经理确认，不静默取值

**立场与清权结论的映射：**

| 整体维权立场 | 风险标记强度 | 清权结论倾向 |
|-----------|-----------|-----------|
| 进取维权型 | 🟡中等以上均重点标注 | 严格；遇任何近似/重叠风险即建议采取行动 |
| 平衡型 | 🟠高/🔴关键重点标注 | 平衡；高度近似/重叠风险建议行动，一般风险备注 |
| 防御型 | 🔴关键才重点标注 | 宽松；仅在明显侵权/无法规避时才建议行动 |

## 四、重跑 / 编辑 / 版本控制

### 4.1 重跑访谈

律师/IP 经理可随时重新运行完整访谈或任一子技能：

- **完整重跑**：`/ip-legal:cold-start-interview` — 按子技能顺序填充全部字段
- **部分更新**：单独触发某一访谈子技能，只更新配置文件的对应章节

### 4.2 直接编辑

律师/IP 经理可直接编辑 `ip-legal/CLAUDE.md`，无需重跑访谈。编辑后即时生效，下次技能调用自动读取新值。

### 4.3 版本控制

- `ip-legal/CLAUDE.md` 纳入 git 版本控制（与代码同仓库）
- 支持通过 git 查看历史版本：`git log --oneline ip-legal/CLAUDE.md`
- 支持通过 git diff 查看变更：`git diff HEAD~1 ip-legal/CLAUDE.md`
- **回滚方式**：`git checkout <commit-hash> -- ip-legal/CLAUDE.md`

## 五、法律事实合理性检查

本节定义访谈子技能在将法律相关字段写入配置文件之前的合理性检查规则。

### 5.1 适用范围

对以下类型的用户口述内容进行合理性检查：

- 具体法规条款号（如「商标法第 58 条」「专利法第 50 条」等）
- 具体期限数值（如「商标续展期 5 年」「专利有效期 15 年」「著作权保护 100 年」等）
- 具体赔偿数额或比率（如「商标侵权法定赔偿上限 X 元」等）
- 引用司法解释条文或部门规章规定

### 5.2 合理性检查规则

**规则 1：IP 法修正条号核查（最重要）**

当用户口述 IP 法（商标法/专利法/著作权法/反不正当竞争法）条款号时，须提示：

> 你提到 [X 条 Y 款]；请注意 IP 法经 2019/2020/2021 年修正，条号可能重排，我将其标为 `[待验证 — IP 法修正后条号重排，须核实现行版本]`。写入配置前请通过 https://flk.npc.gov.cn 核实。

**规则 2：期限数值核查**

当用户口述知识产权期限时，与中度把握锚点对照：

- 「商标续展期 5 年」→ 提示「续展期应为 10 年（规则稳定），请确认」`[前提 flagged — 需验证]`
- 「外观设计专利有效期 10 年」→ 提示「2021 修正后已改为 15 年，请核实申请日期对应的有效期」`[前提 flagged — 需验证]`
- 「著作权保护 100 年」→ 提示「自然人作品为终生+50年，请确认」`[前提 flagged — 需验证]`

**规则 3：时效性标记**

涉及任何 IP 法具体规则配置字段，若距最后更新超过 6 个月，在字段值后附加：

> `[时效性 flagged — 需验证现行 IP 法版本]`

**规则 4：矛盾回答澄清**

若用户在访谈中提供矛盾立场（如「进取维权」但清权结论门槛设为「宽松」），在写入前澄清：

> 你的整体维权立场为「进取维权型」，但清权结论门槛设置为宽松。这是否准确？请确认立场设置。
