# 权属判定（ownership-determination）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

基于 work-type-determination 确定的作品类型，系统分析著作权权属归属：适用创作者原始取得规则，并针对职务作品、委托作品、法人作品、合作作品等各类特殊权属情形，判断著作权的归属主体，为后续登记材料准备（registration-materials）提供权属定性依据。

权属判定的结论直接影响登记申请主体的确定，以及未来著作权主张和转让/许可安排。

**本子技能可由律师/IP 经理单独触发：`/ip-legal:ownership-determination`**

## 详细步骤

### 第一步：收集权属判定所需信息

| 信息项 | 必填/选填 | 说明 |
|-------|---------|------|
| 作者姓名/单位 | 必填 | 实际创作该作品的自然人或单位（法人） |
| 创作时间 | 必填 | 创作完成时间，用于判断是否在职期间 |
| 创作地点 | 选填 | 是否在单位工作场所内 |
| 是否为完成工作任务 | 必填 | 是否为完成用人单位的工作任务而创作（职务作品的核心判断标准） |
| 是否使用单位物质技术条件 | 选填 | 主要利用单位的设备、资金、资料等——影响特殊职务作品认定 |
| 是否有委托合同 | 选填 | 是否存在委托创作合同，及合同中是否有权属约定 |
| 是否为合作创作 | 必填 | 是否有多人参与创作 |
| 是否有权属约定 | 选填 | 雇佣合同、委托合同、合作协议中是否有著作权归属约定 |
| 是否为法人主持创作 | 选填 | 是否由法人主持、代表法人意志创作，法人是否承担责任 |

### 第二步：权属判定流程

**自动保护原则**：著作权自作品创作完成之日起**自动产生**，无需登记（描述规则，条号待验证）。

**权属判定优先级：**

```
优先检查约定（有书面合同约定的，按约定）
  ↓
无约定，适用法定规则：
  ├── 单人创作：创作者原始取得
  ├── 工作任务中创作：职务作品规则
  ├── 受委托创作：委托作品规则
  ├── 法人主持创作：法人作品规则
  └── 多人合作创作：合作作品规则
```

### 第三步：各权属类型详细分析

#### A. 创作者原始取得（自然人作者）

**规则（来自著作权法，描述规则，条号待验证）**：

> 创作作品的自然人是作者，著作权自作品创作完成时起由作者享有。若无相反证据，在作品上署名的自然人推定为作者。

**适用条件**：
- 自然人独立创作
- 非为完成单位工作任务
- 非法人主持的创作

#### B. 职务作品（单位与员工）

**职务作品规则（来自著作权法，描述规则，条号待验证）**：

| 类型 | 定义 | 著作权归属 | 备注 |
|-----|------|---------|------|
| **一般职务作品** | 员工为完成单位工作任务所创作的作品（不符合特殊职务作品条件的） | **作者（员工）享有著作权**；单位有**权利在其业务范围内优先使用**该作品；在作品完成后两年内，作者未经单位同意不得许可第三人以与单位使用相同的方式使用该作品 `[待验证具体规则]` | 员工离职后权利归属须核实合同约定 |
| **特殊职务作品** | 符合以下任一条件：（1）主要利用单位的物质技术条件创作，由单位承担责任的工程设计图、产品设计图、地图、计算机软件等；（2）法律、行政法规规定著作权由单位享有的；（3）合同约定著作权由单位享有的 | **单位（法人）享有著作权**；作者享有**署名权** | 计算机软件常见特殊职务作品情形 |

**职务作品一般 vs 特殊对比表：**

| 比较维度 | 一般职务作品 | 特殊职务作品 |
|---------|-----------|-----------|
| 著作权归属 | 员工 | 单位 |
| 署名权 | 员工 | 员工（署名权不可剥夺） |
| 单位权利 | 优先使用权（业务范围内） | 完整著作权（财产权+人身权除署名外） |
| 典型作品类型 | 一般性文字作品、一般性美术作品 | 工程图、产品设计图、计算机软件（主要使用单位资源者） |
| 约定优先 | 合同约定可改变归属 | 合同约定已是认定条件之一 |

**职务作品判断要点**：
- 「为完成工作任务」是核心标准，非在职期间创作的作品不是职务作品
- 利用单位设备仅是辅助因素（仅利用单位设备但非为工作任务的，不足以认定职务作品）
- 计算机软件：若主要利用单位物质技术条件（服务器、开发平台）且由单位承担技术和经济责任，通常认定为特殊职务作品（著作权归单位）

#### C. 委托作品

**委托作品规则（来自著作权法，描述规则，条号待验证）**：

> 受委托创作的作品，著作权归属由当事人**约定**；**无约定或约定不明**的，著作权由**受委托的创作者（受托人）享有**。

**委托作品权属分析：**

| 情形 | 著作权归属 | 委托方权利 |
|-----|---------|---------|
| 合同约定著作权归委托方 | 委托方（甲方） | 完整著作权 |
| 合同约定著作权归受托方 | 受托方（创作者） | 使用权（按合同约定） |
| 合同有约定但不明确 | 须解释合同，推定归属 | — |
| 合同无约定 | **受托方（创作者）默认** | 委托方可在约定目的范围内使用（须核实约定） |

**实践建议**：委托创作合同若未明确约定著作权归属，可能导致委托方（通常是出资方/企业）无法主张著作权——建议在委托合同中明确约定著作权归属并配套约定转让/独家许可条款，与 commercial-legal 知识产权条款衔接。

#### D. 法人作品

**法人作品规则（来自著作权法，描述规则，条号待验证）**：

> 由**法人主持**、代表法人**意志**创作，并由法人**承担责任**的作品，法人视为作者，著作权由法人享有。

**法人作品认定三要件：**

| 要件 | 说明 |
|-----|------|
| 法人主持 | 由法人（单位）组织、策划、主持创作 |
| 代表法人意志 | 作品体现法人的决策与意志，而非个别员工的独立表达 |
| 法人承担责任 | 作品由法人对外承担法律和技术责任 |

**典型法人作品情形**：法人授权并主导的综合性研究报告、企业官方白皮书、法人以整体名义发表的作品。

#### E. 合作作品

**合作作品规则（来自著作权法，描述规则，条号待验证）**：

> 合作创作的作品，著作权由**全体合作作者共同享有**。
>
> **可分割使用的合作作品**：各部分可以单独使用的合作作品，各部分作者对该部分著作权可以单独行使；但不得侵犯合作作品整体的著作权。
>
> **不可分割使用的合作作品**：不能分割单独使用的，著作权须由合作作者**协商一致**行使，或按比例行使。

**合作作品权属分析表：**

| 情形 | 著作权归属 | 行使规则 |
|-----|---------|---------|
| 两人共同创作，可分割（如合著但各章独立） | 各人对自己部分享有著作权；整体作品著作权共有 | 各部分可单独行使，整体须协商 |
| 两人共同创作，不可分割（如合作绘制一幅画） | 著作权共同享有 | 须协商一致行使 |
| 有约定（合作协议规定各自比例/权利范围） | 按约定 | 按约定 |
| 无约定，按贡献程度 | 通常推定均等共有 | 协商行使 |

### 第四步：权属类型判定决策表

| 问题 | 答案 | 对应权属类型 |
|-----|------|-----------|
| 是否有书面合同关于著作权归属的约定？ | 有约定 → 按约定；无约定 → 继续 | — |
| 是否是法人主持、代表法人意志、由法人承担责任的创作？ | 是 → 法人作品 | 法人作品，著作权归法人 |
| 是否由两人或以上共同参与创作？ | 是 → 合作作品 | 合作作品，著作权共有 |
| 是否是受他人委托而创作的？ | 是，且有权属约定 → 按约定；无约定 → 受托人默认 | 委托作品 |
| 是否是为完成单位工作任务创作的？ | 是 → 进一步判断是否特殊职务作品 | 职务作品（一般/特殊） |
| 以上均否 | — | 创作者（自然人）原始取得 |

## 检查清单

在完成权属判定后，按以下清单验证：

- [ ] 作品创作背景信息已收集（谁创作、创作情境、是否工作任务、是否有约定）
- [ ] 权属约定已优先核查（有约定则按约定）
- [ ] 职务作品判断已完成（一般/特殊），计算机软件已特别注意
- [ ] 委托作品权属已分析，无约定时已确认受托人默认享有著作权
- [ ] 法人作品三要件已逐一核对（主持/意志/责任）
- [ ] 合作作品分割性已判断（可分割/不可分割），行使规则已说明
- [ ] 权属结论明确（归属主体 + 依据类型 + 约定空间）
- [ ] 实践配置已读取，著作权策略与组织类型已确认（或临时模式标注）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 单位与员工权属争议（员工认为非职务作品，单位认为是） | 梳理关键事实（是否为工作任务、是否使用单位资源、是否在工作时间）；提示权属争议须经仲裁/诉讼解决，本分析为初步评估 |
| 委托开发软件（对外委托给开发商） | 须在委托合同中明确约定著作权归委托方（否则默认归开发商）；衔接 commercial-legal 知识产权条款 |
| 员工离职后权属 | 员工离职前为完成工作任务创作的，仍属职务作品；约定竞业限制中的知识产权条款可能影响离职后的作品归属争议 `[待验证]` |
| 合作作品某作者不同意登记 | 不可分割合作作品须全体合作作者同意才能登记；若一方不配合，应记录情况并评估替代路径 |
| 多方合作（三人以上），比例不明 | 建议通过书面合作协议明确各方贡献比例与权利份额；无约定时推定均等 |
| 外包/自由职业者创作的内容 | 通常视为委托作品，著作权默认归受托人（外包方）；务必在外包合同中约定知识产权转让 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 用户主张AI生成内容的著作权 | 提示独创性争议（来自 work-type-determination），目前著作权归属于有人工创作成分的参与者；纯粹AI生成内容的著作权保护不确定 `[待验证]` |
| 委托方认为无需约定、默认归委托方 | 纠正：法定规则为无约定时著作权归受托方（创作者），委托方须明确书面约定才能获得著作权 |
| 用户混淆「著作权人」与「作者」 | 说明：「作者」指实际创作者（自然人或法人）；「著作权人」可包含通过转让/许可获权的主体；两者可以不同 |
