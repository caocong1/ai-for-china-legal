# 作品类型认定（work-type-determination）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

对拟登记（或已存在）的作品进行系统的类型认定：判断独创性是否达到著作权保护门槛、确定法定作品类型、区分受保护的表达与不受保护的思想、识别不受著作权保护的对象。

本子技能的输出将为 ownership-determination（权属判定）提供基础——作品类型直接影响权属归属规则（如计算机软件的特殊职务作品规则），并为 registration-materials（登记材料准备）提供作品类型定性，以确定具体材料要求。

**本子技能可由律师/IP 经理单独触发：`/ip-legal:work-type-determination`**

- **IP 组合**（`## 我们是谁` → 著作权/软件著作权部分）：了解已有作品登记情况

若配置含 `[PLACEHOLDER]` 或文件不存在，使用临时模式：按独创性最低门槛判断，如实认定法定作品类型，标注 `[临时模式]`。

| 子技能 | 分工说明 |
|-------|---------|

## 详细步骤

### 第一步：收集作品基本信息

在开始类型认定前，确认以下作品基本信息：

| 信息项 | 必填/选填 | 说明 |
|-------|---------|------|
| 作品名称 | 必填 | 正式作品名称（或初步名称） |
| 作品内容描述 | 必填 | 作品的主要内容、形式、用途 |
| 创作方式 | 必填 | 作者独立创作 / 合作创作 / 基于模板改编 / AI 辅助创作 / 其他 |
| 表达载体 | 必填 | 文字/图像/代码/音频/视频/建筑图纸/产品设计图/其他 |
| 创作完成时间 | 必填 | 自此时起著作权自动产生（无需登记） |
| 是否已发表 | 选填 | 是/否，发表时间与发表方式（影响保护期计算） |
| 是否含有第三方内容 | 选填 | 如引用文字、图片等——影响独创性判断 |

### 第二步：独创性判断

**著作权保护的核心前提是作品具有独创性（originality）。**

**独创性判断规则（来自著作权法及司法解释，描述规则，条号待验证）**：

> 著作权保护的作品须同时满足：
> 1. **独立创作**（independent creation）：作品须由作者独立创作，非抄袭复制他人作品
> 2. **最低程度的创造性**（minimal creativity）：在表达上具有一定的独特性或创造性，但门槛较低

**独创性程度与作品类型关系：**

| 作品类型 | 独创性门槛 | 说明 |
|---------|---------|------|
| 文字作品（小说/剧本/诗词等） | 较低 | 有一定个性化表达即可，无需文学价值 |
| 美术作品（画作/设计图） | 较低 | 独立创作的视觉艺术表达，最低程度独创性即可 |
| 摄影作品 | 较低 | 对拍摄角度、构图、光线有一定创作性选择 |
| 计算机软件 | 较低 | 对功能实现的具体代码表达具有独创性，不要求算法新颖 |
| 汇编作品（选择与编排） | 中等 | 对内容的选择与编排体现独创性，但各组成部分可能不受保护 |
| 数据库/资料汇编 | 较高（争议） | 纯粹按时间/字母顺序排列的汇编可能不具独创性 |

**独创性不足的常见情形：**

| 情形 | 说明 |
|------|------|
| 纯粹事实/数据的记录 | 如股票价格清单、时刻表、电话号码本——不受著作权保护 |
| 机械性劳动产出 | 无创造性选择的纯复制、测量、排列 |
| 过于简短的表达 | 极短的短语、标语（不足以体现独创性的极简表达） |
| 通用模板的照搬 | 使用他人标准合同模板/标准格式文件，无个性化表达 |
| AI 独立生成（无人工干预） | 目前争议较大，中国著作权法是否保护 AI 独立生成内容尚在发展 `[待验证 — 学说与判例尚在发展]` |

**独创性判断结论：**

| 结论 | 含义 |
|-----|------|
| 具有独创性，受著作权保护 | 可继续进行作品类型认定与登记 |
| 独创性存疑 | 标注并建议咨询律师；谨慎登记 |
| 不具有独创性，不受著作权保护 | 著作权登记无意义；可能受其他保护（商业秘密/专利等） |

### 第三步：法定作品类型认定

**法定作品类型（来自著作权法，描述规则，条号待验证）**：

《著作权法》（2020年修正）规定保护的作品类型包括（条号 `[待验证 — IP 法修正后条号重排，须核实现行版本]`）：

| 作品类型 | 主要特征 | 典型示例 |
|---------|---------|---------|
| 文字作品 | 以文字形式表达的作品，包括书面与口头 | 小说、诗词、论文、剧本、合同文本、报告 |
| 口述作品 | 以口头语言形式创作的作品 | 即兴演讲、讲课录音（未固定的口述形式） |
| 音乐作品 | 歌词、曲谱，含词曲 | 歌曲、器乐曲、歌剧曲谱 |
| 戏剧作品 | 供舞台表演的作品 | 话剧剧本、戏曲脚本 |
| 曲艺作品 | 以说唱为主要艺术形式 | 相声、快板、评书底稿 |
| 舞蹈作品 | 舞蹈动作、姿势、表情的组合 | 编舞作品 |
| 杂技艺术作品 | 杂技艺术动作组合 | 杂技节目 |
| 美术作品 | 以线条、色彩等形式表达的具有审美意义的作品 | 绘画、书法、雕塑、漫画、海报 |
| 建筑作品 | 以建筑物或构筑物形式表现的有审美意义的作品 | 建筑设计方案（有独创性的外观/造型） |
| 摄影作品 | 借助器械在感光材料上记录客观物体形象的艺术作品 | 艺术摄影、纪实摄影（具有独创性的） |
| 视听作品 | 原称「电影作品和以类似摄制电影的方法创作的作品」（2020修正更名） | 电影、电视剧、短视频（具有独创性的）、纪录片 |
| 图形作品和模型作品 | 工程设计图、产品设计图、地图、示意图及三维模型 | CAD 工程图、建筑图纸、地形图、产品三维模型 |
| 计算机软件 | 计算机程序及相关文档 | 源代码、目标代码、软件设计文档（适用软件著作权登记） |
| 其他作品 | 法律、行政法规规定的其他类型 | 如集成电路布图设计（单独立法 `[待验证]`）、体育竞赛规程等争议 |

**作品类型认定决策表：**

| 判断问题 | 是 | 否 |
|---------|---|---|
| 主要是以文字表达的？ | → 文字作品 | 继续 |
| 主要是视觉图像（非文字、非建筑）？ | → 美术作品/摄影作品/图形作品 | 继续 |
| 是计算机代码或软件文档？ | → 计算机软件（走软件著作权登记路径） | 继续 |
| 是可播放的视频/影视内容？ | → 视听作品 | 继续 |
| 是建筑设计或工程图纸？ | → 建筑作品/图形作品 | 继续 |
| 多种媒介组合（含音乐、画面、文字等）？ | → 综合判断，主要类别 + 附属类别 | 继续 |

### 第四步：思想表达二分法应用

**思想表达二分法（idea-expression dichotomy）**——著作权法基本原理，规则稳定：

> **著作权保护作品的具体表达（expression），不保护其中体现的思想、概念、原理、操作方法、数学概念或通用素材（idea）。**

**应用示例表：**

| 作品场景 | 受保护的表达 | 不受保护的思想/概念 |
|---------|-----------|-----------------|
| 一篇合同模板 | 具体的文字措辞、语句结构、独特条款安排 | 合同的法律结构、条款类型的概念、风险分配逻辑 |
| 软件源代码 | 具体的代码实现（语句、函数、类的具体写法） | 软件功能、算法思路、数据结构概念 |
| 教材内容 | 具体文字表述、图表设计、例题编排 | 数学原理、物理规律、教学方法本身 |
| 绘画作品 | 具体的色彩运用、线条、构图方式 | 以夕阳为主题、描绘山水这一题材本身 |
| 商业计划书 | 具体撰写内容、分析逻辑的表达 | 商业模式概念、市场分析框架 |
| 食谱 | 具体文字描述的步骤与烹饪过程 | 烹饪方法本身（方法步骤通常不受著作权保护） |

**合并原则（merger doctrine）**：若某一思想的表达方式极为有限（即思想与表达高度合并），则该表达也可能不受著作权保护，以防止著作权垄断该思想 `[待验证 — 学说争议，司法实践适用程度不一]`。

### 第五步：不受著作权保护的对象清单

以下对象不受著作权法保护（条号待验证，描述规则）：

| 类别 | 典型示例 | 说明 |
|-----|---------|------|
| 法律法规 | 全国人大立法、行政法规、部门规章、地方性法规 | 属于公共领域，任何人可自由使用 |
| 国家机关决议/决定/命令 | 政府文件、行政决定、司法裁判文书 | 公共文件，不受保护 |
| 国家机关认可的翻译作品 | 上述文件的翻译（若官方认可） | 公共资源 |
| 时事新闻 | 报道事实的纯粹新闻信息（仅事实，不含评论）| 事实不受保护；新闻文章的具体撰写可受保护 |
| 历法/通用数表/通用表格/公式 | 日历、元素周期表、数学公式、标准会计报表格式 | 纯信息、公式，无独创性 |

## 检查清单

在完成作品类型认定后，按以下清单验证：

- [ ] 作品基本信息已收集（名称、内容、创作方式、表达载体、创作完成时间）
- [ ] 独创性已判断（独立创作 + 最低程度创造性），结论明确
- [ ] AI 生成内容的独创性争议已单独标注（如涉及）
- [ ] 法定作品类型已认定，类型边界已说明
- [ ] 思想表达二分法已应用，明确说明哪部分是受保护的表达
- [ ] 不受著作权保护的对象已排查（法律法规/国家机关文件/时事新闻/公式）
- [ ] 是否属于计算机软件已判断（影响登记路径：软件著作权登记 vs 一般作品登记）
- [ ] 实践配置已读取，著作权策略已确认（或临时模式标注）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 实用艺术品（同时具有艺术性与实用性的产品） | 须分离判断：艺术性部分（独立欣赏价值的外观设计）可受美术作品保护；纯粹实用性部分不受保护；建议同时考虑外观设计专利保护 `[待验证具体判断标准]` |
| 计算机软件与文字作品的区分 | 软件源代码：走软件著作权登记（中国版权保护中心，《计算机软件著作权登记办法》）；技术文档/用户手册如单独登记可作为文字作品 |
| AI 生成内容的独创性争议 | 目前争议较大，中国著作权法对 AI 生成内容是否具有独创性无明确规定 `[待验证 — 学说与判例尚在发展中，2023年以来部分法院对有人工智能辅助但有人工创作成分的作品给予保护]` |
| 汇编作品 | 整体汇编的选择与编排可受保护；各被汇编内容的著作权状态须单独判断（可能涉及许可） |
| 衍生作品（改编/翻译/注释） | 衍生作品的独创性部分受保护；但须核实对原作品的改编是否合法（是否获得原权利人授权） |
| 跨国作品 | 外国作者在中国受保护（中国缔结或参加的国际条约范围内），保护期与中国著作权法相同 `[待验证具体条约适用]` |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 用户描述不清楚，无法判断作品类型 | 停止判断，询问具体作品内容、表达载体与创作方式；不得凭猜测作出类型认定 |
| 用户主张对纯粹功能性设计享有著作权保护 | 明确说明：纯粹功能性设计（如工业品的功能性形状）通常不受著作权保护，建议考虑外观设计专利或实用新型保护路径 |
| 作品明显是现有作品的简单拷贝 | 提示：拷贝他人作品不具有独创性，且可能构成侵权；须核实原作品著作权状态 |
| 用户将「著作权」与「商标权」混淆（要求对商标的文字/图形进行著作权登记） | 说明：商标图案/文字若具有独创性可单独作为美术作品申请著作权登记；但商标保护须通过商标注册，两种保护路径不同 |
