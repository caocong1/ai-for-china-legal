# 著作权登记（编排入口）

> **迁移说明**：原著作权登记内容（需求收集/法律分析/输出生成三步纲要）已拆分迁移并大幅深化至四个可独立触发的子技能。本文件为编排入口，负责前置护栏和子技能调用顺序。子技能路径：`ip-legal/skills/copyright-registration/<子技能名>/SKILL.md`。

---

## 目的

编排著作权登记全流程，协调四个子技能，对照中国著作权法（2020年修正，2021年6月1日施行）与相关登记办法，系统完成作品类型认定、权属判定、登记材料准备与登记流程指引，输出著作权登记综合方案。

**整体分析流程**：

```
作品类型认定（work-type-determination）
  ↓ 输出：独创性结论 + 法定作品类型 + 思想表达二分分析
权属判定（ownership-determination）
  ↓ 输出：权属类型 + 申请主体 + 约定空间建议
登记材料准备（registration-materials）
  ↓ 输出：按作品类型与权属类型的材料清单（含软件特殊材料）
登记流程（registration-process）
  ↓ 输出：流程步骤 + 证书效力说明 + 著作权登记综合方案
```

### 临时模式

如果用户说「临时模式」，使用以下通用默认值进行分析：
- 维权立场：平衡型（客观判断独创性与权属，如实说明登记价值）
- 角色：律师（输出含「律师工作产物」标题）
- 著作权按独创性最低门槛判断，权属按法定规则（无约定时受托人默认/职务作品按一般规则）

  > 「这是通用默认值运行。运行 `/ip-legal:cold-start-interview` 来获得根据你的真实著作权策略定制的输出。」

## 自愿登记原则提示（入口级）

在开始任何分析前，向用户明确：

> **著作权自愿登记原则**：著作权自作品创作完成之日起**自动产生**，无需登记。著作权登记为**自愿**，非确权手段。**登记证书是著作权归属的初步证据**（可在纠纷中举证便利），但不等于确权——登记证书可被反证推翻。
>
> 著作权法（2020年修正，规则稳定，条号建议复核）规定：
> - 自然人作品保护期：作者**终生及死亡后50年**
> - 法人/视听作品保护期：**首次发表后50年**

## 前置：权利人画像确认

读取 `ip-legal/CLAUDE.md` → `## 我们是谁`，确认：

1. **行业类型**：文创/软件/互联网/制造等——影响作品类型识别（软件走不同登记路径）
2. **著作权策略立场**：是否有主动登记政策，软件是否全部登记
3. **现有著作权登记情况**：已登记数量，主要作品类型

## 子技能编排顺序

律师/IP 经理可单独触发任一子技能，也可通过本入口运行完整著作权登记分析流程：

| 顺序 | 子技能 | 职责 | 子技能路径 | 单独触发命令 |
|------|-------|------|----------|------------|
| 1 | **work-type-determination** | 独创性判断 + 法定作品类型认定 + 思想表达二分法 + 不受保护对象识别 | `skills/copyright-registration/work-type-determination/SKILL.md` | `/ip-legal:work-type-determination` |
| 2 | **ownership-determination** | 权属判定：创作者原始取得/职务/委托/法人/合作作品 + 约定空间分析 + 申请主体确认 | `skills/copyright-registration/ownership-determination/SKILL.md` | `/ip-legal:ownership-determination` |
| 3 | **registration-materials** | 按作品类型与权属类型整理材料清单：申请主体证明/作品说明/权利归属证明/样本（含软件前后各30页，待验证） | `skills/copyright-registration/registration-materials/SKILL.md` | `/ip-legal:registration-materials` |
| 4 | **registration-process** | 登记流程步骤 + 登记证书初步证据效力说明 + 登记后变更/补充/撤销 + 著作权登记综合方案 | `skills/copyright-registration/registration-process/SKILL.md` | `/ip-legal:registration-process` |

**顺序说明**：作品类型认定在前（确定是否具有独创性、适用哪种登记路径）→ 权属判定（确认申请主体与所需权归属证明）→ 材料准备（按前两步结论整理具体材料清单）→ 登记流程（基于材料清单完成提交与证书申领，并输出综合方案）。

**单独触发场景（举例）**：
- 只需判断某作品是否具有独创性：`/ip-legal:work-type-determination [作品描述]`
- 只需分析著作权权属（尤其委托开发软件）：`/ip-legal:ownership-determination [创作背景]`
- 只需整理登记材料清单：`/ip-legal:registration-materials [作品类型] [权属类型]`
- 只需了解登记流程与证书效力：`/ip-legal:registration-process [已有作品类型与权属信息]`

## 入口级护栏

以下护栏在编排入口层执行，不依赖子技能：

### 著作权策略加载

读取 `ip-legal/CLAUDE.md` → `## 维权与申请立场` → `### 著作权策略`：

- 主要作品登记立场（主动/选择性/不登记）——确认本次登记必要性
- 软件著作权登记立场（全部/核心版本/不登记）——软件须走特殊登记路径
- 著作权维权方式优先级（协商/行政投诉/诉讼）——影响登记重要性判断

### 行业识别

当涉及以下行业/作品类型时，须额外注意，标 `[行业 flagged — 需核实主管部门规定]`：

| 行业/类型 | 著作权特别关注 |
|---------|------------|
| 互联网/软件 | 软件著作权登记（中国版权保护中心，《计算机软件著作权登记办法》）；开源许可合规；AI 生成内容独创性争议 `[待验证]` |
| 音乐/影视/出版 | 集体管理组织（中国音乐著作权协会等）；视听作品著作权（2020修正更名） |
| 游戏/动漫 | 美术作品/视听作品/文字作品的综合保护；游戏规则的思想不受保护 |
| 建筑设计 | 建筑作品（有独创性的建筑外观）与图形作品（建筑图纸）的交叉 |
| 文创/设计 | 美术作品与外观设计专利的竞合选择 |

### IP 法时效性提示

> **IP 法修正特别提示**：著作权法经 2020 年修正、2021年6月1日施行，条号重排。本分析中的具体条号均标 `[待验证]`，除保护期规则（规则稳定，条号建议复核）外。在引用法条前建议通过国家法律法规数据库（https://flk.npc.gov.cn）核实现行版本。

### 目的地与特权检查

输出前检查目的地是否在律师-当事人特权圈内。面向对方当事人、法院的内容会使特权保护失效，须提供内部特权版本和脱敏版本两套选项。

### 著作权清权衔接

若用户涉及**作品/素材来源合规**（如使用第三方图片/音乐/代码的许可状态），或需要系统评估拟使用内容是否侵犯他人著作权，可调用 ip-clearance 子技能（06-03 著作权清权/使用权评估），进行更全面的著作权来源与授权链核查。

## 输出

完整分析流程由 `registration-process` 子技能最终汇总，包含：
- **著作权登记综合方案**：作品类型 + 权属 + 材料清单 + 登记路径 + 保护期 + 证书效力说明
- **材料准备状态表**：各类材料的准备情况与注意事项
- **后续里程碑建议**：登记提交时间节点、变更登记提醒、维权路径衔接

工作产物标题继承 `ip-legal/CLAUDE.md` → `## 输出` → `### 工作产物标题`（律师/IP 经理角色）。
