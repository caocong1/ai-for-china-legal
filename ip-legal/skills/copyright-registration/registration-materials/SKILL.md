# 登记材料准备（registration-materials）

> 前置读取：`ip-legal/skills/_shared/legal-basis-conventions.md`（引用规范）、`ip-legal/skills/_shared/ip-law-citations.md`（引用库）、`ip-legal/skills/_shared/practice-profile-schema.md`（配置读取契约）、`ip-legal/CLAUDE.md`（实践配置）

---

## 目的

基于 work-type-determination（作品类型）与 ownership-determination（权属判定）的结论，系统整理著作权登记所需的全部材料，按照中国版权保护中心/地方版权局的登记要求，输出分类材料清单，并对各类材料的具体要求与常见缺陷进行详细说明。

本子技能是登记准备阶段的核心操作步骤，其输出将直接指导实际材料收集与提交工作。

**本子技能可由律师/IP 经理单独触发：`/ip-legal:registration-materials`**

---

## 前置

### 读取实践配置

在开始材料准备前，读取 `ip-legal/CLAUDE.md`：

- **著作权策略**（`## 维权与申请立场` → `### 著作权策略`）：
  - 主要作品登记立场（主动登记/选择性登记）——确认本次登记的必要性
  - 软件著作权登记立场——软件材料要求特殊，须单独处理
- **可用集成**（`## 可用集成` → 版权登记查询）：版权登记查询系统可用性
- **执业角色**（`## 谁在使用`）：决定工作产物标题

若配置含 `[PLACEHOLDER]` 或文件不存在，使用临时模式：按通用材料清单处理，标注 `[临时模式]`。

### 前置依赖

本技能依赖前两个子技能的输出：

| 前置输入 | 说明 |
|---------|------|
| 作品类型（来自 work-type-determination） | 决定样本要求（文字作品 vs 软件 vs 美术等） |
| 权属类型（来自 ownership-determination） | 决定权利归属证明材料（创作者 vs 职务 vs 委托 vs 法人 vs 合作） |
| 申请主体 | 来自权属判定，确认登记申请人身份 |

### 与其他子技能的分工

| 子技能 | 分工说明 |
|-------|---------|
| work-type-determination | 确定作品类型（前置） |
| ownership-determination | 确定权属与申请主体（前置） |
| **本技能（registration-materials）** | 按作品类型与权属类型，系统整理登记材料清单与具体要求 |
| registration-process | 基于本技能整理的材料，完成登记流程提交与证书申领 |

**本子技能可被 copyright-registration 编排入口顺序调用，也可由律师/IP 经理单独触发。**

## 详细步骤

### 第一步：确认登记路径

**著作权登记路径选择（基于作品类型）：**

| 作品类型 | 登记路径 | 受理机构 |
|---------|---------|---------|
| 计算机软件（含源代码） | 软件著作权登记（走《计算机软件著作权登记办法》路径） | 中国版权保护中心 |
| 文字/美术/摄影/视听/图形/音乐等其他作品 | 一般作品著作权登记（走《作品自愿登记办法》路径） | 中国版权保护中心或地方版权局/著作权登记机构 |
| 注意 | 同一作品不重复登记；软件的说明书若单独登记，作为文字作品处理 | — |

### 第二步：通用材料（所有登记类型均需）

**A. 申请主体证明材料**

| 申请主体类型 | 所需证明材料 | 要求 |
|-----------|-----------|------|
| 自然人（个人作者） | 身份证复印件（正反面） | 清晰可辨，与申请表上姓名一致 |
| 法人/其他组织 | 营业执照（复印件并加盖公章）或组织机构代码证（已整合至营业执照的可用三证合一营业执照） | 须在有效期内；与申请表单位名称一致 |
| 外国/港澳台申请人 | 护照/港澳通行证/台胞证 + 若有代理则提供授权委托书 | `[待验证 — 具体要求须核实登记机构最新规定]` |

**B. 作品说明**

| 说明项 | 内容要求 | 注意事项 |
|-------|---------|---------|
| 作品名称 | 正式作品名称，与样本封面/标题一致 | — |
| 作品类别 | 对应法定作品类型（文字/美术/摄影/视听/软件/其他） | 须与 work-type-determination 结论一致 |
| 独创性说明 | 简要说明作品的创作目的、创作过程及独创性（创作者自述） | 不需要超高门槛，客观描述即可 |
| 创作完成时间 | 作品最终完成的日期 | 证明著作权起始时间 |
| 是否首次发表 | 是/否；若已发表，注明首次发表时间与媒介 | 影响保护期计算（法人/视听作品尤其重要） |

**C. 作品样本**

样本是登记的核心材料，要求见「第四步：按作品类型的样本要求」。

### 第三步：按权属类型的权利归属证明材料

**依据权属判定结论，提供对应的权利归属证明：**

| 权属类型 | 权利归属证明材料 | 说明 |
|---------|--------------|------|
| 创作者原始取得（自然人独立创作） | 通常无需单独提交权利归属证明（申请主体即为作者）| 若有争议，可提供创作过程证明（草稿/创作记录/时间戳等） |
| 特殊职务作品（著作权归单位） | 劳动合同（显示作者与单位的劳动关系）+ 单位说明函（说明该作品为职务作品）+ 作者确认声明（若可获得） | 须与 ownership-determination 结论一致 |
| 一般职务作品（著作权归作者，单位优先使用） | 同上；若单位作为申请主体须说明单位基于优先使用权申请登记（情况较少见，通常由作者申请）| — |
| 委托作品（约定归委托方） | 委托合同（含著作权归属约定条款）复印件 + 双方签署 | 若合同不含明确约定，难以主张著作权归委托方 |
| 委托作品（无约定，归受托人） | 无需额外归属证明（申请主体即为受托人/创作者）| 委托合同作为创作背景材料提交即可 |
| 法人作品 | 法人/单位出具的作品创作组织说明 + 作者/参与人员名单 | 说明法人主持、代表法人意志、由法人承担责任 |
| 合作作品 | 全体合作作者共同签署的著作权登记申请表（或授权书）+ 合作协议（如有） | 不可分割合作作品须全体合作作者授权 |
| 继承/受让取得 | 继承文件（遗嘱/继承公证书）或著作权转让合同（经公证或原作者/继承人签署） | 须提供完整权利链文件 |

### 第四步：按作品类型的样本要求

**一般作品样本要求（适用文字/美术/摄影/图形/视听/音乐等）：**

| 作品类型 | 样本形式 | 具体要求 | 注意事项 |
|---------|---------|---------|---------|
| 文字作品 | 作品全文文本（印刷或电子文件） | 通常提交完整全文，短篇提交原文，长篇可提交前后各若干页（具体要求 `[待验证 — 须核实登记机构最新要求]`） | 须有封面/标题页，显示作品名称与作者信息 |
| 美术作品（绘画/雕塑/书法） | 高清图片或照片，彩色打印 | 清晰显示作品全貌；建议附作品原件照片 | 作品图片分辨率须足够辨认细节 |
| 摄影作品 | 高清摄影图片 | 说明拍摄时间、地点；原始文件若有 EXIF 信息更佳 | — |
| 视听作品 | 光盘或U盘（视频文件）+ 截图 | 视频全片或精华部分（具体要求 `[待验证]`）；截图显示作品主要内容 | — |
| 音乐作品 | 曲谱（纸质）+ 录音文件（如有） | 须有词曲完整内容 | — |
| 图形作品（工程图/产品设计图/地图） | 图纸原件或高清扫描件 | 须有设计人员与日期标注 | — |
| 建筑作品 | 建筑设计图 + 建筑物照片（若已建成） | — | — |

**计算机软件样本（特殊要求，单独说明）：**

软件著作权登记对样本有特殊要求（来自《计算机软件著作权登记办法》，描述规则，具体页数 `[待验证 — 须核实登记办法最新要求]`）：

| 材料项 | 具体要求 | 注意事项 |
|-------|---------|---------|
| 源程序样本 | 源代码（连续的源程序文本）：通常要求前后各连续30页，若总页数不足60页则提交全部（具体页数 `[待验证 — 须核实登记办法最新要求]`） | 须为实际源代码，非截图或伪代码；每页须有页码 |
| 软件说明书/文档 | 软件设计说明书或用户手册 | 须包含软件功能描述、运行环境、使用方法等基本信息 |
| 软件标识页 | 封面页：软件名称、版本号、著作权人、开发完成日期 | 须与申请表信息完全一致 |
| 商业秘密处理 | 若源代码含商业秘密，可申请鉴别材料例外（删除商业秘密部分后提交）`[待验证 — 具体例外规则须核实]` | 删除比例须符合登记机构规定 |

### 第五步：代理材料（如委托代理申请）

| 材料 | 要求 |
|-----|------|
| 授权委托书 | 由著作权人（个人签名 / 法人盖章）签署，注明委托代理事项 |
| 代理机构资质证明 | 若由代理机构代办，提供代理机构营业执照（知识产权代理机构） |

### 第六步：材料完整性检查

制作材料完整性检查表，便于提交前核对：

| 材料类别 | 材料名称 | 是否已准备 | 备注 |
|---------|---------|---------|------|
| 申请主体证明 | [身份证/营业执照] | 是/否 | [有效期/清晰度] |
| 作品说明 | 申请表（含独创性说明） | 是/否 | [完整性] |
| 权利归属证明 | [视权属类型而定] | 是/否 | [材料类型] |
| 作品样本 | [视作品类型而定] | 是/否 | [格式/页数] |
| 软件特殊材料 | [如适用] 源程序样本+说明书 | 是/否 | [页数/格式] |
| 代理材料 | [如委托代理] 授权委托书 | 是/否/不适用 | — |

## 检查清单

在完成材料准备后，按以下清单验证：

- [ ] 登记路径已确认（软件著作权登记 vs 一般作品登记）
- [ ] 申请主体证明材料已明确（个人身份证 / 法人营业执照）
- [ ] 权利归属证明已对应权属类型选取（职务/委托/合作等）
- [ ] 作品说明已准备（名称/类别/独创性/创作完成时间/是否发表）
- [ ] 样本已按作品类型确认要求（文字全文/软件源程序前后30页等）
- [ ] 软件登记特殊材料已核查（源程序样本+说明书+商业秘密处理选项）
- [ ] 材料完整性检查表已制作
- [ ] 实践配置已读取，著作权策略已确认（或临时模式标注）

## 边界条件

| 情形 | 处理方式 |
|------|---------|
| 作品样本涉及商业秘密（如软件核心算法） | 软件登记有商业秘密例外申请选项（删除后提交）`[待验证 — 具体例外申请规则须核实最新登记办法]`；一般作品含商业秘密部分建议慎重处理 |
| 未发表作品申请登记 | 可以登记，登记时注明「未发表」；保护期计算以创作完成时间为起点（自然人终生+死后50年；法人/视听作品须注意未发表的特殊处理 `[待验证具体规则]`） |
| 合作作品某合作作者不配合提供材料 | 记录情况；不可分割合作作品通常须全体合作作者授权，若一方不配合则无法申请登记 |
| 作品已在海外登记 | 可以在中国单独登记（两者独立），提交时可说明海外登记情况（非必须）；注意保护期自中国登记之日起的计算方式 `[待验证]` |
| 委托代理申请 | 委托专业代理机构可提高效率；但须核实代理机构资质；授权委托书须由权利人本人签署 |

## 错误处理

| 错误情形 | 处理方式 |
|---------|---------|
| 未完成前序步骤（未有作品类型/权属结论）即触发本技能 | 要求先完成 work-type-determination 与 ownership-determination，或要求用户直接提供作品类型与权属信息 |
| 样本材料不完整（软件仅有截图无源代码） | 明确告知：软件登记须提交源程序文本，截图不满足要求；建议准备连续源代码文本 |
| 用户提交的营业执照已过期 | 标注材料状态为「无效」，要求更新至有效营业执照 |
